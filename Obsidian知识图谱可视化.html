📊 第一部分：图谱类型详解

1.1 全局图谱

定义: 展示整个知识库所有笔记节点及其链接关系



打开方式:

  设置 → 外观 → 右侧边栏 → 开启"图形视图"

  或快捷键: Ctrl/Cmd + Shift + G



视图范围:

  - 所有笔记

  - 所有双向链接

  - 所有标签关系



核心用途:

  1. 宏观看清知识全貌

  2. 发现整体结构问题

  3. 识别知识空白

1.2 本地图谱

定义: 聚焦于当前笔记，显示其直接相连的笔记（1-2层关系）



打开方式:

  当前笔记 → 右上角"更多选项" → "在本图谱中显示"

  或：右键笔记 → "在本图谱中显示"



视图范围:

  - 当前笔记（中心节点）

  - 第一层关联（直接链接）

  - 第二层关联（链接的链接）



核心用途:

  1. 微观分析局部网络

  2. 理解特定概念的关系网络

  3. 发现潜在连接

3.2 路径搜索

【功能】



查找从笔记A到笔记B的所有可能路径



【使用场景】



当你发现两个笔记看似无关，但想探索潜在联系时



【操作步骤】



1. 右键笔记A → "查找到此处的所有路径"

2. 或：搜索框输入 "path:笔记A 笔记B"

3. 系统展示所有中间节点和路径



【解读】



路径长度 = 关联紧密程度

- 1步 = 直接链接

- 2-3步 = 间接关联

- 更多步 = 弱关联或可建立的新连接

3.3 层级过滤

【功能】



控制图谱显示的关联层数



【层级说明】



第0层: 只显示当前笔记

第1层: 显示直接链接（出链/入链）

第2层: 显示链接的链接

第3层+: 依此类推



【推荐设置】



日常使用: 1-2层

全局分析: 2-3层

深度探索: 按需展开

🛠️ 第五部分：高级应用

5.1 Dataview + Graph View

【组合优势】



Dataview: 查询特定条件的笔记

Graph View: 可视化这些笔记的关系



【操作流程】



1. 用Dataview查询

   ```dataview

   TABLE 评分, 标签

   FROM ""

   WHERE 评分 >= 4

   ```



2. 将结果发送到Graph View



3. 观察高质量笔记的分布



4. 发现知识网络中的"明星节点"

5.2 Juggl插件

【Juggl高级功能】



1. 交互式图形

   - 拖拽节点

   - 缩放平移

   - 聚焦跟随



2. 样式控制

   - 节点形状（圆形/方形/图标）

   - 渐变填充

   - 阴影效果



3. 布局算法

   - 力导向布局

   - 层级布局

   - 环形布局



4. 导出功能

   - 导出为PNG/SVG

   - 嵌入到笔记
