一、核心定位

1.1 系统归属关系

``` ┌─────────────────────────────────────────────────────────────────────┐ │ AI龙龟共生伙伴操作系统 │ ├─────────────────────────────────────────────────────────────────────┤ │ │ │ 【调度层】龙心OS（智能中枢） │ │ ├─ 🐉 象思维（0→1原创） │ │ ├─ 📚 知识学习（深度学习） │ │ ├─ 🌈 五色光思维（多维度分析） │ │ ├─ 🤝 人机协同五象限（分工协作） │ │ ├─ 🔄 知行合一（沉淀进化） │ │ └─ 【龙脑OS知识库调用接口】 │ │ │ │ 【知识库层】龙脑OS（被调用层） │ │ ├─ 📚 思维模型库 ← 本Skill所在位置（100个模型·六阶段） │ │ ├─ 📊 麦肯锡思考框架 │ │ ├─ 🔨 五行人格心理学OS │ │ └─ ...其他专业知识体系 │ │ │ └─────────────────────────────────────────────────────────────────────┘ ```

1.2 架构关系说明

思维模型库的定位：
  • 所属系统：龙脑OS知识库（知识资产层）
  • 调用方式：由龙心OS根据场景智能调用
  • 关系性质：被调用方，不直接响应用户输入
  • 生态结构：主Skill + 6个阶段子Skill + 高频独立模型
    • 调用流程： ``` 用户输入 → 龙心OS场景识别 → 判断需要思维模型 → 调用龙脑OS.思维模型库 → 匹配阶段/模型 → 执行分析 → 返回龙心OS整合输出 → 知行合一沉淀 ```

    1.3 核心原则

    1. 龙心OS统一调度：所有分析需求先经过龙心OS的场景识别和引擎路由 2. 每个模型独立存在：100个模型各自独立封装，可被单独调用或组合使用 3. 按需调用，灵活组合：根据场景调用单模型、多模型组合或完整阶段套件 4. 中西合璧，AI赋能：东方智慧（易经/道家/儒家）+ 西方工具（麦肯锡/芒格）+ AI结合方案

    四、使用方式

    4.1 单模型调用

    示例： ``` 用户："帮我用SWOT分析一下我的副业项目"

    龙心OS路由： → 调用龙脑OS.思维模型库.SWOT → 执行SWOT四象限分析 → 返回结构化结果 ```

    4.2 多模型组合

    示例： ``` 用户："我想进入新市场，帮我全面分析一下"

    龙心OS路由： → 调用龙脑OS.思维模型库 → 组合：波特五力 + SWOT + PEST + BCG矩阵 → 返回综合分析报告 ```

    4.3 阶段套件调用

    示例： ``` 用户："我想系统提升自己的能力，从哪些方面入手？"

    龙心OS路由： → 调用龙脑OS.思维模型库.第三阶段 → 输出18个能力模型全景图 → 根据用户情况推荐优先级 ```
