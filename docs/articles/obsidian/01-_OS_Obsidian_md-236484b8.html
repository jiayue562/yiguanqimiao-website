1.3 AI加工详细流程

【第一步：剪藏】

Web Clipper → 选择目标仓库 → 点击保存
→ 自动转换为Markdown格式

【第二步：AI深度加工】

调用DeepSeek API → 输入剪藏内容
→ 系统自动生成：

1. 一句话总结
   格式：基于[主题]，提出[核心观点]

2. 关键点列表
   格式：3-5个Bullet Points
   内容：每个点不超过20字

3. 价值评价
   格式：[rating:: 1-5]
   说明：适用场景、局限性

4. 标签建议
   格式：#标签1 #标签2
   规则：最多5个标签

【第三步：初始标记】

为所有新笔记添加：#未归档
→ 便于后续集中整理

2.2 原子化原则

【原子笔记定义】

一个原子笔记 = 一个核心概念 + 一个主题 + 一个事件

【原子化判断标准】

✅ 一个核心观点 → 独立成篇
❌ 多个观点混杂 → 需要拆分
✅ 可以用一句话总结 → 原子化完成
❌ 需要多句话才能说清 → 继续拆分

【原子化收益】

- 被多次灵活链接
- 复用性提高
- 链接更精准
- 知识网络更清晰

2.3 内容管理三工具

工具一：段落级标签

【定义】
在段落或句子后添加的标签

【语法】
这是重要段落。#金句

这是待处理内容。#待处理

这是心理学内容。#topic/心理学

【用途】
- 快速定位关键片段
- 构建"二次输出素材库"
- 不依赖插件的基础方法

【示例】

"知识不是囤积，而是流动。" #金句 #知识管理

"创新源于连接。" #金句 #创新思维

工具二：内联字段（Dataview）

【定义】
给内容片段附加结构化属性

【语法】
[字段:: 值]

【示例】

【读书笔记模板】

书名:: 《认知觉醒》
评分:: 5
作者:: 周岭
核心观点:: 成长是克服天性的过程

【日志模板】

日期:: 2026-04-03
心情:: 平静
最满意的事:: 完成了Obsidian学习
最不满意的事:: 时间管理

【用途】
- 同一笔记内重复记录同类信息
- 便于Dataview汇总成表格
- 从文本变为可查询数据

工具三：任务列表

【基础语法】
- [ ] 待办事项
- [x] 已完成事项

【Tasks插件增强】

- 待办事项 ← 添加日期/优先级/标签
  @2026-04-05
  🔺high
  #工作

【查询语法】
tasks not done due before next week
→ 从全库自动汇总和过滤任务

【与笔记绑定】

[[项目A]]
- [ ] 完成需求分析 @2026-04-05
- [ ] 编写技术文档 @2026-04-10

优势：学习（笔记）+ 执行（任务）闭环

3.2 对比与交叉阅读

【什么是交叉阅读】

同时打开2-4篇相关笔记进行对照阅读
- 观点互补的笔记
- 观点相反的笔记
- 看似无关但可能关联的笔记

【为什么有效】

并置阅读 ──→ 强制大脑寻找 ──→ 触发
                  ↓
            模式 + 差异 + 联系
                  ↓
            新想法 + 新问题
                  ↓
            写作和创造的直接源泉

【操作步骤】

1. 找到相关笔记（本地图谱/反向链接）
2. 同时打开2-4篇
3. 并排显示
4. 记录差异和联系
5. 生成新想法

3.3 图谱查漏补缺

【每周图谱复盘】

1. 打开全局图谱
2. 寻找孤立节点
3. 分析原因：
   - 为什么孤立？
   - 应该与哪个概念连接？
   - 需要创建什么主题关联？

4. 执行行动：
   - [[孤立笔记]] ──→ [[某个概念]]
   - 或创建新的综述笔记
   - 或合并到已有主题

【目标】
修复和强化知识网络的结构完整性

4.3 仪表盘构建（Dataview）

【今日仪表盘笔记】

TABLE file.ctime as 创建时间 FROM "" WHERE file.ctime >= 2026-04-03 SORT file.ctime DESC LIMIT 10


【金句随机展示】

LIST "#金句" FROM "" WHERE contains(tags, "金句") LIMIT 5


【待办任务汇总】

TASK FROM "" WHERE !completed GROUP BY file.link


优势：
- 知识库主动提供信息
- 从被动检索到主动展示
- 成为真正的"行动面板"
