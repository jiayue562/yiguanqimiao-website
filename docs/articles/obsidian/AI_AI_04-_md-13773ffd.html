二、三分类标注自动化规则

🟢 [AI] 自动标注的条件

以下任务类型无需人工确认，直接标记为🟢[AI]，执行后汇报结果：

条件 示例
数据查询（纯检索，无判断） "昨天营收多少""本月毛利率"
标准化报告生成 "生成上周经营周报"
库存/考勤异常检测 "检查库存预警""生成考勤异常报告"
信息整理/对比 "对比三家门店上周营收"
文档格式转换 "把这份数据转成表格"
跨库知识检索 "搜索相关知识并整理"

🔴 [人] 强制标注的条件

以下任务类型AI绝对不能单独执行，必须标记为🔴[人]并等待：

条件 示例
涉及金额>5,000元的支出决策 "采购5,000元以上的物料"
涉及人员去留/晋升的决策 "录取/辞退/晋升某人"
涉及品牌对外表态 "发布公众号/社交媒体内容"
涉及食品安全判断 "判断食材是否可继续使用"
涉及法律合规判断 "判断某操作是否合法"
涉及文化/价值观判断 "这样做符合味藏文化吗"
新领域/无先例的探索 "从来没有做过的事"

🟡 [⇄] 串联瓶颈标注的条件

以下任务类型AI可以先做，但中间必须等人确认：

条件 示例
方案类产出 AI出方案→人选择→AI细化
内容创作类 AI出草稿→人审核→AI修改→人确认
数据分析+建议类 AI出分析→人判断→AI深化
排班/调度类 AI出方案→人调整→人确认
采购建议类 AI出清单→人确认→AI下单
复杂问题诊断 AI出分析→人给方向→AI深入

三、工作流模板库

模板W1：日报生成（Q1·效率协作者·全自动）

🟢[AI] P-01日反馈智能体采集昨日数据
🟢[AI] 生成日报（营收/客流/客单/TOP菜品/异常标记）
🟢[AI] 异常检测（同比/环比偏离>15%自动标记）
🟢[AI] 推送至对应店长工作群
🔴[人] 店长晨会参考数据

标注模式：🟢🟢🟢🟢 → 🔴（后置阅读）
瓶颈率：0%（无人-AI等待点）

模板W2：排班调整（Q1→Q3·半自动）

🟢[AI] HR-MGR根据客流预测生成排班草案
🟡[⇄] 推送店长确认/调整（2小时内）
🔴[人] 店长审核、处理特殊情况
🟢[AI] 冲突检测 → 如有冲突重新建议
🟢[AI] 确认后排班发布

标注模式：🟢 → 🟡[⇄] → 🔴 → 🟢 → 🟢
瓶颈率：20%（只在店长确认时等待）

模板W3：菜单创新（Q3·思辨对话者·深度协作）

🟡[⇄] 悟空/店长给出创新方向（降本/差异化/时令）
🟢[AI] KC料理长分析现有菜单数据（销量/毛利/趋势）
🟢[AI] PUR-MGR提供食材供应信息（时令/成本/稳定性）
🟢[AI] MKT-MGR提供竞品菜单分析
🟢[AI] 生成3-5个候选方案（含成本/毛利/供应链可行性）
🟡[⇄] 推送决策层讨论
🔴[人] 试菜 → 文化判断 → 最终选择
🟢[AI] KC生成标准SOP → TR-SUP生成培训内容

标注模式：🟡 → 🟢🟢🟢🟢 → 🟡[⇄] → 🔴 → 🟢
瓶颈率：22%（方向确认+最终选择两个握手点）

模板W4：客户投诉处理（B级→D级·分层响应）

🟢[AI] P-01检测投诉（评分≤2星自动标记）
🟢[AI] 分析投诉类型 + 客户历史 + 建议赔偿方案
🟡[⇄] 推送对应店长（一般投诉：4h内/紧急投诉：15min内）
🔴[人] 店长判断严重性 + 决定处理方式

若严重（评分1星/社交媒体曝光）：
  🟢[AI] 升级至GM-CHIEF + CHMN-BRAIN
  🔴[人] 决策层紧急判断
  🟢[AI] 生成对外沟通话术
  🔴[人] 悟空最终审定

标注模式（一般）：🟢 → 🟢 → 🟡[⇄] → 🔴
标注模式（严重）：🟢 → 🟢 → 🟡[⇄] → 🔴 → 🟢 → 🟡[⇄] → 🟢 → 🔴
瓶颈率（一般）：25% | 瓶颈率（严重）：43%

模板W5：月度绩效考核（B级·人主持AI辅助）

🟢[AI] PERF-MGR准备考核数据（完成率/满意度/出勤/7S评分）
🟢[AI] FIN-MGR补充财务维度数据
🟢[AI] HRD生成综合评分 + 个人发展建议
🟡[⇄] 推送各店长审阅（3天内）
🔴[人] 店长逐一面谈（30分钟/人）
🔴[人] 店长确认最终评分 + 下月目标
🟢[AI] PERF-MGR归档 + 更新Trust Score

标注模式：🟢🟢🟢 → 🟡[⇄] (3天) → 🔴🔴 → 🟢
瓶颈率：20%（面谈是人必须做的，不可压缩）

四、路由优先级与冲突处理

优先级排序

P=99 味藏总智能体 > P=98 董事长 > P=97 总经理 > P=96 超级店长Gate 
> P=95 赋能层 > P=94 日反馈 > P=92 SM-CHIEF > P=90 执行层

冲突处理规则

冲突类型 处理规则
两个智能体同时响应同一任务 Gate介入，按职责边界仲裁
紧急任务中断常规任务 P=94日反馈发出中断信号→当前智能体保存状态→响应紧急任务
任务路由到Trust Score<50的智能体 自动降级路由到备选智能体或升级一级
人直接下达的指令vs AI自动路由 人直接下达的指令优先（方向盘在人手中）

路由失败处理

任务路由失败
  ├─ 智能体离线 → 标记+通知维护 + 备选智能体接管
  ├─ 职责边界模糊 → Gate仲裁 + 临时分配 + 事后补充规则
  ├─ 首次出现的新任务类型 → 默认路由到超级店长Gate + 事后注册
  └─ 所有智能体都不匹配 → 升级至GM-CHIEF(总经理) + 建立新路由规则
