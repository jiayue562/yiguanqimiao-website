    五、六层上下文压缩系统

    5.1 Claude Code六种压缩路径

    ``` Claude Code上下文压缩系统：

    1. Auto Compact（自动摘要压缩） → 调用Claude对历史消息做摘要替换

    2. Session Memory Compaction（会话记忆沉淀） → 把重要信息沉淀到session memory

    3. Reactive Compact（被动紧急压缩） → 在prompt_too_long错误时被动触发

    4. Microcompact（增量压缩） → 更轻量的增量压缩，通过cache editing实现

    5. History Snip（边界截断） → 按snip boundary截断历史

    6. Context Collapse（上下文折叠） → 折叠大块上下文

    【关键洞察】 这些压缩路径不是互斥的——snip和microcompact可以同时运行， autocompact在它们之后触发。每一层都可能缩减上下文， 后面的层看到的是前面处理过的结果。 ```
