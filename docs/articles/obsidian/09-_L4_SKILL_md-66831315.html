技能定位

技能名称：L4·五行·分化 (Intelligence）

核心角色：五行人格OS的协调层（L4），负责五大智能体的统一调度与协同执行。

技术架构：

  • L1-L3：通用基础设施（记忆管理、工具调用、基础能力）
  • L4：五行·分化（协调层）← 当前技能包
  • L5-L7：应用层（个人成长、亲密关系、企业管理、教育培训、健康管理）

核心使命：

  • 🎯 统一调度：根据场景动态调用最优智能体组合
  • 🤝 协同执行：协调五行智能体协同工作，实现五行相生相克
  • 🔄 循环优化：通过五行通关点能量循环，实现能量升维
  • 📊 结果整合：整合五大智能体的输出，形成完整的诊断与转化方案

五大智能体

🌿 木智能体

核心特质：生长性 + 觉照性 + 仁德性 + 战略性 + 原创性

一心三界：

  • 一心：仁德本源（生长性 + 觉照性）
  • 三界：灵界（仁/傲慢）、心界（正直/愤懑）、身界（敏捷/僵直）

五行生克：

  • 相生：水生木、木生火
  • 相克：金克木、木克土
  • 化克为生：木克土 → 木生火 → 火生土

九层阶梯：从"仁德舒展"（1级）到"病态破坏"（9级）

核心功能：

  • 木行人格完整分析、诊断、转化
  • 木火共生关系诊断与优化
  • 木土生克关系优化
  • 木行九层阶梯升维指导

五行生克：

  • 相生：木生火、火生土
  • 相克：水克火、火克金
  • 化克为生：火克金 → 火生土 → 土生金

九层阶梯：从"明辨守礼"（1级）到"狂暴毁灭"（9级）

核心调度算法

步骤1：感知 → 提取关键词、推断五行、分析情感、识别场景

步骤2：识别 → 识别任务类型、选择智能体、设计路由策略

步骤3：路由 → 设计最优调度路径（单/双/全系统）

步骤4：调度 → 并行调用智能体、激活五行通关点、实时监控

步骤5：学习 → 分析执行结果、收集用户反馈、优化算法

五行通关点能量循环

核心公式：

木生火（忍辱）→ 火生土（不怨）→ 土生金（感恩）→ 金生水（找好处）→ 水生木（认不是）

通关点含义：

  • 木生火（忍辱）：木行人通过忍辱，滋养火行人的热情
  • 火生土（不怨）：火行人不怨怼，滋养土行人的稳定
  • 土生金（感恩）：土行人感恩，孕育金行人的决断
  • 金生水（找好处）：金行人找好处，引导水行人的智慧
  • 水生木（认不是）：水行人认不是，滋养木行人的仁德

步骤2：感知分析

协调中枢的感知引擎分析用户消息：

  • 提取关键词
  • 推断五行类型
  • 分析情感状态
  • 识别场景类型
  • 评估复杂度

步骤3：智能体调度

根据感知分析结果，协调中枢执行调度：

  • 低复杂度：单智能体路由
  • 中复杂度：双智能体路由

步骤4：协同执行

智能体并行执行，协调中枢监控执行过程：

  • 并行调用智能体

  • 实时监控状态
  • 异常处理

步骤5：结果整合

协调中枢整合智能体输出：

  • 收集智能体结果
