P-01问题路由引擎

来源：TXT v1.0 第四部分 L288-298 标签：#底盘层 #P-01 #问题路由 #基础设施

核心定义

P-01问题路由引擎是味藏AI+人组织架构的底盘层基础设施，负责所有问题的自动分流、升级和闭环追踪。P=95（仅次于董事长P=97和总经理P=96）。

核心功能

1. 三级分流机制

级别 路由方向 问题类型 闭环要求 触发条件
L1 组长自解 操作偏差/排班调整/物料补充 当日闭环 影响范围≤本组
L2 店长决策 SOP突破/人员违纪/客诉升级 24h闭环 跨组协调/需罚单
L3 公司支持 制度修订/资源不足/系统故障 72h响应 需跨部门/总部资源

2. 自动升级条件

  • 超时未闭环 → 自动升级一级
  • 重复发生≥3次 → 强制升级至L2
  • 安全/合规/舆情 → 直接升级至L3

3. 日反馈主动推送（实线）

P-01 → 每日定时采集POS/巡检/客诉/会员数据
     → 自动标记异常
     → 实时推送至对应岗位Skill
     → 触发日反馈生成流程

4. 问题被动升级（虚线）

问题发生 → 组长判断 → 无法解决 → L1路由到店长
                        → 店长判断 → 无法解决 → L2路由到公司
                                    → 公司判断 → L3响应

与其他Skills的接口

  • 上级：超级店长Skill（P=94宪法层·Gate4人机分工边界校验）
  • 同级：店长7S总控（P=92方法论层·日反馈数据源）
  • 下级：18个岗位任务Skills（问题来源+推送目标）

关键设计原则

  • P4·人做决策确认：P-01只做路由和推送，不替代人做判断
  • P5·赋能可归因：每次路由记录可追溯
  • 实线=主动推送（日反馈数据），虚线=被动响应（问题升级）
