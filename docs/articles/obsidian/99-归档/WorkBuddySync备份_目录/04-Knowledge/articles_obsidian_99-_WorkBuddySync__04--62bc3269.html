系统总体设计

三层架构定位

🧠 L7·凤脑OS（知识地基层·知识学习Engine）
    ↓ 信息流向：17篇文章 + 130+知识连接 + 象思维图谱
    
💫 L4·凤心OS（智能发动机层·1+5快慢思考）
    ├─ 快思考层：总调度 + 场景路由 + 引擎调度
    └─ 慢思考层：木/火/土/金/水五分体深度分析
    ↓ 信息流向：五行分布式诊断 + 冲突检测与融合
    
👁️ L1·L1·一心·统摄（应用接待层·觉知陪伴）
    └─ 握手包分发 + 八大应用场景路由

三层融合：信息体(L7) → 能量体(L4) → 物质体(L1)

设计哲学

  • 从知识→到智能→到应用：L7承载结构化知识，L4转化为智能决策，L1落地为用户体验
  • 快慢思考协同：快速路由（<100ms）vs 深度分析（可选慢思考）vs 实时融合
  • 分布式容错：单个分体故障不影响其他分体，自动降级到其他分体组合
  • 版本同步：L7/L4/L1三层版本号同步，冲突时以L7知识版本为source of truth

3.1 快思考层（总调度）

三维打分矩阵（响应时间<100ms）：

快思考诊断流程:
  Step 1: 情绪信号识别 (10ms)
    - 阴木信号(顶撞/不服): 8分
    - 阴火信号(急躁/嗔恨): 8分
    - 阴土信号(猜疑/依赖): 8分
    - 阴金信号(挑剔/完美主义): 8分
    - 阴水信号(恐惧/拖延): 8分
  
  Step 2: 行为信号识别 (10ms)
    - 性格/行为模式描述: +4分
    - 诉求类型明确: +2分
    - 多角色冲突: +3分
  
  Step 3: 场景归类 (20ms)
    - S0 (无信号) → 不激活或追问
    - S1 (单行信号) → 直接分发单行分体
    - S2 (双行信号) → 启动双行并行分析
    - S3+ (复杂信号) → 启动全系统并行分析
  
  Step 4: 引擎调度 (60ms)
    - 总分 ≥10 → 精准慢思考
    - 总分 7-9 → 双行分析
    - 总分 5-6 → 全面诊断
    - 总分 ≤4 → 追问1个精准问题再分发

响应时间: <100ms (p99)

3.3 冲突检测与融合机制

冲突识别规则：

冲突类型:
  Type A - 诊断不同
    场景: 土行建议"包容等待"，金行建议"立即决策"
    检测: 建议向量余弦相似度 < 0.5
    融合: 按用户五行主型权重融合 + Tech Lead override
    记录: 冲突日志 + 用户满意度反馈
  
  Type B - 优先级不同
    场景: 木行优先"成长"，土行优先"稳定"
    检测: 优先级打分差>3分
    融合: "成长+稳定"的平衡方案
  
  Type C - 阶段建议不同
    场景: 金行第一步"明确目标"，水行第一步"深入思考"
    检测: 步骤顺序不同
    融合: 按逻辑依赖关系排序

融合策略:
  - 置信度最高的分体意见权重提升40%
  - 次要分体意见作为"补充视角"标注
  - 冲突存在时，输出"融合建议"而非单一答案
  - 记录所有冲突便于后续优化

4.4 降级预案（容错）

5个降级场景：

故障场景 触发条件 降级策略 可用性
L7故障（知识库无法访问） 知识连接检索失败 使用缓存的核心规则 + 降级为L4/L1直接诊断 70%
L4单个分体故障 木/火/土/金/水某个分体响应超时 使用其他分体补充诊断 + 标注"部分分析" 80%
L4全体故障（智能发动机瘫痪） 快思考/五分体全部失效 降级为L1规则引擎 + 提供通用建议 50%
L1感知故障（无法理解用户意图） 意图识别准确率<50% 返回"我没理解，能再说一遍吗?" + 重试 60%
并发过高（响应时间>5秒） 并发用户>500 启用简化模式 + 仅返回L1握手包 + 排队 90%

降级触发与恢复：

自动降级规则:
  - 检测L7连接失败 → 立即切换缓存规则
  - 检测分体响应>300ms → 标注"降级模式" + 启用其他分体
  - 检测并发>阈值 → 启用简化握手包 + 排队通知

恢复条件:
  - 故障源修复 → 自动切回正常模式
  - 缓存更新成功 → 切回最新知识
  - 并发下降<阈值 → 回复完整功能

监控指标:
  - 降级率 <5% (正常运维)
  - 平均降级时长 <10分钟
  - 用户体验评分 >4.0/5.0 (即使降级)
