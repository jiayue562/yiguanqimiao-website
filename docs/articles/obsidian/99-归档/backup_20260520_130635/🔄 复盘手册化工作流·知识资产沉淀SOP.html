🔄 复盘手册化工作流·知识资产沉淀SOP v1.0

适用场景：当你完成了一个复杂的项目/系统搭建/深度学习后，需要将经验固化为可交付的标准化手册时，使用此工作流。

核心价值：把"我做过了"变成"别人也能做"——将个人经验升华为组织资产。

龙心OS路由：S7系统规划 → 🐉×📚×🔄（象思维+知识学习+知行合一）

总耗时：2-3小时（复杂项目）/ 1-1.5小时（中等项目）

二、触发条件（满足任意一条）

  • ✅ 完成了一个持续3天以上的复杂项目/系统搭建
  • ✅ 经历了一个从0到1的创新突破过程
  • ✅ 积累了他人无法轻易复现的经验（有踩坑、有调整）
  • ✅ 下次遇到类似事情，希望能做得更快更好
  • ✅ 需要把经验传递给他人（培训、交接、教学）

三、五步执行流程（详细SOP）

Phase 0 · 素材收集（20分钟）

目标：把散落的经验记录集中到一处，建立"原材料仓库"

执行清单：

  • [ ] 翻阅工作记忆（今日日志 + MEMORY.md），摘录关键里程碑
  • [ ] 整理聊天记录中的重要决策节点
  • [ ] 列出所有涉及的文件/工具/路径
  • [ ] 识别"做对了什么"（5-7项）
  • [ ] 识别"做错了什么/踩了哪些坑"（5-7项）

输出物：一份原始素材清单（可以是草稿，不需要工整）

Phase 1 · 骨架构建（15分钟）

目标：用象思维抓住核心结构，设计手册框架

执行步骤：

Step 1：定义手册目标读者

这份手册给谁看？他们的起点是什么？他们需要达到什么终点？

Step 2：提炼核心结构（象思维·物象→意象→原象）

  • 物象层：这件事涉及的具体步骤有哪些？
  • 意象层：这些步骤背后的逻辑是什么？
  • 原象层：这件事最本质的规律是什么（用一句话概括）？

Step 3：设计七大章节框架（标准模板）

1. 适用场景与前提条件
2. 整体架构图（全景视角）
3. 阶段化执行步骤（含时间估算）
4. MVP节点与验收标准
5. 做对了什么（可复制）
6. 避坑指南（做错了什么 + 正确做法）
7. 附录（模板/清单/资源）

输出物：手册目录大纲（1页）

Phase 2 · 血肉填充（45-90分钟）

目标：将原材料按框架组织成完整内容

核心原则（五色光思维）：

  • ⬜ 白光（事实层）：数据要准确，路径要具体，时间要真实
  • 🟥 红光（洞察层）：每个阶段的关键决策是什么？为什么这样做？
  • 🟨 黄光（价值层）：每步操作的价值是什么？不做的代价是什么？
  • 🟩 绿光（创新层）：有没有更好的做法？留出改进空间
  • 🟦 蓝光（风险层）：每步的常见错误是什么？如何规避？

每个阶段的内容要包含：

  1. 是什么（What）：这个阶段做什么
  2. 为什么（Why）：为什么在这个时间点做
  3. 怎么做（How）：具体操作步骤
  4. 验收标准（Check）：怎么判断这步做好了

MVP节点设计原则：

  • 每3-5天设一个MVP节点
  • 每个MVP节点必须有"可见可测"的输出物
  • MVP节点是暂停点——在此验收是否需要调整方向

输出物：完整手册草稿

Phase 3 · 质量验收（20分钟）

目标：用"陌生人视角"审查手册可用性

验收四问（MUST FOLLOW）：

问题1：零基础可执行？

一个从未接触过此类项目的人，按照这份手册，能否独立完成80%以上的工作？

问题2：踩坑已覆盖？

你亲自踩过的所有坑，手册里都有对应的避坑方法吗？

问题3：时间估算真实？

每个阶段的时间估算，是基于你的实际耗时，而不是理想状态？

问题4：MVP节点可验收？

每个MVP节点的验收标准，是否具体到"做了X就算通过"？

如果有任何"不是"→回到Phase 2补充完善

Phase 4 · 双路径存储（10分钟）

目标：将手册存入两个必须的位置
