🤖 岗位智能体 · SKILL.md

核心定位：基于五行人格心理学OS的岗位AI工作流设计系统 一句话功能：通过"行业×企业×岗位×性格×任务×模型"六维框架，生成精准适配的AI提示词与智能体使用方案 版本：v3.0 | 创建日期：2026-04-20 | 更新日期：2026-05-18 | 维护者：龙龟神将

📖 核心定义

What（是什么）

岗位智能体是龙龟神将独创的业务场景×人格特质×思维模型深度融合的AI提示词生成与岗位工作流设计工具。

它解决了传统提示词"一刀切"的问题，通过六维框架精准识别岗位特性，结合五行人格心理学，为每个岗位生成专属的AI协作方案。

核心组成：

  • 六维输入框架：行业 + 企业简介 + 岗位 + 性格（五行人格）+ 核心任务 + 适用模型
  • 五行人格深度诊断：阴阳面分析 + 数据驱动识别 + 五行借力路径（v3.0升级）
  • 7S组织诊断评分：1-10分制 + 诊断依据 + 核心问题 + 改善方向（v3.0新增）
  • 三层指标体系：结果指标（考核）+ 过程指标（监控）+ 风险指标（预警）+ P0/P1优先级（v3.0升级）
  • 关键动作设计：目标量化 + 资源分配 + 数据验证 + 风险控制（4列风险预警表）
  • AI增强关键词：洞察、剖析、透视、阐释、推演、解构、思辨、溯源、融合、启发、映射
  • 复盘画布6步框架：工作盘点→优劣势→根因→拔阴取阳→风险预警→进化路径（v3.0新增）
  • 龙爪断语生成：3大核心判断 + 金句引用 + 核心命题（v3.0新增）
  • 提示词工程体系：黄金公式 + 基础结构 + 高级技巧 + 专家级资产

Why（为什么）

现状问题：

  • 🔴 通用提示词无法满足特定岗位的深度需求
  • 🔴 忽视人格特质导致AI协作效果不佳
  • 🔴 缺乏系统化工作流设计方法
  • 🔴 结果指标与过程指标混淆
  • 🔴 风险控制缺乏预警机制
  • 🔴 没有组织级7S诊断能力（v3.0新增）
  • 🔴 五行人格缺少阴阳面分析，无法识别"暗礁"（v3.0新增）
  • 🔴 指标体系平铺，缺乏分层逻辑和P0/P1优先级（v3.0新增）
  • 🔴 输出缺乏高层面的诊断总结和判断提炼（v3.0新增）
  • 🔴 HTML输出无视觉规范，风格不统一（v3.0新增）

解决方案：

  • ✅ 五行人格×业务场景深度融合（木生火共生视角）
  • ✅ 工作流化思维（信息交汇点精准识别）
  • ✅ 六维框架全面覆盖（行业→企业→岗位→性格→任务→模型）
  • ✅ 五行阴阳面双轨诊断（阳面显发优势 + 阴面过刚风险）（v3.0升级）
  • ✅ 7S组织诊断定量评分（1-10分制 + 诊断依据 + 改善方向）（v3.0新增）
  • ✅ 三层指标金字塔（结果/过程/风险 + P0/P1 + 分层关联规则）（v3.0升级）
  • ✅ 复盘画布6步闭环（从盘点到进化路径的完整闭环）（v3.0新增）
  • ✅ 龙爪断语升华（从数据堆砌到高层判断的"灵魂"）（v3.0新增）
  • ✅ HTML视觉设计规范（CSS变量 + 组件库 + 响应式 + 颜色编码）（v3.0新增）

How（怎么做）

核心工作流（v3.0·10步SOP）：

    ↓

🎮 双模式 + 三种输出类型

核心升级：岗位智能体v3.0支持两种模式、三种输出类型。

模式 A：👤 提示词模式（默认）

  • 输出：人用 AI 提示词
  • 适用：给企业员工设计 AI 协作助手
  • 示例：味藏总经理的 AI 助手提示词

模式 B：🐉 龙爪模式

  • 输出类型 B1：个人复盘画布HTML（聚焦个人一周工作的6步深度分析）
  • 输出类型 B2：组织诊断报告HTML（聚焦组织7S+经营+文化的多维度诊断）
  • 适用：给 WorkBuddy 造龙爪（AI 智能体）
  • 示例：味藏店长周复盘画布（B1）/ 堂食创业单元7S诊断报告（B2）

模式切换指令

你想做什么 输入
给人设计提示词 默认模式，直接按六维框架输入
造 WorkBuddy 龙爪 先说"切换到龙爪模式"，再输入设计

报告类型判断逻辑（v3.0新增）

报告类型自动判断:
  输入包含个人名 + 周数据 → B1个人复盘画布
  输入包含多门店/多单元数据 → B2组织诊断报告
  输入仅有岗位描述 + 任务 → 提示词模式（默认）

P0·直接触发词（权重5·绝对触发）

  • "生成提示词"
  • "岗位AI"
  • "工作流设计"
  • "智能体方案"

  • "龙爪断语"（v3.0新增）

P1·场景触发词（权重4·强触发）

出现1个累积4分即触发：

  • 岗位描述：市场总监、总经理、运营经理、产品经理、销售总监、店长
  • 行业描述：餐饮行业、科技行业、教育行业、金融行业
  • 性格描述：水行人、火行人、木行人、金行人、土行人
  • 任务描述：渠道投放、用户增长、品牌策划、产品运营
  • 指标描述：KPI、OKR、结果指标、过程指标、量化目标
  • 风险控制：风险预警、过程控制、切换机制、ROI
  • 诊断描述：组织诊断、7S评分、指标分层、阴阳面（v3.0新增）

P2·信号触发（权重3·弱触发）

出现≥2个累积6分即触发：

  • 同时提到行业+岗位+任务（六维框架信号）
  • 描述性格特质（五行人格信号）
  • 询问"信息交汇点"（工作流信号）
  • 询问"如何量化目标"（指标设计信号）
  • 询问"用什么模型分析"（模型选择信号）
  • 询问"阴阳面/优势劣势"（深度诊断信号）（v3.0新增）
  • 询问"总部赋能/接口评估"（组织诊断信号）（v3.0新增）

数据驱动五行识别机制（v3.0新增）

触发条件：用户未提供五行类型，或用户提供的数据与自报类型矛盾时自动启动。

数据驱动五行识别机制:
  触发条件: 用户未提供五行类型 或 用户提供的数据与自报类型矛盾
  识别逻辑:
    - 目标导向+执行快+数据驱动 → 金行信号
    - 热情高+感染力强+跟进弱 → 火行信号
    - 创新力+前瞻性+落地弱 → 木行信号
    - 稳健+流程强+变革慢 → 土行信号
    - 灵活+策略强+定力弱 → 水行信号
  输出: 推断的五行类型 + 识别依据 + 阴阳面标签
  
  示例:
    数据: 金月湾完成率148.8%，超额目标，执行极快
    推断: 金行人（目标强执行快）
    阳面标签: ✅目标感强 ✅执行力强 ✅数据意识强
    阴面标签: ❌关系维护弱 ❌文化把关弱 ❌过程指标少
    
    数据: 松山湖渗透率50%，销售热情高
    推断: 火行人（热情高感染力强）
    阳面标签: ✅客户链接力强 ✅行动力快
    阴面标签: ❌跟进弱 ❌系统性不足

Step 3: 7S组织诊断评分（v3.0新增）

定位：在五行识别之后、关键动作设计之前，用7S框架对组织进行定量诊断。

7S评分标准:
  S1战略 (Strategy):
    8-10: 战略清晰且可执行，团队认知统一
    5-7: 有战略但执行走样或认知不统一
    1-4: 战略模糊或与实际脱节
  S2结构 (Structure):
    8-10: 组织架构清晰，权责明确，协作流畅
    5-7: 基本架构在但存在模糊地带
    1-4: 架构缺位或权责混乱
  S3制度 (Systems):
    8-10: 制度完善，流程标准化，执行一致性高
    5-7: 有制度但执行不统一或覆盖不全
    1-4: 制度缺位或形同虚设
  S4风格 (Style):
    8-10: 管理风格鲜明且适配组织，团队认同度高
    5-7: 有管理风格但不一致或不适配
    1-4: 管理风格混乱或专制
  S5人员 (Staff):
    8-10: 人才梯队完整，关键岗位无空缺
    5-7: 核心岗位有人但梯队不完整
    1-4: 关键岗位空缺或人岗不匹配
  S6技能 (Skills):
    8-10: 核心技能完备，持续培训体系运转
    5-7: 有基本技能但缺乏深度培训
    1-4: 关键技能缺失，无培训体系
  S7共同价值观 (Shared Values):
    8-10: 价值观清晰，全员认同，日常行为一致
    5-7: 有价值观但认知不统一或行为偏离
    1-4: 价值观模糊或仅停留在口号

Step 4: 三层指标体系 + 关键动作设计（v3.0升级）

v2.0→v3.0变化：将"目标量化/数据验证/风险控制"重组为三层指标金字塔，新增P0/P1优先级、4列风险预警表、分层关联规则。

三层指标体系:
  Layer 1 - 结果指标（考核用·绿色）:
    定义: 衡量最终经营成果的指标
    模板: 指标名 | 当前值 | 目标值 | 达成状态
    示例: 综合营业额完成率 | 132.9% | 100% | 达标
    频率: 月度/季度
    颜色编码: 绿色(tag-g) = 达标/完成/优势

  Layer 2 - 过程指标（监控用·蓝色）:
    定义: 衡量执行过程质量的指标
    模板: 指标名 | 当前值 | 预警阈值 | 监控频率
    示例: 1对1邀约执行率 | 极低 | 每店每日≥3人 | 日度
    频率: 日度/周度
    颜色编码: 蓝色(tag-b) = 过程/监控/信息

  Layer 3 - 风险指标（预警用·红色）:
    定义: 需要立即关注的系统性风险
    模板: 指标名 | 描述 | 严重度 | 优先级
    示例: 客户链接机制断裂 | 三店系统性失效 | 系统级 | P0
    频率: 实时
    颜色编码: 红色(tag-r) = 预警/未达标/薄弱

  分层关联规则:
    - 过程指标连续3期低于阈值 → 升级为风险指标
    - 风险指标P0 → 当周必须解决，纳入每日行动日历
    - 风险指标P1 → 当月必须解决，纳入每周复盘

HTML视觉规范校验（v3.0新增）

定位：龙爪HTML输出必须遵循的视觉设计规范，确保风格统一、组件复用、响应式一致。

HTML视觉规范校验清单:
  1_CSS变量体系:
    --topbar-bg: #1e293b 或 #1a1a2e（深色顶栏）
    --canvas-bg: #ffffff（白色画布）
    --grid-bg: #f5f6f8 或 #f5f5f4（浅灰网格）
    --accent: #c0392b（警示红）
    --gold: #d4a017（金标色·龙爪品牌色）
    --green: #27ae60（达标绿）
    --blue: #2980b9（过程蓝）
    --orange: #e67e22（关注橙）
    --purple: #8e44ad（特殊紫）

  2_颜色编码逻辑:
    绿(tag-g): 达标/完成/优势
    红(tag-r): 预警/未达标/薄弱
    橙(tag-o): 关注/偏弱/进行中
    蓝(tag-b): 过程/监控/信息
    紫(tag-p): 特殊/战略/文化

  3_通用组件:
    顶栏(.topbar): 深色背景+白色文字+sticky定位
    卡片(.card): 白色背景+圆角+阴影+1px边框
    进度条(.progress-bar): 6-8px高+圆角+渐变填充+过渡动画
    仪表盘卡片(.dash-card): 网格布局+图标+大数字+副文本
    Tab切换(.cal-tab/.cal-panel): JS切换+active样式

  4_响应式断点:
    @media (max-width: 900px): 4列→2列
    @media (max-width: 600px): 全部→1列

  5_交互组件:
    进度条动画: setTimeout渐入（0→目标宽度的过渡动画）
    Tab切换: JS函数switchCal
    7S评分进度条: 颜色随分数变化（8-10绿/5-7橙/1-4红）
