测试场景一：客户流失分析

输入

情境：某电商平台发现月活用户流失率上升，构建逻辑树诊断原因

期望输出

【逻辑树 · 客户流失原因分析】

问题树结构：

              [客户流失率上升]
                    /    \
              用户侧     平台侧
              /    \     /    \
          新客   老客   产品   服务
           |      |      |      |
         获取   留存   体验   售后
         成本高  活跃低  浏览   物流
                      体验差  投诉

MECE检验：
✓ 各分支相互独立，无重叠
✓ 分支覆盖用户侧+平台侧全部
✓ 可进一步分解到可执行层级

验证标准

  • [ ] 问题树结构清晰
  • [ ] MECE检验通过
  • [ ] 分解到可执行层级

测试场景二：成本超支分析

输入

情境：某建筑项目成本超支30%，用逻辑树分解原因

期望输出

【逻辑树 · 项目成本超支分析】

              [成本超支30%]
                    /    \
              直接成本     间接成本
              /    \       /    \
          材料   人工   管理   风险
           |      |      |      |
        涨价   窝工   协调   设计
        损耗   效率低  沟通   变更
                       低效

进一步分解（直接成本）：
- 材料涨价：市场波动、供应链问题
- 材料损耗：管理不当、技术问题
- 人工窝工：天气、调配、配合
- 人工效率：技能、培训、激励
