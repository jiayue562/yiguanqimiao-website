📖 核心定义

What（是什么）

上下文工程是管理AI跨会话任务连续性的专业智能体，通过"上下文快照+接力机制"实现任务状态的保存与恢复。

核心能力：

  1. 智能检测 - 自动识别需要保存上下文的场景
  2. 快照生成 - 结构化记录任务完整状态
  3. 状态恢复 - 新会话快速恢复工作脉络
  4. 接力协同 - 支持多会话间的任务延续

Why（为什么）

现状痛点：

  • 🔴 会话过期导致任务状态丢失
  • 🔴 长周期项目难以持续跟踪
  • 🔴 复杂任务中断后无法接续
  • 🔴 AI"答非所问"（缺乏上下文背景）

解决方案：

  • ✅ 自动检测保存时机
  • ✅ 结构化快照记录
  • ✅ 一键恢复任务状态
  • ✅ 跨会话无缝衔接

How（怎么做）

一句话工作流：

检测触发 → 收集状态 → 生成快照 → 存储备份 → 检测恢复 → 加载状态 → 继续执行

⚡ 触发条件

P0·直接触发词（权重5·绝对触发）

  • "保存上下文"、"生成快照"、"任务中断"
  • "恢复任务"、"上下文工程"、"context快照"
  • "保存进度"、"任务接力"、"继续上次"

P1·场景触发词（权重4·强触发）

  • 长周期项目需要中断
  • 会话即将过期
  • 复杂任务阶段完成
  • 需要跨会话继续工作

P2·信号触发（权重3·弱触发）

  • 对话持续超过30分钟
  • 涉及多个文件操作
  • 用户说"下次继续"
  • 任务明显未完结

P3·条件触发（权重2·条件触发）

  • 检测到龙心OS S5/S7场景（重大决策/系统规划）
  • 任务涉及3个以上Skills协同

🎯 核心流程

阶段一：触发检测

自动触发场景：

  1. 任务执行时间 > 30分钟
  2. 涉及多文件操作（≥3个文件）
  3. 用户明确说"保存进度"/"下次继续"
  4. 会话即将过期（系统提示）
  5. 龙心OS S5/S7场景（重大决策/系统规划）

手动触发方式：

  • 用户说："保存上下文" / "生成快照"

阶段二：状态收集

收集维度：

项目信息:
  - 项目名称: string
  - 任务类型: enum(开发/写作/分析/规划/其他)
  - 优先级: enum(P0/P1/P2/P3)
  - 开始时间: datetime

进度状态:
  - 已完成事项: array
  - 累计进度: percentage
  - 当前阶段: string

关键决策:
  - 决策内容: string
  - 决策依据: string
  - 决策时间: datetime
  - 相关文件: array

待办事项:
  - 高优先级: array
  - 中优先级: array
  - 低优先级: array

AI状态:
  - 已加载Skills: array
  - 激活引擎: array
  - 知识库引用: array

风险提示:
  - 已知问题: array
  - 解决方案: array

阶段三：快照生成

文件命名规则：

context-{项目名}-{YYYYMMDD}-{HHMMSS}.md

阶段四：存储备份

存储策略： | 优先级 | 存储位置 | 说明 | |--------|---------|------| | P0紧急 | IMA笔记 + 本地 | 移动端可访问，双保险 | | P1重要 | IMA笔记 + Obsidian | 短期记忆+长期知识库 | | P2普通 | 本地文件 | 仅本地存储 |

存储路径：

  • IMA笔记：上下文快照/{项目名}/
  • Obsidian：00-总索引与导航/上下文快照/
  • 本地：~/.workbuddy/context-snapshots/

阶段五：恢复检测

新会话启动时：

  1. 检测是否有历史快照
  2. 判断快照时效性（<7天为有效）
  3. 列出可用快照供选择

恢复方式：

  • 单快照 → 自动加载
  • 多快照 → 列表选择
  • 过期快照 → 提示确认

阶段六：状态恢复

恢复流程：

加载快照文件
    ↓
解析项目信息
    ↓
恢复AI记忆状态（加载Skills、激活引擎）
    ↓
展示已完成事项和待办事项
    ↓
提示关键决策和风险
    ↓
询问是否继续执行

阶段七：继续执行

恢复后状态：

  • AI已"回忆"起任务背景
  • 待办事项已加载
  • 可以继续上次的工作

🚫 边界条件

不触发场景

  • ✅ S0简单问答（单次完成）
  • ✅ 纯信息查询（无需延续）
  • ✅ 已明确完结的任务

异常处理

异常情况 处理方式
存储失败 提示用户，尝试备用存储路径
快照损坏 尝试读取上一个有效快照
信息不完整 提示补充关键信息后再生成
用户取消 记录日志，停止生成

💡 核心原则

  1. 双向友好 - 既方便人类理解，又便于AI检索
  2. 自动化优先 - 智能检测触发，减少人工干预
  3. 多层级存储 - IMA短期+Obsidian长期，双重保障
  4. 渐进式恢复 - 从概览到细节，逐步加载状态
  5. 持续进化 - 基于使用反馈优化快照模板
