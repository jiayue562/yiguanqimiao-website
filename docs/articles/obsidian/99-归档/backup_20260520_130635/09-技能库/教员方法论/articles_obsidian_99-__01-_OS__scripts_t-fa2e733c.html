测试用例一：餐饮企业问题诊断

场景描述

某连锁餐饮企业面临"门店盈利下滑"问题，需要系统诊断根本原因。

测试步骤

Step 1: 矛盾定位

输入: 门店盈利下滑现象 预期输出:

  • 根本矛盾识别：不是简单的"客流减少"，而是三维结构失衡
    • 物质体：食材成本上涨15%，但定价策略未调整
    • 能量体：员工流失率30%，服务品质下降
    • 信息体：缺乏数字化运营能力，决策滞后

验证标准:

  • [ ] 是否识别出三维结构失衡？
  • [ ] 是否发现15%阈值突破？
  • [ ] 是否超越表面现象（客流）找到根本原因？

Step 2: 结构分层

输入: 上述三维矛盾 预期输出:

                    [核心结论：运营体系结构性失衡]
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   [成本结构失衡]      [服务能力下降]      [决策机制滞后]
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │         │        │         │        │         │
[食材]   [人工]   [培训]   [激励]   [数据]   [流程]

验证标准:

  • [ ] 金字塔结构是否符合MECE原则？
  • [ ] 各层级之间是否有逻辑支撑关系？
  • [ ] 结论是否有数据支撑？

Step 3: 逻辑验证

输入: 上述金字塔结构 预期输出:

  • 假设验证：
    • 假设1：食材成本上涨是主因 → 验证：占比从30%→35%，但毛利率仍可达60%
    • 假设2：员工流失是主因 → 验证：相关系数0.82，强相关
    • 假设3：数字化滞后是主因 → 验证：决策周期7天→竞品1天
  • 核心结论：员工流失导致服务质量下降是主要矛盾

验证标准:

  • [ ] 是否进行了假设驱动分析？
  • [ ] 是否有数据支撑（相关系数>0.7）？
  • [ ] 逻辑链条是否自洽？

Step 4: 实践闭环

输入: 核心结论 预期输出:

  • Plan：3个月降低员工流失率至15%以下
  • Do：优化薪酬结构+完善培训体系+建立晋升通道
  • Check：每月监测流失率、顾客满意度
  • Act：根据反馈调整激励方案

验证标准:

  • [ ] PDCA各环节是否完整？
  • [ ] 是否有明确的量化指标？
  • [ ] 是否有时间节点？

测试用例二：培训课程设计

场景描述

为中层管理者设计"高效团队建设"培训课程。

测试步骤

Step 1: 需求分析（矛盾定位）

输入: 中层管理者团队建设能力现状 预期输出:

  • 根本矛盾：管理者有意愿但缺乏系统方法论
    • 物质体：缺乏实用工具和模板
    • 能量体：时间碎片化，难以深度学习
    • 信息体：知识零散，缺乏体系

验证标准:

  • [ ] 是否准确识别学员现状与期望的差距？
  • [ ] 三维分析是否到位？

Step 2: 内容架构（金字塔原理）

输入: 培训需求 预期输出:

                    [核心结论：成为高效团队领导者]
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   [识人用人]          [激励沟通]          [冲突管理]
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │         │        │         │        │         │
[五行人格] [岗位匹配] [动机理论] [反馈技巧] [矛盾定位] [调解方法]

验证标准:

  • [ ] 模块之间是否MECE？
  • [ ] 是否符合成人学习规律？
  • [ ] 时长分配是否合理？

Step 3: 方法验证（金线原理）

输入: 教学设计 预期输出:

  • 教学方法组合：
    • 理论讲授30%（概念框架）
    • 案例分析40%（真实场景）
    • 实操演练20%（工具应用）
    • 小组讨论10%（经验分享）
  • 效果验证：课后30天行为改变跟踪

验证标准:

  • [ ] 教学方法是否多样？
  • [ ] 是否有效果验证机制？
  • [ ] 是否符合70-20-10学习法则？

Step 4: 实施闭环（PDCA）

输入: 完整课程设计 预期输出:

  • Plan：课前调研→课件制作→物料准备
  • Do：按计划执行教学
  • Check：课后满意度调查+知识测试+行为跟踪
  • Act：根据反馈优化课程

验证标准:

  • [ ] 课前准备是否充分？
  • [ ] 课后评估是否完整？
  • [ ] 是否有持续改进机制？

测试用例三：企业战略规划

场景描述

一家成长期科技企业需要制定3年战略规划。

测试步骤

Step 1: 战略环境分析（矛盾定位）

输入: 企业现状与市场环境 预期输出:

  • 阶段定位：扩张期→成熟期过渡
  • 根本矛盾：快速增长与组织能力不匹配
    • 物质体：资金充足但投资效率待提升
    • 能量体：团队扩张但文化稀释
    • 信息体：战略清晰但执行走样

验证标准:

  • [ ] 阶段定位是否准确？
  • [ ] 六维分析是否全面？
  • [ ] 是否识别出真正的战略瓶颈？

Step 2: 目标体系（金字塔原理）

输入: 战略分析结果 预期输出:

                    [愿景：成为行业领导者]
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   [市场地位]          [盈利能力]          [组织能力]
   目标：前三          目标：利润率20%      目标：文化落地
        │                  │                  │
   ┌────┴────┐        ┌────┴────┐        ┌────┴────┐
   │         │        │         │        │         │
[Y1]     [Y2]     [Y1]     [Y2]     [Y1]     [Y2]

验证标准:

  • [ ] 目标是否符合SMART原则？
  • [ ] 各层级目标是否相互支撑？
  • [ ] 资源是否匹配？

Step 3: 可行性验证（金线原理）

输入: 战略目标 预期输出:

  • 关键假设验证：
    • 市场增长率假设：8%（行业数据支撑）
    • 技术领先优势假设：保持18个月（研发能力支撑）
    • 人才供给假设：关键岗位可招聘到位（薪酬竞争力支撑）
  • 风险识别：核心技术人员流失、竞争加剧

验证标准:

  • [ ] 假设是否有数据支撑？
  • [ ] 风险识别是否全面？
  • [ ] 应对策略是否可行？

Step 4: 执行体系（PDCA）

输入: 战略方案 预期输出:

  • 战略地图：财务→客户→流程→学习成长
  • 关键举措：
    • Y1：组织能力建设、核心产品升级
    • Y2：市场扩张、生态构建
    • Y3：行业整合、平台化转型
  • 监控机制：季度经营分析会、战略仪表盘

验证标准:

  • [ ] 战略地图是否完整？
  • [ ] 关键举措是否具体可执行？
  • [ ] 监控机制是否有效？
