KJ法 · 信息分组与创意涌现

Skill版本: v1.0 创建时间: 2026-04-10 维护者: 龙龟神将 核心分类: 创意思考 · 创意涌现

📌 一句话定义

KJ法（亲和图法）由川喜田二郎创立，是一种将繁杂的碎片化信息进行直觉分组，从而产生全新创意或发现隐藏问题的发散性思维工具。

🎯 核心定义

1.1 理论基础

"直觉先行"——让潜意识发挥作用，不要过早进行理性分析

1.2 核心特征

  • 自下而上分组
  • 依赖直觉判断
  • 适合质性数据

📊 详细解析

2.1 操作步骤

┌─────────────────────────────────────────────────────────────┐
│                    KJ法六步法                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1️⃣ 明确主题                                           │
│  ─────────────────────────────────────                      │
│  确定要分析的中心主题                                         │
│                                                             │
│  Step 2️⃣ 制作卡片                                          │
│  ─────────────────────────────────────                      │
│  • 将收集到的每一个想法、事实、意见或问题                      │
│  • 分别写在一张卡片上                                         │
│  • 语言要简洁、直观                                           │
│                                                             │
│  Step 3️⃣ 直观分组                                          │
│  ─────────────────────────────────────                      │
│  • 将所有卡片散开摊在桌面上                                   │
│  • 反复浏览，依靠直觉                                         │
│  • 将觉得"好像有联系"的卡片放在一起                           │
│  ⚠️ 这是自下而上的过程，无需预设分类标准                       │
│                                                             │
│  Step 4️⃣ 命名标签                                          │
│  ─────────────────────────────────────                      │
│  • 为每个卡片组找到能概括核心内容的标签（标题）                 │
│  • 写在另一张卡片上，置于该组之上                              │
│                                                             │
│  Step 5️⃣ 构建层级                                          │
│  ─────────────────────────────────────                      │
│  • 如果分组较多                                               │
│  • 可以继续对分组卡片进行归纳                                   │
│  • 形成更高层级的标签                                          │
│  • 最终形成树状结构                                            │
│                                                             │
│  Step 6️⃣ 阐释与洞察                                         │
│  ─────────────────────────────────────                      │
│  • 分析各组之间的关系                                         │
│  • 解读标签的含义                                             │
│  • 发现问题的结构                                             │
│  • 基于此产生新创意或解决方案                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

案例：分析用户对APP的负面评价

优先级1：修复视觉交互体验
