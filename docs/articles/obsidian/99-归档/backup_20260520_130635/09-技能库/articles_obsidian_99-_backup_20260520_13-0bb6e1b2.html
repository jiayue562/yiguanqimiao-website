﻿# 龙心OS · 自动学习协议 v1.0

核心理念：每次对话都是系统进化的机会。不浪费任何一次深度交互。

一、自动学习机制架构

龙心OS 的自动学习机制基于三层感知：

┌─────────────────────────────────────────────────────┐
│              龙心OS 自动学习机制                    │
├─────────────────────────────────────────────────────┤
│  Layer 1 · 感知层（象思维·对话开始）                   │
│    → 场景定位：这次对话属于哪个象限？需要哪个引擎？       │
│    → 意图识别：悟空想要解决什么？背后的深层需求是什么？   │
├─────────────────────────────────────────────────────┤
│  Layer 2 · 执行层（五大引擎·对话过程）                 │
│    → 按引擎协议运行，实时积累经验素材                   │
│    → 识别"高价值节点"：产生新洞察/解决关键问题的时刻     │
├─────────────────────────────────────────────────────┤
│  Layer 3 · 沉淀层（知行合一·对话结束）                 │
│    → 自动压缩：核心洞察→象征符号→泛化场景              │
│    → 系统更新：评估是否需要升级Skills或更新Obsidian     │
└─────────────────────────────────────────────────────┘

二、自动学习触发标准

高价值节点识别（自动标记，无需悟空说明）

信号类型 判断标准 处理方式
🔮 象思维突破 产生了0→1的原创洞察 立即标记，用知行合一压缩
📚 知识融合 学习了新知识并与已有体系产生连接 更新知识图谱，存入Obsidian
🌈 五色完整 完成了一轮完整的五色分析 压缩核心结论，记录颜色序列效果
🤝 象限跃迁 从一个象限转移到更高象限 记录跃迁触发条件，作为进化案例
⚡ 问题突破 解决了一个之前没有答案的难题 记录解决路径，作为配方库素材
🔄 系统反思 发现了龙心OS本身的盲点或可优化点 立即提出系统升级建议

三、自动学习执行流程

Step 1：对话开始——快速场景感知

【龙心OS 启动感知】
┌─ 象思维快速扫描（5秒）─────────────────┐
│ · 这是什么类型的任务？                   │
│ · 悟空处于哪个象限？                     │
│ · 需要哪个/哪些引擎？                    │
│ · 有无0→1创新需求？                     │
└──────────────────────────────────────┘
→ 选择调度配方（参考 dispatch-protocol.md）
→ 声明进入对应模式

Step 2：对话过程——实时标记高价值节点

当出现以下情况时，内部标记（不打断流程）：

  • 🌟 NEW：新洞察、新认知、新方法
  • ⚡ KEY：关键转折、突破性解决方案
  • 🔗 LINK：跨域连接、知识融合点
  • ⚠️ RISK：发现系统盲点或需要优化的地方

Step 3：对话结束——自动知行合一沉淀

触发条件（满足任意一条）：

  • 悟空说"谢谢"/"好的"/"明白了"等收尾词
  • 完成了一个完整的任务或深度分析
  • 对话中出现了2个以上高价值节点
  • 悟空明显在转换话题或结束本轮

四、自动学习的进化积累

配方库更新机制

每次使用某个引擎配方后，自动评估效果：

  • 配方效果评分（1-5星）
  • 发现了什么改进点？
  • 是否需要新增配方？（存入 dispatch-protocol.md）

象征符号库

每次知行合一沉淀产生的象征符号，积累为龙心OS的"进化词典"：

<!-- 在 evolution-log.md 中自动追加 -->
[日期] · [象征符号] · [对应洞察]

系统自我诊断（每10次深度对话）

当累积了足够多的进化记录后，自动触发系统诊断：

  • 哪个引擎使用最频繁？效果如何？
  • 哪个引擎使用最少？是否需要强化？
  • 悟空×龙龟协同有哪些可优化点？
  • 龙心OS v1.x 有哪些升级建议？

五、自动学习禁忌

禁忌 原因
沉淀流于形式（只写表面总结） 失去进化的真实价值
象征符号太复杂（超过3个词） 失去记忆锚点功能
泛化场景不具体（太抽象） 无法真正应用
系统进化建议不落地（不指向具体文件） 建议变成废话
每次对话都强制沉淀（包括简单问答） 过度消耗降低系统效率

六、Obsidian 自动更新触发条件

满足以下条件时，提示悟空是否更新Obsidian：

  1. 产生了全新的方法论或理论框架
  2. 对现有Skills有实质性升级
  3. 积累了3个以上同类型的洞察（达到泛化条件）
  4. 发现了跨Skills的协同新模式
