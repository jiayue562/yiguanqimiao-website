---

📌 一句话定义

PREP法是一种"先说结论"的高效逻辑表达框架，通过Point（结论）→ Reason（理由）→ Example（事例）→ Point（结论）的结构，确保信息传达的高效与清晰。

---

🎯 核心定义

1.1 名称解释

PREP = Point（结论）+ Reason（理由）+ Example（事例）+ Point（结论）

1.2 核心价值

> "在经营活动等时间宝贵的场合，必须先说结论，再阐述理由和事例，最后重申结论，以确保信息传达的高效与清晰。"

---

📊 详细解析

2.1 PREP结构图示

``` ┌─────────────────────────────────────────────────────────────┐ │ PREP法结构 │ ├─────────────────────────────────────────────────────────────┤ │ │ │ P ── Point（结论） │ │ │ │ │ │ 开门见山，清晰、简洁地陈述你的核心观点或主张 │ │ │ │ │ ↓ │ │ R ── Reason（理由） │ │ │ │ │ │ 提供支持结论的1-3个核心理由 │ │ │ 确保理由与结论有直接的逻辑联系 │ │ │ │ │ ↓ │ │ E ── Example（事例） │ │ │ │ │ │ 用具体的事实、数据、案例或个人经历来印证理由 │ │ │ 使抽象的理由变得生动可信 │ │ │ │ │ ↓ │ │ P ── Point（结论） │ │ │ │ │ │ 再次强调核心结论，呼应开头，形成闭环 │ │ │ 可在此处提出行动呼吁 │ │ │ │ │ │ └─────────────────────────────────────────────────────────────┘ ```

2.2 操作步骤

| 步骤 | 要素 | 操作要点 | |------|------|---------| | 1 | P（结论） | 开门见山，清晰简洁陈述核心主张 | | 2 | R（理由） | 提供1-3个核心理由，与结论直接关联 | | 3 | E（事例） | 用事实、数据、案例印证理由 | | 4 | P（结论） | 重申核心结论，形成闭环 |

2.3 应用案例

案例：提案削减某项市场活动的预算

``` 【P - 结论】 "我建议将本次活动的预算削减30%。"

【R - 理由】 "原因有两点： ① 该活动历史ROI持续低于平均水平 ② 当前公司现金流紧张，需要优先保证核心业务投入"

【E - 事例】 "事实依据： ① 去年同季度，该活动带来每元收入是2元，而行业平均是5元 ② 本季度公司现金流预测为负，如不削减非核心支出，可能影响Q3的研发投入"

【P - 结论】 "因此，建议削减30%预算，将资源集中到ROI更高的渠道和核心业务上。" ```

2.4 注意事项

| 要点 | 说明 | |------|------| | 结论先行的勇气 | 必须有勇气先抛出结论，再组织论据 | | 理由要精炼 | 1-3个核心理由，不要堆砌 | | 事例要贴切 | 紧扣理由，提供有力支撑 | | 重申结论要坚定 | 呼应开头，形成闭环 |
