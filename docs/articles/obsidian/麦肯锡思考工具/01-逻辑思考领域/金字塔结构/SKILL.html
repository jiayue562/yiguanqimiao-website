📌 一句话定义

金字塔结构是一种"结论先行"的论据支撑体系，通过逐层列举论据以支撑核心论点，确保逻辑清晰、说服力强。

---

🎯 核心定义

1.1 核心思想

> "结论先行"——在沟通开始就清晰提出中心思想，然后自上而下将其分解为若干相互支撑的论点。

金字塔结构呈现信息的经典形式：

  • 顶端：主要结论
  • 下一层：支撑结论的关键论点
  • 再下一层：支持这些论点的数据和事实

    • 1.2 四大基本原则

    | 原则 | 说明 | 操作要点 | |------|------|---------| | 结论先行 | 每篇文章只有一个中心思想，并置于最前 | 先抛出结论，再组织论据 | | 以上统下 | 每一层次思想是对下一层次思想的总结概括 | 上一层统领下一层 | | 归类分组 | 每一组思想必须属于同一逻辑范畴 | 同层论点必须同类型 | | 逻辑递进 | 每一组思想必须按逻辑顺序排列 | 时间/重要性/结构顺序 |

    2.2 操作步骤

    ``` ┌─────────────────────────────────────────────────────────────┐ │ 金字塔结构构建五步法 │ ├─────────────────────────────────────────────────────────────┤ │ │ │ Step 1️⃣ 提出核心论点/结论 │ │ ───────────────────────────────────── │ │ 这是金字塔的塔尖，必须： │ │ • 清晰、明确 │ │ • 具体、可行动 │ │ • 结论先行 │ │ │ │ Step 2️⃣ 构建支撑论点 │ │ ───────────────────────────────────── │ │ 思考"为什么核心论点成立？"（Why So?） │ │ 列出直接支撑核心论点的几个关键子论点 │ │ ⚠️ 这些子论点之间需符合MECE原则 │ │ │ │ Step 3️⃣ 填充论据事实 │ │ ───────────────────────────────────── │ │ 对每个子论点，继续追问"为什么？" │ │ 提供更具体的数据、事实或案例作为支撑 │ │ │ │ Step 4️⃣ 自下而上验证 │ │ ───────────────────────────────────── │ │ 思考"所以呢？"（So What?） │ │ 下层论据综合起来能推导出什么结论？ │ │ 验证逻辑链条是否稳固 │ │ │ │ Step 5️⃣ 检查逻辑结构 │ │ ───────────────────────────────────── │ │ • 上下层级之间逻辑连贯，无矛盾 │ │ • 同一层级的子论点属于同一逻辑范畴 │ │ • 按逻辑顺序（时间/重要性/结构）排列 │ │ │ └─────────────────────────────────────────────────────────────┘ ```

    2.3 应用案例

    案例：是否进军东南亚市场的提案

    ``` 【核心结论】 "公司应立即进军东南亚市场"

    【支撑论点】 ├── 论点1：东南亚市场增长迅速（宏观机会） ├── 论点2：公司产品具有适配性（内部优势） └── 论点3：竞争对手尚未充分布局（竞争窗口期）

    【论据事实】 ├── 论点1论据： │ • 东南亚GDP增速6.2%（高于全球平均） │ • 中产阶级扩大至1.5亿人 │ • 电商渗透率年增30% │ ├── 论点2论据： │ • 产品已通过当地认证 │ • 价格区间匹配当地消费水平 │ • 本地化团队已组建 │ └── 论点3论据： • 主要竞争对手A尚未进入 • 竞争对手B仅布局新加坡 • 市场存在2-3年的窗口期 ```

    2.4 PREP法与金字塔的结合

    | PREP要素 | 对应金字塔层级 | 说明 | |---------|---------------|------| | P（结论） | 塔尖 | 核心论点 | | R（理由） | 第二层 | 支撑论点 | | E（事例） | 第三层 | 事实数据 | | P（结论） | 塔尖+呼应 | 重申结论 |

    2.5 注意事项

    | 要点 | 说明 | |------|------| | 避免确认偏误 | 必须避免只收集支持自身观点的信息 | | 主动寻找反面论据 | 进行辩证思考，确保结论客观性 | | 不仅是表达工具 | 金字塔结构更是思考工具 | | 先梳理后表达 | 写作或汇报前，先用金字塔梳理思路 |
