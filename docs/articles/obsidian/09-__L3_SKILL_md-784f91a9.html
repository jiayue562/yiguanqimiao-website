L3智能调度内核.skills - 真正读懂意图的中枢大脑

定位

L3智能调度内核是五行人格心理学操作系统的真正中枢大脑，负责：

  • 意图理解：不只是关键词匹配，而是理解用户真正想要什么
  • 场景识别：精细化场景分类（S0-S9）
  • 智能路由：根据场景自动调度最优引擎组合

核心能力

1. 意图理解引擎

四层意图解析：

  1. 表层意图：用户直接说的（如"分析一下木行人"）
  2. 深层意图：用户真正想要的（如想知道性格特点+如何相处）
  3. 潜在意图：用户没明说但相关的（如五行关系对合作的影响）
  4. 元意图：用户想要的学习/成长目的

意图分类矩阵： | 类型 | 示例 | 响应策略 | |------|------|---------| | 诊断型 | "我是什么五行" | 直接调用L4五行分智能体 | | 咨询型 | "木行人怎么相处" | 调用L4+提供关系建议 | | 学习型 | "给我讲讲五行人格" | 调用L2知识体系+深度学习 | | 成长型 | "我想改变自己" | 调用L5记忆层+制定干预方案 | | 关系型 | "我们五行合吗" | 调用L4+五行关系分析 | | 评估型 | "我最近状态如何" | 调用L5记忆层+诊断评估 |

2. 场景识别矩阵（S0-S9精细版）

S0 简单问答：直接回答，不需要引擎
S1 信息查询：调用L2知识库检索
S2 深度理解：调用L2+知识学习引擎
S3 创意创新：调用象思维+五色光绿蓝
S4 分析决策：调用五色光完整序列
S5 重大决策：全系统+象思维
S6 任务执行：人机协同执行
S7 系统规划：象思维+人协同+五色光
S8 修行文化：象思维+知识溯源
S9 系统升级：知行合一+系统更新

3. 智能路由决策树

用户输入
    ↓
意图理解（四层解析）
    ↓
场景分类（S0-S9）
    ↓
复杂度评估（低/中/高）
    ↓
引擎选择
    ├── 单引擎 → 直接调用
    ├── 双引擎 → 组合调用
    ├── 多引擎 → 序列调用
    └── 全系统 → A型配方
    ↓
执行反馈
    ↓
学习优化（记录到L5记忆层）

与其他层级的协同

L3调度层 → L4五行分智能体

  • 识别到五行分析需求 → 自动调度对应分智能体
  • 如："我最近脾气大" → 判断可能火行人 → 调度火行分智能体

L3调度层 → L5记忆层

  • 每次交互记录到记忆层
  • 识别到历史相关 → 调用记忆层
  • 学习用户偏好 → 优化路由

L3调度层 → L2底层Skills

  • 需要深度学习 → 调用知识学习引擎
  • 需要创新 → 调用象思维引擎
  • 需要决策 → 调用五色光引擎

自动触发关键词

意图理解触发：

  • 我想了解、我想知道、我想学习
  • 帮我分析、帮我看看、帮我诊断
  • 我们关系如何、我们合吗、怎么相处
  • 我最近状态、最近情况、状态如何
  • 怎么办、怎么处理、如何选择

场景识别触发：

  • 简单问题（S0）：是什么、有什么区别
  • 信息查询（S1）：查一下、告诉我
  • 深度理解（S2）：深入分析、详细说说
  • 创意创新（S3）：有什么新想法、创新方案
  • 分析决策（S4）：分析利弊、哪个好
  • 重大决策（S5）：要不要、是否应该
  • 任务执行（S6）：帮我做、帮我完成
  • 系统规划（S7）：如何构建、如何设计
  • 修行文化（S8）：修行、悟道、心性
  • 系统升级（S9）：优化、升级、改进
