一、架构关系定义

1.1 系统层级

``` ┌─────────────────────────────────────────────────────────────────────┐ │ 系统架构全景图 │ ├─────────────────────────────────────────────────────────────────────┤ │ │ │ 【用户层】 │ │ 悟空（木行人）←→ 龙龟神将（火行人） │ │ │ ├─────────────────────────────────────────────────────────────────────┤ │ │ │ 【调度层】龙心OS（智能中枢） │ │ ├─ 意图识别模块 │ │ ├─ 场景归类模块（S0-S9） │ │ ├─ 引擎路由模块 │ │ └─ 结果整合模块 │ │ │ ├─────────────────────────────────────────────────────────────────────┤ │ │ │ 【知识库层】龙脑OS（知识资产） │ │ ├─ 📊 麦肯锡思考框架 ← 本Skill所在位置 │ │ ├─ 🔨 五行人格心理学OS │ │ ├─ 📚 知识学习理论体系 │ │ ├─ 🌈 五色光思维体系 │ │ └─ ...其他专业知识体系 │ │ │ ├─────────────────────────────────────────────────────────────────────┤ │ │ │ 【执行层】躯体系统 │ │ CLI → Skills → MCP → Framework │ │ │ └─────────────────────────────────────────────────────────────────────┘ ```

1.2 核心关系

龙心OS（调度中枢）：
  • 唯一入口：所有用户输入首先由龙心OS处理
  • 智能路由：根据场景判断调用哪个知识库
  • 协议转换：将用户需求转换为知识库调用指令
  • 结果整合：将知识库输出与五大引擎整合
    • 龙脑OS（知识库）：
  • 被调用方：不直接响应用户，只响应龙心OS的调用
  • 独立封装：每个知识库独立维护，保持完整性
  • 标准接口：提供统一的调用接口和输出格式
    • 麦肯锡思考框架：
  • 位置：龙脑OS知识库中的一个独立组件
  • 功能：提供西方商业分析框架的完整知识体系
  • 特点：六大类框架独立存在，可按需调用

    二、调用协议

    2.1 调用流程

    ``` 用户输入 ↓ 龙心OS意图识别 ↓ 场景归类（S0-S9） ↓ 是否需要西方分析框架？ ├─ 是 → 路由至龙脑OS麦肯锡框架库 │ ↓ │ 选择具体框架 │ ↓ │ 执行分析 │ ↓ │ 返回结构化结果 │ ↓ │ 龙心OS整合输出 │ ↓ │ 知行合一沉淀 │ └─ 否 → 调用其他引擎（象思维/五色光等） ```

    2.2 调用接口

    龙心OS调用麦肯锡框架的标准格式：

    ```yaml

    调用请求

    调用方: 龙心OS 被调用方: 龙脑OS.麦肯锡思考框架 调用参数: 场景类型: [战略规划/问题分析/组织诊断/营销客户/逻辑表达/创意解决] 具体框架: [SWOT/MECE/七步法/...] # 可选，如未指定则自动匹配 输入内容: [用户原始输入] 上下文: [对话历史摘要]

    调用响应

    响应方: 龙脑OS.麦肯锡思考框架 响应内容: 推荐框架: [具体框架名称] 框架定义: [一句话定义] 操作步骤: [Step 1 → Step 2 → ...] 输出模板: [结构化输出格式] 注意事项: [使用要点] ```

    2.3 场景路由表

    | 龙心OS场景 | 麦肯锡框架调用 | 组合建议 | |-----------|---------------|---------| | S2深度分析 | 根据内容类型选择 | + 知识学习 | | S3创意创新 | SCAMPER、曼陀罗 | + 象思维 | | S4分析决策 | SWOT、决策矩阵、逻辑树 | + 五色光 | | S5重大决策 | 多框架组合 | + 全系统 | | S6任务执行 | 七步法、流程图 | + 人机协同 | | S7系统规划 | 7S、BCG、安索夫矩阵 | + 象思维+五色光 |

    三、与五大引擎的协同

    3.1 标准协同流程

    ``` ┌─────────────────────────────────────────────────────────────┐ │ 中西合璧分析流程 │ ├─────────────────────────────────────────────────────────────┤ │ │ │ 阶段1：0→1原创（东方智慧） │ │ 🐉 象思维 → 直觉洞察问题本质 │ │ ↓ │ │ 阶段2：1→N展开（西方工具） │ │ 📊 麦肯锡框架 → 逻辑结构化分析 │ │ ↓ │ │ 阶段3：多维度验证（整合工具） │ │ 🌈 五色光思维 → 五色分治同频共振 │ │ ↓ │ │ 阶段4：沉淀进化（系统升级） │ │ 🔄 知行合一 → 经验压缩与泛化 │ │ │ └─────────────────────────────────────────────────────────────┘ ```

    3.2 各引擎与麦肯锡框架的关系

    | 引擎 | 与麦肯锡框架的关系 | 协同方式 | |------|------------------|---------| | 象思维 | 前置引擎 | 象思维0→1洞察 → 麦肯锡框架1→N展开 | | 知识学习 | 并行引擎 | 深度学习麦肯锡方法论本身 | | 五色光思维 | 验证引擎 | 麦肯锡分析结果用五色光验证 | | 人机协同五象限 | 分工引擎 | 复杂分析任务的人机分工 | | 知行合一 | 收尾引擎 | 麦肯锡分析经验沉淀为系统能力 |

    四、独立框架清单

    4.1 每个框架都是独立的

    麦肯锡思考框架库包含六大类、25+个独立框架，每个框架都可以被单独调用：

    ``` 麦肯锡思考框架库/ ├── 1.战略规划与管理类/ │ ├── SWOT分析.md │ ├── BCG矩阵.md │ ├── PEST分析.md │ ├── STP模型.md │ └── 安索夫矩阵.md │ ├── 2.项目管理与执行类/ │ ├── 七步问题解决法.md │ ├── 逻辑树.md │ ├── 流程图.md │ ├── 重要度紧急度矩阵.md │ └── 假设思考.md │ ├── 3.组织与文化分析类/ │ ├── 麦肯锡7S框架.md │ ├── PESTLE分析.md │ └── 愿景与使命定义.md │ ├── 4.市场营销与客户分析类/ │ ├── 五力分析.md │ ├── 3C分析.md │ ├── 客户旅程图.md │ ├── 用户画像.md │ └── 品牌资产金字塔.md │ ├── 5.逻辑思考与结构化分析类/ │ ├── MECE原则.md │ ├── 金字塔结构.md │ ├── 五色光思维.md │ ├── 归纳演绎.md │ └── 议题树.md │ └── 6.创意与问题解决类/ ├── SCAMPER.md ├── 曼陀罗思考法.md ├── 利弊均衡法.md └── 决策矩阵.md ```

    4.2 框架调用优先级

    当龙心OS调用麦肯锡框架库时，按以下优先级选择具体框架：

    1. 用户明确指定 → 调用指定框架 2. 场景关键词匹配 → 调用最匹配的1-2个框架 3. 复杂度评估 → 简单问题用单框架，复杂问题用组合 4. 历史使用记录 → 参考该场景下的历史成功案例

    五、输出标准

    5.1 麦肯锡框架输出格式

    所有框架的输出必须遵循以下标准格式：

    ```markdown

    📊 [框架名称]分析结果

    1. 框架定义

    [一句话定义该框架]

    2. 分析过程

    [Step-by-step的分析步骤和中间结果]

    3. 核心发现

    [3-5个关键发现，用 bullet points]

    4. 行动建议

    [具体的、可执行的建议]

    5. 风险提示

    [需要注意的风险和局限]

    --- 分析框架: [框架名称] 适用场景: [场景描述] 置信度: [高/中/低] ```

    5.2 与龙心OS整合输出

    麦肯锡框架的输出会被龙心OS整合到最终回复中：

    ``` 龙心OS最终输出 = 象思维洞察（如有）+ 麦肯锡框架分析结果 + 五色光验证（如有）+ 人机协同建议（如有）+ 知行合一沉淀建议 ```
