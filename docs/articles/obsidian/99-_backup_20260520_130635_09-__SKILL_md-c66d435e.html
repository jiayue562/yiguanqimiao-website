Task Board Skill（任务看板）

📖 技能定义

Task Board是AI龙龟共生伙伴操作系统的任务看板，提供拖拽式任务管理、多维度任务视图、实时更新任务状态的功能。

核心定位：

  • 拖拽式：直观的拖拽操作
  • 多维度：灵活的视图切换
  • 实时更新：任务状态实时同步

核心原则：

拖拽式任务管理，让任务状态一目了然。

🎯 核心功能

1. 拖拽式任务管理

看板布局（默认视图）：

+------------+------------+------------+------------+------------+
|   待办     |   进行中   |   审核中   |   已完成   |   归档     |
+------------+------------+------------+------------+------------+
| TASK-001   | TASK-003   | TASK-005   | TASK-002   | TASK-010   |
| P0         | P1         | P2         | P0         | P3         |
| 创建Skill  | 部署应用   | 审核代码   | 完成文档   | 归档项目   |
|            |            |            |            |            |
+------------+------------+------------+------------+------------+

拖拽操作：

  • 拖拽到右侧：任务状态推进
    • 待办 → 进行中
    • 进行中 → 审核中
    • 审核中 → 已完成
    • 已完成 → 归档
  • 拖拽到左侧：任务状态回退
    • 归档 → 已完成
    • 已完成 → 审核中
    • 审核中 → 进行中
    • 进行中 → 待办

2. 多维度任务视图

视图类型：

按状态视图（默认）：

  • 待办 / 进行中 / 审核中 / 已完成 / 归档

按优先级视图：

  • P0 / P1 / P2 / P3

按类型视图：

  • 开发 / 设计 / 测试 / 部署 / 文档

按负责人视图：

  • 龙龟神将 / 悟空 / 其他

按时间视图：

  • 今日 / 本周 / 本月 / 全部

按标签视图：

  • 根据标签分组（如：#AI #开发 #文档）

3. 实时更新任务状态

更新触发：

  • 拖拽任务后自动更新状态
  • 手动编辑任务信息后自动保存
  • 子任务完成时自动更新进度
  • 截止日期临近时自动预警

实时同步：

  • 所有客户端实时同步任务状态
  • 显示最后更新时间和操作人
  • 冲突检测和解决机制

🔄 与其他Skills的协同

1. 与任务控制台Skill协同

  • 任务看板：拖拽式任务管理
  • 任务控制台：任务详情和操作历史
  • 协同：任务看板管理状态→任务控制台展示详情

2. 与心跳巡检Skill协同

  • 心跳巡检：监控任务进度和截止日期
  • 任务看板：展示任务状态和进度
  • 协同：心跳巡检预警→任务看板高亮显示

3. 与长期记忆Skill协同

  • 长期记忆：记录任务历史和经验总结
  • 任务看板：展示当前任务状态
  • 协同：任务完成后→长期记忆归档

🔔 任务提醒与预警

1. 截止日期预警

预警级别：

  • 🟢 绿色：截止日期 > 7天
  • 🟡 黄色：截止日期 3-7天
  • 🟠 橙色：截止日期 1-3天
  • 🔴 红色：截止日期 < 1天或已逾期

预警方式：

  • 任务卡片颜色标记
  • 看板统计中高亮显示
  • 推送通知（P0任务）

2. 逾期任务处理

逾期处理流程：

  1. 逾期任务自动标记为红色
  2. 通知负责人
  3. 记录逾期原因
  4. 更新截止日期或取消任务
  5. 生成逾期报告

🚫 使用禁忌

1. 不要做的

  • ❌ 随意拖拽任务，不确认状态
  • ❌ 忽视逾期任务
  • ❌ 修改任务但不记录原因
  • ❌ 删除任务不归档
  • ❌ 过度细分任务

2. 必须做的

  • ✅ 拖拽后确认任务状态
  • ✅ 及时处理逾期任务
  • ✅ 记录任务变更原因
  • ✅ 完成后归档任务
  • ✅ 保持任务卡片清晰

💡 最佳实践

1. 任务管理

任务拆分：

  • 大任务拆分为小任务
  • 每个子任务独立完成
  • 子任务完成后自动更新主任务进度

任务优先级：

  • P0：立即处理（阻塞其他任务）
  • P1：尽快处理（重要但不紧急）
  • P2：按计划处理（一般任务）
  • P3：延后处理（非必要任务）

任务标签：

  • 使用标签分类任务
  • 标签要简洁明确
  • 避免过多标签

2. 看板维护

定期清理：

  • 及时归档已完成任务
  • 删除无用任务（记录原因）
  • 整理任务标签

视图切换：

  • 根据场景选择合适视图
  • 状态视图：日常管理
  • 优先级视图：紧急任务
  • 时间视图：时间规划

3. 团队协作

任务分配：

  • 明确任务负责人
  • 合理分配任务量
  • 避免过度分配

任务沟通：

  • 使用评论功能沟通
  • @相关人员通知
  • 记录重要决策

🎯 核心原则总结

三大铁律

  1. 拖拽式管理
    • 直观的操作方式
    • 快速切换任务状态
    • 自动保存和同步
  2. 多维度视图
    • 灵活切换视图
    • 满足不同场景需求
    • 快速定位任务
  3. 实时更新
    • 任务状态实时同步
    • 实时统计和预警
    • 及时通知相关人员

核心价值

  • 直观易用：拖拽操作，简单直观
  • 灵活高效：多维度视图，快速定位
  • 实时同步：状态更新，实时可见
  • 团队协作：清晰分工，高效沟通
