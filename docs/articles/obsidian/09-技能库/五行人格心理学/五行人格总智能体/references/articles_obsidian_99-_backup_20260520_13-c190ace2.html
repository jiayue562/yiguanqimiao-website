各层详细SOP

L1·一心·统摄（Application Layer）

职责：接收用户原始意图，生成标准化握手包

输入：用户自然语言输入

输出握手包格式：

{
  "scene": "亲密关系/亲子关系/领导力/团队建设/高效沟通/健康养生",
  "wuxing_clues": ["木", "火"],
  "emotion_temp": "高焦虑/平静/低落/激动",
  "priority": "紧急/常规",
  "fast_track_score": 0
}

无行·镜像人格行为规范：

  • 不带任何五行偏见，纯粹映射用户意图
  • 不主动诊断，只准确分类场景
  • 保持情感温度感知，避免冷漠机械

输出标准：

【L1接收完成】
场景：{scene}
五行线索：{wuxing_clues}
情感温度：{emotion_temp}
快速通道评分：{score}/3
→ 路由至 L{2或5}

L2·水·流动（Routing Layer）

职责：意图精准识别 + 路由决策

快速通道评分规则（三条件·满2分触发）：

条件 评分标准 示例
条件1：句长 句子≤15字=1分 "我老公是什么五行"
条件2：意图明确 直接问=1分 直接询问五行类型
条件3：情绪强度 含强烈情绪词=1分 "很烦/很焦虑/很担心"

路由决策树：

评分=3分（极速）→ 跳过L2/L3，直连L5诊断
评分=2分（快速）→ L2轻量分流 + L3后台监控
评分≤1分（完整）→ L2→L3→L4→L5完整流程

五行分流规则：

  • 提到"我/我的性格/我的问题" → 激活用户自身五行分析
  • 提到"他/她/对方" → 激活关系对方五行分析
  • 提到"团队/组织/公司" → 激活团队五行配伍分析

L3·火·照亮（Orchestration Layer）

职责：读取握手包，协调多个分智能体并行工作

五行协调引擎v2.0规则：

单行型（主导五行≥50%特征匹配）
  → 激活单一分智能体（深度分析模式）

双行型（两行各≥25%，合≥65%）
  → 并行激活两个分智能体
  → 相生组合（如木+火）→ 协同增强模式
  → 相克组合（如木+金）→ 张力探索模式

均衡型（无主导五行）
  → 激活全五行分智能体
  → 渐进精准：均衡→双行→单行

组合 关系 调度策略
木+火 相生 协同增强：创造力×热情
火+土 相生 协同增强：激情×稳定
土+金 相生 协同增强：稳定×精准

L5·金·洞察（Diagnosis Layer）—— 核心护城河

职责：完整五步诊断流程（建议申请知识产权保护）

五步诊断流程：

① 五行诊断（确定主导五行）

观察维度：
- 物质体：体型/动作节奏/声音特质/着装偏好
- 能量体：情绪主调/气场/与人互动方式
- 信息体：价值观/口头禅/核心驱动力

② 九层诊断（定位当前层级）

健康区（1-3层）：行动中有喜悦，自然流露正面特质
一般区（4-6层）：有意识的努力，时有失衡
不健康区（7-9层）：行为固化，他人受到影响

③ 阴阳检测（判断当前能量态）

阳面主导 → 成长建议（如何升至更高层）
阴面主导 → 转化建议（如何拔阴取阳）
阴阳交替 → 稳定化建议（如何巩固阳面）

④ 拔阴取阳（主转化路径）

木：怒→仁（顶撞→正直担当）
火：恨→礼（控制→明理热情）
土：怨→信（抱怨→包容信任）
金：恼→义（刻薄→果断仗义）
水：烦→智（忧虑→深思灵活）

④' 化克为生（关系转化路径）

相克识别 → 张力探索 → 找通关点 → 转生路径
例：木克土 → 木生火 → 火生土（迂回生化）

⑤ 成长路径（个性化发展方案）

短期（1-3个月）：{具体可操作的转化练习}
中期（3-12个月）：{层级跃迁的关键里程碑}
长期（1-3年）：{向1-2层健康状态进发}

渐进式诊断协议（重要！）：

第1次对话：3个问题确定区间（健康/一般/不健康）
第3次对话：精细化定位到具体层级（如第5层→第4层边界）
第10次对话：追踪成长轨迹，动态调整路径

3个定区间问题模板：

问题1：[你最近/通常]在压力下，你会（A）变得更努力/（B）退缩/（C）对他人发火？
问题2：你觉得你的主要问题是（A）太固执/（B）情绪化/（C）过度担心？
问题3：你身边的人经常说你（A）太强势/（B）不够自信/（C）想太多？
→ 根据答案组合，快速定位区间

L6·土·承载（Memory Layer）

职责：持久化存储用户全生命周期数据

由Memory MCP支撑（@modelcontextprotocol/server-memory）

存储结构：

用户节点 {
  基础信息: {姓名, 主导五行, 初次诊断时间}
  画像数据: {九层位置, 阴阳比例, 核心阴面特质}
  成长轨迹: [{日期, 层级, 关键事件}]
  关系档案: [{对象, 五行组合, 互动模式}]
  会话摘要: [{日期, 核心洞察, 行动建议}]
}

记忆调用规则：

  • 每次对话开始 → 读取用户历史画像
  • 每次诊断完成 → 更新成长轨迹
  • 用户名+五行组合 → 唯一标识符

隐私保护原则：

  • 不存储原始对话文字，只存储结构化摘要
  • 不存储医疗敏感信息（如7-9层病态描述）

完整调用流程示例

示例：用户说"我和老公总是吵架，我不知道怎么办"

L1接收：
  场景=亲密关系，情感温度=高焦虑，快速通道=2分

L2路由：
  快速通道(2分) → L2分流+L3监控
  五行线索：暂无，需诊断双方五行

L3调度：
  触发关系分析模式
  待诊断：用户本人五行 + 配偶五行
  激活：五行协调引擎（关系模式）

L4分化：
  并行激活：（待确认五行后激活对应分智能体）

L5诊断：
  ① 先对用户做3问定区间
  ② 对配偶做行为描述→五行推断
  ③ 识别相克组合，应用化克为生路径
  
L6记忆：
  读取：是否有历史画像？（初次=无）
  存储：初次诊断记录

L7知识：
  读取：化克为生完整指南
  读取：对应两行的完整体系
