🔬 反推方法

输入：两份龙爪实测HTML（狮山店周复盘画布 + 堂食创业单元7S诊断报告）
  → 对照：岗位智能体v2.0的8步SOP + 两种输出模板
  → 识别：龙爪实际输出中哪些内容是岗位智能体无法生成的
  → 归因：定位Skill层面的缺失（流程缺步/模板缺项/框架缺维）
  → 输出：13项修改清单（含优先级/修改原因/具体内容/预期效果）

核心发现：岗位智能体v2.0的8步SOP覆盖了"从输入到输出骨架"的流程，但在以下三个层面存在系统性缺口：

  1. 深度分析框架（7S评分、指标分层、五行阴阳面）
  2. 多维度诊断输出（复盘画布、龙爪断语）
  3. 视觉化呈现规范（HTML仪表盘设计）

本次升级从v2.0→v3.0，核心目标是让岗位智能体能够直接生成龙爪级别的深度诊断报告。

🔴 P0 修改项（7项·必须修改）

P0标准：龙爪实测输出中已验证的核心功能，但岗位智能体v2.0完全无法生成或仅有空壳定义。

P0-1：新增7S评分体系与诊断依据模板

⚡ 龙爪实测证据

堂食创业单元7S诊断报告中，7S维度各有1-10分评分 + 进度条 + 评级标签（强/良/中/弱）+ 诊断依据文字 + 核心问题 + 改善方向。例如S2结构评4分，诊断依据为"组织架构调整连续两周未完成，权责不清；三店执行差异巨大（完成率93.9%~148.8%）"。

当前岗位智能体Step 2仅有"五行-岗位适配表"，完全没有7S评分方法论。SKILL.md中7S仅在龙爪模式输出模板中被提及为"核心理论体系"，但无评分标准、无诊断依据模板、无评级逻辑。

🔧 具体修改

新增 Step 2.5：7S组织诊断评分（插入在五行识别之后、关键动作设计之前）

  1. 新增7S评分标准表（1-10分制，每分对应的行为描述）
  2. 新增7S诊断依据模板：评分 + 诊断依据 + 核心问题 + 改善方向
  3. 新增7S可视化规范：进度条颜色（8-10绿/5-7橙/1-4红）+ 评级标签逻辑
  4. 新增7S综合评分计算方法：7维加权平均（默认等权，可自定义权重）
# 新增 Step 2.5：7S组织诊断评分
7S评分标准:
  S1战略:
    8-10: 战略清晰且可执行，团队认知统一
    5-7: 有战略但执行走样或认知不统一
    1-4: 战略模糊或与实际脱节
  S2结构:
    8-10: 组织架构清晰，权责明确，协作流畅
    5-7: 基本架构在但存在模糊地带
    1-4: 架构缺位或权责混乱
  # ... S3-S7 类似标准

7S诊断依据模板:
  维度: [S1-S7]
  评分: [1-10]
  诊断依据: [基于数据/行为的具体证据]
  核心问题: [一句话概括瓶颈]
  改善方向: [具体可执行的改善建议]

7S可视化规范:
  评分8-10: 绿色进度条 + "强/良"标签
  评分5-7: 橙色进度条 + "中/偏弱"标签
  评分1-4: 红色进度条 + "弱"标签
  综合评分: 7维加权平均（默认等权）

📈 预期效果

岗位智能体可直接输出带评分的7S诊断报告，而非仅列出7个维度的概念。用户可基于评分快速识别组织薄弱点，龙爪输出从"定性描述"升级为"定量+定性双重诊断"。

P0-2：新增复盘画布模块（6步深度分析框架）

⚡ 龙爪实测证据

狮山店周复盘画布采用了6步深度分析框架：

步骤 内容 示例
①工作全量盘点 含7S维度标签 6项任务100%达成
②五行优劣势剖析 阳面5项+阴面5项，附行为证据 ✅目标感强 ❌关系维护弱
③原因剖析 根因链条，3条因果链 起因→过程→结果
④拔阴取阳行动方案 阴面→取阳方向→具体行动→借力五行 4列表格
⑤下周风险预警 任务→风险→预警信号→应对预案 4列风险预警表
⑥进化路径 当前→目标，4级跃迁 进化阶梯可视化

当前岗位智能体完全没有复盘分析模块。8步SOP聚焦"如何生成提示词/龙爪"，而非"如何用龙爪进行深度复盘"。

🔧 具体修改

新增 Step 3.5：复盘画布生成（龙爪模式专属）

  1. 新增6步复盘画布框架定义
  2. 新增每步的输入/输出/分析逻辑
  3. 新增"7S维度标签"机制——工作盘点时自动关联7S维度
  4. 新增"五行借力"方法论——从阴面到阳面的五行转化路径表
# 新增 Step 3.5：复盘画布生成
复盘画布6步框架:
  Step A: 工作全量盘点
    输入: 本周完成事项清单
    处理: 每项标注7S维度标签
    输出: 带7S标签的工作盘点表 + 完成率进度条

  Step B: 五行优劣势剖析
    输入: 五行类型 + 工作盘点数据
    处理: 阳面=五行显发优势（附行为证据），阴面=过刚/过柔劣势（附行为证据）
    输出: 双栏对比（优势5项 vs 劣势5项）

  Step C: 原因剖析（根因链条）
    输入: 阴面劣势项
    处理: 每个劣势追溯因果链（至少3层），直到可执行的根因
    输出: 根因链条卡片（起因→过程→结果）

  Step D: 拔阴取阳行动方案
    输入: 根因链条 + 五行类型
    处理: 阴面→取阳方向→具体行动→借力五行
    输出: 4列表格（阴面/取阳方向/具体行动/借力五行）

  Step E: 下周风险预警
    输入: 下周任务 + 阴面劣势
    处理: 每个任务识别风险→预警信号→应对预案
    输出: 4列风险预警表

  Step F: 进化路径
    输入: 当前优势 + 阴面改善方向
    处理: 设计3-4级进化阶梯（当前→目标），每级有核心关键词
    输出: 进化路径可视化 + 当前定位 + 目标定位

五行借力路径表:
  金行过刚 → 借水行润泽（加倾听、加柔性）
  金行过刚 → 借火行礼明（加仪式、加情感设计）
  金行过刚 → 借土行承载（加预案、加弹性）
  金行自身 → 数据补数据（过程追踪、打卡点）
  金行过刚 → 借木行生发（加追问、加深度）
  # 其他五行借力路径类似设计

📈 预期效果

龙爪模式可直接生成周复盘画布HTML，包含6步深度分析。用户从"写了什么工作"升级为"工作背后的原因是什么→如何改善→进化方向在哪"的完整闭环。

P0-3：升级五行人格适配——增加阴阳面分析与数据驱动识别

⚡ 龙爪实测证据

堂食创业单元7S诊断报告中，三店店长的五行识别全部基于经营数据和行为数据，而非自报类型：金月湾148.8%超额→"目标强执行快"→金行；松山湖50%渗透率→"销售热情高"→火行。且每店都有阳面3-4项 + 阴面2-3项的阴阳面标签（如：✅目标感强 ❌关系维护弱）。

当前岗位智能体Step 2仅有5种五行类型的一维适配表（核心特质→适配岗位→适配任务），没有阴阳面分析，且假设用户自报五行类型，没有数据驱动识别机制。

🔧 具体修改

升级 Step 2：五行人格识别与适配 → 五行人格深度诊断

  1. 新增"数据驱动五行识别"机制——当用户无法自报五行类型时，从行为/数据特征自动推断
  2. 新增"五行阴阳面分析表"——每种五行的阳面3-5项优势 + 阴面3-5项风险，附行为证据模板
  3. 新增"阴面警示话术"——当识别到阴面时，龙爪输出警示文字
  4. 保留原有一维适配表作为"基础层"，阴阳面分析作为"深度层"
# 升级 Step 2：新增深度层
五行阴阳面分析表:
  金行人:
    阳面（显发）:
      - 目标导向极强: [行为证据模板]
      - 执行速度极快: [行为证据模板]
      - 原则清晰公正: [行为证据模板]
      - 数据意识强: [行为证据模板]
    阴面（过刚）:
      - 团队沟通偏指令式: [行为证据模板]
      - 文化把关弱: [行为证据模板]
      - 弹性排班不足: [行为证据模板]
      - 过程指标少: [行为证据模板]
      - 复盘深度浅: [行为证据模板]
    阴面警示话术: "金行过刚——用数量忙碌掩盖质量溃败，做更多不如做更深"
  # 木/火/土/水类似设计

数据驱动五行识别机制:
  触发条件: 用户未提供五行类型 或 用户提供的数据与自报类型矛盾
  识别逻辑:
    - 目标导向+执行快+数据驱动 → 金行信号
    - 热情高+感染力强+跟进弱 → 火行信号
    - 创新力+前瞻性+落地弱 → 木行信号
    - 稳健+流程强+变革慢 → 土行信号
    - 灵活+策略强+定力弱 → 水行信号
  输出: 推断的五行类型 + 识别依据 + 阴阳面标签

📈 预期效果

龙爪不再依赖用户自报五行类型，可从行为数据自动推断。阴阳面分析让诊断从"你适合什么"升级为"你的强项和暗礁分别是什么"，直接支撑复盘画布和拔阴取阳方案。

P0-4：新增指标分层预警体系（结果/过程/风险三层）

⚡ 龙爪实测证据

堂食创业单元7S诊断报告中，指标体系被清晰地分为三层：

层级 用途 示例
✅结果指标 考核用 综合营业额完成率132.9%、高价值占比40.6%
🔄过程指标 监控用 桌桌链接覆盖率、1对1邀约执行率
🚨风险指标 预警用 客户链接断裂(P0)、品控效果不可验证(P0)

每层有独立的进度条、颜色、标签。且风险指标有P0/P1优先级。

当前岗位智能体Step 3"关键动作设计"中有"目标量化"和"数据验证"和"风险控制"三个独立小节，但没有分层逻辑，没有P0/P1优先级，没有视觉呈现规范，三个小节之间也缺乏层级关联。

🔧 具体修改

升级 Step 3：关键动作设计 → 三层指标体系 + 关键动作设计

  1. 将"目标量化/数据验证/风险控制"重组为"三层指标体系"
  2. 新增P0/P1优先级标签系统
  3. 新增每层指标的视觉呈现规范（颜色/进度条/标签）
  4. 新增"指标分层关联规则"——过程指标异常如何上溯到风险指标
# 升级 Step 3：三层指标体系
三层指标体系:
  Layer 1 - 结果指标（考核用·绿色）:
    定义: 衡量最终经营成果的指标
    模板: 指标名 | 当前值 | 目标值 | 达成状态
    示例: 综合营业额完成率 | 132.9% | 100% | 达标
    频率: 月度/季度

  Layer 2 - 过程指标（监控用·蓝色）:
    定义: 衡量执行过程质量的指标
    模板: 指标名 | 当前值 | 预警阈值 | 监控频率
    示例: 1对1邀约执行率 | 极低 | 每店每日≥3人 | 日度
    频率: 日度/周度

  Layer 3 - 风险指标（预警用·红色）:
    定义: 需要立即关注的系统性风险
    模板: 指标名 | 描述 | 严重度 | 优先级
    示例: 客户链接机制断裂 | 三店系统性失效 | 系统级 | P0
    频率: 实时

分层关联规则:
  - 过程指标连续3期低于阈值 → 升级为风险指标
  - 风险指标P0 → 当周必须解决，纳入每日行动日历
  - 风险指标P1 → 当月必须解决，纳入每周复盘

📈 预期效果

指标体系从"平铺列表"升级为"三层金字塔"，用户可一眼区分"考核什么、监控什么、预警什么"。P0/P1标签让资源分配有优先级依据。

P0-5：新增总部赋能接口激活状态评估机制

⚡ 龙爪实测证据

堂食创业单元7S诊断报告中，5大总部赋能接口各有激活状态评估：数据赋能=已激活、培训赋能=部分激活、资源赋能=待加强、问题赋能=薄弱、反馈赋能=未激活。每个接口还有✅已完成项和❌缺失项的具体列举。文末有龙爪判断："5大接口中仅数据赋能基本到位，反馈赋能完全空缺"。

当前岗位智能体和味藏店长龙爪SKILL.md都定义了5大接口的功能，但都没有激活状态评估方法论——如何判断某个接口是"已激活"还是"未激活"？评估标准是什么？证据来源是什么？

🔧 具体修改

新增 Step 4.5：总部赋能接口激活状态评估

  1. 新增5级激活状态定义（已激活/部分激活/待加强/薄弱/未激活）
  2. 新增每个接口的激活评估标准——哪些能力项达标=已激活
  3. 新增"已完成/缺失"清单模板
  4. 新增"接口激活判断"输出格式
# 新增 Step 4.5：总部赋能接口激活状态评估
5级激活状态:
  已激活: 核心能力项100%可用，门店可直接调用
  部分激活: 部分能力可用，存在关键缺失项
  待加强: 基础能力在建设中，核心流程未跑通
  薄弱: 仅有个别触点，系统性支撑不足
  未激活: 该接口完全空缺，无任何可用能力

各接口激活评估标准:
  数据赋能:
    已激活: 营业额+成本+客诉数据可获取
    缺失信号: 品控数据/满意度/培训考核数据缺失
  培训赋能:
    部分激活: 有标准化培训流程
    缺失信号: 话术素材/实操模板/深度培训缺失
  # ...其他3个接口类似

输出格式:
  每个接口:
    状态: [5级之一]
    ✅ 已完成: [列举]
    ❌ 缺失: [列举]
  龙爪判断: [综合评估 + 优先打通建议]

📈 预期效果

龙爪不再仅"描述接口功能"，而能"评估接口是否生效"。用户可直观看到总部赋能的缺口在哪里，优先打通哪个接口。

P0-6：新增龙爪断语模块（诊断总结与核心判断）

⚡ 龙爪实测证据

两份HTML都有诊断总结模块：

  • 狮山店复盘画布："一周一句话"——"俞腾芳是味藏最锋利的剑——本周6项任务100%达成证明了这一点。但锋利的剑需要火来淬炼温度，需要水来磨出柔韧。"
  • 堂食创业单元7S诊断："龙爪断语"——3大核心判断 + 金句引用 + 1个核心命题

当前岗位智能体的输出模板（无论提示词模式还是龙爪模式）都没有诊断总结模块。输出到指标/风险就结束了，缺乏高层面的判断和提炼。

🔧 具体修改

新增 Step 7.5（龙爪模式专属）：龙爪断语生成

  1. 新增"龙爪断语"模板：3大核心判断 + 1个核心命题
  2. 新增"一周一句话"模板：基于五行人格的意象化总结
  3. 新增金句引用机制——从分析数据中提炼可引用的金句
# 新增 Step 7.5：龙爪断语生成
龙爪断语模板:
  标题: 🐉 龙爪断语 · [分析对象]
  核心判断（3条）:
    一、[判断1——从结果指标提炼]
    二、[判断2——从过程/风险指标提炼]
    三、[判断3——从7S/文化维度提炼]
  金句引用:
    "[从数据中提炼的警句]"——[解释]
  核心命题:
    "[下周/下阶段的核心命题，一句话]"

一周一句话模板:
  [人物]是味藏最[五行意象]的[隐喻]——
  [阳面成就]。但[五行意象]需要[借力五行]来[转化方向]。
  [核心行动方向]，不是做更多，而是做更深。

📈 预期效果

龙爪输出有了"灵魂"——不仅是数据堆砌，更有高层面的判断和提炼。用户读完整份报告后，最后看到的断语能让其立刻抓住核心命题。这是AI从"工具"到"军师"的关键升级。

P0-7：新增HTML输出格式与视觉设计规范

⚡ 龙爪实测证据

两份HTML都采用了悟空偏好的深色顶栏 + 白色画布 + 浅灰网格布局，包含：CSS变量体系（--topbar-bg/--canvas-bg/--grid-bg等）、响应式设计（@media断点）、进度条动画（setTimeout渐入）、Tab切换（JS函数switchCal）、颜色编码标签体系（tag-g/tag-r/tag-o/tag-b/tag-p）。

当前岗位智能体完全没有视觉输出规范。龙爪模式输出模板仅有Markdown结构定义，没有HTML/CSS/JS规范。龙爪每次输出都需要AI"凭感觉"写HTML，导致风格不统一、布局不一致。

🔧 具体修改

新增 附录A：龙爪HTML输出视觉设计规范

  1. 新增CSS变量体系定义（必须遵循）
  2. 新增通用组件库：卡片/表格/标签/进度条/仪表盘卡片/Tab切换
  3. 新增响应式断点规范
  4. 新增颜色编码逻辑（绿=达标/橙=关注/红=预警/蓝=过程/紫=特殊）
  5. 新增交互组件规范（Tab切换、进度条动画）
# 新增 附录A：HTML输出视觉设计规范

  绿(tag-g): 达标/完成/优势

P1-8：升级经营分析框架——增加多门店对比与经营仪表盘

⚡ 龙爪实测证据

堂食创业单元7S诊断报告中有8张仪表盘卡片（三店总销售/综合完成率/高价值销售/风险门店/三店分别/预警），支持三店同时对比（完成率93.9%~148.8%的跨店比较）。

当前岗位智能体有"经营分析框架"但仅有yaml字段定义，没有多门店对比模板，没有仪表盘卡片布局规范。假设的是单店/单岗位场景。

🔧 具体修改

  1. 经营分析框架新增"多门店/多单元对比"模式
  2. 新增"经营仪表盘"卡片布局模板（4列网格，每卡片=图标+标签+大数字+副文本）
  3. 新增"风险门店/薄弱单元"自动识别逻辑——完成率最低的自动标红

P1-9：升级心文化落地评估——从清单到诊断

⚡ 龙爪实测证据

堂食创业单元7S诊断报告中心文化评估采用了双栏结构：左栏=本周文化落地检视（6项，每项有颜色圆点标识通过/未通过/待提升），右栏=下周心文化行动（5项，每项有日历图标+具体行动）。

当前岗位智能体和味藏店长龙爪中的心文化落地引擎仅有每日/每周/每月的执行清单，没有评估/诊断机制——无法回答"文化落地到底做到什么程度了"。

🔧 具体修改

  1. 心文化落地引擎新增"文化落地检视"模块——6-8项检视点，每项有颜色标识（●绿=通过 ●红=未通过 ●橙=待提升）
  2. 新增"价值观行为化"评估逻辑——将"匠心·严谨·温暖"等抽象价值观转化为可检查的行为标准
  3. 新增"文化行动"模板——与检视结果一一对应的改善行动
  4. 新增心文化六维评估法（以心为本/化克为生/拔阴取阳/礼法合治/差序格局/知行合一），每维1-10分

P1-10：升级三层行动日历——增加攻坚标记与新增项标注

⚡ 龙爪实测证据

堂食创业单元7S诊断报告中的三层行动日历，每日清单有【攻坚】标记（品控抽检松山湖）和【新增】标记（1对1邀约确认、话术演练时间、门店反馈表填写），且用颜色区分（攻坚=红色、新增=蓝色）。每月规划有【硬截止】标记。

当前味藏店长龙爪的三层行动日历仅有时段+事项+勾选框，没有攻坚/新增/硬截止标记，无法区分"常规动作"与"重点攻坚"。

🔧 具体修改

  1. 行动日历新增3种标记：【攻坚】红色 + 【新增】蓝色 + 【硬截止】红色粗体
  2. 新增"标记注入逻辑"——P0风险指标→每日清单中自动注入攻坚项；上周缺失项→自动注入新增项；有截止日的任务→自动注入硬截止项
  3. 每日清单模板从4时段细化为6时段（新增14:30-15:00话术演练时段、21:00收店后增加反馈表填写）

P1-11：升级风险控制——增加预警信号层与任务锚定

⚡ 龙爪实测证据

狮山店复盘画布中的风险预警表有4列：下周任务 → 潜在风险 → 预警信号 → 应对预案。其中"预警信号"是关键的中间层——不是直接从风险跳到预案，而是先定义"看到什么信号就知道风险发生了"。例如：预警信号="11:30前等位超5桌"→才知道"客流超预期导致翻台率不足"。

当前岗位智能体Step 3风险控制仅有3列：风险点 → 预警条件 → 应对策略。缺少"预警信号"层（可观测的具体现象），缺少"下周任务"锚定（风险是相对于哪个任务的）。

🔧 具体修改

  1. 风险控制模板从3列升级为4列：下周任务 → 潜在风险 → 预警信号 → 应对预案
  2. 新增"预警信号"定义逻辑——必须是可观测、可量化、有时效的现象
  3. 新增"任务锚定"——每个风险必须关联到一个具体任务

🐉 龙爪断语 · 岗位智能体v3.0升级方向

岗位智能体v2.0解决了"从0到1"的问题——六维框架、五行适配、双模式输出，让造龙爪有了标准化流程。

但两份龙爪实测报告揭示了一个核心命题：

"岗位智能体造出了龙爪的骨架，但龙爪真正咬下去时，还需要牙齿——7S评分是牙齿、阴阳面分析是牙齿、指标分层是牙齿、复盘画布是牙齿、龙爪断语是牙齿。"

v3.0的核心升级方向只有一个：从"能造龙爪"到"能造出咬得动问题的龙爪"。

13项修改，7P0+4P1+2P2，核心逻辑线：7S评分定基 → 五行阴阳面定性 → 指标分层定量 → 复盘画布结构化 → 龙爪断语升华。这条线打通后，岗位智能体输出的不再是一个"框架"，而是一份"可以直接用于经营决策的诊断报告"。
