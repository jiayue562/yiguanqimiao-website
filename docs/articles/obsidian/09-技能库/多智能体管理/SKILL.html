核心升级：1+5并行模式 + 五行协同 版本：v2.0 | 升级日期：2026-04-15 | 维护者：龙龟神将

一、核心升级：从串行到并行

v1.0 vs v2.0 对比

维度 v1.0 v2.0
架构模式 主从串行 1+5并行模式
子Agent数量 N个（可配置） 固定5个（五行）+ N个扩展
协同方式 顺序调用 并行+串行混合
五行整合 无 原生支持五行协同
结果整合 简单聚合 智能整合（五行相生）
横向通信 无 原生支持横向协作

二、核心定义

什么是1+5并行模式？

1+5并行模式是多智能体协同的最优架构：

         ┌─────────────────────────────────────┐
         │           主Agent（协调者）          │
         │  任务分解 │ 调度分配 │ 结果整合  │
         └──────────────┬──────────────────────┘
                        │
         ┌──────────────┼──────────────┬──────────────┐
         │              │              │              │
         ▼              ▼              ▼              ▼
    ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
    │ 木Agent │   │ 火Agent │   │ 土Agent │   │ 金Agent │ ←─ 子Agent并行执行
    │ (生长) │   │ (光明) │   │ (承载) │   │ (决断) │
    └─────────┘   └─────────┘   └─────────┘   └─────────┘
                                                   │
                                              ┌─────────┐
                                              │ 水Agent │ ←─ 子Agent
                                              │ (润泽) │
                                              └─────────┘

    子Agent之间可横向通信（协同）

五行子Agent职责

Agent 五行属性 核心职责 思维特征
主Agent — 协调、分解、整合 圆满觉知
木Agent 木 创新、规划、生长 0→1原创
火Agent 火 热情、行动、转化 光明洞察
土Agent 土 稳定、承载、整合 信实厚德
金Agent 金 决断、精准、分析 清明规则
水Agent 水 智慧、流动、适应 润下变通

四、核心功能 v2.0

1. 任务分解引擎

功能：将复杂任务智能分解为5个五行子任务

# 任务分解算法
任务分解:
  输入:
    - 原始任务描述
    - 任务类型分类
    - 复杂度评估
    
  分解策略:
    五行分配:
      木行子任务: "创新、规划、战略方向"
      火行子任务: "热情、行动、执行驱动"
      土行子任务: "稳定、整合、资源协调"
      金行子任务: "精准、分析、决策支持"
      水行子任务: "智慧、风险、灵活适应"
      
  输出:
    - 5个标准化的子任务
    - 每个子任务的：描述、目标、约束，成功标准

2. 并行调度引擎

功能：支持并行+串行的智能调度

# 并行调度配置
并行调度:
  模式: "parallel"  # 主模式
  
  执行策略:
    - 木Agent: "并行执行"
    - 火Agent: "并行执行"
    - 土Agent: "并行执行"
    - 金Agent: "并行执行"
    - 水Agent: "并行执行"
    
  依赖管理:
    显式依赖:
      - "木→火"  # 木完成后火可开始
      - "火→土"  # 火完成后土整合
      - "土→金"  # 土整合后金决策
      - "金→水"  # 金决策后水评估风险
      
    隐式依赖:
      - "自动检测横向依赖"
      - "动态调整执行顺序"
      
  超时控制:
    子Agent超时: 60000ms
    全局超时: 300000ms
    部分结果接受: true

3. 横向协作引擎

功能：支持子Agent之间的横向通信

# 横向协作配置
横向协作:
  启用: true
  
  协作模式:
    - 数据共享: "子Agent可共享中间结果"
    - 能力调用: "子Agent可调用其他Agent的能力"
    - 洞察传递: "子Agent可将洞察传递给其他Agent"
    
  协作协议:
    - 发起方: "sender_agent"
    - 接收方: "receiver_agent"
    - 消息类型: "data_request|result_sharing|acknowledgment"
    
  冲突处理:
    - 检测冲突: "分析子Agent结果的冲突"
    - 解决策略: "木生火→火生土→土生金→金生水→水生木"

4. 五行结果整合引擎

功能：智能整合5个子Agent的结果

# 五行整合算法
结果整合:
  整合策略: "五行相生"
  
  相生整合流程:
    1. 木生火:
       - 木Agent结果 → 注入火Agent的洞察燃料
       - 火Agent融合 → 生成更有热情的洞察
       
    2. 火生土:
       - 火Agent结果 → 注入土Agent的行动能量
       - 土Agent融合 → 生成更稳定的执行方案
       
    3. 土生金:
       - 土Agent结果 → 注入金Agent的分析基础
       - 金Agent融合 → 生成更精准的决策
       
    4. 金生水:
       - 金Agent结果 → 注入水Agent的评估视角
       - 水Agent融合 → 生成更智慧的风险评估
       
    5. 水生木:
       - 水Agent结果 → 反馈给木Agent
       - 木Agent整合 → 生成最终创新方案
       
  整合输出:
    - 主报告: "五行综合分析"
    - 子报告: "5个子Agent独立报告"
    - 洞察图谱: "五行关联洞察"

5. 异常处理引擎

功能：全面的异常检测和处理

# 异常处理配置
异常处理:
  检测机制:
    子Agent故障:
      - 无响应超时: 60000ms
      - 错误率阈值: 10%
      - 资源耗尽: cpu>90%|memory>90%
      
    结果冲突:
      - 检测置信度差异: >0.3
      - 检测方向冲突: "木vs金"
      
    死锁:
      - 循环依赖检测
      - 无限等待检测
      
  处理策略:
    自动恢复:
      - 重试: "最多3次"
      - 降级: "使用缓存结果"
      - 跳过: "继续执行其他子Agent"
      
    人工干预:
      - 严重故障通知
      - 冲突解决请求

六、调度策略 v2.0

智能调度算法

# 调度决策树
调度决策:
  输入任务评估:
    - 复杂度: "低|中|高|极高"
    - 时间敏感度: "低|中|高"
    - 并行度: "串行|部分并行|完全并行"
    
  调度策略选择:
    低复杂度+低敏感:
      - 策略: "单Agent快速处理"
      - 选择: "最空闲Agent"
      
    中复杂度+中敏感:
      - 策略: "1+2模式"
      - 选择: "主Agent + 2个相关五行Agent"
      
    高复杂度+高敏感:
      - 策略: "1+5完全并行"
      - 选择: "主Agent + 全部5个五行Agent"
      
    极高复杂度:
      - 策略: "1+5+嵌套"
      - 选择: "主Agent + 5个五行Agent + 各Agent的子任务Agent"
