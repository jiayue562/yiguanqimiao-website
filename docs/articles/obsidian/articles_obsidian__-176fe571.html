二、Step 1: 感知层（输入预处理）

1.1 关键词提取

``` 输入文本 │ ├─ 直接关键词（五行标识） │ ├─ 木行/木行人/木属性 → 木 │ ├─ 火行/火行人/火属性 → 火 │ ├─ 土行/土行人/土属性 → 土 │ ├─ 金行/金行人/金属性 → 金 │ └─ 水行/水行人/水属性 → 水 │ ├─ 特征关键词 │ ├─ 愤怒/激动/热情 → 火行 │ ├─ 抑郁/抱怨/固执 → 土行 │ ├─ 焦虑/多疑/变通 → 水行 │ ├─ 完美/挑剔/决断 → 金行 │ └─ 行动/创造/生长 → 木行 │ └─ 场景关键词 ├─ "是什么" → S1 ├─ "如何" → S2/S4 ├─ "为什么" → S4 ├─ "帮我" → S6 └─ "构建" → S7 ```

1.2 情感状态识别

``` 情感向量 = [焦虑, 愤怒, 抑郁, 恐惧, 迷茫, 平静, 喜悦] │ ├─ 焦虑向量 > 0.6 → 水行关注 ├─ 愤怒向量 > 0.6 → 火行关注 ├─ 抑郁向量 > 0.6 → 土行关注 ├─ 恐惧向量 > 0.5 → 水行关注 └─ 迷茫向量 > 0.5 → 水行+木行 ```

1.3 用户身份推断

``` 根据历史对话/用户画像 │ ├─ 用户是木行人 → 侧重木行分析 ├─ 用户是火行人 → 侧重火行分析 ├─ 用户是土行人 → 侧重土行分析 ├─ 用户是金行人 → 侧重金行分析 ├─ 用户是水行人 → 侧重水行分析 └─ 未确定 → 五行全面扫描 ```

三、Step 2: 场景归类

2.1 决策规则

``` IF 输入长度 < 10 AND 无关键信息 THEN → S0 (简单问答) ELSE IF 包含 "什么是" / "定义" / "解释" THEN → S1 (信息查询) ELSE IF 包含 "深入分析" / "学习" / "理解" THEN → S2 (深度理解) ELSE IF 包含 "创新" / "新想法" / "结合" THEN → S3 (创意创新) ELSE IF 包含 "分析" / "评估" / "诊断" THEN → S4 (分析决策) ELSE IF 包含 "方向" / "转型" / "构建系统" THEN → S5 (重大决策) ELSE IF 包含 "帮我" / "制定" / "执行" THEN → S6 (任务执行) ELSE IF 包含 "构建" / "升级" / "整合" THEN → S7 (系统规划) ELSE → S4 (默认分析决策) END ```

2.2 复杂度评估

``` 复杂度 = f(涉及五行数量, 场景类型, 意图强度)

低复杂度：

  • 单五行 + S0/S1 → 直接回答/轻量查询

    • 中复杂度：

  • 单五行 + S2/S4/S6 → 标准分析

    • 高复杂度：

  • 多五行 + S3/S5/S7 → 深度分析+方案设计
  • 任意五行 + S7 → 系统规划
    • ```

    四、Step 3: 引擎路由

    3.1 单引擎选择

    ``` S0: 不启动任何引擎 S1: 知识学习引擎 S2: 知识学习引擎 → 象思维 S3: 象思维 → 五色光(绿) S4: 五色光思维(完整序列) S5: 全系统(5引擎) S6: 人机协同四象限 S7: 象思维 → 人机协同 → 五色光 → 知行合一 ```

    3.2 分智能体选择

    ``` IF 明确指定某一行 THEN 调用对应分智能体 ELSE IF 情感/特征指向某一行 THEN 调用对应分智能体 + 总智能体协调 ELSE IF 涉及两行关系 THEN 调用两个相关分智能体 ELSE IF 需要全面分析 THEN 启动 1+5 模式：总智能体 + 5个分智能体 END ```

    3.3 五行分智能体调度规则

    ``` 【木行触发条件】

  • 直接：木行/木行人/木属性
  • 特征：愤怒、行动、创造、生长、规划
  • 场景：想做大事、创业、扩张
  • 关系：木生火、木克土

    • 【火行触发条件】

  • 直接：火行/火行人/火属性
  • 特征：热情、激动、表达、礼仪、明亮
  • 场景：社交活动、情绪管理、领导力
  • 关系：火生土、火克金

    • 【土行触发条件】

  • 直接：土行/土行人/土属性
  • 特征：抱怨、固执、信任、稳定、承载
  • 场景：人际关系、稳定发展、信任问题
  • 关系：土生金、土克水

    • 【金行触发条件】

  • 直接：金行/金行人/金属性
  • 特征：完美、挑剔、决断、原则、清明
  • 场景：决策分析、制度设计、品质把控
  • 关系：金生水、金克木

    • 【水行触发条件】

  • 直接：水行/水行人/水属性
  • 特征：焦虑、多疑、变通、智慧、流动
  • 场景：选择困难、情绪困扰、智慧提升
  • 关系：水生木、水克火

    • → 升级为S5 ```

    八、1+5 模式详解

    8.1 模式定义

    1+5模式是五行人格心理学操作系统的核心调度机制：
  • 1：总智能体（中枢大脑，负责协调）
  • 5：木、火、土、金、水五个分智能体

    • 8.2 调度类型

    | 模式 | 描述 | 示例 | 调用 | |------|------|------|------| | 1+0 | 总智能体独立处理 | "五行是什么？" | 知识学习引擎 | | 1+1 | 总+单分智能体 | "分析我的水行特质" | 水行人分智能体 | | 1+2 | 总+双分智能体 | "木生火怎么实现" | 木+火分智能体 | | 1+3 | 总+三分智能体 | "土金水循环" | 土+金+水分智能体 | | 1+5 | 全面分析模式 | "分析我的人格" | 总+5个分智能体 |

    8.3 自动判断规则

    ``` IF 用户明确指定某一行 THEN → 1+1模式 ELSE IF 用户涉及两行关系 THEN → 1+2模式 ELSE IF 用户需要全面分析 THEN → 1+5模式 ELSE IF 用户只是随便问问 THEN → 1+0模式（轻量响应） END ```
