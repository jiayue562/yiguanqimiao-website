一、战略规划与管理类框架详解

1.1 SWOT分析

定义: 对企业内外部环境进行系统性评估的经典框架。

四个维度:

  • S (Strengths) 优势: 内部积极因素，如核心竞争力、资源优势
  • W (Weaknesses) 劣势: 内部消极因素，如资源短板、能力缺陷
  • O (Opportunities) 机会: 外部积极因素，如市场增长、政策支持
  • T (Threats) 威胁: 外部消极因素，如竞争加剧、技术变革

分析矩阵:

              积极              消极
        ┌─────────────┬─────────────┐
   内部 │   优势(S)    │   劣势(W)    │
        │             │             │
        ├─────────────┼─────────────┤
   外部 │   机会(O)    │   威胁(T)    │
        │             │             │
        └─────────────┴─────────────┘

策略组合:

  • SO策略: 利用优势抓住机会（增长型）
  • WO策略: 克服劣势抓住机会（扭转型）
  • ST策略: 利用优势规避威胁（多元化）
  • WT策略: 减少劣势规避威胁（防御型）

使用步骤:

  1. 列出所有内部优势
  2. 列出所有内部劣势
  3. 列出所有外部机会
  4. 列出所有外部威胁
  5. 构建SWOT矩阵
  6. 制定策略组合

1.2 BCG矩阵（波士顿矩阵）

定义: 用于业务组合分析和资源分配决策的工具。

两个维度:

  • 市场增长率（纵轴）: 高/低
  • 相对市场份额（横轴）: 高/低

四类产品:

        高市场增长率
              │
    ┌─────────┼─────────┐
    │  问题类  │  明星类  │
    │(Question)│  (Star)  │
    │  选择性  │  投资发展 │
低市├─────────┼─────────┤高市
场份│  瘦狗类  │  金牛类  │场份
额  │  (Dog)   │  (Cash)  │额
    │  淘汰/收割│  维持/收割│
    └─────────┴─────────┘
        低市场增长率

策略建议:

  • 明星类: 加大投资，保持领先地位
  • 金牛类: 维持地位，收割现金流
  • 问题类: 选择性投资，转化为明星或淘汰
  • 瘦狗类: 逐步淘汰或收割

1.3 PEST分析

定义: 宏观环境扫描工具，分析影响企业的外部因素。

四个维度:

  • P (Political) 政治: 政策、法规、政治稳定性、政府态度
  • E (Economic) 经济: GDP、利率、通胀、汇率、消费能力
  • S (Social) 社会: 人口结构、文化习惯、生活方式、价值观
  • T (Technological) 技术: 技术发展趋势、创新速度、技术替代

扩展版PESTLE:

  • +L (Legal) 法律: 劳动法、环保法、知识产权法
  • +E (Environmental) 环境: 环保要求、可持续发展、气候变化

使用要点:

  • 每个维度列出3-5个关键因素
  • 评估每个因素的影响程度（高/中/低）
  • 评估每个因素的变化趋势（正向/负向/稳定）
  • 识别关键机会和威胁

1.4 STP模型

定义: 市场细分、目标市场选择、定位的经典营销框架。

三个步骤:

S (Segmentation) 市场细分:

  • 地理细分: 区域、城市、气候
  • 人口细分: 年龄、性别、收入、教育
  • 心理细分: 生活方式、价值观、个性
  • 行为细分: 使用频率、品牌忠诚度、购买动机

T (Targeting) 目标市场选择:

  • 无差异营销: 整个市场
  • 差异化营销: 多个细分市场
  • 集中营销: 单一细分市场
  • 微营销: 特定个体/地区

P (Positioning) 定位:

  • 确定差异化优势
  • 选择定位策略（领导者/挑战者/跟随者/利基）
  • 构建定位陈述
  • 传播定位信息

定位陈述模板:

对于[目标市场]，我们的[产品/品牌]是[品类]中，能够提供[关键利益]的[差异化特征]，因为[支持理由]。

1.5 安索夫矩阵

定义: 用于制定增长战略的工具，分析产品和市场的组合。

两个维度:

  • 产品: 现有产品 / 新产品
  • 市场: 现有市场 / 新市场

四种增长策略:

              现有产品          新产品
        ┌─────────────┬─────────────┐
        │   市场渗透   │   产品开发   │
  现有  │  (Penetrate) │  (Product)   │
  市场  │             │             │
        ├─────────────┼─────────────┤
        │   市场开发   │   多元化    │
  新市场 │  (Market)    │ (Diversify) │
        │             │             │
        └─────────────┴─────────────┘

策略详解:

  • 市场渗透: 现有产品+现有市场 → 提高市场份额
  • 市场开发: 现有产品+新市场 → 地理扩张/新细分市场
  • 产品开发: 新产品+现有市场 → 产品创新/升级
  • 多元化: 新产品+新市场 → 相关多元化/非相关多元化

风险等级: 市场渗透 < 市场开发 ≈ 产品开发 < 多元化

二、项目管理与执行类框架详解

2.1 七步问题解决法

定义: 麦肯锡经典的问题解决流程，系统化、结构化处理问题。

七个步骤:

步骤1: 界定问题

  • 明确问题的范围和边界
  • 区分症状与根本原因
  • 确定问题的紧迫性和重要性

步骤2: 分解问题

  • 使用逻辑树或议题树拆解
  • 遵循MECE原则
  • 识别关键子问题

步骤3: 优先排序

  • 评估各子问题的影响程度
  • 评估解决难度
  • 确定优先解决的子问题

步骤4: 分析计划

  • 确定分析方法
  • 明确数据来源
  • 制定时间计划

步骤5: 关键分析

  • 收集数据
  • 进行深入分析
  • 验证假设

步骤6: 综合建议

  • 整合分析结果
  • 形成结论
  • 提出具体建议

步骤7: 沟通实施

  • 准备汇报材料
  • 使用金字塔结构表达
  • 推动实施落地

2.2 逻辑树

定义: 将复杂问题分解为可管理子问题的层级结构工具。

三种类型:

议题树 (Issue Tree):

  • 用于: 问题分解、假设生成
  • 结构: 问题 → 子议题 → 具体议题
  • 特点: 开放式，探索性

假设树 (Hypothesis Tree):

  • 用于: 假设验证、快速决策
  • 结构: 假设 → 子假设 → 验证点
  • 特点: 结论先行，验证导向

决策树 (Decision Tree):

  • 用于: 方案选择、风险评估
  • 结构: 决策点 → 选项 → 结果
  • 特点: 量化评估，概率分析

MECE原则:

  • M (Mutually Exclusive) 相互独立: 各分支不重叠
  • CE (Collectively Exhaustive) 完全穷尽: 全覆盖无遗漏

构建步骤:

  1. 明确核心问题
  2. 确定分解维度
  3. 逐层分解至可执行
  4. 检查MECE原则
  5. 标注优先级

2.3 重要度/紧急度矩阵

定义: 任务优先级排序工具，又称艾森豪威尔矩阵。

四个象限:

              紧急
              │
    ┌─────────┼─────────┐
    │  第一象限 │  第二象限 │
    │ 重要且紧急│ 重要不紧急│
    │  立即做   │  计划做   │
高重├─────────┼─────────┤低重
要度│  第三象限 │  第四象限 │要度
    │ 紧急不重要│ 不重要不紧急│
    │  委托做   │  不做     │
    └─────────┴─────────┘
              不紧急

策略建议:

  • 第一象限: 立即处理，但尽量减少
  • 第二象限: 重点投入，预防第一象限
  • 第三象限: 委托他人或快速处理
  • 第四象限: 消除或减少

2.4 假设思考

定义: 基于假设快速推进，通过数据验证或证伪的分析方法。

核心流程:

生成假设:

  • 基于经验和直觉快速形成假设
  • 假设应该可验证、可证伪
  • 通常生成3-5个 competing hypotheses

定义验证标准:

  • 明确验证假设需要的数据
  • 确定证明/证伪的标准
  • 评估数据获取难度

快速验证:

  • 优先验证关键假设
  • 使用80/20原则
  • 快速迭代调整

形成结论:

  • 整合验证结果
  • 接受/拒绝/修正假设
  • 形成最终建议

适用场景:

  • 时间紧迫的决策
  • 信息不完整的分析
  • 创新性问题解决

三、逻辑思考与结构化分析类框架详解

3.1 MECE原则

定义: 结构化思维的核心原则，确保分析的完整性和独立性。

核心要义:

  • 相互独立 (Mutually Exclusive): 各分类之间无重叠
  • 完全穷尽 (Collectively Exhaustive): 所有分类加起来全覆盖

常见分解方法:

公式法:

  • 利润 = 收入 - 成本
  • 收入 = 价格 × 数量

流程法:

  • 按时间顺序分解
  • 按业务流程分解

要素法:

  • 按组成部分分解
  • 按影响因素分解

矩阵法:

  • 两个维度交叉
  • 形成分类矩阵

检查清单:

  • 是否有重叠? → 调整分类边界
  • 是否有遗漏? → 补充遗漏项
  • 是否在同一层级? → 调整层级关系

3.2 金字塔结构

定义: 结论先行的逻辑表达结构，适用于汇报、写作、沟通。

核心原则:

结论先行:

  • 先给出核心观点
  • 再展开支持论据
  • 符合听众认知习惯

以上统下:

  • 上层是下层的概括
  • 下层是上层的支撑
  • 层级关系清晰

归类分组:

  • 同组内容属于同一范畴
  • 按逻辑顺序排列
  • 遵循MECE原则

逻辑递进:

  • 时间顺序: 步骤、流程
  • 结构顺序: 空间、组成
  • 程度顺序: 重要性、优先级

金字塔结构图示:

              ┌─────────┐
              │  核心观点 │  ← 结论先行
              │ (G: Goal)│
              └────┬────┘
                   │
       ┌───────────┼───────────┐
       │           │           │
   ┌───┴───┐   ┌───┴───┐   ┌───┴───┐
   │ 论据A  │   │ 论据B  │   │ 论据C  │  ← 以上统下
   │       │   │       │   │       │
   └───┬───┘   └───┬───┘   └───┬───┘
       │           │           │
   ┌───┴───┐   ┌───┴───┐   ┌───┴───┐
   │支撑A1 │   │支撑B1 │   │支撑C1 │  ← 归类分组
   │支撑A2 │   │支撑B2 │   │支撑C2 │
   └───────┘   └───────┘   └───────┘

SCQA序言结构:

  • S (Situation) 情境: 背景信息
  • C (Complication) 冲突: 存在的问题
  • Q (Question) 问题: 核心疑问
  • A (Answer) 答案: 核心观点

3.3 五色光思维（原六顶思考帽）

定义: 悟空独创的结构化、同频化集体思考与决策系统，与六顶思考帽高度同构。

六顶帽子:

颜色 思维方向 关键问题 五色光对应
⬜ 白帽 客观事实 我们知道什么? 需要什么信息? ⬜ 白光
🔴 红帽 直觉感受 我的直觉是什么? 有什么预感? 🔴 红光
🟡 黄帽 积极价值 有什么好处? 价值在哪里? 🟡 黄光
🟢 绿帽 创新方案 还有什么可能性? 如何创新? 🟢 绿光
🔵 黑帽 风险控制 风险是什么? 有什么问题? 🔵 蓝光
🔵 蓝帽 过程控制 我们思考到哪里了? 下一步是什么? 🎯 主持人

使用方式:

  • 单独使用: 特定阶段聚焦特定思维
  • 序列使用: 按顺序切换不同帽子
  • 团队使用: 不同成员戴不同帽子

经典序列:

  • 白→绿→黄→黑→红→蓝（全面分析）
  • 蓝→白→绿→黄→黑→红→蓝（结构化讨论）

四、创意与问题解决类框架详解

4.1 SCAMPER法

定义: 系统性创新检查清单，用于改进现有产品/流程。

七个检查点:

字母 英文 中文 关键问题
S Substitute 替代 什么可以替代? 谁可以替代?
C Combine 组合 可以组合什么? 如何组合?
A Adapt 适应 可以适应什么? 借鉴什么?
M Modify 修改 可以修改什么? 放大/缩小?
P Put to other uses 他用 还有什么用途? 新场景?
E Eliminate 消除 可以消除什么? 简化什么?
R Reverse/Rearrange 逆转/重排 可以逆转什么? 重新排列?

使用步骤:

  1. 选择改进对象
  2. 依次检查7个维度
  3. 记录所有想法
  4. 评估可行性
  5. 形成改进方案

4.2 曼陀罗思考法

定义: 九宫格联想拓展工具，用于创意发散和系统思考。

结构:

┌─────┬─────┬─────┐
│  8  │  1  │  2  │
├─────┼─────┼─────┤
│  7  │  主题 │  3  │
├─────┼─────┼─────┤
│  6  │  5  │  4  │
└─────┴─────┴─────┘

使用方式:

  1. 中心填写核心主题
  2. 周围8格填写相关要素
  3. 每个要素可再扩展为新的曼陀罗
  4. 形成多层次思考网络

应用场景:

  • 创意发散
  • 计划制定
  • 问题分析
  • 学习整理

4.3 决策矩阵

定义: 多方案量化评估工具，用于复杂决策。

构建步骤:

确定评估标准:

  • 列出所有相关标准（5-8个）
  • 确定标准权重（总和100%）
  • 确保标准MECE

列出备选方案:

  • 收集所有可行方案
  • 确保方案互斥
  • 考虑创新方案

评分:

  • 每个方案在每个标准上评分（1-5或1-10）
  • 可使用多专家评分取平均
  • 记录评分依据

计算加权得分:

方案总分 = Σ(标准得分 × 标准权重)

决策:

  • 比较各方案总分
  • 考虑敏感性分析
  • 结合直觉判断
  • 做出最终决策

示例矩阵: | 标准 | 权重 | 方案A | 方案B | 方案C | |------|------|-------|-------|-------| | 成本 | 30% | 4 | 3 | 5 | | 效果 | 40% | 5 | 4 | 3 | | 风险 | 20% | 3 | 5 | 4 | | 速度 | 10% | 4 | 3 | 4 | | 加权总分 | | 4.1 | 3.9 | 3.8 |

六、与东方思维的融合

6.1 麦肯锡框架 vs 象思维

维度 麦肯锡框架 象思维
起点 结构化分解 整体直觉
过程 逻辑推演 观物取象
终点 量化结论 悟道明理
优势 1→N规模化 0→1原创性
适用 已知问题 未知问题

融合方式:

  1. 象思维洞察本质（0→1）
  2. 麦肯锡框架展开（1→N）
  3. 五色光验证完善

6.2 麦肯锡框架 vs 五行人格

五行 擅长框架类型 应用优势
木 创新类（SCAMPER、头脑风暴） 创意发散、突破常规
火 表达类（金字塔、五色光思维） 感染力强、推动共识
土 执行类（七步法、流程图） 落地稳健、持续跟进
金 分析类（MECE、逻辑树） 逻辑严谨、精准切割
