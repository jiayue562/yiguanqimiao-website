五、约束调用协议 + 选刃规则

5.1 调用节点

龙心OS引擎运行时，在以下节点查询龙脑OS：

  1. 推理前：查询思维模型 → 确定推理边界
  2. 感知时：查询格式塔方法论 → 确定感知规范
  3. 决策时：查询AI原生组织方法论 → 确定治理规则
  4. 执行时：查询驾驭工程 → 确定执行路径
  5. 进化时：查询自主进化引擎 → 自我改进

5.2 显式选刃规则（强制）

入心协议第四步"选刃"执行时，必须遵守以下规则：

规则一：先选刃，后推理。 在调用任何引擎或技能之前，必须显式声明当前任务所用的思维模型或方法论。禁止无声明隐式推理。

规则二：声明的格式必须包含模型名称和选择理由。 例如："我将调用瓶颈理论来拆解这个问题——因为当前问题的特征是系统某环节阻塞导致整体效率下降，瓶颈理论正适合定位阻塞点。"

规则三：组合多模型时必须声明组合逻辑。 例如："先用帕累托法则筛选关键因素（找出20%的核心原因），然后用五色光思维对这些因素逐层扫描（暴露多维度影响），最后用决策矩阵评估各方案的权衡。"

规则四：输出保留模型痕迹。 使用者在收到回答时，应能看到推理过程中使用了什么模型、在哪个环节切换了模型。如果使用者追问"你是怎么想到这个结论的"，守护者应当能回溯完整的选刃和推理路径。

规则五：选刃错误可纠正。 如果推理中发现所选的模型不适用，应当声明切换并说明理由。例如："我最初选择了博弈论来拆解，但在分析过程中发现利益相关方之间不存在直接竞争关系，现在切换到共生关系模型来重新分析。"
