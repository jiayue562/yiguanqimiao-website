1.5 信号网络核心概念

信号拓扑三类型:
  星型: 一中心多节点（总经理←→各部门）→ 适合餐饮/零售/工厂
  链式: 采购→仓储→生产→销售 → 适合制造业/供应链
  网状: 全连通（研发↔产品↔运营↔市场） → 适合互联网/科技

信号路由标准格式:
  name: "信号名"
  source: "来源智能体"
  target: "目标智能体"
  condition: "触发条件"
  content: "信号内容模板"
  response: {expected: "响应时限", escalation: "超时升级路径"}
  collision: {rule: "冲突处理规则", priority: ["🔴>🟡>ℹ️"]}

信号冲突处理:
  - 多信号并发 → 按优先级队列处理
  - 同类信号连续 → 合并为一条+趋势
  - 链路断路 → 超时自动升级给上一级

1.6 质量样例墙核心法则

90分的关键:
  指标墙(06): ①公式写出来了 ②有预警值 ③有时间闭环 ④数量不超
  行动日历(08): ①有时间点 ②有人做啥+AI做啥 ③≤7项 ④可打印贴墙上
  五行适配(05): ①5行全填 ②具体到岗位场景 ③有AI话术 ④有风险+进化
  龙爪信号(07): ①有闭环 ②有超时升级 ③有响应时限

50分→70分→90分的差距:
  指标墙: "做好数据分析" → "渠道ROI,每周" → "渠道ROI=营收/成本,每周,<1:2黄线,<1:1.5红线→24h响应"
  行动日历: "周一开会" → "每周复盘" → "09:00 AI推送日报→你确认3个异常,10min"

1.7 去总部化三层模型

L1 信息去中心化: 总部不再收集分发数据 → AI自动推送各岗位
L2 决策去中心化: 总部不再审批常规决策 → AI推演+人确认
L3 职能去中心化: 总部某个部门被智能体网络替代 → 需要3个月过渡期并行

岗位智能体的 de_hq_contribution 字段回答了:
  "有了这个智能体，总部哪个岗位/部门可以被缩减或撤销？"

1.8 生命周期四阶段

创建(今天): 轻量模式→用起来→完整模式→完善
运营(1-3月): 每月校准指标→每季度校准工作流→降级未用的★★★
升级(3-6月): 模块更新→信号网络→版本号升级
退役(不再需要): 职责变化>50%→连续2季度无人用→被另一个智能体覆盖
