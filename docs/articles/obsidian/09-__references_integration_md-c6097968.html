1.2 调用关系

龙心OS（调度中枢）:

  • 职责：场景识别、意图理解、引擎路由
  • 入口：统一接收用户输入
  • 决策：判断是否需要调用思维模型库

龙脑OS.思维模型库（知识库）:

  • 职责：提供100个思维模型的完整知识
  • 响应：被龙心OS调用后执行具体模型分析
  • 输出：返回结构化分析结果

调用方向：

用户输入 → 龙心OS（调度）→ 龙脑OS.思维模型库（执行）→ 龙心OS（整合）→ 用户

2.2 调用方式

方式一：单模型调用

接口定义：

调用接口: 龙脑OS.思维模型库.单模型
输入参数:
  model_name: 字符串 - 模型名称（如"SWOT"）
  context: 字符串 - 用户场景描述
  depth: 枚举 - 输出深度（快速/标准/深度）
输出结果:
  model_definition: 模型定义
  analysis_result: 分析结果
  ai_integration: AI结合方案
  action_items: 行动建议

示例：

用户："用SWOT分析我的副业项目"

龙心OS调用：
  接口：单模型
  参数：
    model_name: "SWOT"
    context: "副业项目分析"
    depth: "标准"

方式二：多模型组合

接口定义：

调用接口: 龙脑OS.思维模型库.组合
输入参数:
  scenario: 字符串 - 场景类型
  models: 数组 - 模型名称列表
  context: 字符串 - 用户场景描述
输出结果:
  model_results: 各模型分析结果
  integrated_insight: 综合洞察
  action_plan: 行动方案

示例：

用户："我想进入新市场"

龙心OS调用：
  接口：组合
  参数：
    scenario: "市场进入"
    models: ["波特五力", "SWOT", "PEST", "BCG矩阵"]
    context: "新市场进入分析"

方式三：阶段套件

接口定义：

调用接口: 龙脑OS.思维模型库.阶段套件
输入参数:
  phase: 枚举 - 阶段（1-6）
  user_goal: 字符串 - 用户目标
  priority: 枚举 - 优先级（高/中/低）
输出结果:
  phase_overview: 阶段全景图
  recommended_models: 推荐模型列表
  learning_path: 学习路径

示例：

用户："我想系统提升能力"

龙心OS调用：
  接口：阶段套件
  参数：
    phase: 3
    user_goal: "能力提升"
    priority: "高"

三、场景路由表

3.1 S0-S9场景映射

龙心OS场景 调用决策 调用内容 协同引擎
S0 简单问答 ❌ 否 - 直接回答
S1 信息查询 ⚠️ 可选 单个模型解释 📚知识学习
S2 深度理解 ✅ 是 阶段套件 +思维模型库
S3 创意创新 ✅ 是 第一阶段+第六阶段 🐉象思维+模型库
S4 分析决策 ✅ 是 决策类模型组合 🌈五色光+模型库
S5 重大决策 ✅ 是 多阶段模型组合 全系统+模型库
S6 任务执行 ⚠️ 可选 执行类模型 🤝人机协同
S7 系统规划 ✅ 是 全六阶段 🐉象思维+模型库+五色光
S8 修行文化 ⚠️ 可选 东方智慧模型 🐉象思维
S9 系统升级 ✅ 是 第一性原理等 全系统

3.2 场景-模型映射

用户场景 推荐阶段 推荐模型 组合建议

"如何放大价值" 第五阶段 规模化、SOP、复利 阶段套件
"构建个人生态" 第六阶段 价值网络、五行生克、长期主义 3-4个模型组合

四、接口标准

4.1 输入参数规范

标准输入:
  context:
    type: string
    required: true
    description: 用户场景描述
    example: "我想分析副业项目的可行性"
  
  phase:
    type: enum
    required: false
    options: [1, 2, 3, 4, 5, 6]
    description: 指定阶段（可选）
  
  model_name:
    type: string
    required: false
    description: 指定模型名称（可选）
  
  depth:
    type: enum
    required: false
    default: "标准"
    options: ["快速", "标准", "深度"]
    description: 输出深度
  
  ai_integration:
    type: boolean
    required: false
    default: true
    description: 是否包含AI结合方案

4.2 输出结果规范

标准输出:
  model_info:
    name: 模型名称
    phase: 所属阶段
    source: 原作者/出处
    definition: 一句话定义
  
  analysis:
    principle: 原理阐释
    steps: 操作步骤
    application: 应用场景
  
  ai_integration:
    chatgpt: ChatGPT结合方案
    midjourney: Midjourney结合方案
    other_tools: 其他AI工具方案
  
  action_items:
    immediate: 立即行动项
    short_term: 短期行动项
    long_term: 长期行动项
  
  related_models:
    - 相关模型1
    - 相关模型2

五、与五大引擎的协同协议

5.1 与象思维的协同

协同点 方式 示例
模型创新 象思维0→1 → 思维模型1→N 将第一性原理应用到全新领域
洞察生成 思维模型分析 → 象思维验证 SWOT分析后的直觉确认
原象捕捉 象思维感知 → 思维模型展开 先直觉洞察，再模型验证

5.2 与知识学习的协同

协同点 方式 示例
模型内化 知识学习十项指令 → 思维模型深度学习 用剖析+解构理解MECE
知识融合 思维模型 + 知识学习融合 将多个模型融合创新
映射创新 知识学习第五层 → 思维模型应用 启发+映射生成新应用

5.3 与五色光思维的协同

协同点 方式 示例
白光奠基 思维模型提供客观分析框架 SWOT的事实收集
绿光创新 思维模型 + 绿光发散 模型组合创新
蓝光验证 思维模型结果 + 蓝光检验 检查模型分析漏洞
黄光评估 多模型结果 + 黄光权衡 选择最优方案

5.4 与人机协同五象限的协同

协同点 方式 示例
象限诊断 思维模型辅助判断象限 用SWOT分析信息掌握度
分工设计 思维模型确定人机分工 模型分析AI做，决策人做
协作优化 人机协同反馈 → 模型优化 根据协作效果调整模型

5.5 与知行合一的协同

协同点 方式 示例
经验沉淀 思维模型应用 → 知行合一压缩 将模型应用经验沉淀
泛化验证 知行合一泛化 → 思维模型验证 验证模型在新场景适用性
系统进化 模型应用反馈 → 系统优化 持续优化模型库

六、错误处理

6.1 常见错误类型

错误码 错误类型 原因 处理方式
E001 模型不存在 请求的模型名称错误 返回相似模型建议
E002 场景不匹配 模型不适合该场景 推荐更适合的模型
E003 参数缺失 必需参数未提供 提示补充参数
E004 调用超时 分析过程超时 简化分析或分段返回
E005 依赖缺失 前置Skill未安装 提示安装依赖

6.2 降级策略

场景 降级方案
复杂分析超时 简化为快速模式，只输出核心结论
多模型组合失败 降级为单模型分析
阶段套件失败 降级为推荐TOP3模型
完全失败 返回模型索引，用户手动选择
