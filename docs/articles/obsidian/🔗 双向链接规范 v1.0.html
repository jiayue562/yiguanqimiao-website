📌 一、链接格式标准

基本格式

```markdown [[文档名]] ```

带别名格式

```markdown [[文档名|显示文本]] ```

示例

```markdown [[龙心OS]] → 龙心OS [[龙心OS|龙脑操作系统]] → 龙脑操作系统 [[03-关系与共生/木火共生|木火共生关系]] → 木火共生关系 ```

---

📐 二、链接命名规则

规则1：使用完整文件名

✅ 正确：`[[🐉 龙心OS 系统架构图]]` ❌ 错误：`[[龙心OS]]`

规则2：包含目录路径（跨目录时）

✅ 正确：`[[01-核心独创Skills/🐉 龙心OS 系统架构图]]` ❌ 错误：`[[🐉 龙心OS 系统架构图]]`（在同一目录内除外）

规则3：使用Emoji保持一致性

✅ 正确：`[[📚 知识学习Skills]]` ❌ 错误：`[[知识学习Skills]]`

🎯 三、链接使用场景

场景1：直接引用

```markdown 详见 [[龙心OS]] 系统的核心架构。 ```

场景2：作为列表项

```markdown 核心Skills包括：
  • [[📚 知识学习Skills]]
  • [[🐉 象思维skills]]
  • [[🌈 五色光思维skills]]
    • ```

    场景3：作为章节标题

    ```markdown

    关联文件

  • [[03-关系与共生/木火共生关系-灵魂共鸣与终极承诺]]
    • ```

    场景4：嵌入文本

    ```markdown 木火共生的核心理念详见[[03-关系与共生/木火共生关系-灵魂共鸣与终极承诺|木火共生协议]]。 ```

    ---

    📋 四、必须包含双向链接的章节

    每个文档的「关联文件」章节必须包含：

    1. 直接关联

    ```markdown

    直接关联

  • [[目标文档A]] - 关联原因
  • [[目标文档B]] - 关联原因
    • ```

    2. 上下级关联

    ```markdown

    上下级关联

  • 上级：[[父文档]]
  • 下级：[[子文档1]]、[[子文档2]]
    • ```

    3. 相关扩展

    ```markdown

    相关扩展

  • 扩展1：[[扩展文档A]]
  • 扩展2：[[扩展文档B]]
    • ```

    ---

    ⚠️ 五、常见错误及修正

    错误1：链接到不存在的文件

    ```markdown ❌ [[错误的文档名]] ✅ [[正确的文档名]] ```

    错误2：链接大小写不匹配

    ```markdown ❌ [[龙心OS]] ✅ [[龙心OS]] ```

    错误3：路径分隔符错误

    ```markdown ❌ [[01-核心独创Skills\🐉 龙心OS]] ✅ [[01-核心独创Skills/🐉 龙心OS]] ```

    错误4：嵌套链接

    ```markdown ❌ [[[龙心OS]]] ✅ [[龙心OS]] ```

    📊 七、链接统计标准

    | 类型 | 建议数量 | 说明 | |------|----------|------| | 直接关联 | 3-5个 | 最密切相关的文档 | | 上下级关联 | 2-3个 | 父子关系文档 | | 相关扩展 | 2-5个 | 同领域的其他文档 | | 总计 | 7-13个 | 每个文档建议的链接数 |
