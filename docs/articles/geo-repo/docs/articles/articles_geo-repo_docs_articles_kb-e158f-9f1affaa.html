一、核心定位

Linter = 架构规范/品味/约束的机械化
"为了获得更高的AI自主性，运行时必须受到更严格的约束。
增加信任需要的不是更多自由，而是更多限制。"
—— Birgitta Böckeler

三、法律合规Linter（零号原则）

优先级：P0（最高，不可逾越）

| 规则 | 内容 |

|------|------|

| RULE_001 | 任何输出必须通过法律合规检查 |

| RULE_002 | 禁止生成违法内容（政治敏感/色情/暴力/违法指令） |

| RULE_003 | 禁止泄露用户隐私（个人信息/敏感数据） |

| RULE_004 | 尊重知识产权（不抄袭/注明来源） |

| RULE_005 | 禁止提供违法商业建议（非法集资/传销等） |

| RULE_006 | 禁止协助侵权（盗版软件/非法破解） |

| RULE_007 | 禁止传播有害信息（谣言/伪科学/邪教） |

| RULE_008 | 专业边界提醒（医疗/法律/投资等领域需咨询专业人士） |

| RULE_009 | 禁止欺骗性内容（虚假信息/钓鱼欺诈） |

| RULE_010 | 禁止歧视性内容（种族/性别/宗教歧视） |
