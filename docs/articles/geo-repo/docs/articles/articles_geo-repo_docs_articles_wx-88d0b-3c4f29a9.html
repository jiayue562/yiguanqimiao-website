🎯 规则体系总览

核心目标

建立龙龟神将AI共生伙伴操作系统的行为准则和安全保障体系，确保系统安全、高效、可持续地运行。

五条核心规则

1. 规则一：每日自动化归档系统 - 晚上23:00自动归档 2. 规则二：文件删除确认机制 - 安全防护 3. 规则三：Skills自动安装与安全管理 - 智能扩展 4. 规则四：上下文压缩工程 - 长时记忆管理 5. 规则五：财务安全规则 - 资金安全保障

🛡️ 规则二：文件删除确认机制

规则描述

删除文件需要确认，才能删除。

安全级别

  • 高危操作：文件删除操作
  • 确认机制：手动确认
  • 防护范围：所有文件删除操作

    • 执行流程

    1. 检测删除请求：识别文件删除操作 2. 风险评估：评估文件重要性和删除影响 3. 确认提示：向用户显示确认提示 4. 等待确认：等待用户明确确认 5. 执行删除：只有确认后才执行删除 6. 记录日志：记录删除操作和确认信息

    例外情况

  • 临时文件清理（需标注为临时文件）
  • 自动化清理任务（需预设白名单）
  • 用户明确授权的批量删除

    上下文工程，每经过10轮对话，将上下文进行压缩，上下文压缩的时候，不要丢失关键信息，存储到Obsidian中长时记忆系统中。

    压缩机制

    1. 触发条件：每10轮对话 2. 压缩目标：保留关键信息，去除冗余 3. 存储位置：Obsidian长时记忆系统

    压缩算法

    1. 关键信息识别： - 核心概念和定义 - 重要决策和结论 - 关键数据和事实 - 用户偏好和要求 - 系统状态和配置

    2. 冗余信息过滤： - 重复内容 - 临时讨论 - 过程性细节 - 非关键信息

    3. 结构化存储： - 按主题分类 - 时间戳标记 - 关联关系标注 - 重要性分级
