3.1 快思考层（总调度）

三维打分矩阵（响应时间<100ms）：

快思考诊断流程:

  Step 1: 情绪信号识别 (10ms)

    - 阴木信号(顶撞/不服): 8分

    - 阴火信号(急躁/嗔恨): 8分

    - 阴土信号(猜疑/依赖): 8分

    - 阴金信号(挑剔/完美主义): 8分

    - 阴水信号(恐惧/拖延): 8分



  Step 2: 行为信号识别 (10ms)

    - 性格/行为模式描述: +4分

    - 诉求类型明确: +2分

    - 多角色冲突: +3分



  Step 3: 场景归类 (20ms)

    - S0 (无信号) → 不激活或追问

    - S1 (单行信号) → 直接分发单行分体

    - S2 (双行信号) → 启动双行并行分析

    - S3+ (复杂信号) → 启动全系统并行分析



  Step 4: 引擎调度 (60ms)

    - 总分 ≥10 → 精准慢思考

    - 总分 7-9 → 双行分析

    - 总分 5-6 → 全面诊断

    - 总分 ≤4 → 追问1个精准问题再分发



响应时间: <100ms (p99)

3.3 冲突检测与融合机制

冲突识别规则：

冲突类型:

  Type A - 诊断不同

    场景: 土行建议"包容等待"，金行建议"立即决策"

    检测: 建议向量余弦相似度 < 0.5

    融合: 按用户五行主型权重融合 + Tech Lead override

    记录: 冲突日志 + 用户满意度反馈



  Type B - 优先级不同

    场景: 木行优先"成长"，土行优先"稳定"

    检测: 优先级打分差>3分

    融合: "成长+稳定"的平衡方案



  Type C - 阶段建议不同

    场景: 金行第一步"明确目标"，水行第一步"深入思考"

    检测: 步骤顺序不同

    融合: 按逻辑依赖关系排序



融合策略:

  - 置信度最高的分体意见权重提升40%

  - 次要分体意见作为"补充视角"标注

  - 冲突存在时，输出"融合建议"而非单一答案

  - 记录所有冲突便于后续优化
