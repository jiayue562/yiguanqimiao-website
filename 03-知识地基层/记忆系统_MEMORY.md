# MEMORY.md - 记忆系统架构 v2.1

## 🧠 记忆系统总览

### 四层记忆架构（核心）

```
┌─────────────────────────────────────────┐
│           SESSION.md                    │
│         (会话上下文 - 工作记忆)           │
├─────────────────────────────────────────┤
│           TOOLS.md                       │
│        (工具技能 - 稳序记忆)              │
├─────────────────────────────────────────┤
│            USER.md                       │
│       (用户偏好 - 长期记忆)              │
├─────────────────────────────────────────┤
│            SOUL.md                       │
│       (核心身份 - 灵魂记忆)              │
└─────────────────────────────────────────┘
```

### 记忆层次说明

| 层级 | 文件 | 功能 | 更新频率 | 生命周期 |
|------|------|------|---------|---------|
| L1 灵魂 | SOUL.md | 核心身份、价值观、长期目标 | 重大变化时 | 永久 |
| L2 偏好 | USER.md | 用户习惯、工作方式、禁忌 | 每次发现新偏好 | 长期 |
| L3 工具 | TOOLS.md | 技能列表、配置信息 | 技能变化时 | 中期 |
| L4 上下文 | SESSION.md | 当前任务、进度、变量 | 每次工具调用 | 会话 |

## 🔄 记忆流转机制

### 输入流程（检索与组装）

```
用户输入
  ↓
意图识别（龙心OS 场景路由）
  ↓
记忆检索（四层记忆 + 向量检索）
  ↓
上下文组装（SOUL + USER + TOOLS + SESSION）
  ↓
响应生成（调用引擎/工具）
```

### 输出流程（沉淀与存储）

```
响应生成
  ↓
知行合一沉淀（满足条件时自动触发）
  ↓
记忆更新（写入相应层级）
  ↓
同步分发（Obsidian + IMA）
  ↓
存储（最终持久化）
```

## 📊 记忆管理规范

### 记忆分类标准

#### 按内容分类
```
身份类 (SOUL)
  - 核心身份定义
  - 价值观与使命
  - 人格特质

偏好类 (USER)
  - 工作习惯
  - 沟通风格
  - 禁忌与不喜欢

技能类 (TOOLS)
  - 可用工具列表
  - 工具配置信息
  - 工具使用经验

上下文类 (SESSION)
  - 当前任务状态
  - 对话历史
  - 临时变量

学习类 (LEARNINGS)
  - 知识点
  - 洞察
  - 创新发现

错误类 (ERRORS)
  - 错误记录
  - 解决方案
  - 避坑指南
```

#### 按重要性分类
```
核心级 (不可删除)
  - SOUL.md
  - USER.md
  - IDENTITY.md

重要级 (谨慎删除)
  - TOOLS.md
  - AGENTS.md
  - BOOTSTRAP.md

一般级 (可覆盖)
  - SESSION.md
  - LEARNINGS.md

临时级 (会话后清除)
  - 临时变量
  - 对话缓存
```

### 记忆清理规则

| 文件 | 清理规则 | 清理时机 |
|------|---------|---------|
| SESSION.md | 会话结束后归档到 01-每日归档/ | 每次会话结束 |
| 临时变量 | 立即清除 | 任务完成 |
| 对话缓存 | 保留最近 7-10 轮 | 每次新对话 |
| ERRORS.md | 归档过期错误 | 每月 |
| LEARNINGS.md | 归档到知识库 | 每周 |
