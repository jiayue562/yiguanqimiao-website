## 🌟 核心功能

### 功能 1: 知识编译（Ingest）

**脚本**: `compiler/ingest.py`

**流程**:
```
原始资料 → Layer 1 (保存)
    ↓
提取核心概念 (3-5 个)
    ↓
生成 Wiki 页面 → Layer 2
    ↓
更新索引
```

**输出**:
- 原始资料归档（带哈希）
- 实体/概念页面生成
- 双向链接建立
- 索引更新

---

### 功能 2: 健康检查（Lint）

**脚本**: `compiler/lint.py` + `scripts/health-check.py`

**检查指标**:
| 指标 | 计算方式 | 健康范围 |
|------|----------|----------|
| 链接率 | 有链接页面/总页面 | >80% |
| 孤立率 | 无链接页面/总页面 | <5% |
| 平均链接数 | 总链接数/总页面 | ≥3 |

**输出**:
- 健康分数 (0-100)
- 问题列表
- 修复建议
- 详细报告

---

### 功能 3: 索引重建（Rebuild）

**脚本**: `scripts/rebuild-index.py`

**流程**:
```
扫描所有页面 → 提取概念 → 构建索引
    ↓
生成 INDEX.md + link-graph.json
```

**输出**:
- 概念索引（按字母排序）
- 页面列表
- 链接图谱（JSON）
- 高连接度页面排行

---

### 功能 4: 三层架构

**架构规范**: `architecture/layer3-schema/structure.md`

**Layer 1: 原始资料层**
- 只读的事实来源
- 保持原始性，不修改
- 归档带元数据（来源/时间/哈希）

**Layer 2: Wiki 本体层**
- LLM 维护的知识图谱
- 实体/概念/对比/摘要页面
- 双向链接网络

**Layer 3: Schema 配置层**
- 机器的行为准则
- 结构规范/质量规则/触发规则

## 🎯 使用示例

### 示例 1: 录入文档到 Wiki

```python
# 使用 ingest.py
from compiler.ingest import ingest_document

content = """
# React Query 核心概念

React Query 是一个强大的数据获取库...
"""

result = ingest_document(
    content=content,
    source_url="https://tanstack.com/query",
    source_type="article"
```

## ⚠️ 关键规则

### 编译规则

1. **原始性保护**: Layer 1 原始资料只读，不修改
2. **语义完整**: 编译不丢失核心语义
3. **链接建立**: 每个页面至少 3 个双向链接
4. **版本标注**: 明确标注来源和版本

### 质量规则

5. **一致性**: 概念定义无冲突
6. **完整性**: 核心概念无遗漏
7. **可检索**: 索引完整，支持快速查找
8. **可进化**: 支持 Linting 自我修复

### 协同规则

9. **与记忆系统分工**: LLM Wiki=知识生产，记忆系统=经验沉淀
10. **与 self-learning 协同**: self-learning 学习 → LLM Wiki 编译
11. **与 IMA 协同**: 本地 Wiki → IMA 云端同步
