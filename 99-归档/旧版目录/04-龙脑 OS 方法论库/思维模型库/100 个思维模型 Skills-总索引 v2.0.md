## 📋 架构说明

### 设计理念

> **先搭建框架，再按需完善**——所有思维模型建立目录索引和简版内容，达到可根据实际情况单个或组合使用的目的。详细 Skill 封装在实际应用中按需完善。

### 三层架构

| 层级 | 模型数 | 状态 | 用途 |
|------|--------|------|------|
| **核心层** | 9 个 | ✅ 完整版 | 高频本能反应，龙心 OS 核心能力 |
| **扩展层** | 20 个 | 📝 简版 | 中频场景激活，按需完善 |
| **工具层** | 71 个 | 📝 简版 | 低频按需加载，用时完善 |

### 简版 Skill 结构

```
[思维模型名称]/
├── SKILL.md (简版)          # 核心定义 + 触发词 + 基本用法
├── references/
│   └── theory.md (待完善)    # 理论文档
└── triggers/
    └── trigger-rules.yaml   # 触发规则
```
