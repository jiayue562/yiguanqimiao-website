修复方案：Clean URL文件Content-Type问题

问题诊断

| URL类型 | HTTP状态 | Content-Type | 浏览器行为 | |---------|----------|-------------|-----------| | .html 文件 | 200 OK | text/html ✅ | 正常渲染 | | Clean URL（无扩展名） | 200 OK | application/octet-stream ❌ | 触发下载 |

影响AI爬取：✅ 是。application/octet-stream会被AI爬虫跳过，不解析内容。

修复方案

✅ 方案A已成功实施

结果：_headers 精确路径规则生效，所有Clean URL的Content-Type从 application/octet-stream 变为 text/html。

| 路径 | 前 | 后 | |------|-----|-----| | /articles/obsidian/* | application/octet-stream 🚫 | text/html ✅ | | /articles/wechat-articles/* | application/octet-stream 🚫 | text/html ✅ | | /articles/geo-repo/* | application/octet-stream 🚫 | text/html ✅ |

关键经验：之前用 /articles/*（broad）导致路由异常；用具体目录路径则正常。

方案A：创建 _headers 文件（推荐，优先尝试）

在 docs/_headers 中添加精确路径规则（避免之前 /articles/* 导致的路由异常）：

/articles/obsidian/*
  Content-Type: text/html
/articles/wechat-articles/*
  Content-Type: text/html
/articles/geo-repo/*
  Content-Type: text/html
注意：上次用 /articles/* 导致所有文章返回首页。这次使用更窄、更具体的路径，应该不会触发该bug。

方案B（兜底）：删除无扩展名副本，使用 .html URL

如果方案A的 _headers 仍然导致路由异常，则： 1. 删除 docs/ 下所有无扩展名文件副本（3546个） 2. 修改 generate_indexes.py 生成 .html URL链接 3. 删除首页js中的 .replace(/\.html$/, '') 4. 重新部署

执行步骤

1. 创建 _headers 文件（方案A） 2. Wrangler部署 (仅更新少量文件) 3. 验证Content-Type → 检查是否为 text/html 4. 如果方案A失败 → 执行方案B 5. 最终验证: 浏览器打开链接，不再触发下载

涉及文件

| 文件 | 操作 | |------|------| | docs/_headers | 创建（方案A）或忽略 | | scripts/generate_indexes.py | 可能需要修改URL格式（方案B） | | docs/index.html | 可能需要修改js（方案B） |

验证方法

验证Clean URL的Content-Type

curl -s -I "https://yiguanqimiao-website.pages.dev/articles/obsidian/WorkBuddySync_overview" | grep -i "content-type"

期望结果：Content-Type: text/html
