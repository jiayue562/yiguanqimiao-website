岗位智能体 · 阶段二架构设计

构建日期: 2026-04-20
Skill Builder版本: v1.1
阶段: 第二阶段·架构设计（Step 5-8）

一、Skill粒度判断（Step 5）

1.1 判断矩阵

| 维度 | 判断标准 | 本Skill情况 | 结果 | |------|----------|-------------|------| | 操作步数 | 3-7步→单体，8步以上→生态 | 7步 | ✅ 单体 | | 决策点数 | 1-2个→单体，3个以上→生态 | 4个 | ⚠️ 接近边界 | | 依赖关系 | 独立→单体，有依赖链→生态 | 依赖五行人格识别 | ⚠️ 外部依赖 | | 触发复杂度 | 简单→单体，复杂→生态 | 中等 | ✅ 单体 |

1.2 粒度决策

最终判断: 单体Skill（Monolithic Skill） 决策理由: 1. 核心流程7步，符合单体Skill范围 2. 4个决策点虽略多，但逻辑清晰、层次递进 3. 依赖五行人格识别是调用现有Skill（外部依赖），不是内部子模块 4. 触发条件相对明确，无需复杂路由 未来扩展路径: 
v1.0: 单体Skill（当前）
    ↓ 如果使用场景增加
v2.0: 可升级为生态Skill
    ├─ 岗位智能体-主Skill（调度）
    ├─ 岗位智能体-提示词生成子Skill
    ├─ 岗位智能体-工作流设计子Skill
    ├─ 岗位智能体-指标体系子Skill
    └─ 岗位智能体-风险预警子Skill

二、触发机制设计（Step 6-7）

2.1 四维触发矩阵

P0·直接触发词（权重5·绝对触发）

| 序号 | 触发词 | 场景示例 | |------|--------|----------| | 1 | 岗位智能体 | "启动岗位智能体" | | 2 | 生成提示词 | "帮我生成提示词" | | 3 | 岗位AI | "设计一个岗位AI" | | 4 | 工作流设计 | "需要工作流设计" | | 5 | 智能体方案 | "给我智能体方案" | | 6 | 构建岗位助手 | "构建一个市场总监助手" | | 7 | AI岗位赋能 | "AI赋能岗位" |

触发条件: 出现任意1个P0词 → 立即触发（5分）

P1·场景触发词（权重4·强触发）

| 序号 | 场景类型 | 触发词 | 示例 | |------|----------|--------|------| | 1 | 岗位描述场景 | 市场总监/总经理/运营经理 | "我是市场总监，需要..." | | 2 | 行业描述场景 | 餐饮行业/科技行业/教育行业 | "我在餐饮行业工作" | | 3 | 性格描述场景 | 水行人/火行人/木行人 | "我是水行人，性格灵活" | | 4 | 任务描述场景 | 渠道投放/用户增长/品牌策划 | "负责精准渠道投放" | | 5 | 指标描述场景 | KPI/OKR/结果指标/过程指标 | "需要设定KPI指标" | | 6 | 风险控制场景 | 风险预警/过程控制/切换机制 | "ROI低于预期怎么办" |

触发条件: 出现1-2个P1词 → 累积4分 → 触发

P2·信号触发（权重3·弱触发）

| 序号 | 信号类型 | 识别特征 | 权重 | |------|----------|----------|------| | 1 | 六维框架信号 | 用户同时提到行业+岗位+任务 | 3分 | | 2 | 五行人格信号 | 用户描述性格特质（如"我很固执"） | 3分 | | 3 | 工作流信号 | 用户询问"信息交汇点"或"AI使用时机" | 3分 | | 4 | 指标设计信号 | 用户询问"如何量化目标"或"如何考核" | 3分 | | 5 | 模型选择信号 | 用户询问"用什么模型分析" | 3分 |

触发条件: 出现≥2个P2信号 → 累积6分 → 触发

P3·条件触发（权重2·特殊场景）

| 序号 | 条件类型 | 触发条件 | 权重 | |------|----------|----------|------| | 1 | 多轮对话累积 | 连续3轮对话涉及岗位+任务 | 2分 | | 2 | 否定现有方案 | 用户说"这个提示词不够精准" | 2分 | | 3 | 优化诉求 | 用户说"帮我优化这个岗位的AI使用" | 2分 |

触发条件: 出现≥4个P3条件 → 累积8分 → 触发

2.2 触发决策树

用户输入
    ↓
[计算触发分数]
    ↓
┌─────────────────────────────────────────┐
│ if 包含P0关键词（岗位智能体/生成提示词等）│
│    → 【立即触发】进入岗位智能体模式     │
│    → 总分 = 5分（绝对触发）             │
└─────────────────────────────────────────┘
    ↓ 否
┌─────────────────────────────────────────┐
│ elif P1场景信号 ≥ 1个                   │
│    → 【强触发】询问是否需要岗位智能体   │
│    → 总分 = 4分                         │
└─────────────────────────────────────────┘
    ↓ 否
┌─────────────────────────────────────────┐
│ elif P2行为信号 ≥ 2个                   │
│    → 【弱触发】观察后建议               │
│    → 总分 = 6分（3分×2）                │
└─────────────────────────────────────────┘
    ↓ 否
┌─────────────────────────────────────────┐
│ elif P3条件累积 ≥ 4个                   │
│    → 【条件触发】特殊场景处理           │
│    → 总分 = 8分（2分×4）                │
└─────────────────────────────────────────┘
    ↓ 否
┌─────────────────────────────────────────┐
│ else                                    │
│    → 【不触发】静默观察                 │
│    → 总分 < 3分                         │
└─────────────────────────────────────────┘

协同协议:

1. 与象思维协同: - 触发条件: 创新类岗位任务 - 调用方式: 岗位智能体推荐→用户确认→象思维激活 - 信息交换: 岗位智能体提供任务框架，象思维提供0→1突破

2. 与五色光思维协同: - 触发条件: 复杂岗位问题分析 - 调用方式: 岗位智能体内置五色光分析模块 - 信息交换: 五色光提供白/红/黄/绿/蓝光分析

3. 与知行合一协同: - 触发条件: 岗位智能体方案使用后 - 调用方式: 自动触发（alwaysApply） - 信息交换: 岗位智能体输出→知行合一三阶段沉淀

4. 与知识学习协同: - 触发条件: 需要深度学习岗位知识时 - 调用方式: 用户主动请求或自动建议 - 信息交换: 岗位智能体生成学习任务→知识学习执行

5. 与人机协同五象限协同: - 触发条件: 设计人机协作模式时 - 调用方式: 岗位智能体推荐协作象限 - 信息交换: 岗位智能体提供岗位特性，五象限推荐协作模式
