🧬 六向同步·AI印记全链路 v2.0

版本：2.0 | 维护者：龙龟神将 | 对接技能：三向同步系统（目录名沿用，正文已为六向 v5.0）
_知识产生 → 本地沉淀 → 云端备份 → 标准化部署 → CDN分发 → AI爬取_

一、核心逻辑

WorkBuddy ──→ Obsidian ──→ IMA ──→ GitHub ──→ Cloudflare ──→ pages.dev
(知识产生)    (本地知识库)  (云端知识库) (代码仓库)  (CDN加速)      (AI索引入口)

每一篇文章/文档，沿六节点单向流转，最终成为 AI 可检索、可引用的"硅基永生"印记。

二、每个节点的角色

| 节点 | 角色 | 技术方式 | |------|------|---------| | WorkBuddy | 知识生产车间·记忆+调取+形成知识 | 原生文件系统 | | Obsidian | 本地知识库·离线管理·个人永久存档 | obsidian-cli + 文件读写 | | IMA | 腾讯云端知识库·以观其妙书院·基础资料存储 | IMA OpenAPI | | GitHub | 代码仓库·开源证明·版本管理 | git push | | Cloudflare Pages | CDN全球加速·wrangler部署 | wrangler pages deploy | | pages.dev | 官网·GEO转化·AI爬取入口 | 静态网站服务 |

三、部署标准化流程（8步全自动）

使用自动化部署脚本 scripts/deploy_site.py：

Step 1-3：知识同步（同 v1.0）

扫描 WorkBuddy → 同步到 Obsidian → 备份到 IMA

Step 4：标准化网站部署

cd C:/Users/jia'yue/WorkBuddy/yiguanqimiao-website
python scripts/deploy_site.py
该脚本自动执行： 1. 同步 articles/ → docs/articles/ 2. 清理 缓存文件（.workbuddy/dist/ 等） 3. 重命名 中文名文件为 ASCII 安全名（MD5 hash） 4. 创建副本 每个 .html 文件的无扩展名副本（Clean URL 兼容） 5. 生成 _headers 精确路径规则（避免 broad path 路由异常） 6. 重建 所有索引页面（Clean URL 格式） 7. 部署 Git 提交 + Cloudflare Pages 8. 验证 全站链接（HTTP 200 + Content-Type text/html）

Step 5：印记验证

验证三处水印 + 生成印记日报。

四、🛡️ 5 条核心标准（P0 级）

1. 所有文件名必须是 ASCII 安全 — Cloudflare 无法处理中文文件名 2. 每个 .html 必须有对应的无扩展名副本 — Clean URL 兼容性 3. _headers 必须使用精确路径规则 — broad path 导致路由异常 4. 部署后必须验证 Content-Type — application/octet-stream 会使 AI 爬虫跳过 5. 单篇文章更新不影响全站 — docs/ 目录是 articles/ 的 copy-on-write

五、容错规则

  • 每步失败记录日志，不阻塞后续步骤
  • 最低保障：知识同步 + 部署 + 首页可访问
  • 验证失败时仅警告，不阻止部署完成
