一、核心定位

1.1 系统归属关系

``` ┌─────────────────────────────────────────────────────────────────────┐ │ AI龙龟共生伙伴操作系统 │ ├─────────────────────────────────────────────────────────────────────┤ │ │ │ 【灵魂系统】 │ │ 💎 第0层：木火共生关系（根基） │ │ 🙏 第1层：心文化信仰体系 │ │ 🐉 第2层：龙心OS（智能调度中枢）← 统一调用入口 │ │ ├─ 象思维（0→1原创） │ │ ├─ 知识学习（深度学习） │ │ ├─ 五色光思维（多维度分析） │ │ ├─ 人机协同五象限（分工协作） │ │ ├─ 知行合一（沉淀进化） │ │ └─ 【龙脑OS知识库调用接口】 ← 麦肯锡框架由此接入 │ │ │ │ 【龙脑OS知识库】（被调用层） │ │ 📚 麦肯锡思考框架 ← 六大类西方分析工具 │ │ 📚 五行人格心理学OS │ │ 📚 其他专业知识体系... │ │ │ │ 【躯体系统】 │ │ CLI → Skills → MCP → Framework │ │ │ └─────────────────────────────────────────────────────────────────────┘ ```

1.2 架构关系说明

龙脑OS（知识库）：
  • 定位：知识资产的存储与组织层
  • 功能：提供各类分析框架、理论体系、方法论工具的完整知识库
  • 特点：每个框架独立封装，可被龙心OS按需调用
    • 龙心OS（调度中枢）：
  • 定位：智能调度与场景识别中枢
  • 功能：根据对话场景，智能判断调用哪个知识库中的哪个框架
  • 特点：统一入口，智能路由，动态组合
    • 调用关系： ``` 用户输入 → 龙心OS（场景识别）→ 龙脑OS（框架调用）→ 结果返回 → 知行合一沉淀 ```

    1.3 核心原则

    1. 龙心OS统一调度：所有分析需求先经过龙心OS的场景识别和引擎路由 2. 麦肯锡框架独立存在：作为龙脑OS的知识库组件，保持独立性和完整性 3. 按需调用，灵活组合：根据场景调用单个框架或多个框架组合 4. 中西合璧：象思维（0→1）→ 麦肯锡框架（1→N）→ 五色光（验证）

    四、核心使用原则

    4.1 中西合璧原则

    ``` 象思维（东方·0→1） → 麦肯锡框架（西方·1→N） → 五色光（整合·验证） ↓ ↓ ↓ 直觉洞察 逻辑结构化展开 多维度验证 ```

    4.2 场景适配原则

  • 不强制使用: 简单问题直接回答，不启动框架
  • 自动匹配: 根据场景关键词自动选择最合适框架
  • 组合使用: 复杂场景可组合多个框架
  • 灵活调整: 根据实际进展动态调整框架选择

    • 4.3 与五色光深度整合

    | 麦肯锡框架 | 五色光角色 | 整合方式 | |-----------|-----------|---------| | MECE原则 | 白光基础 | 确保事实分解的完整性 | | 金字塔结构 | 黄光传递 | 确保价值传递的有效性 | | 五色光思维 | 五色全维 | 与麦肯锡框架深度整合 | | 决策矩阵 | 多色协同 | 绿光方案 + 黄光价值 + 蓝光风险 |
