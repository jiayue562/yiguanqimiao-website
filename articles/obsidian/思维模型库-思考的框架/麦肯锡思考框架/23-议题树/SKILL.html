一、核心定义

1.1 一句话定义

议题树是将复杂议题/问题结构化分解为子议题，通过层层追问Why/How，穷尽议题维度，确保分析全面不遗漏。

1.2 与逻辑树的区别

| 维度 | 逻辑树 | 议题树 | |------|--------|--------| | 核心 | 问题分解 | 议题分解 | | 重点 | 解决问题 | 全面覆盖议题 | | 结构 | Why（原因） | Why/How（原因/解决方案） | | 适用 | 问题诊断 | 议题分析 |

---

二、结构类型

2.1 Why型议题树（原因导向）

``` [核心议题] 为什么？ / | \ Why1 Why2 Why3 / \ | / \ ... ... ... ... ...

示例： [销售额下降] Why？ / | \ 需求降 供给降 竞争强 ↓ ↓ ↓ Why? Why? Why? ```

2.2 How型议题树（方案导向）

``` [目标] How？ / | \ How1 How2 How3 / \ | / \ ... ... ... ... ...

示例： [提升销售额] How？ / | \ 提单价 提销量 开新渠道 ↓ ↓ ↓ How? How? How? ```

---

三、构建步骤

3.1 标准流程

``` 第一步：明确核心议题/问题 ↓ 第二步：确定分解角度（Why或How） ↓ 第三步：第一层分解（3-5个维度） ↓ 第四步：第二层分解（追问） ↓ 第五步：检验MECE（无遗漏、无重复） ↓ 第六步：标注优先级（关键维度） ```

3.2 分解检查清单

``` □ 分解维度是否相互独立？ □ 是否覆盖了议题的主要方面？ □ 每个分支是否可以进一步分解？ □ 分解是否到达可执行层级？ □ 是否有遗漏的重要维度？ ```

四、实战模板

4.1 市场战略议题树

``` 【议题树 · 市场战略选择】

目标：选择最优市场进入策略 How？ / | \ 深度做 广度做 创新做 ↓ ↓ ↓ 现有 新市场 新模式 城市 下沉 差异化 ↓ ↓ ↓ How? How? How?

■ 深度做（现有城市） - 如何提升份额？ - 如何深耕用户？ - 如何建立壁垒？

■ 广度做（开拓新城市） - 如何选择城市？ - 如何快速复制？ - 需要多少资源？

■ 创新做（新模式） - 有哪些创新可能？ - 风险如何？ - 如何验证？ ```

4.2 组织问题议题树

``` 【议题树 · 员工离职率高】

核心议题：为什么离职率高？ Why？ / | \ 钱少 发展 文化 ↓ ↓ ↓ Why? Why? Why?

■ 钱少 - 薪酬低于市场？ - 薪酬结构不合理？ - 晋升调薪慢？

■ 发展 - 看不到晋升空间？ - 学不到新东西？ - 能力被埋没？

■ 文化 - 工作氛围差？ - 领导管理方式？ - 公司价值观不合？ ```
