📖 核心定义

What（是什么）

龙脑OS是龙心OS的知识库智能体，采用三层架构升级方案：

``` ┌─────────────────────────────────────────┐ │ 第1层：概念层（文档澄清） │ │ ├── 明确声明：龙脑OS本质是智能体 │ │ └── 区分"龙脑OS智能体"vs"普通Skill" │ ├─────────────────────────────────────────┤ │ 第2层：架构层（Framework突破） │ │ ├── 内部通过Framework实现自主调度 │ │ ├── 突破Skill框架限制 │ │ └── 实现智能体核心特征 │ ├─────────────────────────────────────────┤ │ 第3层：载体层（Skill结构） │ │ ├── 利用WorkBuddy基础设施 │ │ ├── 触发机制、版本管理、知识库集成 │ │ └── 保持与现有生态兼容性 │ └─────────────────────────────────────────┘ ```

智能体 vs Skill 的本质区别：

| 维度 | 智能体（龙脑OS） | Skill（普通工具） | |------|-----------------|------------------| | 核心特征 | 自主感知、场景理解、引擎路由、持续学习 | 被动触发、确定性流程、固定输出 | | 运行方式 | 主动感知上下文，自主决策 | 等待用户明确指令 | | 学习能力 | 持续学习，动态优化 | 固定逻辑，无自学习 | | 载体形式 | 用Skill结构承载智能体灵魂 | 纯工具化封装 |

Why（为什么）

问题：龙心OS需要调用多种知识库（思维模型、五行人格、专业知识），如何统一管理和智能调度？ 解决方案：
  • ✅ 统一入口：所有知识库调用都经过龙脑OS
  • ✅ 智能路由：根据场景自动匹配最合适的知识库
  • ✅ 组合能力：支持多知识库组合分析
  • ✅ 持续学习：根据使用反馈优化路由策略

    • How（怎么做）

    一句话工作流： ``` 用户输入 → 龙心OS场景识别 → 龙脑OS智能路由 → 匹配知识库 → 执行分析 → 结果整合 → 知行合一沉淀 ``` 核心流程： 1. 场景识别：龙心OS识别用户场景和需求类型 2. 知识库路由：龙脑OS判断调用哪个知识库 3. 模型匹配：在知识库中选择具体模型或框架 4. 执行分析：调用选中模型执行分析 5. 结果整合：整合分析结果返回龙心OS
