人机协同五象限标准操作流程（SOP）

> 📋 标准化操作流程 · 确保协作质量

---

📋 SOP 概述

目的

  • 标准化人机协同五象限的使用流程
  • 确保协作质量和效能
  • 支持经验沉淀与持续改进

    • 适用范围

  • 所有人机协作场景
  • 所有使用五象限模型的用户
  • 所有需要优化 AI 协作的任务

    • 核心流程

    ``` 任务接收 → 场景分析 → 象限匹配 → 协作执行 → 效能评估 → 经验沉淀 ```

    ---

    🔄 标准操作流程

    流程 1：任务接收与分类

    步骤 1.1：接收任务 ``` 输入：用户任务描述 处理：记录任务基本信息 输出：任务卡片 ``` 任务卡片模板： ```markdown

    任务卡片

    基本信息

  • 任务 ID：[自动生成]
  • 任务名称：[名称]
  • 接收时间：[时间]
  • 期望完成：[时间]

    • 任务描述

    [详细描述]

    任务特征

  • 类型：[执行/学习/思考/探索/创造]
  • 复杂度：[低/中/高/极高]
  • 不确定性：[确定/部分不确定/高度不确定]
  • 紧急度：[低/中/高]

    • 初步判断

  • 推荐象限：[第 X 象限]
  • 预计时长：[时长]
  • 备注：[特殊说明]
    • ``` 步骤 1.2：任务分类 ``` 决策树： 1. 任务是否规则明确？ - 是 → 第一象限 - 否 → 继续判断

    2. 任务是否以获取知识为主？ - 是 → 第二象限 - 否 → 继续判断

    3. 任务是否需要深度思考？ - 是 → 第三象限 - 否 → 继续判断

    4. 任务是否探索未知？ - 是 → 第四象限 - 否 → 第五象限 ```

    ---

    流程 2：场景分析与象限匹配

    步骤 2.1：场景分析 ``` 分析维度： 1. 任务类型分析 2. 复杂度评估 3. 不确定性评估 4. 人类参与度需求 5. AI 能力匹配度 ``` 场景分析表： ```markdown | 维度 | 评估选项 | 选择 | 说明 | |------|----------|------|------| | 任务类型 | 执行/学习/思考/探索/创造 | [选择] | [说明] | | 复杂度 | 低/中/高/极高 | [选择] | [说明] | | 不确定性 | 确定/部分不确定/高度不确定 | [选择] | [说明] | | 人类参与度 | 低/中/高 | [选择] | [说明] | | AI 能力匹配 | 高/中/低 | [选择] | [说明] | ``` 步骤 2.2：象限匹配 ``` 匹配规则：
  • 执行类 + 低复杂度 + 确定 → 第一象限
  • 学习类 + 中复杂度 + 部分不确定 → 第二象限
  • 思考类 + 高复杂度 + 部分不确定 → 第三象限
  • 探索类 + 极高复杂度 + 高度不确定 → 第四象限
  • 创造类 + 极高复杂度 + 未知 → 第五象限
    • ``` 象限匹配报告： ```markdown

    象限匹配报告

    推荐象限

    第 [X] 象限：[协作模式名称]

    匹配理由

    1. [理由 1] 2. [理由 2] 3. [理由 3]

    角色定位

  • AI 角色：[角色名称]
  • 人类角色：[角色名称]

    • 备选方案

  • 备选象限：[第 Y 象限]
  • 切换条件：[什么情况下切换]
    • ```

    流程 3：协作方案设计

    步骤 3.1：协作流程设计 ``` 设计要素： 1. 协作步骤（分阶段） 2. 人机分工（谁做什么） 3. 交接点（何时交接） 4. 验收标准（如何验收） ``` 协作流程模板： ```markdown

    协作流程设计

    阶段划分

    | 阶段 | 任务 | 负责人类 | 负责 AI | 输出 | |------|------|----------|--------|------| | 1 | [任务] | [职责] | [职责] | [输出] | | 2 | [任务] | [职责] | [职责] | [输出] | | 3 | [任务] | [职责] | [职责] | [输出] |

    交接点

  • 交接点 1：[条件] → [交接内容]
  • 交接点 2：[条件] → [交接内容]

    • 验收标准

  • 阶段 1 验收：[标准]
  • 阶段 2 验收：[标准]
  • 最终验收：[标准]
    • ``` 步骤 3.2：效能指标设计 ``` 指标类别： 1. 效率指标（时间节省） 2. 质量指标（成果质量） 3. 学习指标（认知扩展） 4. 创新指标（新想法数量） 5. 满意度指标（人类满意度） ``` 效能指标表： ```markdown | 指标类别 | 具体指标 | 目标值 | 测量方式 | |----------|----------|--------|----------| | 效率 | 时间节省倍数 | [X] 倍 | 人工时间/AI 时间 | | 质量 | 一次性通过率 | [X]% | 验收通过比例 | | 学习 | 新概念掌握 | [X] 个 | 前后测试 | | 创新 | 新想法数量 | [X] 个 | 计数 | | 满意度 | 满意度评分 | [X]/5 | 评分 | ```

    流程 4：协作执行与监控

    步骤 4.1：启动协作 ``` 启动检查清单： □ 任务卡片已填写完整 □ 象限匹配报告已生成 □ 协作流程已设计 □ 效能指标已定义 □ 人类已准备好 □ AI 已就绪 ``` 步骤 4.2：执行协作 ``` 执行原则： 1. 按协作流程执行 2. 记录关键节点 3. 监控效能指标 4. 及时调整优化 ``` 协作日志模板： ```markdown

    协作日志

    任务信息

  • 任务 ID：[ID]
  • 协作象限：第 [X] 象限
  • 开始时间：[时间]

    • 执行记录

    | 时间 | 阶段 | 人类行动 | AI 行动 | 产出 | 备注 | |------|------|----------|--------|------|------| | :00 | 阶段 1 | [行动] | [行动] | [产出] | [备注] | | :00 | 阶段 2 | [行动] | [行动] | [产出] | [备注] |

    效能监控

    | 指标 | 目标值 | 当前值 | 状态 | |------|--------|--------|------| | [指标 1] | [目标] | [当前] | ✅/⚠️/❌ | | [指标 2] | [目标] | [当前] | ✅/⚠️/❌ |

    调整记录

    | 时间 | 调整内容 | 调整原因 | 效果 | |------|----------|----------|------| | [时间] | [调整] | [原因] | [效果] | ``` 步骤 4.3：异常处理 ``` 异常情况处理流程：

    1. 效能低下 - 识别：效能指标低于阈值 - 处理：重新评估象限匹配 - 行动：调整协作模式

    2. 任务变更 - 识别：任务需求变化 - 处理：更新任务卡片 - 行动：重新设计协作流程

    3. AI 能力不足 - 识别：AI 无法完成任务 - 处理：人类接管或降级象限 - 行动：调整人机分工

    4. 人类过度依赖 - 识别：人类放弃主导权 - 处理：提醒人类主导权 - 行动：调整分工界面 ```
