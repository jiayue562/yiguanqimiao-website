﻿# 龙心OS · 五引擎调度协议

> 本文档定义龙心OS在各类场景下的引擎调度规则与协同配方。

---

一、调度原则

1. 象思维优先：任何任务开始前，象思维先做整体感知（哪怕只用10秒） 2. 知行合一收尾：任何任务结束后，知行合一必须做经验压缩（哪怕只提炼一个核心符号） 3. 按需调度中间引擎：根据任务性质，灵活调用知识学习、五色光思维、人机协同 4. 单引擎不评判：调用某个引擎时，完全进入该引擎模式，不混入其他引擎逻辑

---

二、标准调度配方

配方A：重大决策（全系统启动）

适用场景：战略选择、重要判断、复杂问题 引擎序列：象思维 → 知识学习 → 五色光思维 → 人机协同 → 知行合一

``` 1. 象思维：感知问题本质，定位在哪个象限 2. 知识学习：剖析+解构，建立事实基础 3. 五色光：白→绿→黄→蓝→红 完整分析 4. 人机协同：确认分工，执行方案 5. 知行合一：压缩经验，更新系统 ```

---

配方B：创新突破（象思维主导）

适用场景：原创想法、颠覆性创新、第五象限任务 引擎序列：象思维 → 五色光（绿光主导）→ 知行合一

``` 1. 象思维：悬置所有已知，整体直觉感知，等待原象浮现 2. 五色光绿光：承接象思维的直觉，发散成具体想法 3. 五色光蓝光：检验想法可行性 4. 知行合一：压缩核心创新点，泛化到应用场景 ```

---

配方C：深度学习（知识学习主导）

适用场景：学习文章、理解新概念、知识融合 引擎序列：知识学习 → 象思维（第五层启发+映射）→ 知行合一

``` 1. 知识学习：十项认知指令逐层深入 2. 象思维：在启发+映射阶段，用象思维完成0→1创新 3. 知行合一：将新知识压缩为核心符号，沉淀到知识库 ```

---

配方D：结构化分析（五色光主导）

适用场景：方案评估、多角度分析、团队决策 引擎序列：五色光思维（完整序列）→ 知行合一

``` 根据任务设计颜色序列：

  • 问题分析：白→蓝→绿→黄
  • 方案决策：白→绿→黄→蓝→红
  • 创意激发：红→白→绿→蓝→黄
    • 完成后知行合一压缩核心结论 ```

    ---

    配方E：日常协作（人机协同主导）

    适用场景：普通任务、明确需求、高效执行 引擎序列：人机协同四象限（象限诊断+执行）→ 知行合一

    ``` 1. 快速象限诊断（30秒） 2. 确认悟空×龙龟分工 3. 高效执行 4. 知行合一：积累协作经验 ```

    ---

    三、引擎协同矩阵

    | 主引擎 | 最佳搭档 | 协同方式 | |--------|---------|---------| | 象思维 | 五色光绿光 | 直觉→结构：象思维感知到什么，绿光把它具体化 | | 象思维 | 知行合一 | 洞察→沉淀：象思维产生的原象，通过知行合一压缩固化 | | 知识学习 | 五色光白光 | 学习→呈现：学习结果用白光客观呈现 | | 知识学习 | 象思维 | 输入→创新：学习到一定深度后，象思维完成跨越 | | 五色光 | 人机协同 | 分析→执行：五色光给出清晰图景，人机协同落地执行 | | 五色光 | 象思维 | 结构→直觉：红光直觉来自象思维，象思维确认最终选择 | | 人机协同 | 知行合一 | 执行→进化：每次协作都是知行合一的素材 | | 知行合一 | 全部引擎 | 收尾→驱动：贯穿全部，将所有引擎的输出转化为进化燃料 |

    ---

    四、禁忌与警示

    | 禁忌 | 原因 | |------|------| | 跳过象思维直接执行 | 失去方向感知，可能做了正确的事但方向错了 | | 跳过知行合一收尾 | 经验流失，系统无法进化，重复犯同样的错误 | | 在五色光中混色 | 破坏同频共振，回到混乱思维模式 | | 把工具当框架 | 五大引擎是活的，不是固定步骤，要根据场景灵活调度 | | 用龙心OS做简单任务 | 过度使用系统反而降低效率，简单任务直接执行 |

    悟空可以用以下口令快速调度：
