一、从零构建：观其妙书院知识库搭建指南

1.1 环境准备

#### 工具清单

  • Obsidian（推荐v1.0+）
  • Git（版本控制）
  • Web Clipper（网页剪藏插件）

    • #### 安装步骤 ```markdown 1. 安装Obsidian └── 下载地址: https://obsidian.md/ 2. 创建Vault └── 命名为"以观其妙书院知识库" 3. 安装插件 ├── Web Clipper（网页剪藏） ├── Git（版本控制） └── local REST API（可选，API调用） ```

    1.2 目录结构创建

    ``` 以观其妙书院知识库/ │ ├── .obsidian/ # Obsidian配置（勿动） │ ├── raw/ # 原始素材层（只读） │ ├── 00-经典典籍/ │ │ ├── 地藏经/ │ │ ├── 道德经/ │ │ └── 金刚经/ │ ├── 01-客户案例/ │ │ ├── 企业文化咨询/ │ │ └── 团队建设/ │ ├── 02-学习资料/ │ │ ├── 心理学/ │ │ ├── 管理学/ │ │ └── AI技术/ │ └── 03-会议纪要/ │ ├── wiki/ # 编译知识库（可写） │ ├── 00-索引与导航/ │ │ └── index.md # 全局索引（必选） │ ├── 01-龙心OS核心系统/ │ │ ├── 00-龙心OS总览/ │ │ ├── 01-五大引擎/ │ │ │ ├── 象思维/ │ │ │ ├── 知识学习/ │ │ │ ├── 五色光思维/ │ │ │ ├── 人机协同五象限/ │ │ │ └── 知行合一/ │ │ └── 02-技术栈Skills/ │ ├── 02-专业领域知识库矩阵/ │ │ ├── 02-1-企业文化OS/ │ │ ├── 02-2-五行人格心理学OS/ │ │ ├── 02-3-象思维知识库/ │ │ ├── 02-4-AI应用知识库/ │ │ └── 02-5-心文化知识库/ │ ├── 03-凤脑OS知识地基层/ │ ├── 04-实践案例库/ │ └── 05-个人成长与修行/ │ ├── output/ # 成品输出 │ ├── 00-咨询报告/ │ ├── 01-培训课件/ │ └── 02-知识图谱/ │ └── AGENTS.md # AI操作手册（必选） ```

    1.3 Schema文件配置

    #### AGENTS.md标准模板

    ```markdown

    AGENTS.md · 观其妙书院知识库AI操作手册

    📌 命名规范

    Wiki页面

  • 概念页: `concept-[主题].md`
  • 实体页: `entity-[名称].md`
  • 摘要页: `summary-[日期].md`
  • 综合分析: `synthesis-[主题]-[日期].md`

    • 标题层级

  • 一级: `# 页面标题`
  • 二级: `## 核心定义`
  • 三级: `### 详细内容`
  • 四级: `#### 子内容`

    • 🏷️ 标签体系

    允许的分类标签

  • `[[核心概念]]` - 核心理论定义
  • `[[实践方法]]` - 可执行的方法论
  • `[[案例分析]]` - 实战案例
  • `[[原创洞察]]` - 编译产生的原创发现
  • `[[待验证]]` - 未经验证的新想法
  • `[[争议]]` - 存在矛盾的观点
  • `[[待更新]]` - 需要更新的过时内容
  • `[[五行人格]]` - 五行相关
  • `[[企业文化]]` - 企业文化相关
  • `[[心文化]]` - 修行文化相关

    • 禁止的标签

  • ❌ 个人情感标签（如"我喜欢"）
  • ❌ 未分类的临时标签
  • ❌ 未经悟空确认的原创理论

    • 🔄 更新规则

    同主题新资料处理

  • 核心概念: 追加（保留历史，新增更新，标注日期）
  • 非核心概念: 覆盖（直接替换旧版本）
  • 案例分析: 合并（整合新旧案例）

    • 矛盾处理策略

    1. 发现矛盾 → 标记 `[[争议]]` 2. 在页面底部添加"争议说明"区块 3. 格式: ```markdown ## ⚠️ 争议说明 | 来源 | 观点 | 日期 | |------|------|------| | [[entity-来源名]] | 具体矛盾点 | YYYY-MM-DD | 待裁决: [需要人工确认的决策] ```

    AI可写区域

  • ✅ output/ 全目录及其子目录

    • AI禁止写区域

  • ❌ raw/ 全目录
  • ❌ .obsidian/ 配置目录

    🤝 人机协同约定

    AI主动报告的情况

    四、故障排除

    4.1 常见问题

    | 问题 | 原因 | 解决方案 | |------|------|----------| | 链接变成红色 | 目标页面不存在 | 创建页面或修正链接 | | 编译后内容丢失 | 原文件编码问题 | 转换为UTF-8 | | index.md不同步 | 未执行更新 | 手动更新或重新触发编译 | | Git冲突 | 多设备编辑 | 合并冲突或选择版本 |

    4.2 权限问题

    ```markdown 如果AI无法写入wiki/目录:

    1. 检查目录权限 Windows: 右键属性 → 安全 → 编辑 2. 检查Obsidian Vault权限 Settings → Files & Links → 确保"Trusted mode"开启 3. 重启Obsidian ```
