LLM Wiki 知识编译 SOP

📝 标准操作流程 · v1.0

标签: #LLMWiki #知识编译 #SOP #操作流程

📋 核心目标

将原始资料 (文章/PDF/文档) 编译为结构化 Wiki 页面，建立双向链接，执行健康检查，实现知识复利增长。

⏰ 适用场景

  • 深度阅读学术文章/专业文档
  • 建立领域知识库
  • 知识沉淀与方法论提炼
  • 跨文章知识关联与综合

🔧 操作流程

阶段一：准备 (2 分钟)

Step 1: 确认原始资料

检查项目:
[ ] 原始资料位置 (raw/articles/)
[ ] 文件格式 (PDF/MD/TXT)
[ ] 文件大小 (<10MB 为佳)
[ ] 版权许可 (可编译)

Step 2: 确定编译目标

问题:
1. 这篇文章的核心主题是什么？
2. 需要提炼几个核心概念？
3. 与哪些现有知识相关？

阶段二：编译 (15-30 分钟)

Step 1: 读取原始素材

# 使用 markitdown-local 技能读取
markitdown-local "raw/articles/文章名.pdf"

输出: Markdown 格式的原始内容

Step 2: 提炼核心概念

使用知识学习 Skill 的十项认知指令:

L1 基础解析:
- 剖析：拆解核心概念
- 解构：提取语义单元

L2 深度理解:
- 透视：把握本质
- 阐释：多角度展开

L3 批判建构:
- 推演：逻辑外推
- 思辨：批判性分析

L4 跨域整合:
- 溯源：追根溯源
- 融合：跨学科整合

L5 创新生成:
- 启发：生成新问题
- 映射：框架重构

输出: 核心概念清单 (3-5 个)

Step 3: 生成 Wiki 页面

# [概念名]

> 一句话定义 (30 字内)

---

## 核心定义

[200-300 字详细定义]

## 学术谱系

| 理论 | 来源 | 贡献 |
|------|------|------|
| ... | ... | ... |

## 核心观点

- 观点 1
- 观点 2
- 观点 3

## 应用场景

1. 场景 1
2. 场景 2

## 双向链接

- [[相关概念 1]]
- [[相关概念 2]]
- [[相关理论]]

## 标签

#标签 1 #标签 2 #标签 3

---

_创建：YYYY-MM-DD · 来源：[原始资料]_

输出: wiki/concepts/概念名.md

Step 4: 建立双向链接

检查清单:
[ ] 在新生成的页面中添加 [[相关概念]]
[ ] 在相关概念页面中添加 [[新页面]]
[ ] 检查链接是否有效 (无 404)
[ ] 更新分类索引

阶段三：健康检查 (5-10 分钟)

Step 1: 一致性检查

检查项目:
[ ] 概念定义是否一致？
[ ] 术语使用是否统一？
[ ] 引用格式是否规范？
[ ] 标签是否符合规范？

Step 2: 完整性检查

检查项目:
[ ] 核心概念是否有页面？
[ ] 重要链接是否建立？
[ ] 索引是否更新？
[ ] 元数据是否完整？

Step 3: 连接性检查

# 使用 obsidian-agent 检查
obsidian-agent orphans
obsidian-agent broken-links

目标:

  • 孤立页面 <20%
  • 断裂链接 <5 个

Step 4: 生成健康检查报告

# Wiki 健康检查报告

**检查日期**: YYYY-MM-DD
**检查范围**: [范围]

## 统计

| 指标 | 数值 | 变化 |
|------|------|------|
| 总页面数 | | |
| 孤立页面 | | |
| 断裂链接 | | |

## 问题

1. [问题描述]
2. [问题描述]

## 建议

1. [建议]
2. [建议]

阶段四：索引更新 (3-5 分钟)

Step 1: 更新分类索引

# [分类] 索引

## 新增页面

- [[页面 1]] - 简短说明
- [[页面 2]] - 简短说明

## 最近更新

- [[页面 3]] - YYYY-MM-DD 更新

Step 2: 更新总索引

# 00-知识库总索引

## 最近更新

- YYYY-MM-DD: [更新内容]

阶段五：反哺同步 (可选，5-10 分钟)

Step 1: 询问用户

"是否将本次编译的知识同步到三库？"

选项:
[ ] 同步到 OpcClaw 知识库
[ ] 同步到 IMA 知识库
[ ] 仅保留在 Obsidian

Step 2: 执行同步

# 使用三向同步脚本
cd ~/.agents/skills/qclaw-knowledge-sync/scripts
.\sync-all.ps1 -Direction obsidian-to-opclaw
.\sync-all.ps1 -Direction obsidian-to-ima
