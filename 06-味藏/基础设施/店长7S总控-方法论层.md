## 调度逻辑

### 双向调度（DAG驱动）
```
店长驱动（从下往上）：
  门店需求 → 赋能工单 → SM-CHIEF路由 → 对应职能部门
  → 4h响应 → 24h方案 → 48h执行 → 门店评价

总部赋能（从上往下）：
  战略目标 → SM-CHIEF分解 → 各岗位Tasks
  → 执行反馈 → 日反馈→周复盘→月分析
```

### 日反馈路由器
```
每日定时（10:00-22:30）：
  POS数据 → 采集 → 标记异常 → 填充模板 → 推送到店长
  店长补充判断 → 确认发布 → 闭环归档
```
