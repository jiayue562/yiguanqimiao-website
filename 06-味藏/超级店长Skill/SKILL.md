# 超级店长Skill · SUPER-SM 总控系统

> **核心公式（v1.7.0 共识版）**：**超级店长 = 总控Skill(大脑) × [ N=26~33个模块Skills(手脚) ] × 7S v1.7.0骨架(脊柱) × 五行人格(灵魂) × 龙龟喂养(进化)**
> **本质定义**：超级店长不是替代店长，不是工具箱，不是被动响应——是围绕店长岗位的认知操作系统。Skill本身=数字分身，人工是使用者，不进入Skill公式。
> **质量评级**：⭐⭐⭐⭐⭐⭐⭐⭐⭐⭐ (10/10 卓越级)
> **源文档**：A:0521录音分析报告 + B:7S×龙爪OS架构 + C:小组讨论决策 → 建立步骤.pdf（9页·4463字）

## 零、系统定位：7S理论守门人

### 为什么需要超级店长Skill？

> "店长可能有100件事，混乱无序。要把这一骨碌东西归拢到7S里面去——这个东西无非就在7S里面的某一块。"

超级店长Skill是**味藏7S理论的守门人**。所有子Skill必须在7S骨架内运行，任何偏离7S理论的内容都会被拦截和纠正。

### 子智能体自动调度

```
超级店长Skill (父Skill·总控大脑·7S守门人)
    │ auto_trigger_sub = true
    ├──→ 店长7S总控 (SM-CHIEF) ← 自动触发·调度技能
    │       ├──→ 16个子Skill (日/周/月/常驻)
    │       └──→ F1-F5五职能引擎
    ├──→ [未来] N个模块Skills ← 待构建
    └──→ 每个子Skill必须通过7S分类校验
```

## 一、核心公式四层分解

| 层级 | 组件 | 功能 | 核心原理 |
|------|------|------|---------|
| **大脑** | 总控Skill | 统一调度、信息汇总、决策路由 | AI负责信息收集→分析→方案输出 |
| **手脚** | N个模块Skills | 具体事务执行（日/周/月） | 一个Skill只解决一件完整的事 |
| **脊柱** | 7S骨架 | 所有工作的分类归拢框架 | 无序工作→有序7S分类→可SOP化 |
| **灵魂** | 五行人格 | 个性化管人方式、适配方案 | 按性格特点出不同方案，实现超级个性化 |

## 二、三大定位澄清（不是 / 是）

| ❌ 不是 | ✅ 是 |
|--------|------|
| 替代店长 | **共生伙伴**：AI做AI擅长的，店长做店长擅长的 |
| 工具箱 | **认知操作系统**：不是给一堆散装工具，是围绕岗位的一体化智能 |
| 被动响应 | **主动认知**：自动收集数据、分析问题、给出方案，店长只做判断和执行 |

## 三、80/20 分工法则

### 核心逻辑

```
店长：80%解决人的问题 + 20%解决业务问题
    ├── 人员档案（招人/管人/育人）
    └── 客户档案（客户关系/会员运营）

三驾马车（料理长/前厅经理/线上）：80%解决业务问题 + 20%解决人的问题（培训）
    ├── 料理长 → 产品搞定
    ├── 前厅经理 → 服务搞定
    └── 线上/外卖 → 外卖搞定
```

### 店长核心抓两个档案

| 档案 | 内容 | AI支撑 |
|------|------|--------|
| **人员档案** | 谁适合干什么、性格特点、能力边界 | 五行人格诊断+面谈记录+九宫格 |
| **客户档案** | 老饕分类、消费偏好、关系维护 | RFM分析+储值追踪+感动案例 |

> "人是一切资源。店长一定要招能解决问题的人进来，而不是找一个人进来。"

## 五、7S骨架 — 工作归拢框架（守门人核心）

### 核心原则

> "把所有店长工作按7S分类 → 每个S下面形成多个Skills → 每个S变身一个'龙爪'"

### 7S-to-Skills 映射框架

| S模块 | 覆盖范围 | 预计Skills数 | AI/人分工倾向 |
|------|---------|:--:|:--:|
| S1 战略/目标/计划 | 日目标·月目标·KPI拆解 | 3-5 | AI 70% / 人 30% |
| S2 组织/人员/档案 | 人员档案·五行匹配·师徒制 | 5-8 | AI 40% / 人 60% |
| S3 制度/流程/SOP | 日反馈·巡检·退菜追踪 | 4-6 | AI 60% / 人 40% |
| S4 数据/分析/日报 | 日报·周报·月报·驾驶舱 | 4-5 | AI 90% / 人 10% |
| S5 客户/会员/营销 | 会员运营·储值·老饕维护 | 5-7 | AI 50% / 人 50% |
| S6 产品/出品/品质 | 金枪鱼管理·匠心·三昧真火 | 3-5 | AI 40% / 人 60% |
| S7 财务/成本/绩效 | 成本率·净利率·KPI考核 | 3-4 | AI 70% / 人 30% |

### 龙爪概念

> "你说'日报'哗哗哗说完，激活日报Skill，自动给你总结、判断哪个地方有问题。"

每个S模块 = 一个**龙爪容器**，内含N个具体Skills。店长只需用自然语言触发，AI自动激活对应Skill执行。

## 六、Skill构建标准（9要素·四步法）

### 一个Skill的定义

> "一个Skill只解决一件事。一件完整的事。日总结就是一件事，周复盘就是一件事。"

| 标准 | 说明 |
|------|------|
| **粒度** | 一件事 = 一个完整的、可独立完成的任务 |
| **检验标准** | 把真实数据扔进去，能否解决现实问题？ |
| **好坏判断** | AI干的比人干得好 → 合格；还不如人 → 失败 |

### SOP/Skill 必备9要素

| 序号 | 要素 | 说明 | 示例 |
|:--:|------|------|------|
| ① | **岗位定位** | 这个Skill服务于什么角色 | 店长/料理长/前厅经理 |
| ② | **性格特点** | 五行人格匹配，个性化输出 | 火行人→发散型方案 |
| ③ | **核心任务** | 这个Skill解决什么问题 | 日经营数据复盘 |
| ④ | ⭐**分析模型** | 最关键：用什么模型分析 | RFM/WIME-5D/PDCA |
| ⑤ | **关键动作** | 必须执行的步骤 | 数据采集→清洗→对比→结论 |
| ⑥ | **量化标准** | 结果指标/过程指标 | 客单价/翻台率/会员活跃率 |
| ⑦ | **数据验证** | 如何验证产出是否正确 | 环比/同比/目标达成率 |
| ⑧ | **风险控制** | 触发报警的底线 | 当日营业额低于X→红色预警 |
| ⑨ | **输出格式** | 最终产出的形式 | 日报模板/分析图表/行动建议 |

> **"没有分析模型，你的SOP是假的。"**

### 构建流程（四步法）

```
第一步：列出店长全部工作（按最理想版本，不是当前版本）
    ↓
第二步：按7S框架归拢分类
    ↓
第三步：每个工作判断 AI干 vs 人干 → 能SOP化的全部SOP化
    ↓
第四步：构建Skill → 测试 → 优化 → 上线
```

> "按未来的样子去罗列，按最理想的来。因为未来AI就能实现——原先没这个能力，AI就有这个能力。"

## 七、组织架构变革：倒三角赋能

### 核心理念

> "总部全部变成大运营部，全部下到一线。从管理变成赋能。"

```
旧模式：                    新模式（倒三角）：
董事长                      一线门店（店长）
  ↓                            ↑↑↑
总经理                      总部大运营部（每人赋能1-N个S）
  ↓
总部各部门
  ↓
一线门店
```

### 总部职能重构

| 旧角色 | 新角色 | 赋能哪些S |
|--------|--------|---------|
| 人力部 | **一线人力赋能师** | S2（人员）的1-2个子模块 |
| 市场部 | **一线营销赋能师** | S5（客户/会员）的2-3个子模块 |
| 财务部 | **数据中台** | S4（数据）、S7（财务） |
| 培训部 | **AI知识库维护者** | 所有S的知识更新 |

> "总部人员赋能人家其中几个S。只有店长、总经理、董事长他们是最全的，剩下我们都是服务人家其中3个S、2个S。"

### 理想时间线

| 年份 | 目标 | 含义 |
|------|------|------|
| 2027 | **解放董事长** | 董事长不再困于日常管理，专注产品创新/全国考察 |
| 2028 | **解放总经理** | 总经理通过AI系统自主运转 |
| 2029 | **解放3位老总** | 店长层级全面AI化 |

## 九、五行人格 × 个性化管理

### 为什么需要五行人格

> "店长抓的是什么？抓人。你对他自身都不了解，你抓什么？"

五行人格心理学的作用：
1. **招聘**：招他本身就是好的，而不是要改他
2. **管理**：火行人有火行人的管法，水行人有水行人的管法
3. **方案适配**：同一问题，对不同性格的人出不同方案

### 典型场景

| 五行 | 管人方式 | AI输出适配 |
|------|---------|-----------|
| 火 | 天生老大，不爱请示 | 方案给方向少细节，鼓励自主 |
| 水 | 稳重型，需要安全感 | 方案给足数据和风险分析 |
| 木 | 成长型，需要认可 | 方案带成长路径和里程碑 |
| 金 | 规则型，需要清晰边界 | 方案结构化，标准明确 |
| 土 | 关系型，需要归属感 | 方案强调团队和氛围 |

> "性格排屁股——招他本身就是好的，而不是要改他。"

## 十、7S守门人校验规则

### 子Skill注册必须通过7S分类校验

任何新建子Skill在注册到超级店长Skill时，必须通过以下校验：

| 校验项 | 规则 | 不通过后果 |
|--------|------|----------|
| **S归属** | 必须明确属于哪个7S模块 | 拒绝注册 |
| **覆盖范围** | 不能超出味藏7S体系定义范围 | 提示修正 |
| **AI/人分工** | 必须声明AI/人分工比例 | 返回补充 |
| **分析模型** | 必须包含至少1个分析模型 | 标记为不合格 |
| **五行适配** | 必须声明五行输出风格 | 返回补充 |
| **输出格式** | 必须定义输出格式 | 返回补充 |

### 子智能体自动触发规则

| 触发条件 | 激活方式 |
|---------|---------|
| 超级店长Skill被调用 | 店长7S总控自动加载 |
| 新建子Skill注册 | 自动7S分类校验 |
| 子Skill输出偏离7S | 自动拦截+提示修正 |
| 关键词：超级店长/店长AI/门店助手 | 全系统激活 |

## 十二、5×7=35种五行适配矩阵（灵魂层）

> "性格排屁股——招他本身就是好的，而不是要改他。"

同一个"月度经营分析Skill"，对不同五行人格的店长，输出方式不同：

| 五行 | 店长特点 | Skill输出适配 |
|------|---------|-------------|
| **木** | 成长型，需要认可 | 输出带成长路径和里程碑，强调"进步了多少" |
| **火** | 天生领导，不爱琐碎 | 输出精简，只给方向和关键决策点，不堆数据 |
| **土** | 关系型，重归属感 | 输出强调团队表现和氛围指标，用"我们"叙事 |
| **金** | 规则型，重边界清晰 | 输出结构化表格+明确标准+黑白判断 |
| **水** | 稳重型，需安全感 | 输出足量数据和风险分析，强调"稳"和"可控" |

### 注入方式

不是给每个Skill写5个版本。而是在Skill的SOP九要素之②"性格特点"中定义参数，让AI在运行时自动适配：

```
Skill运行时参数:
personality_type: "火" | "木" | "水" | "金" | "土"
→ 影响：输出详细度、语言风格、数据密度、决策建议方式
```

## 十四、龙龟喂养标准模板（进化层）

> "龙龟的本质，就是要把你的经验SOP化。一次受苦，以后自动化。"

每个Skill配套的喂养机制：

| 喂养项 | 频率 | 输入方式 | AI学习目标 |
