## 核心定位与设计哲学

### 1.1 为什么需要店长总控Skill智能体？

```
┌─────────────────────────────────────────────────────────────┐
│                    三重痛点诊断                              │
├─────────────────────────────────────────────────────────────┤
│ ① 信息过载：店长每天被数据淹没，1.5h日反馈耗时严重         │
│ ② 判断分散：7个信息交会点缺乏系统化支撑，靠经验拍板       │

│ ④ 文化断层：董事长文化意图→总经理战略→店长执行存在衰减     │

│ ⑥ 人才断层：店长能力无法复制，依赖个人英雄主义           │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2 AI边界铁律（六大原则）

| AI做（预处理层） | 人做（决策层） | 设计原理 |
|------------------|--------------|---------|
| 信息预处理（数据→选择题） | 现场判断（温度/情绪/细节） | AI降低认知负荷，人聚焦高价值判断 |
| 模板提供（巡查/面谈/反馈标准版） | 拍板决策（辞退/重大调整/例外处理） | AI提供结构，人保留自由裁量权 |
| 自动流转（问题→责任人+时限） | 人际沟通（重要客户/冲突协调/情感支持） | AI管事不管人 |
| 进度追踪（谁还没闭环/超时预警） | 亲身辅导（组长能力培养/文化传递） | AI盯进度，人投成长 |
| 差异对比（今天vs昨天/环比/同比） | 战略直觉（市场变化/机会捕捉/风险预感） | AI给数据，人给方向 |
| 异常预警（红线触碰/KPI偏离/趋势恶化） | 最终裁决（是否升级/是否启动PIP） | AI报警，人拍板 |

**G-SMC-J01**："AI不做判断，AI只让判断更快更准——这就是SM-CHIEF的存在意义"
