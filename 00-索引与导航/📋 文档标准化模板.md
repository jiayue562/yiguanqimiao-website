# 📋 以观其妙书院文档标准化模板

## 📝 标准文档结构

每篇文档必须包含以下六个部分：

```markdown
---
title: "文档标题"
type: 类型标识
tags: [标签1, 标签2, 标签3]
created: YYYY-MM-DD
version: 1.0
author: 作者
aliases: [别名1, 别名2]
---

# 文档标题

> 核心定义（一句话概括）

---

## 1. 核心定义

**关键概念的定义和解释**

---

## 2. 详细内容

**主题的详细展开**

### 2.2 子主题2

- [[关联文档2]]

### 3.2 相关概念
- [[概念1]]
- [[概念2]]

---

## 4. 核心金句

> "关键引用或总结"

---
```

## 🔗 双向链接规范

### 链接语法
```markdown
[[文件名]]           # 标准链接
[[文件名|显示文本]]   # 带显示文本的链接
```

### 链接位置
| 位置 | 说明 |
|------|------|
| 关联文件 | 必须包含，与核心内容相关 |
| 文中引用 | 适当引用，避免过度链接 |
| 脚注 | 详细说明可放脚注 |

### 链接密度要求
- **核心文档**: 至少 5 个双向链接
- **普通文档**: 至少 3 个双向链接
- **短文档**: 至少 1 个双向链接

## 📊 内容规范

### 标题层级
```
# 一级标题 - 文档标题
## 二级标题 - 主要章节
### 三级标题 - 子章节
#### 四级标题 - 细节内容
```

### 列表格式
- 使用 `-` 或 `1.` 进行列表
- 保持同一层级的一致性

### 引用格式
- 使用 `>` 进行引用
- 核心定义使用加粗

### 代码格式
- 使用 ` ```语言 ` 进行代码块
- 配置文件使用代码块
