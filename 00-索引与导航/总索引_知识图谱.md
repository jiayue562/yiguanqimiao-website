## 📋 什么是知识图谱

知识图谱（Graph View）是 Obsidian 对双向链接网络的可视化呈现，让知识网络从抽象变为直观。

| 特性 | 说明 |
|------|------|
| **节点** | 每个文件显示为一个点 |
| **连线** | 双向链接显示为线 |
| **颜色** | 可按标签/分类区分 |
| **大小** | 链接越多节点越大 |

## 🎨 五行配色方案

### 推荐配色

| 五行 | 颜色 | 标签 | RGB |
|------|------|------|-----|
| **木** | 绿色 | #wood | 76, 175, 80 |
| **火** | 红色 | #fire | 244, 67, 54 |
| **土** | 黄色 | #earth | 255, 193, 7 |
| **金** | 白色 | #metal | 250, 250, 250 |
| **水** | 黑色 | #water | 33, 33, 33 |

### 设置方法

1. 打开 Obsidian 设置
2. 进入"外观" → "图谱"
3. 添加颜色组：
   - 组名：木属性
   - 颜色：#4CAF50
   - 匹配：标签包含 `wood`

## 🔍 图谱查看模式

### 全局图谱

**用途**: 查看整个知识库的网络结构

**操作**:
1. 点击左侧菜单"图谱"
2. 选择"全局图谱"
3. 使用鼠标滚轮缩放
4. 拖拽查看不同区域

**适用场景**:
- 审视知识库整体结构
- 发现知识空白区域
- 识别核心节点

### 本地图谱

**用途**: 查看当前文件的直接关联

**操作**:
1. 打开任意文件
2. 点击右侧面板"本地图谱"
3. 查看 1-2 层关联节点

**适用场景**:
- 理解单个文件的上下文
- 发现相关概念
- 浏览知识网络

## 📊 图谱分析指标

### 网络密度

```
网络密度 = (实际链接数 / 最大可能链接数) × 100%
```

| 密度 | 说明 | 建议 |
|------|------|------|
| <5% | 链接稀疏 | 增加双向链接 |
| 5-10% | 中等密度 | 继续优化 |
| >10% | 链接丰富 | 保持并精选 |

### 核心节点

**识别方法**: 节点越大，链接越多

**Top 核心节点** (目标):
1. [[象思维总论]]
2. [[五行分类图谱]]
3. [[阴阳原理]]
4. [[易经基础]]
5. [[总索引]]

### 孤立节点

**识别方法**: 无连线的小节点

**处理建议**:
- 添加相关双向链接
- 合并到相关主题
- 或删除无用内容

## 🎯 图谱优化技巧

### 1. 筛选显示

```
按标签筛选：#wood OR #fire
按目录筛选：path:"03-知识地基层"
按链接数筛选：links:>5
```

### 2. 深度控制

| 深度 | 说明 | 适用 |
|------|------|------|
| 1 层 | 直接关联 | 本地图谱 |
| 2 层 | 关联的关联 | 局部浏览 |
| 全部 | 完整网络 | 全局审视 |

### 3. 节点分组

**按五行分组**:
- 木属性节点：绿色
- 火属性节点：红色
- 土属性节点：黄色
- 金属性节点：白色
- 水属性节点：黑色

**按层级分组**:
- 能量层 (A 层): 最大节点
- 时空层 (B 层): 中等节点
- 实践层 (C 层): 较小节点

## 📈 知识网络健康检查

### 检查清单

- [ ] 无孤立节点（或<5%）
- [ ] 网络密度>10%
- [ ] 核心节点清晰可辨
- [ ] 五行分布均衡
- [ ] 三层结构可见

### 运行验证

```bash
# 运行链接验证脚本
python scripts/verify_links.py
```

**查看输出**:
- 总文件数
- 总链接数
- 网络密度
- 断裂链接
- 孤立文件

## 🎨 高级可视化

### 使用 Juggl 插件

**安装**:
1. 设置 → 社区插件 → 浏览
2. 搜索"Juggl"
3. 安装并启用

**功能**:
- 交互式图谱编辑
- 动态节点展开
- 自定义样式
- 导出图谱图片

### 使用 Dataview 插件

**安装**:
1. 设置 → 社区插件 → 浏览
2. 搜索"Dataview"
3. 安装并启用

**功能**:
- 查询知识网络
- 动态生成列表
- 统计链接数据

**示例查询**:
```dataview
TABLE file.links as 链接数
FROM "03-知识地基层"
SORT length(file.links) DESC
```

## 📝 图谱使用场景

### 场景 1: 新知识入库

```
1. 创建新笔记
2. 添加双向链接到相关笔记
3. 查看本地图谱确认关联
4. 查看全局图谱确认位置
```

### 场景 2: 发现知识空白

```
1. 查看全局图谱
2. 识别稀疏区域
3. 分析是否需要补充
4. 创建新笔记填补空白
```

### 场景 3: 复习与回顾

```
1. 打开核心节点
2. 查看本地图谱
3. 遍历关联笔记
4. 更新过时内容
```

### 场景 4: 创作与输出

```
1. 确定主题
2. 查看相关节点
3. 整合关联内容
4. 生成新的创作
```
