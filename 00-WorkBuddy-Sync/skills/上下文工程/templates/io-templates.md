# 输入输出模板

> 📥📤 标准化的输入输出格式

---

## 📥 输入模板

### 保存上下文请求模板

```markdown
## 上下文保存请求

**项目名称**: [必填] 项目/任务名称
**任务类型**: [必填] 开发/研究/写作/分析/其他
**优先级**: [可选] 高/中/低
**中断原因**: [可选] 会话超时/主动中断/其他

**当前状态描述**:
[用户描述当前进展到哪里了]

**需要特别记住的**:
[用户强调的关键信息]

**相关文件**:
[可选：相关文件路径列表]
```

### 恢复任务请求模板

```markdown
## 上下文恢复请求

**快照文件**: [必填] context-YYYYMMDD-HHMM.md 或 "最新"
**恢复深度**: [可选] 完整/摘要/仅待办

**特别关注**:
[用户想优先了解的部分]

**状态更新**:
[可选：快照保存后完成的工作]
```

### 管理快照请求模板

```markdown
## 快照管理请求

**操作类型**: [必填] list/delete/merge/export

**操作对象**:
- list: 无需额外参数
- delete: [文件名]
- merge: [文件名列表]
- export: [格式：zip/tar/md]

**附加选项**:
[可选：额外参数]
```

## 📤 输出模板

### 保存成功输出模板

```markdown
## ✅ 上下文已保存

**快照文件**: `[文件名]`
**存储位置**: `[完整路径]`
**快照大小**: [X.X KB]

### 保存摘要
- **项目**: [项目名称]
- **任务类型**: [类型]
- **已完成**: [X] 项
- **待办**: [Y] 项
- **关键决策**: [Z] 条
- **相关文件**: [N] 个

### 关键信息速览
| 类别 | 内容 |
|------|------|
| 当前进度 | [X]% |
| 下一步 | [建议 1] |
| 风险点 | [注意事项] |

### 下次恢复
下次会话时输入以下任一命令即可恢复：
- `恢复任务`
- `继续上次的项目`
- `加载最新快照`
- `加载快照 [文件名]`

---
*快照生成时间：YYYY-MM-DD HH:MM*
*建议：重要项目建议每次会话后都保存快照*
```

### 快照列表输出模板

```markdown
## 📋 可用快照列表

| 文件名 | 项目 | 类型 | 时间 | 大小 | 进度 |
|--------|------|------|------|------|------|
| [文件名] | [项目] | [类型] | [时间] | [大小] | [进度] |
| [文件名] | [项目] | [类型] | [时间] | [大小] | [进度] |

**统计信息**:
- 总快照数：[X] 个
- 总大小：[X.X MB]
- 最早快照：[日期]
- 最新快照：[日期]

**操作建议**:
- 恢复最新：`恢复任务`
- 恢复特定：`加载快照 [文件名]`
- 删除过期：`管理快照 delete [文件名]`
- 导出备份：`管理快照 export zip`

---
*需要恢复哪个快照？或输入"最新"加载最近的*
```

## 🔄 状态转换模板

### 保存流程状态机

```
[等待触发] → [收集状态] → [生成快照] → [存储文件] → [输出确认] → [完成]
     ↑                                                              │
     └────────────────────── [错误处理] ←───────────────────────────┘
```

### 恢复流程状态机

```
[等待触发] → [检测快照] → [加载内容] → [解析状态] → [输出摘要] → [完成]
     ↑                                                              │
     └────────────────────── [错误处理] ←───────────────────────────┘
```
