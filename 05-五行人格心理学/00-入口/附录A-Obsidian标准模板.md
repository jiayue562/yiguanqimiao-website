# 附录A：Obsidian知识文档标准模板

> 确保所有凤脑OS文章学习后创建的知识文档保持一致性

---

## 📄 文档头部元数据（YAML）

```yaml
---
title: "[文章标题]"
created: "YYYY-MM-DD"
version: "1.0"
tags: [五行人格, 理论基石, [具体标签]]
---
```

---

## 📖 正文结构标准

### 第一部分：核心定义（必须）

```
# [文章标题]

## 核心定义

> 一句话概括本文章的核心概念

**关键词**：[关键词1] · [关键词2] · [关键词3]

**所属模块**：凤脑OS · 理论基石
```

### 第二部分：详细内容（根据文章内容）

```
## 核心内容

### [主题一]
[详细阐述]

### [主题二]
[详细阐述]

...

## 应用场景

### [场景一]
[具体应用]

### [场景二]
[具体应用]
```

### 第三部分：跨域连接（隐秘联系记录）

```
## 跨域连接

### 与西方心理学/认知科学连接
- [连接点]：[具体说明]
- [理论对应]：[如：对应卡尼曼系统2]
- [实践意义]：[如何应用]

### 与传统文化/修行体系连接
- [连接点]：[具体说明]
- [传统智慧]：[如：对应大圆满见地]
- [修行意义]：[如何体证]

### 与AI系统/人机协同连接
- [连接点]：[具体说明]
- [AI应用]：[如：对应人机协同第五象限]
- [技术意义]：[如何优化]
```

### 第四部分：核心洞察（5-10条）

```
## 核心洞察

1. [洞察一]（用自己的话表达，不超过30字）
2. [洞察二]
3. [洞察三]
...
```

---

## 🔗 文档间链接规则

### 标准链接格式

```
[[文章名]] - 用于引用其他文章
#章节名 - 用于文档内部跳转
![[图片名]] - 用于插入图片
```

### 双向链接原则

- **双向链接**：如果A文章提到B概念，则在B文章中链接回A
- **集群化**：相关文章形成双向链接集群
- **索引链接**：每篇文章链接到总索引 `[[📚 五行人格心理学总索引]]`

---

## 📊 联系编号分配规则

### 编号规则

```
联系编号 = 3位数字（001-999）

分配方式：
- 新文章：从001开始
- 跨域连接：按顺序分配
- 补充联系：在现有编号后追加
```

### 联系记录格式

```
| 编号 | 联系 | 涉及概念 | 来源文章 |
|------|------|---------|---------|
| 001 | [联系描述] | [概念1] / [概念2] | [文章名] |
```

## ✅ 文档质量检查清单

发布前检查：

- [ ] YAML头部完整（title/created/version/tags）
- [ ] 核心定义清晰（一句话概括）
- [ ] 关键词提取准确（3-5个）
- [ ] 正文结构分层（H1-H4清晰）
- [ ] 跨域连接至少3条（西方/传统/AI）
- [ ] 核心洞察5-10条（每条<30字）
- [ ] 双向链接建立（相关文章相互引用）
- [ ] 联系编号记录在知识图谱中
- [ ] 文件命名规范（见下节）

---

## 📁 文件命名规范

### 标准格式

```
[序号]-[简明标题].md
```

### 示例

```
01-一心即纯粹觉知.md
02-三界理论全息阐释.md
13-金行人格全谱系解析.md
```

### 分类文件夹

```
00-入口/          - 索引和总览
01-理论基石/       - 17篇核心文章
02-融合体系/       - 龙爪OS·一心·统摄
03-应用场景/       - 八大场景专属知识
04-人格类型/       - 五行分智能体知识
```

---

## 🔄 文档更新机制

### 更新触发条件

- 新增跨域连接时
- 补充核心洞察时
- 修正错误内容时
- 增加应用案例时

### 更新记录

```yaml
version: "1.0"
updates:
  - date: "YYYY-MM-DD"
    version: "1.1"
    changes: "新增XX跨域连接"
```
