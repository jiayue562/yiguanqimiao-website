## 📋 本次复盘：做对与做错

### ✅ 做对的事（保留并强化）

| # | 做对的事 | 价值 |
|---|---------|------|
| 1 | **先建地基·再起高楼**（L7→凤心OS→L1顺序正确） | 根深方能叶茂，没有L7就没有一切 |
| 2 | **每篇学习后立刻沉淀**（Obsidian+知识图谱+每日日志三联动） | 知行合一的正确实践，经验不流失 |
| 3 | **从独立包到全局Rule的演进**（alwaysApply:true·永远在线） | 全局化是系统化的前提 |
| 4 | **130条跨域联系的积累** | 知识不是孤岛而是网络，打通五行与西方认知科学 |
| 5 | **凤心OS从关键词触发升级为上下文感知** | 最大技术突破，真正的智能调度 |

### ❌ 做错的事（下次避免）

| # | 做错的事 | 代价 | 下次如何做 |
|---|---------|------|-----------|
| 1 | **架构反复重构**（四层→七层→三层） | 大量返工，内容迁移成本高 | **第0天就确定三层架构，之后不再变动大方向** |
| 2 | **L7积累17篇才封装L4/L1** | 理论和应用层分离太久，后期整合成本高 | **L7每完成5篇，同步推进一次MVP迭代** |
| 3 | **IMA同步被遗留为手动任务** | 三向同步实际只有两向在运行 | **第0天先测试IMA权限，有问题立刻解决** |
| 4 | **命名不统一**（凤心OS/总智能体/L4混用） | 理解成本高，协作时产生混乱 | **第0天定好命名规范，全程统一使用** |
| 5 | **龙心OS与凤心OS边界不清晰** | Skills调用关系混乱，有时互相调用 | **第0天先画边界图，之后严格遵守** |

## ⚡ 第二阶段：凤心OS搭建（Day 6-9，与L7并行！）

> **关键原则**：不要等L7全部完成才开始！L7完成第5篇后立刻开始！

### Step 5：凤心OS Skills创建

**文件结构**：
```
~/.workbuddy/skills/凤心OS/
├── SKILL.md（主文件·核心定义）
└── references/
    ├── scene-router.md（场景路由矩阵·S0-S8）
    ├── fast-slow-thinking.md（快慢思考协调协议）
    └── wuxing-agents-protocol.md（五行分智能体协作协议）
```

**SKILL.md必须包含的核心内容（6项缺一不可）**：

```
1. 核心定位：发动机·心脏·神经中枢（1+5模式）
2. 快思考层：五维感知矩阵
   - 情绪信号（阴木愤怒/阴火急躁/阴土猜疑/阴金挑剔/阴水恐惧·各8分）
   - 行为信号（各4分）
   - 诉求类型（辅助·各2分）
3. 慢思考层：七步标准分析流程
   ① 一心状态定位 → ② 三界扫描 → ③ 五行生克诊断
   → ④ 九层定位 → ⑤ 转化方案 → ⑥ B=MAP落地 → ⑦ 输出报告
4. 路由决策规则：
   - 总分≥10 → 激活对应五行分智能体（精准慢思考）
   - 总分7-9 → 双行分析（主行+副行并行）
   - 总分5-6 → 全面诊断（五行总体分析）
   - 总分≤4  → 先追问1个精准问题，确认后路由
5. 龙心OS七模块整合表
6. 不触发场景（避免误触发）
```

### Step 6：凤心OS全局Rule创建

```
路径：~/.workbuddy/rules/凤心OS自动触发.mdc
关键设置：alwaysApply: true
触发层级：
  第一层：上下文感知（情绪信号·行为信号）→ 最高优先级
  第二层：直接关键词（五行相关词汇）
```

### Step 7：凤心OS MVP验证

```
测试场景：输入含情绪信号的对话
样例："我最近很烦躁，总跟同事吵架，他们就是不理解我"

期望结果：
  - 凤心OS感知到阴火信号（烦躁=急躁·8分）
  - 自动路由到火行分智能体
  - 输出火行三界诊断+拔阴取阳建议

通过标准：无需用户输入「凤心OS」等关键词，系统自动感知并分析
未通过：检查Rule文件，调整感知矩阵分值，确认alwaysApply:true
```

## 📋 附录C：常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| Skills在某些对话不触发 | Rule不是alwaysApply | 检查Rule文件，确认`alwaysApply: true`，Rules放在用户全局目录 |
| 凤心OS和龙心OS冲突 | 没有明确边界 | 龙心OS处理通用认知；凤心OS处理五行人格专项；两者不互相嵌套 |
| L7知识调用不准确 | 知识标签不规范 | 统一标签体系，每篇都加五行标签+场景标签+L7根系标签 |
| IMA无法执行 | Windows权限问题 | 手动运行Python脚本，或直接使用IMA网页端 |
| 分智能体分析不准确 | L7根系不够深 | 回到L7，补充该行人格的详细内容，特别是九层阶梯和转化技术 |
| 系统越来越复杂难维护 | 没有统一的文档索引 | 用`📚 五行人格心理学总索引.md`做统一入口，每次新增文档都更新索引 |
| 用户反馈不准确 | 缺乏反馈闭环 | 每次分析后收集用户打分，用于优化感知矩阵的分值权重 |

```

7. 不要孤立    ：每篇L7文章都要强制寻找跨域联系，建立知识网络而非知识孤岛
```
