📜 核心理论

理论定位

知识学习 Skill是一套基于深度认知操作的文章学习与知识重构方法论，实现 AI 作为"超级读者"与"创新催化剂"的双重角色。

核心愿景：让 AI 不仅学习单篇文章，更要发现不同文章之间的关系，形成新的想法、思路与创新。

🎯 十项认知指令

认知指令体系


L1 基础解析：剖析、解构

L2 深度理解：透视、阐释

L3 批判建构：推演、思辨

L4 跨域整合：溯源、融合

L5 创新生成：启发、映射

L1 基础解析

1. 剖析（Analysis）

定义：对文章进行精准的结构性拆解，识别核心元素。

操作：

  • 文本预处理与清洗
  • 章节结构识别（摘要/引言/方法/结论）
  • 关键元素标注（图表/公式/参考文献）
  • 提取核心研究问题与假设

输出：结构化元素清单

提示词示例：


请对以下文章进行剖析：

1. 识别文章的章节结构

2. 提取核心研究问题

3. 列出所有关键概念

4. 标注重要图表和公式

2. 解构（Deconstruction）

定义：将宏观论述拆解为微观的、可操作的语义单元。

操作：

  • 提取核心实体（人物/机构/概念）
  • 提取关键陈述（研究问题/假设/结论）
  • 分解方法论为步骤清单
  • 分解结论为独立可验证的陈述

输出：语义单元清单（JSON 格式）

提示词示例：


请对文章进行解构性分析：

1. 提取核心研究问题与核心假设

2. 列出主要概念及文中定义

3. 将研究方法分解为连续步骤

4. 将结论分解为独立可验证的陈述

以 JSON 格式输出，确保可追溯至原文段落

L2 深度理解

3. 透视（Insight）

定义：超越表面词汇，把握文章的深层逻辑、论证脉络和潜在假设。

操作：

  • 生成论证结构图/逻辑依赖图
  • 分析推理链条的合理性
  • 识别潜在弱点
  • 发现隐含的新颖关系苗头

输出：论证结构图 + 逻辑分析报告

提示词示例：


分析该文章的论证链条：

从初始观察（A）→核心假设（B）→验证方法（C）→数据结果（D）→结论（E）

分析 B→C、C→D、D→E 每一步推理的合理性及潜在弱点

4. 阐释（Elaboration）

定义：对关键概念进行深化，置于具体语境并联系更广阔的知识背景。

操作：

  • 概念的上下文含义解释
  • 联系更广阔的知识背景
  • 多角度展开讨论
  • 知识图谱查询与多轮追问

输出：概念阐释报告

提示词示例：


请阐释文章中的核心概念 [X]：

1. 在本文语境下，该概念具体如何应用？

2. 相对于文中提到的其他方法，其优势是什么？

3. 该概念在更广阔领域中的起源、演变和主要变体

L3 批判建构

5. 推演（Deduction）

定义：基于现有内容进行逻辑外推，生成假设与模拟后果。

操作：

  • 假设生成（条件变化后的结果演变）
  • 后果模拟（应用到相邻领域的连锁反应）
  • 不确定性识别

输出：假设集 + 推演报告

提示词示例：


基于本文关于现象 X 的结论，进行三步逻辑推演：

1. 如果技术参数 Y 提升一个数量级，该结论是否会强化或弱化？

2. 将研究框架应用于领域 Z（文中未提及但相关），预测核心现象

3. 指出推演过程中最大的不确定性来源

6. 思辨（Speculation）

定义：进行批判性思考和多角度分析，建设性的审视。

操作：

  • 方法论思辨（实验设计/对照组/样本偏差）
  • 结论思辨（数据支持度/普适性/局限性）
  • 立场与范式思辨（理论范式/基本假设）
  • 伦理与影响思辨（社会/伦理/环境影响）

输出：批判分析报告

提示词示例：


请从"批判性朋友"的角度：

1. 找出本文论证中最脆弱的三个环节

2. 为每个环节提供加强论证或替代解释的思路

3. 引用其他领域的研究范例作为类比支撑

L4 跨域整合

7. 溯源（Tracing）

定义：追根溯源，构建学术谱系，理解知识的来龙去脉。

操作：

  • 参考文献分析（奠基性文献/前沿文献）
  • 概念演化追踪（历史时期/学者著作/定义演变）
  • 问题溯源（经典问题/长期困境）

输出：学术谱系图 + 概念演化史

提示词示例：


绘制本文核心概念 [X] 的学术简史：

1. 在本文中，[X] 的明确定义是什么？

2. 找出最早关于 [X] 的文献，简述其核心观点

3. 描述过去二十年中，[X] 概念的内涵迁移和拓展

8. 融合（Integration）

定义：整合不同领域、不同文章的知识和视角，产生新见解。

操作：

  • 类比思维与概念迁移
  • 跨学科知识嫁接
  • 强制关联法
  • 多篇文章/多领域知识碰撞

输出：融合洞察报告

提示词示例：


现在你有两篇文章：

A 文（关于神经网络架构搜索）

B 文（关于生物进化中的自然选择）



请进行融合思考：

1. A 文的"性能评估"和"架构变异"如何类比于 B 文的"环境适应"和"基因突变"？

2. 从进化生物学中，为改进神经网络架构搜索提出三条技术启发

3. 这种跨领域融合可能催生怎样的新交叉研究方向？

L5 创新生成

9. 启发（Heuristics）

定义：开拓新思路，生成开放性的、前瞻性的输出。

操作：

  • 研究空白启发（未被充分探索的"空白区"或"矛盾点"）
  • 反事实场景启发（"如果历史不是这样"的思考）
  • 未来趋势启发（基于技术发展曲线的预测）

输出：创新方向清单

提示词示例：


基于对多篇文章的深度分析，请启发：

1. 当前领域未被充分探索的"空白区"或"矛盾点"

2. 反事实场景："如果 [关键条件] 不是这样，会如何？"

3. 未来 5-10 年可能的研究范式转变或解决方案突破

10. 映射（Mapping）

定义：将一篇文章或一系列文章的内容，融合到新的框架结构中，形成新成果。

操作：

  1. 主题与输入确定
    1. 多维分析（运用所有十项指令）
      1. 框架生成（3-5 个核心维度 + 定义 + 指标 + 关系）
        1. 迭代与验证
          1. 成果格式化

            2. 输出：新框架/文章草稿

            提示词示例：

            
基于以上对所有输入文章的分析，请创建新框架：



框架名称：[新框架名称]



包含：

1. 3-5 个核心维度及其明确定义

2. 每个维度的理论依据（引用或融合输入文章观点）

3. 每个维度下 2-3 个可操作的测量指标

4. 不同维度之间的权衡或协同关系



确保该框架是对输入文章的超越与整合，而非简单罗列

            三层架构详解

            Layer 1：原始资料层

            功能：存储原始文章/文档

            格式：

            • PDF/DOCX/TXT 等原始格式
            • 元数据（标题/作者/日期/来源）
            • 初步标签（主题/领域/关键词）

            操作：

            • 文档解析与清洗
            • 结构识别
            • 初步标注

            Layer 2：Wiki 本体层

            功能：结构化知识表示

            格式：

            • 概念节点（定义/属性/关系）

            • 学术谱系（引用关系/演化路径）

            操作：

            • 十项认知指令处理
            • 知识图谱构建
            • 跨文档关联

            Layer 3：Schema 配置层

            功能：知识编译规则与配置

            内容：

            • 认知指令配置（优先级/权重）
            • 输出格式模板
            • 质量检验标准
            • 人机协同规则

            操作：

            • 规则配置
            • 模板管理
            • 质量审核

                      🎯 使用场景

                      场景 1：单篇文章深度学习

                      目标：全面深入理解一篇文章

                      指令组合：剖析 + 解构 + 透视 + 阐释 + 思辨

                      输出：

                      • 结构化元素清单
                      • 论证结构图
                      • 批判分析报告

                      场景 2：跨文章关联综合

                      目标：发现多篇文章之间的关系

                      指令组合：溯源 + 融合 + 启发

                      输出：

                      • 学术谱系图
                      • 融合洞察
                      • 研究空白识别

                      场景 3：理论框架构建

                      目标：基于多篇文章创建新理论框架

                      指令组合：全部十项指令 + 映射

                      输出：

                      • 新理论框架（3-5 个核心维度）
                      • 框架详细阐述
                      • 测量指标
                      • 文章草稿

                      场景 4：研究方案制定

                      目标：基于文献综述制定研究方案

                      指令组合：剖析 + 溯源 + 融合 + 启发 + 映射

                      输出：

                      • 研究空白
                      • 研究问题
                      • 研究方法建议
                      • 研究方案草稿

                      ⚠️ 边界条件

                      人类主体性原则

                      必须遵循：

                      • 人类负责设定战略目标
                      • 人类提供价值判断
                      • 人类进行最终决策
                      • 人类把握伦理边界

                      AI 角色：

                      • 执行战术性的深度分析
                      • 提供海量选项
                      • 模拟推演结果
                      • 辅助而非替代

                      学术诚信原则

                      必须遵循：

                      • 引用溯源准确
                      • 不伪造数据/结论
                      • 明确标注 AI 辅助
                      • 尊重知识产权
