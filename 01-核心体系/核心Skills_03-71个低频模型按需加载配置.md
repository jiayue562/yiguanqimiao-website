# 🧠 71个低频模型按需加载配置 · v1.0

## 一、核心定位

**71个低频模型**是龙心OS思维模型感知雷达的第三层级（工具层），在核心层（9个高频本能）和模式层（20个中频）之后，用于需要精准调用特定思维工具的场景。

| 层级 | 模型数 | 响应层级 | 触发方式 |
|------|--------|---------|---------|
| **核心层** | 9个高频本能模型 | 即时本能反应 | 感知即激活 |
| **模式层** | 20个中频模型（四大模式） | 场景本能反应 | 识别场景后激活 |
| **工具层** | 71个低频模型 | 精准工具调用 | 明确指令后加载 |

## 四、按需加载配置

### 4.1 触发机制

```yaml
# 工具层模型触发规则

触发条件:
  - 用户明确提到具体模型名称
  - 用户请求"使用XX模型分析"
  - 用户说"用XX思维"

加载策略:
  - 单模型：直接加载对应模型
  - 多模型组合：并行加载，组合输出
  - 场景套件：加载整个场景的5-6个模型

响应时间:
  - 单模型：<0.5秒
  - 多模型：<1秒
  - 场景套件：<2秒
```

## 五、指令调用协议

### 5.1 单模型调用

```
用户指令格式：
- "用[模型名]分析[问题]"
- "[模型名]思维"
- "从[模型名]角度看[问题]"

龙心OS响应：
→ 加载对应模型
→ 执行分析
→ 输出结果
```

### 5.2 多模型组合调用

```
用户指令格式：
- "用[模型A]和[模型B]分析[问题]"
- "从多角度分析：模型A、模型B、模型C"

龙心OS响应：
→ 并行加载多个模型
→ 组合分析
→ 整合输出
```

### 5.3 场景套件调用

```
用户指令格式：
- "[场景名]分析"
- "用[场景套件名]处理"

龙心OS响应：
→ 加载整个套件
→ 系统性分析
→ 完整报告
```

## 六、与上层的协同

```
核心层（9个）：
  - 感知即激活
  - 直接穿透本质
  ↓ 组合使用
模式层（20个）：
  - 场景识别后激活

```

  → 决策分析模式：风险评估

工具层调用（指令）：
