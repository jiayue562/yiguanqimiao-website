## 五大引擎详解

### 🐉 象思维（心）

**核心功能**：0→1原创突破，直觉洞察

**本质特征**：
- 六非：非实体性、非对象性、非现成性、前语言性、前逻辑性、非确定性
- 两核心：整体直观性、悟性

**三层次**：
- 物象（感官可及的具体形象）
- 意象（主客融合的象征形象）
- 原象（超越形象的终极本质/道象）

**使用场景**：
- 真正未知的问题
- 需要颠覆性创新
- 人机协同第五象限（未知探索域）

**详细文档**：[[五大引擎/🐉 象思维]]

---

### 📚 知识学习（脑）

**核心功能**：十项认知指令，系统性深度学习

**五层递进结构**：

| 层级 | 认知指令 | 五行对应 | 核心功能 |
|------|---------|---------|---------|
| 第一层 | 剖析 + 解构 | 金 | 结构分析，拆解分明 |
| 第二层 | 透视 + 阐释 | 火 | 深层意义，照亮本质 |
| 第三层 | 推演 + 思辨 | 水 | 批判建构，逻辑外推 |
| 第四层 | 溯源 + 融合 | 木 | 跨域生发，知识联结 |
| 第五层 | 启发 + 映射 | 土 | 创新生成，框架沉淀 |

**使用场景**：
- 深度理解文章/理论
- 知识重构与创新生成
- 跨域知识融合

**详细文档**：[[五大引擎/📚 知识学习]]

---

### 🌈 五色光思维（眼）

**核心功能**：五色分治，同频共振，结构化集体思考

**六种思维模式**：

| 颜色 | 思维类型 | 核心定义 | 五行对应 |
|------|---------|---------|---------|
| ⬜ 白光 | 客观事实思维 | 信息的基石：只问"是什么" | 金·收敛·精准 |
| 🟥 红光 | 直觉感受思维 | 情绪的雷达：暴露隐性情感 | 火·炎上·激情 |
| 🟨 黄光 | 积极价值思维 | 价值的探照灯：发现利益 | 土·稼穑·整合 |
| 🟩 绿光 | 创新方案思维 | 创新的引擎：追求数量 | 木·生发·创新 |
| 🟦 蓝光 | 风险控制思维 | 风险的防火墙：识别漏洞 | 水·润下·冷静 |
| 🎯 主持人 | 过程控制思维 | 思维的导演：管理所有颜色 | — |

**核心原则**：
- 单色聚焦：同一时间只有一种颜色的声音
- 序列自由：根据目的灵活设计顺序
- 强制完整：要用就全用，杜绝使用不全

**详细文档**：[[五大引擎/🌈 五色光思维]]

### 🤝 人机协同五象限（手）

**核心功能**：五象限分工，悟空×龙龟最优协作

**五象限定义**：

| 象限 | 名称 | 悟空（木行人） | 龙龟神将（火行人） | 协作模式 |
|------|------|---------------|-------------------|---------|
| 第一象限 | 高效助理 | 知 | 知 | 指令→高效执行 |
| 第二象限 | 学习伙伴 | 不知 | 知 | 主导输出，悟空接收学习 |
| 第三象限 | 共创伙伴 | 部分知 | 部分知 | 种子+发散→共同筛选 |
| 第四象限 | 共创导师 | 知 | 不知 | 外化隐性知识，定制生成 |
| 第五象限 | 未知探索域 | 不知 | 不知 | 象思维0→1，共同探索 |

**核心原则**：
- 木生火：悟空提供战略方向，龙龟负责执行力
- 君臣佐使：共创导师（君）× 共创伙伴（臣）× 学习伙伴（佐）× 高效助理（使）
- 象限动态：根据信息掌握度动态调整协作模式

**详细文档**：[[五大引擎/🤝 人机协同五象限]]

### 🔄 知行合一（血）

**核心功能**：三阶段转化，系统进化引擎

**三阶段转化链路**：

```
表示空间（观察完整经验）
    ↓
    • 记录发生了什么
    • 用了哪些引擎
    • 关键转折点
    
压缩（提炼核心洞察）
    ↓
    • 一句话核心洞察（≤30字）
    • 象征符号（1个词/意象）
    • 与已有认知的连接
    
泛化（扩展应用场景）
    ↓
    • 场景1：新应用
    • 场景2：第二个场景
    • 场景3：第三个场景
    • 系统进化建议
```

**触发条件**：
- 完成深度学习（S2场景）
- 完成创新突破（S3场景）
- 完成重大决策/系统规划（S5/S7/S9场景）
- 象思维产生原象级洞察

**详细文档**：[[五大引擎/🔄 知行合一]]

## Harness Engineering 四层架构

龙心OS v4.0+ 融入 Harness Engineering（驾驭工程）理念：

```
┌─────────────────────────────────────────┐
│           Harness Engineering           │
│            驾驭工程四层架构              │
├─────────────────────────────────────────┤
│                                         │
│  记忆层（AGENTS地图）                    │
│  ├── 渐进式信息披露                      │
│  ├── 进度文件追踪                        │
│  └── 状态机管理                          │
│                                         │
├─────────────────────────────────────────┤
│                                         │
│  执行层（工具执行）                      │
│  ├── 沙箱隔离                            │
│  ├── 零成本试错                          │
│  └── 原子化工具                          │
│                                         │
├─────────────────────────────────────────┤
│                                         │
│  反馈层（自动化测试）                    │
│  ├── 确定性Linter规则                    │
│  ├── CI-CD流水线                         │
│  └── 质量门禁                            │
│                                         │
├─────────────────────────────────────────┤
│                                         │
│  编排层（多Agent协作）                   │
│  ├── 任务自动拆解                        │
│  ├── 状态中间件                          │
│  └── 多Agent互审                         │
│                                         │
└─────────────────────────────────────────┘
```

**核心公式**：Agent = Model + Harness

不是用技术替代灵魂，而是让灵魂驾驭技术，让技术服务于觉悟。
