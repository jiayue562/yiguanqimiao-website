# 🎯 07-内容创作系统 v2.0

> **定位**：观其妙书院微信公众号内容创作的「知识编译 + 数据驱动」生产系统
> **赛道**：五行人格心理学 / 心文化
> **平台**：微信公众号
> **核心痛点解决**：好内容找不到、无法复用 + 知识碎片化无法沉淀
> **V2.0 升级**：基于 Karpathy 编译方法论，新增 raw/ → wiki/ → outputs/ 三层知识编译架构

---

## 🏗️ 双轨架构

```
=== 知识管理层（Karpathy 编译架构） ===

URL / 剪藏 / 播客 / 推文
 ↓ 「抓取链接」
raw/（原始资料，不改）
 ↓ 「编译」（三步：浓缩→质疑→对标）
wiki/summaries/ + wiki/concepts/ + wiki/indexes/
 ↓
outputs/qa/（问答沉淀）+ outputs/health/（健康检查）

=== 内容创作层（生产系统） ===

碎片想法 → 选题库 → 素材检索 → 文稿生成
    → 标题/封面 → 发布 → 归档 → 数据复盘 → 方法论迭代

=== 两条线的关系 ===

知识管理线 → 为内容创作线提供编译好的知识素材
内容创作线 → 发布后数据反馈 → 迭代方法论 → 指导知识编译的重点方向
```

## 🔬 三步编译法

| 步骤 | 方法 | 核心操作 | 输出 |
|------|------|----------|------|
| 浓缩 | 第一性原理+剃刀法则 | 先抓主线，删减无关 | 3条核心结论+证据+术语 |
| 质疑 | 芒格反驳法 | 逻辑链+前提假设+反例 | 前提清单+不适用场景 |
| 对标 | 它山之石可以攻玉 | 跨域类比+迁移应用 | 跨域映射+新连接 |

## 📋 快速上手

1. **保存灵感**：说"记录选题：XXX"→ 自动创建选题文件
2. **抓取文章**：发一个 URL → 自动存入 raw/ 并编译
3. **写文稿**：说"深化选题"→ AI 检索全部知识资产 → 生成大纲
4. **优化标题**：说"生成标题"→ 5维评分排序
5. **复盘迭代**：说"记录数据"→ 自动分析 + 更新方法论
