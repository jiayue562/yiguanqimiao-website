# CLAUDE.md · 内容创作 AI 自动化配置 v2.1

> **V2.1 升级**：新增智能触发规则，8条规则实现全自动/半自动/定时三层自主运行
> **V2.0 升级**：基于 Karpathy「编译」方法论，新增 raw/ → wiki/ → outputs/ 三层编译架构
> **双轨运行**：知识管理层（编译）+ 内容创作层（生产）相互驱动
> **AI 读取此文件后，能自动感知场景并执行，无需手动指令**

---

## 📌 AI 行为准则

1. **领域专精**：所有内容围绕「五行人格心理学」和「心文化」两大领域
2. **素材复用优先**：每次创作前先检索已有素材库，复用 > 重新创作
3. **双向链接**：所有内容必须使用 `[[概念名]]` 链接到知识库已有文档
4. **数据驱动**：复盘必须基于真实数据，不做无数据的主观判断
5. **沉淀增量**：每次创作都在给系统添砖加瓦，不能"用完即弃"
6. **编译优于整理**：原始资料只存入 raw/，经三步编译后才进入 wiki/

## ⚡ 智能触发规则（自动执行，无需手动指令）

> **核心理念**：Karpathy "机器编译、人类审核"——AI 自动感知场景并执行，人类只需审核关键节点。

### 🔴 全自动触发（AI 自主判断，无需你说话）

#### 规则 1：URL 自动抓取
**触发条件**：你在对话中发送了一个 URL 链接（http/https）
**AI 自动执行**：
1. 识别 URL → 使用 Web Fetch 抓取内容
2. 转为 Markdown + 添加 YAML frontmatter
3. 存入 `raw/articles/YYYY-MM-DD-标题关键词.md`
4. 更新 `wiki/indexes/All-Sources.md`
5. 简短回复：「✅ 已抓取并存入 raw/，文件名：xxx。需要编译吗？」

#### 规则 2：知识问答自动沉淀
**触发条件**：你提问涉及知识库已有概念/理论，AI 检索 wiki/ 后回答
**AI 自动执行**：
1. 检索 wiki/concepts/ + wiki/summaries/ + outputs/qa/
2. 生成带来源标注的回答
3. **自动**将问答沉淀到 `outputs/qa/问答-YYYY-MM-DD-问题关键词.md`
4. 不需要额外确认，静默执行

#### 规则 3：创作时自动素材检索
**触发条件**：你开始写文稿、构思选题、深化内容
**AI 自动执行**：按 V2.0 检索顺序（wiki/concepts → wiki/summaries → outputs/qa → 素材库）自动检索并匹配素材，不需要你手动说"找素材"

#### 规则 4：编译后自动索引更新
**触发条件**：完成一次编译后
**AI 自动执行**：自动更新 All-Sources.md 和 All-Concepts.md，无需手动触发

### 🟡 半自动触发（AI 提议，你确认后执行）

#### 规则 5：编译提议
**触发条件**：
- raw/ 中存在未编译素材（没有对应 wiki/summaries/ 文件）
- 你发完 URL 抓取后，AI 主动提议
**AI 行为**：
- 检测到未编译素材时，主动提议：「检测到 N 条未编译素材，需要编译吗？」
- 你说"编译"或"处理" → AI 执行三步编译法
- 编译完成后，展示提取的概念和选题启发，**等待你确认**后再写入 wiki/concepts/ 和 01-选题库/

#### 规则 6：选题启发提议
**触发条件**：编译完成后产生了新的选题灵感
**AI 行为**：编译摘要末尾列出选题启发 → 征求你的意见 → 你确认后才写入 01-选题库/

#### 规则 7：方法论迭代提议
**触发条件**：数据复盘中发现新的标题/开头规律
**AI 行为**：提出方法论更新建议 → 你确认后才更新 05-方法论迭代/

### 🟢 定时自动（不需要你参与）

#### 规则 8：每周健康检查
**触发机制**：可配置自动化任务，每周日自动执行
**AI 自动执行**：


## 🔬 核心编译方法：三步编译法

每次编译 raw/ 中的内容，必须严格按以下顺序执行三步：

### 第一步：浓缩（第一性原理 + 剃刀法则）

> 基于认知负荷理论（人类工作记忆最多维持20秒，同时处理2-4个信息块）

- 先抓主线，再深度质疑（顺序不能反）
- 用剃刀法则删减：删掉它会影响理解吗？不会就删
- **输出**：核心结论（不超过3条）+ 关键证据 + 核心术语表

### 第二步：质疑（芒格反驳法）

> "如果我不能比别人更好地反驳自己的观点，我就不配拥有这个观点。"

- 逻辑链检查：推理链在哪里断了？
- 前提假设：结论依赖哪些前提？列出每个隐含前提及"如果该前提不成立会怎样"
- 不适用场景：这个观点在什么情况下不成立？写出具体反例
- **输出**：前提假设清单 + 不适用场景 + 反例

### 第三步：对标（它山之石可以攻玉）

- 跨域类比：其他领域有类似现象吗？建立映射表
- 迁移应用：这些知识还能用在哪儿？
- 它山之石：其他领域的方法论能否迁移到当前领域？
- **输出**：跨域映射表 + 可迁移洞察 + 新连接

### 🔄 工作流 2：选题深化

**触发词**：深化选题、展开选题、选题大纲、帮我构思

**AI 执行流程**：
1. 读取已有选题文件
2. 检索 `02-素材库/` 中的相关素材
3. 检索 `wiki/concepts/` 中的概念条目（含质疑和跨域洞察）
4. 检索 `wiki/summaries/` 中的已编译知识

### 🔄 工作流 3：标题生成

**触发词**：生成标题、起标题、标题优化、帮我取标题

**AI 执行流程**：
1. 读取文稿内容（或选题信息）
2. 参考 `05-方法论迭代/标题方法论.md` 中的验证公式
3. 生成 5-8 个候选标题
4. 按评分排序

**标题评分标准**：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 好奇心 | 30% | 是否引发"我想点开看看"的冲动 |
| 价值感 | 25% | 是否暗示读者能获得具体价值 |
| 识别度 | 20% | 是否与五行人格/心文化领域相关 |
| 简洁度 | 15% | 15字以内为佳，不超过22字 |
| 情绪共鸣 | 10% | 是否触及目标读者的情绪痛点 |

---

### 🔄 工作流 4：开头优化

## 📐 编译产物规范

### 摘要文件（wiki/summaries/）
- 文件名格式：`YYYY-MM-DD-关键词.md`
- 必须包含三步编译的完整输出
- 末尾列出：提取的概念、关联素材、选题启发

### 概念条目（wiki/concepts/）
- 文件名格式：`概念名.md`
- YAML frontmatter：aliases, created_at, updated_at, sources, related_concepts
- 必须包含：定义、第一性原理、证据链、质疑、跨域连接、适用场景、典型表达、反向链接
- 新概念建条目，老概念补新证据

### 索引维护
- 每次编译后自动更新 All-Sources.md（追加来源）
- 每次编译后自动更新 All-Concepts.md（新增/更新概念）
