# LLM Wiki 实战方法论：用AI管理个人知识库

> **位置**: 龙脑OS·知识地基层 | **方法来源**: Andrej Karpathy + 个人实践
> _别让AI每次都从零开始，让它帮你把知识攒起来_

---

## 一、核心问题

同一个问题换个session问AI，它又从头推导一遍。之前聊过的上下文全丢了。

**Karpathy的解法**: 让LLM把每次对话中产生的知识沉淀成Markdown文件。下次再问，AI先翻已有的笔记，在这个基础上往前推。知识像滚雪球一样越攒越多，交叉引用越来越密，AI对你的领域理解越来越深。

> 维护知识库最累的不是阅读和思考，而是记账。

读文章、判断价值、决定方向 — 这些是人该干的事。但整理格式、更新链接、维护索引、检查哪篇过时了 — 这些bookkeeping的活，交给AI干最合适。

---

## 二、三层架构 vs 五层架构

### Karpathy的三层

| 层 | 干什么 | 谁管 |
|----|--------|------|
| Raw Sources | 原始资料丢进来，只读不改 | 人选材料 |
| The Wiki | LLM生成的知识产物：摘要、概念页、对比页 | LLM维护 |
| The Schema | 规则文件，告诉LLM该怎么干活 | 人和LLM一起维护 |

### 我的五层（Obsidian + OpenClaw + Claude Code）

```
Notes/（输入层）
  ├── Clippings/    网页剪藏
  ├── Inbox/        碎片想法
  └── Conversation/ 跟AI的有价值对话
        ↓ Ingest 分发
Knowledge/（知识层）    方法论、读书笔记、原创思考
Software/（技能层）      工具技巧、开发思考、产品研发计划
LifeOS/（行动层）        投资、健康、保险、联系人
        ↓ 素材调用
Writing/（产出层）       视频脚本、文章、运营SOP
```

**关键差异**:
- Karpathy的Wiki是终点，知识沉淀下来就完了
- 我做内容创作，知识存着不用等于没有 → 多了产出层
- 行动层意味着知识要落地（投资笔记要指导交易决策）
- 产出层意味着知识要变成产品（视频、文章、课程）

## 三、三大操作流程

### 3.1 Ingest（入库）：先报计划，再执行

用户说「处理一下Notes/」，Agent就开始干活。

**第一步：列方案。** 扫描Notes/下所有未处理内容，列出表格：

| 文件 | 建议放哪 | 怎么处理 | 为什么 |
|------|---------|---------|--------|
| xxx.md | Software/Skills/ | 合并到已有文件 | 内容跟现有文档重叠 |
| yyy.md | Knowledge/ | 新建 | 全新主题 |
| zzz.md | 留着 | 等补充 | 内容太短了 |

等我确认后才执行。

**第二步：分发。** 按内容性质判断目标：
- 软件工具、代码技巧 → Software/
- 投资、保险、健康 → LifeOS/
- 方法论、读书笔记 → Knowledge/
- 内容创作方法论 → Writing/Knowledge/
- 选题素材 → Writing/
- 判断不了的 → 留在Notes/，等我定

**第三步：加工。** 读原文写摘要，创建或更新页面，补交叉引用。

**第四步：清理。** 原始文件从Notes/删掉。

### 3.2 Query（查询）：搜索 + 答案 + 反哺

用户提问，Agent在相关目录里搜索内容，综合多篇文件给答案。

**关键在第三步 — 反哺**：如果搜索过程中发现两篇文件内容互补但没互相链接，Agent会说出来，确认后就补上。

> 示例：问「创业怎么找前1000个用户」，Agent找到了《头1000名用户》和《做产品的经验和感悟》。前者讲获客，后者讲产品打磨，互相补充但从来没链接过。发现这个关联后，补上了链接。

Query的价值不只是回答问题，还在于暴露知识库的结构缺陷。

### 3.3 Lint（健康检查）：定期扫雷

四项检查：

| 查什么 | 标准 |
|--------|------|
| 孤立页面 | 没有任何链接指向 |
| 过时内容 | 超过90天没动过 |
| 缺失引用 | 讲同一个概念但没互相链接 |
| Notes/积压 | 超过7天没处理的输入 |

跑完出报告，不自动改。

**实践经验**：按目录分别跑，不要一次全跑。Software/这种工具备忘目录，每篇本来就是独立的操作手册，报一堆「孤立页面」没意义。Knowledge/才是Lint真正能发挥价值的地方。

## 五、两套方案对比

| 维度 | Karpathy | 我的方案 |
|------|---------|---------|
| 知识库定位 | 终点，沉淀就完了 | 弹药库，要变成产品 |
| AI角色 | 全权管家，自主操作 | 参谋，提建议我拍板 |
| 知识流向 | 单向：资料→Wiki | 多层：输入→沉淀→产出 |
| Ingest | LLM自主分类更新 | 先列方案→确认→执行 |
| Query | 查询→综合→自动反哺 | 搜索→综合→建议补链接 |
| Lint | 检查矛盾过时孤立 | 四项检查+按目录分跑 |
| 特殊文件 | index.md(文件级)+log.md | index.md(目录级) |
| 操作规则 | CLAUDE.md | AGENTS.md(多Agent兼容) |
| 对话沉淀 | 无 | Notes/Conversation/ |
