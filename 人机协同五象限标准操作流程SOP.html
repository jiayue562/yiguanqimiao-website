人机协同五象限标准操作流程（SOP）

📋 标准化操作流程 · 确保协作质量

📋 SOP 概述

目的

  • 标准化人机协同五象限的使用流程
  • 确保协作质量和效能
  • 支持经验沉淀与持续改进

适用范围

  • 所有人机协作场景
  • 所有使用五象限模型的用户
  • 所有需要优化 AI 协作的任务

核心流程


任务接收 → 场景分析 → 象限匹配 → 协作执行 → 效能评估 → 经验沉淀

🔄 标准操作流程

流程 1：任务接收与分类

步骤 1.1：接收任务


输入：用户任务描述

处理：记录任务基本信息

输出：任务卡片

步骤 1.2：任务分类


决策树：

1. 任务是否规则明确？

   - 是 → 第一象限

   - 否 → 继续判断



2. 任务是否以获取知识为主？

   - 是 → 第二象限

   - 否 → 继续判断



3. 任务是否需要深度思考？

   - 是 → 第三象限

   - 否 → 继续判断



4. 任务是否探索未知？

   - 是 → 第四象限

   - 否 → 第五象限

流程 2：场景分析与象限匹配

步骤 2.1：场景分析


分析维度：

1. 任务类型分析

2. 复杂度评估

3. 不确定性评估

4. 人类参与度需求

5. AI 能力匹配度

| AI 能力匹配 | 高/中/低 | [选择] | [说明] |

步骤 2.2：象限匹配


匹配规则：

- 执行类 + 低复杂度 + 确定 → 第一象限

- 学习类 + 中复杂度 + 部分不确定 → 第二象限

- 思考类 + 高复杂度 + 部分不确定 → 第三象限

- 探索类 + 极高复杂度 + 高度不确定 → 第四象限

- 创造类 + 极高复杂度 + 未知 → 第五象限

步骤 4.3：异常处理


异常情况处理流程：



1. 效能低下

   - 识别：效能指标低于阈值

   - 处理：重新评估象限匹配

   - 行动：调整协作模式



2. 任务变更

   - 识别：任务需求变化

   - 处理：更新任务卡片

   - 行动：重新设计协作流程



3. AI 能力不足

   - 识别：AI 无法完成任务

   - 处理：人类接管或降级象限

   - 行动：调整人机分工



4. 人类过度依赖

   - 识别：人类放弃主导权

   - 处理：提醒人类主导权

   - 行动：调整分工界面
