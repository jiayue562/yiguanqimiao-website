🔴 全自动触发（AI 自主判断，无需你说话）

规则 1：URL 自动抓取

触发条件：你在对话中发送了一个 URL 链接（http/https）

AI 自动执行：

  1. 识别 URL → 使用 Web Fetch 抓取内容

  2. 转为 Markdown + 添加 YAML frontmatter

  3. 存入 raw/articles/YYYY-MM-DD-标题关键词.md

  4. 更新 wiki/indexes/All-Sources.md

  5. 简短回复：「✅ 已抓取并存入 raw/，文件名：xxx。需要编译吗？」

规则 2：知识问答自动沉淀

触发条件：你提问涉及知识库已有概念/理论，AI 检索 wiki/ 后回答

AI 自动执行：

  1. 检索 wiki/concepts/ + wiki/summaries/ + outputs/qa/

  2. 生成带来源标注的回答

  3. 自动将问答沉淀到 outputs/qa/问答-YYYY-MM-DD-问题关键词.md

  4. 不需要额外确认，静默执行

规则 3：创作时自动素材检索

触发条件：你开始写文稿、构思选题、深化内容

AI 自动执行：按 V2.0 检索顺序（wiki/concepts → wiki/summaries → outputs/qa → 素材库）自动检索并匹配素材，不需要你手动说"找素材"

规则 4：编译后自动索引更新

触发条件：完成一次编译后

AI 自动执行：自动更新 All-Sources.md 和 All-Concepts.md，无需手动触发

🟢 定时自动（不需要你参与）

规则 8：每周健康检查

触发机制：可配置自动化任务，每周日自动执行

AI 自动执行：

  1. 一致性检查（概念定义冲突）

  2. 完整性检查（缺失字段）

  3. 孤岛检测（入链出链 < 2）

  4. 报告存入 outputs/health/健康检查-YYYY-MM-DD.md

  5. 下次对话时简短汇报：「上周健康检查发现 N 个问题，已生成报告」

碎片想法 → 选题库 → 素材检索 → 文稿生成

    → 标题/封面 → 发布 → 归档 → 数据复盘 → 方法论迭代



=== 两条线的关系 ===



知识管理线 → 为内容创作线提供编译好的知识素材

内容创作线 → 发布后数据反馈 → 迭代方法论 → 指导知识编译的重点方向

🔬 核心编译方法：三步编译法

每次编译 raw/ 中的内容，必须严格按以下顺序执行三步：

第一步：浓缩（第一性原理 + 剃刀法则）

基于认知负荷理论（人类工作记忆最多维持20秒，同时处理2-4个信息块）

  • 先抓主线，再深度质疑（顺序不能反）

  • 用剃刀法则删减：删掉它会影响理解吗？不会就删

  • 输出：核心结论（不超过3条）+ 关键证据 + 核心术语表

第二步：质疑（芒格反驳法）

"如果我不能比别人更好地反驳自己的观点，我就不配拥有这个观点。"

  • 逻辑链检查：推理链在哪里断了？

  • 前提假设：结论依赖哪些前提？列出每个隐含前提及"如果该前提不成立会怎样"

  • 不适用场景：这个观点在什么情况下不成立？写出具体反例

  • 输出：前提假设清单 + 不适用场景 + 反例

第三步：对标（它山之石可以攻玉）

  • 跨域类比：其他领域有类似现象吗？建立映射表

  • 迁移应用：这些知识还能用在哪儿？

  • 它山之石：其他领域的方法论能否迁移到当前领域？

  • 输出：跨域映射表 + 可迁移洞察 + 新连接

🔄 工作流 2：选题深化

触发词：深化选题、展开选题、选题大纲、帮我构思

AI 执行流程：

  1. 读取已有选题文件

  2. 检索 02-素材库/ 中的相关素材

  3. 检索 wiki/concepts/ 中的概念条目（含质疑和跨域洞察）

  4. 检索 wiki/summaries/ 中的已编译知识

  5. 检索知识库根目录（02-五行人格心理学/、02-信仰文化体系/）中的理论文档

  6. 输出内容大纲 + 素材匹配清单

🔄 工作流 4：开头优化

触发词：优化开头、开头改写、帮我写开头、钩子优化

开头类型库：

| 类型 | 适用场景 | 示例框架 |

|------|----------|----------|

| 痛点型 | 解决具体问题 | "你是不是也遇到过..." |

| 数据型 | 反常识/颠覆认知 | "90%的人都不知道..." |

| 故事型 | 情感共鸣 | "上周一位学员问我..." |

| 金句型 | 价值观输出 | "五行不是算命，是..." |

| 场景型 | 日常代入 | "开会时领导说了一句话..." |

| 提问型 | 引发思考 | "为什么有些人天生就有领导力？" |

🔄 工作流 6：数据复盘

触发词：记录数据、发布复盘、数据记录、这篇文章表现如何

AI 执行流程：

  1. 创建文件：04-数据复盘/复盘-YYYY-MM-DD.md

  2. 填写数据记录

  3. 分析亮点与不足

  4. 提炼方法论更新建议

  5. 更新 05-方法论迭代/ 中的方法论文件

🔄 工作流 8：「编译」

触发词：编译、compile、处理新内容、编译知识

AI 执行流程：

  1. 扫描 raw/ 找未编译素材（没有对应 wiki/summaries/ 文件的）

  2. 三步编译法（浓缩→质疑→对标）

  3. 存入 wiki/summaries/YYYY-MM-DD-标题关键词.md

  4. 提取概念 → 在 wiki/concepts/ 中建立或更新条目

  5. 更新 wiki/indexes/All-Sources.md（追加来源）

  6. 更新 wiki/indexes/All-Concepts.md（新增/更新概念）

  7. 如有选题启发 → 追加到 01-选题库/

增量编译：只处理没有对应 wiki/summaries/ 文件的 raw 素材，不全量重建

🔄 工作流 10：「知识问答」

触发词：问知识库、基于知识库回答、知识查询

AI 执行流程：

  1. 检索 wiki/concepts/ + wiki/summaries/ + outputs/qa/

  2. 生成带来源的回答（标注引用来源）

  3. 以 Markdown 存入 outputs/qa/问答-YYYY-MM-DD-问题关键词.md

  4. 标注不确定性（哪些地方还不确定）
