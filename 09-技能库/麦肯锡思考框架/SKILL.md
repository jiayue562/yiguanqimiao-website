### 1.2 架构关系说明

**龙脑OS（知识库）**：
- 定位：知识资产的存储与组织层
- 功能：提供各类分析框架、理论体系、方法论工具的完整知识库
- 特点：每个框架独立封装，可被龙心OS按需调用

**龙心OS（调度中枢）**：
- 定位：智能调度与场景识别中枢
- 功能：根据对话场景，智能判断调用哪个知识库中的哪个框架
- 特点：统一入口，智能路由，动态组合

**调用关系**：
```
用户输入 → 龙心OS（场景识别）→ 龙脑OS（框架调用）→ 结果返回 → 知行合一沉淀
```

### 1.3 核心原则

1. **龙心OS统一调度**：所有分析需求先经过龙心OS的场景识别和引擎路由
2. **麦肯锡框架独立存在**：作为龙脑OS的知识库组件，保持独立性和完整性
3. **按需调用，灵活组合**：根据场景调用单个框架或多个框架组合
4. **中西合璧**：象思维（0→1）→ 麦肯锡框架（1→N）→ 五色光（验证）

## 四、核心使用原则

### 4.1 中西合璧原则

```
象思维（东方·0→1） → 麦肯锡框架（西方·1→N） → 五色光（整合·验证）
     ↓                        ↓                        ↓
  直觉洞察              逻辑结构化展开            多维度验证
```

### 4.2 场景适配原则

- **不强制使用**: 简单问题直接回答，不启动框架
- **自动匹配**: 根据场景关键词自动选择最合适框架
- **组合使用**: 复杂场景可组合多个框架
- **灵活调整**: 根据实际进展动态调整框架选择

### 4.3 与五色光深度整合

| 麦肯锡框架 | 五色光角色 | 整合方式 |
|-----------|-----------|---------|
| MECE原则 | 白光基础 | 确保事实分解的完整性 |
| 金字塔结构 | 黄光传递 | 确保价值传递的有效性 |
| 五色光思维 | 五色全维 | 与麦肯锡框架深度整合 |
| 决策矩阵 | 多色协同 | 绿光方案 + 黄光价值 + 蓝光风险 |

```
│                    调用层级架构                              │

│  ├─ 判断：是否需要西方分析框架？                               │

│                                                             │
│  第三层：麦肯锡框架执行                                        │
│  ├─ 加载框架定义与操作步骤                                     │
│  ├─ 执行分析流程                                              │
│  └─ 输出结构化结果                                            │
│                                                             │
│  第四层：龙心OS整合输出                                        │
│  ├─ 与象思维洞察整合                                          │
│  ├─ 与五色光验证整合                                          │
│  └─ 知行合一沉淀                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 触发信号（由龙心OS识别）

**龙心OS识别到以下信号时，路由至麦肯锡框架库**：

| 触发级别 | 权重 | 触发条件 | 龙心OS响应 |
|---------|------|---------|-----------|
| **P0·直接触发** | 5 | 明确提到框架名称（SWOT、MECE、七步法等） | 路由至麦肯锡框架库 |

| **P2·信号触发** | 3 | 多信号组合（战略+竞争、问题+拆解） | 智能匹配最佳框架 |
