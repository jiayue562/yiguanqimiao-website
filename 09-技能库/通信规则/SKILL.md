# Communication Skill（通信规则）

## 📖 技能定义

Communication是AI龙龟共生伙伴操作系统的**通信规则**，定义Discord交互规范、信息传递格式和标准，避免沟通混乱和误解。

**核心定位**：
- **规范化**：统一的通信格式
- **标准化**：明确的信息传递规则
- **可追溯**：完整的沟通历史

**核心原则**：
> 清晰、简洁、准确的信息传递，避免沟通混乱。

## 🎯 Discord交互规范

### 1. 提及规则

**@指定人员**：
- 所有需要指定人员的消息必须@相关人员
- 不要泛发消息到频道
- 明确接收人

**格式示例**：
```
@龙龟神将 请完成TASK-001的创建工作
```

**禁止示例**：
```
❌ 请完成TASK-001的创建工作（未指定接收人）
❌ @所有人 有新任务分配（不要滥用@所有人）
```

### 2. Discord ID规则

**纯文本格式**：
- 使用Discord ID（纯文本）
- 不使用@提及（除非需要通知）
- ID格式：`<数字ID>`

**格式示例**：
```
接收人: 123456789012345678
```

**禁止示例**：
```
❌ 接收人: @用户名（不要使用@提及）
```

### 3. 回复规则

**直接回复**：
- 回复相关消息时使用直接回复功能
- 保持消息上下文关联
- 不要在频道中泛发

**格式示例**：
```
[回复] @龙龟神将 TASK-001已完成，请审核
```

### 4. 文件分享规则

**直连文件链接**：
- 分享文件时使用直连链接
- 不要使用PR（拉取请求）链接
- 确保链接可访问

**格式示例**：
```
文件: https://example.com/files/task-001.md
```

**禁止示例**：
```
❌ 文件: https://github.com/user/repo/pull/1（不要使用PR链接）
```

## 📧 信息传递格式

### 1. 消息格式

**标准格式**：
```markdown
[类型] @接收人 消息内容

示例：
[任务] @龙龟神请 完成TASK-001的创建工作

[通知] @所有人 系统将在今晚22:00进行维护
```

**消息类型**：
- `[任务]`：任务分配和更新
- `[通知]`：系统通知和公告
- `[问题]`：问题和疑问
- `[回复]`：回复和确认
- `[讨论]`：讨论和交流

### 2. 任务消息格式

**任务分配**：
```markdown
[任务] @负责人 任务名称

[详情]
任务ID: TASK-001
任务描述: 创建任务控制台Skill
优先级: P0
截止日期: 2026-03-23
预计时长: 2h

[附件]
相关文档: https://example.com/docs/task.md
```

**任务更新**：
```markdown
[更新] @负责人 任务名称

[状态]
任务ID: TASK-001
状态: 进行中 → 已完成
进度: 0% → 100%
完成时间: 2026-03-23 14:23

[备注]
所有功能已实现，等待审核
```

### 3. 问题消息格式

**问题报告**：
```markdown
[问题] @相关人员 问题描述

[详情]
问题ID: ISSUE-001
问题描述: 任务看板拖拽功能异常
严重程度: P0
影响范围: 所有用户

[复现步骤]
1. 打开任务看板
2. 拖拽任务到其他列
3. 任务状态未更新

[环境]
系统版本: v4.1
浏览器: Chrome 120
```

**问题解决**：
```markdown
[解决] @相关人员 问题描述

[详情]
问题ID: ISSUE-001
解决时间: 2026-03-23 14:30
解决方式: 修复前端拖拽逻辑
```

### 4. 通知消息格式

**系统通知**：
```markdown
[通知] @相关人员 通知内容

[详情]
通知ID: NOTIF-001
通知类型: 系统维护
维护时间: 2026-03-23 22:00-23:00
影响范围: 所有系统功能

[备注]
请提前保存工作内容
```

**公告**：
```markdown
[公告] @所有人 公告内容

[详情]
公告ID: ANN-001
公告类型: 版本更新
版本号: v4.2
更新时间: 2026-03-24
更新内容:
- 新增任务看板功能
- 优化任务控制台界面
- 修复已知问题

[附件]
更新日志: https://example.com/changelog.md
```

## 🔔 消息提醒

### 1. 实时提醒

**提醒级别**：
- 🔴 P0：立即提醒（重要任务、P0问题）
- 🟠 P1：5分钟内提醒（一般任务、P1问题）
- 🟡 P2：30分钟内提醒（次要任务、P2问题）
- 🟢 P3：1小时内提醒（非必要任务、P3问题）

**提醒方式**：
- Discord通知
- 邮件通知（重要消息）
- 手机推送（P0消息）

### 2. 未读消息提醒

**未读消息统计**：
- 未读消息数量
- 未读消息类型分布
- 最近未读消息时间

**提醒策略**：
- 未读消息 > 10条：每30分钟提醒一次
- 未读消息 > 50条：每10分钟提醒一次
- 未读消息 > 100条：每5分钟提醒一次

- 明确消息类型和接收人
