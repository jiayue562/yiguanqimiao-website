## 🎯 核心能力（v2.0）

### 1. 五层架构

| 层级 | 目录 | 功能定位 |
|------|------|---------|
| **第一层** | Notes/ | 输入层·统一入口（Clippings/Inbox/Conversation） |
| **第二层** | Knowledge/ | 知识层·方法论沉淀（方法论/读书笔记/原创思考） |
| **第三层** | Software/ | 技能层·工具与开发（AI工具/开发技巧/产品研发） |
| **第四层** | LifeOS/ | 行动层·生活管理（投资/健康/保险/人脉） |
| **第五层** | Writing/ | 产出层·内容产品（脚本/文章/SOP） |

### 2. 八大设计原则

| # | 原则 | 核心理念 |
|---|------|---------|
| 1 | 一个入口，多个终点 | 所有输入走 Notes/，统一流程分发 |
| 2 | 结构够用就行 | 目录分到能找到东西就够了 |
| 3 | AI 建议，你拍板 | 只有你自己最清楚该放哪 |
| 4 | 链接宜少不宜多 | 两三条强关联比十条弱关联有用 |
| 5 | 对话也是知识 | 跟 AI 的深度讨论本身就在生产知识 |
| 6 | 存着不用等于没有 | 知识库的终极检验：有没有帮你产出过东西 |
| 7 | 索引到目录，不到文件 | 目录结构稳定后，索引几乎零维护 |
| 8 | 不绑定工具 | AGENTS.md 兼容多种 Agent |

### 3. 核心操作流程

- **Ingest（入库）**：扫描 → 方案表 → 用户确认 → 分发执行 → 加工处理 → 清理
- **Query（查询）**：用户提问 → 搜索 Wiki → 综合回答 → 反哺建议
- **Lint（健康检查）**：孤立页面 → 过时内容 → 缺失引用 → Notes/ 积压

## 🌟 象思维核心洞察（v2.0）

### 核心原象：知识是弹药库，产出才是开枪

**象思维隐喻**：
- 知识库 ≠ 图书馆（存着看）
- 知识库 = 弹药库（备着用）
- 产出 = 枪声（打完才见真章）

**核心洞见**：
> 读过了 ≠ 会了
> 写出来才算数
> 存着不用等于没有
> 知识库的终极问题是：它有没有帮你产出过东西？

**五行关联**：
- 🔥 火（产出之光）：Writing/ 是知识燃烧转化为光芒的产出层
- 🌏 土（承载稳定）：五层架构承载知识从输入到产出的完整生命周期
- ⚔️ 金（收敛精准）：强关联链接策略，收敛而非铺开
- 🌊 水（流动智慧）：Ingest/Query/Lint 形成知识的活水循环
