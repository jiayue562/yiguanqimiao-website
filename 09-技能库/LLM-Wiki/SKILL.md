# LLM Wiki Skills · 活体维基知识编译系统 v1.0

> **核心定位**：LLM Wiki 是基于卡帕西(Karpathy)「活体维基」体系的知识编译系统，将知识管理从"被动检索"升级为"主动编译"。
> 
> **版本**：v1.0 | **创建日期**：2026-04-11 | **维护者**：龙龟神将
> 
> **核心哲学**："知识不是被检索的，而是被编译的；知识库不是静态仓库，而是活的生命体"

---

## 📖 核心定义

### What（是什么）

LLM Wiki Skills 是一套将**原始素材**通过AI编译为**结构化知识网络**的系统化方法，实现：
- ✅ 知识越用越聪明（复利增长）
- ✅ 查询无需向量检索（直接读Wiki）
- ✅ 全生命周期AI自动化（编译→纠错→反哺）

### Why（为什么）

**传统RAG的三大死穴**：
| 死穴 | 表现 | 后果 |
|------|------|------|
| 🔴 即时编译 | 每次提问都要重新读原始素材 | Token消耗爆炸 |
| 🔴 碎片断裂 | Chunk之间无逻辑关联 | 输出割裂不连贯 |
| 🔴 零沉淀 | 关闭对话知识清零 | 无复利效应 |

**LLM Wiki的三大突破**：
| 突破 | 原理 | 效果 |
|------|------|------|
| ✅ 提前编译 | 知识被预编译为Wiki摘要 | Token消耗降低70%+ |
| ✅ 网状结构 | 双向链接天然关联 | 逻辑连贯一致 |
| ✅ 答案反哺 | 优质答案写回Wiki | 越用越聪明 |

### How（怎么做）

**一句话工作流**：
```
原始素材 → [AI编译] → Wiki结构化知识 → [查询] → 高质量答案 → [反哺] → 更聪明的Wiki
```

## ⚙️ 七步核心操作流程

### Step 1: 三目录体系建立

**目标**：创建raw/wiki/output三目录结构

```markdown
## 操作清单

### 1.1 创建主目录结构
obsidian-vault/
├── raw/                    # 原始素材（只读）
│   ├── pdf/               # PDF文档
│   ├── web/               # 网页剪藏
│   ├── doc/               # Word/文档
│   └── notes/             # 速记素材
│
├── wiki/                   # 编译知识库（可写）
│   ├── index.md           # 全局索引（必选）
│   ├── concepts/          # 概念页
│   ├── entities/          # 实体页
│   ├── summaries/         # 摘要页
│   └── references/        # 交叉引用
│
└── output/                 # 成品输出
    ├── reports/           # 分析报告
    ├── slides/            # 幻灯片
    └── charts/            # 图表可视化
```

**检验标准**：
- ✅ 三目录齐全
- ✅ 权限配置正确（raw只读，wiki/output可写）
- ✅ index.md存在

### Step 2: Schema配置部署

**目标**：部署AGENTS.md规则文件，控制AI操作边界

```markdown
## AGENTS.md 标准模板

# 知识库Schema · AI操作手册

## 📌 结构规范

### 命名约定
- 概念页: `concept-[主题].md`
- 实体页: `entity-[名称].md`
- 摘要页: `summary-[日期].md`

### 标题层级
```
# 页面标题（页面名）
## 核心定义
### 详细内容
#### 子内容
```

## 🏷️ 标签体系

### 允许的分类标签
- `[[核心概念]]` - 核心理论定义
- `[[实践方法]]` - 可执行的方法论
- `[[案例分析]]` - 实战案例
- `[[待验证]]` - 未经验证的新想法
- `[[争议]]` - 存在矛盾的观点

### 禁止的标签
- ❌ 个人情感标签
- ❌ 未分类的临时标签

## 🔄 更新规则

### 同主题新资料处理
- 核心概念: **追加**（保留历史，新增更新）
- 非核心概念: **覆盖**（直接替换）
- 案例分析: **合并**（整合新旧案例）

### 矛盾处理策略
- 发现矛盾 → 标记 `[[争议]]`
- 在页面底部添加"争议说明"区块
- 等待人工确认后决定保留哪个

## 🛡️ 权限边界

### AI可写区域
- ✅ wiki/ 全目录
- ✅ output/ 全目录

### AI禁止写区域
- ❌ raw/ 全目录
- ❌ 核心概念页（需人工审核）

### 核心概念页（需人工审核）
- `concept-五行识人.md`
- `concept-大圆满见地.md`
- `concept-龙心OS.md`
```

### Step 3: Ingest原始资料录入

**目标**：将新素材录入raw/，触发编译流程

```markdown
## Ingest操作流程

### 3.1 素材收集
1. 使用Obsidian Web Clipper抓取网页
2. 下载PDF到raw/pdf/
3. 手动放入doc/文件夹

### 3.2 触发编译
向AI发送指令：
> "请处理 raw/[文件名]，编译进入wiki/"

### 3.3 AI执行
1. 读取原始素材全文
2. 提炼核心观点
3. 判断归属目录
4. 生成/更新Wiki页面
5. 建立交叉引用
```

### Step 4: Wiki编译生成

**目标**：AI将原始素材编译为结构化Markdown

```markdown
## 编译输出模板

### 概念页模板
```markdown
# concept-[主题名]

## 核心定义
[一句话定义，不超过30字]

## 详细阐述
[2-3段深度解释]

## 与其他概念的关系
- 相关: [[概念A]]、[[概念B]]
- 对比: [[概念C]]（异同点）

## 实践应用
[如何应用的场景]

## 争议与讨论
[如有矛盾观点，在此标记]

## 参考来源
- [[原始素材文件名]]
```

### 实体页模板
```markdown
# entity-[实体名]

## 基本信息
- 类型: [人物/产品/技术/事件]
- 领域: [所属领域]
- 重要度: ⭐⭐⭐

## 核心特征
[3-5个关键点]

## 历史沿革
[时间线，如有]

## 相关项目
- [[关联实体A]]
- [[关联实体B]]

## 原始资料
- [[raw/原始文件名]]
```

### Step 5: 交叉引用构建

**目标**：建立实体页/概念页之间的双向链接

```markdown
## 引用构建规则

### 5.1 实体页关联
当多个文档提及同一实体时：
1. 生成统一的entity-[实体名].md
2. 在各相关页面添加 [[entity-实体名]]
3. 确保实体页包含所有引用片段

### 5.2 概念页关联
当概念之间存在逻辑关系时：
- 因果关系: A导致B → [[A]]→[[B]]
- 对比关系: A vs B → [[A]]与[[B]]对比
- 包含关系: A包含B → [[A]]包含[[B]]

### 5.3 索引更新
每次新增引用后，更新 index.md：
```markdown
## 最近更新
- [[entity-新实体]] - [日期]
- [[concept-新概念]] - [日期]

## 知识图谱概览
[用列表展示顶层分类]
```
```

### Step 6: 健康检查维护

**目标**：定期Lint扫描，保持知识库健康

```markdown
## 健康检查清单

### 6.1 死链接检测
- 扫描所有 [[链接]]
- 确认目标页面存在
- 标记孤立页面

### 6.2 矛盾检测
- 扫描 [[争议]] 标签
- 检查同一主题的多个版本
- 标记需要人工裁决的冲突

### 6.3 过时标记
- 检查6个月未更新的页面
- 添加 [[待更新]] 标签
- 在index.md中显示"可能过时"

### 6.4 链接建议
- 扫描孤立页面
- 建议可能的关联对象
- 生成"待关联"清单

## Lint执行频率
- 增量检查: 每次Ingest后
- 全面检查: 每周一次
```

---

### Step 7: 答案反哺归档

**目标**：优质答案回写Wiki，实现复利增长

```markdown
## 反哺机制

### 7.1 什么答案值得反哺？
- 跨3个以上原始素材的综合分析
- 原创性洞察（非简单摘要）
- 可复用的实践模板

### 7.2 反哺格式
```markdown
# synthesis-[主题]-[日期]

## 原创洞察
[来自这次分析的独特发现]

## 综合来源
- [[entity-实体A]]
- [[entity-实体B]]
- [[concept-概念C]]

## 应用建议
[如何应用到实践中]

## 原创者
LLM Wiki编译 · [日期]
```

### 7.3 反哺流程
1. 分析完成后，询问"是否写入Wiki?"
2. 用户确认 → 生成synthesis页
3. 自动更新index.md
4. 记录为一次"知识复利"
```
