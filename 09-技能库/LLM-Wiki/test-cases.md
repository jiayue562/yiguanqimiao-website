## TC-001：标准场景 - PDF编译

### 场景描述

用户上传了一个关于"五行识人基础理论"的PDF文档，需要编译进入知识库。

### 测试数据

```markdown
输入:
  - 文件: raw/02-学习资料/五行识人基础-20260411.pdf
  - 内容: 五行识人理论概述（5000字）
  - 所属领域: 五行人格心理学

预期输出:
  - 新页面: wiki/02-专业领域知识库矩阵/02-2-五行人格心理学OS/concept-五行识人.md
  - 摘要页: wiki/02-专业领域知识库矩阵/02-2-五行人格心理学OS/summary-五行识人基础-20260411.md
  - 交叉引用: 3-5个
```

### 执行流程

```markdown
Step 1: 触发编译
  用户: "请编译 raw/02-学习资料/五行识人基础-20260411.pdf"

Step 2: AI响应
  📥 输入: raw/02-学习资料/五行识人基础-20260411.pdf
  🎯 目标: wiki/02-专业领域知识库矩阵/02-2-五行人格心理学OS/
  
  ⏳ 执行中...
  [1/5] 读取原始素材... ✓
  [2/5] 提炼核心观点... ✓
  [3/5] 判断归属目录... → 02-2-五行人格心理学OS/
  [4/5] 生成Wiki页面... ✓
  [5/5] 建立交叉引用... ✓
  
Step 3: 输出确认
  ✅ 编译完成！
  
  📄 生成文件:
  ├── concept-五行识人.md
  └── summary-五行识人基础-20260411.md
  
  🔗 交叉引用:
  ├── [[concept-龙心OS]]
  ├── [[concept-五行人格总论]]
  └── [[entity-悟空]]
  
  📊 编译报告:

## TC-002：复杂场景 - 多文档综合

### 场景描述

用户要求对"企业文化"相关的多个文档进行综合分析，生成系统性知识体系。

### 测试数据

```markdown
输入:
  - 文件夹: raw/01-客户案例/企业文化/
  - 文件数: 5个PDF（每个2000-5000字）
  - 总字数: 约20,000字
  - 领域: 企业文化咨询

预期输出:
  - 综合分析页: synthesis-企业文化综合分析-YYYYMMDD.md

## TC-003：边界场景 - 矛盾检测

### 场景描述

用户编译的新文档与知识库中已有文档存在观点矛盾，系统应正确检测并标记。

### 测试数据

```markdown
输入:
  - 新文档: raw/02-学习资料/五行识人新观点-20260411.pdf
    - 内容: 提出与现有"五行识人"不同的观点
  - 冲突点: "木行人是否一定是创新者?"
    - 旧观点: 木行人是创新者的代表
    - 新观点: 木行人也可能是保守者

预期输出:
  - 矛盾检测: 成功
  - 标记方式: [[争议]] 标签
  - 争议说明: 页面底部添加说明
```

### 执行流程

```markdown
Step 1: 触发编译
  用户: "请编译 raw/02-学习资料/五行识人新观点-20260411.pdf"

Step 2: AI执行编译
  ⏳ 执行中...
  [检测到潜在矛盾]
  
  ⚠️ 矛盾警告:
  ┌─────────────────────────────────────────────────┐
  │ 新文档与现有知识存在观点冲突                   │
  ├─────────────────────────────────────────────────┤
  │ 冲突点: 木行人与创新的关系                     │
  │                                                  │
  │ 旧观点（[[concept-木行人]]）:                   │
  │ "木行人是创新者的代表，具有生长性和开创力"    │
  │                                                  │
  │ 新观点（当前文档）:                             │
  │ "木行人也可能是保守者，关键在于阴阳平衡"      │
  │                                                  │
  │ 建议处理方式:                                   │
  │ [A] 标记为 [[争议]]，保留双方观点              │
  │ [B] 更新旧观点，合并为新理解                   │
  │ [C] 创建新页面，专门论述"木行人的双面性"      │
  └─────────────────────────────────────────────────┘
  
  请选择处理方式: [A/B/C]

Step 3: 悟空选择
  悟空: "A"

Step 4: 生成页面
  📄 生成文件:
  └── concept-木行人.md (已更新)
      - 添加 [[争议]] 标签
      - 页面底部添加"争议说明"区块

Step 5: 争议说明格式
  ```markdown
  ## ⚠️ 争议说明
  
  | 来源 | 观点 | 日期 |
  |------|------|------|
  | [[concept-木行人]] | 木行人是创新者代表 | 2026-04-01 |
  | [[raw/五行识人新观点-20260411.pdf]] | 木行人也可能是保守者 | 2026-04-11 |
  
  待裁决: 需要进一步研究木行人与创新的关系，或结合具体案例验证
  
  💡 建议: 可以在下次咨询案例中收集相关数据
  ```
```

### 验收标准

| 检查项 | 预期 | 实际 | 结果 |
|--------|------|------|------|
| 矛盾检测 | 成功识别 | 成功 | ✅ |
| 冲突警告 | 明确展示 | 明确 | ✅ |
| 处理选项 | 提供选择 | 3个选项 | ✅ |
| 标记方式 | [[争议]] | [[争议]] | ✅ |
| 争议说明 | 包含来源和日期 | 包含 | ✅ |
