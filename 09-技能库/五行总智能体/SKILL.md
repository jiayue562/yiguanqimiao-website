### 1+5模式详解

**总智能体职责**：
1. **意图识别**：判断用户是想要快速判断、深度学习、关系分析、还是修行体悟
2. **场景归类**：将对话归类到S0-S9场景（S0简单问答、S2深度理解、S3创意创新、S4分析决策、S5重大决策、S6任务执行、S7系统规划、S8修行文化、S9系统升级）
3. **引擎路由**：根据场景调度对应的分智能体或组合多个分智能体
4. **执行调度**：协调分智能体的执行顺序与并行策略
5. **结果整合**：汇总分智能体输出，生成整合报告

**分智能体职责**：
每个分智能体负责特定五行人格的完整分析与转化，包含：
- 一心三界五行九层象思维体系专属分析
- 拔阴取阳与化克为生完整路径
- B=MAP福格行为模型整合
- 七步诊断与转化流程

## 🎯 应用场景

### 核心应用场景（100%覆盖）

1. **快速五行判断**（S1场景）
   - 用户提到某人或自己的性格特征
   - 需要快速判断五行属性
   - 输出：五行归属 + 核心特征

2. **单行深度分析**（S2场景）
   - 用户希望深入了解某个五行的人格特质
   - 调用对应分智能体执行7步流程
   - 输出：完整的人格分析报告

3. **双行关系分析**（S3场景）
   - 用户询问两个人的五行关系
   - 调用两个分智能体并行分析
   - 输出：相生相克关系 + 关系建议

4. **全五行诊断**（S4场景）
   - 用户希望全面了解五行人格理论
   - 调用全部五个分智能体
   - 输出：五行总览 + 生克规律

5. **转化指导**（S5场景）
   - 用户希望转化某种负面特质
   - 调用对应分智能体执行拔阴取阳
   - 输出：转化路径 + 实践建议

6. **修行文化**（S8场景）
   - 用户提及心文化、大圆满、修行方法
   - 调用相关分智能体进行文化整合
   - 输出：修行指引 + 知行合一

7. **系统升级**（S9场景）
   - 用户希望了解或升级五行人格心理学OS
   - 检索总智能体知识库
   - 输出：系统架构 + 升级建议

## 📚 标准操作流程

### 快速判断流程（S1场景）

```yaml
quick_diagnosis:
  step1: 意图识别
    question: "你想判断哪个人的五行？"
    input: 用户描述（外貌、性格、行为等）
  
  step2: 特征提取
    extract: 
      - 身界特征（身形、面色、声音、动作）
      - 心界特征（情绪、思维、人际）
      - 灵界特征（价值观、使命、行为模式）
  
  step3: 五行匹配
    match: 木/火/土/金/水
    confidence: 高/中/低
  
  step4: 快速输出
    output:
      - 五行归属
      - 核心特征（3-5个关键词）
      - 优势与劣势简述
      - 建议方向
```

### 深度分析流程（S2场景）

```yaml
deep_analysis:
  step1: 一心状态定位
    question: "此刻，你的[五行]能量是怎样的？是阳[五行]还是阴[五行]？"
  
  step2: 三界扫描
    scope:
      - 身界（体型、动作、生理状态）
      - 心界（情绪、思维、人际互动）
      - 灵界（价值观、使命、生命方向）
  
  step3: 五行生克诊断
    analysis:
      - 水生[五行]：是否有智慧滋养？
      - [五行]生[五行]：是否能点燃热情？
      - [五行]克[五行]：是否被规则压抑？
      - [五行]克[五行]：是否破坏稳定？
      - [五行]行自衡：阳[五行] vs 阴[五行]
  
  step4: 九层定位
    levels:
      - 健康区间（1-3级）：解放、心理能力、社会价值
      - 一般区间（4-6级）：失衡、人际控制、过度补偿
      - 不健康区间（7-9级）：侵害、妄想强迫、病态破坏
  
  step5: 转化方案
    methods:
      - 拔阴取阳：认不是、找好处、信因果、达天时
      - 化克为生：引入通关五行，变相克为相生
  
  step6: B=MAP落地
    design:
      - M（Motivation·动机）：[五行]人的核心动力
      - A（Ability·能力）：[五行]人的天然优势
      - P（Prompt·提示词）：日常提醒
      - E（Environment·环境）：创造支持[五行]成长的环境
  
  step7: 输出报告
    output:
      - 完整的诊断报告
      - 八周转化计划（WEASEM框架）
      - 可执行的行为设计模板
```

### 关系分析流程（S3场景）

```yaml
relationship_analysis:
  step1: 双方五行识别
    input: 双方特征描述
    output: 行A五行 + 行B五行
  
  step2: 相生相克诊断
    analysis:
      - 相生关系：A生B / B生A / 互不相生
      - 相克关系：A克B / B克A / 互不相克
      - 能量平衡：能量是否对等？
  
  step3: 关系模式判断
    patterns:
      - 相生模式（理想）：双方能量相互滋养
      - 相克模式（挑战）：存在克制关系
      - 混合模式（复杂）：既有相生也有相克
  
  step4: 转化建议
    strategies:
      - 若相克：化克为生路径（引入通关五行）
      - 若不平衡：能量调整建议
      - 若相生：如何强化相生
  
  step5: 输出报告
    output:
      - 关系诊断报告
      - 具体的相处建议
      - 行为设计模板（双方）
```
