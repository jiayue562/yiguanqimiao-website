# MEMORYmd Skill（长期记忆档案）

## 📖 技能定义

MEMORYmd是AI龙龟共生伙伴操作系统的**长期记忆管理系统**，通过结构化存储提炼后的核心洞见、项目状态和关键决策，确保重要信息永久可检索。

**核心定位**：
- **不是**原始日志的堆砌
- **而是**精心整理的长期记忆库
- **目标**：总长度严格控制在100行以内，只保留精华

**核心原则**：
> 文字胜过大脑记忆——没写下来，就等于没发生过。

---

## 🎯 核心功能

### 1. 主档案（MEMORY.md）

**内容结构**：
- 核心洞见（最重要的认知）
- 项目状态（当前进行中的项目）
- 关键决策（重大决策记录）
- 核心规则（不可逾越的原则）
- 重要联系（关键人员/机构）

**长度控制**：
- **严格限制在100行以内**
- 每条信息不超过3行
- 删除冗余、合并同类、精简表述

### 2. 子文件存储细节信息

**文件结构**：
```
memory/
├── MEMORY.md          # 主档案（100行以内）
├── projects.md        # 项目详情
├── people.md          # 人员信息
├── decisions.md       # 决策记录
├── rules.md           # 规则清单
└── YYYY-MM-DD.md      # 每日日志
```

**子文件内容**：
- `projects.md`：项目的完整背景、目标、进展、风险
- `people.md`：人员的详细信息、关系、联系方式
- `decisions.md`：决策的前因后果、逻辑依据、执行结果
- `rules.md`：所有规则的完整描述、适用场景、例外情况

### 3. 每日日志（YYYY-MM-DD.md）

**记录内容**：
- 每一个决策
- 每一次结果
- 每一个错误
- 每一次收获

**记录格式**：
```markdown
## [时间] [任务/事件]

**背景**：...
**决策/行动**：...
**结果**：...
**关键洞察**：...
```

## 🔄 维护流程

### 1. 实时记录

**触发条件**：
- 完成重要任务后
- 做出关键决策时
- 遇到错误或异常
- 产生新洞察

**执行动作**：
- 立即写入当日日志文件（`memory/YYYY-MM-DD.md`）
- 包含完整背景、行动、结果、洞察

### 2. 定期复盘（每3-5天）

**复盘流程**：
1. 读取近期的每日日志
2. 提炼核心内容
3. 归档到主档案（MEMORY.md）
4. 归档到对应子文件
5. 清理过期日志（>30天）

**归档标准**：
- ✅ 核心洞见→ MEMORY.md
- ✅ 项目状态→ MEMORY.md + projects.md
- ✅ 关键决策→ MEMORY.md + decisions.md
- ✅ 重要人员→ MEMORY.md + people.md
- ❌ 日常琐事→ 不归档
- ❌ 重复信息→ 已有则覆盖

### 3. 主档案维护

**优化规则**：
- 保持总长度<100行
- 删除过期信息
- 合并同类信息
- 精简表述
- 定期清理（每两周）

## 📊 记忆分类体系

### 1. 按重要性分类

| 优先级 | 定义 | 存储位置 |
|--------|------|---------|
| **P0** | 核心决策、关键洞见 | MEMORY.md + 对应子文件 |
| **P1** | 重要项目、主要人员 | MEMORY.md + 对应子文件 |

- 泛化场景（应用范围）
- 关联引擎（龙心OS哪个引擎产生）

**项目状态**：
- 项目名称
- 当前进度（百分比）
- 关键里程碑
- 风险预警
- 下一步行动

**关键决策**：
- 决策时间
- 决策背景
- 决策内容
- 决策依据
- 执行结果
- 复盘反思

## 🎯 核心原则总结

### 三大铁律

1. **文字胜过大脑记忆**
   - 没写下来，就等于没发生过
   - 记忆文件是唯一可靠的记忆

2. **主档案<100行**
   - 只保留精华，不存冗余
   - 定期清理，保持精简

3. **定期复盘归档**
   - 每3-5天复盘一次
   - 及时归档，不要拖延
   - 清理过期日志

### 核心价值

- **永久可检索**：重要信息不会丢失
- **上下文注入**：新任务快速获取背景
- **避免重复错误**：历史决策和洞见可复用
- **知识沉淀**：将经验转化为长期资产

## 十二、规划规则：写下来，拒绝脑记（对标OpenClaw #9 Planning）

> **核心铁律**：任何需要记住的内容，都必须写入文件，不依赖大脑记忆。

### 12.1 规划规则定义

**规划规则（Planning Rules）**：
- 定义：所有需要留存的信息必须文件化，禁止依赖"脑记"
- 目的：杜绝信息遗漏，确保跨会话信息完整性
- 本质：**文件持久 > 上下文临时**——文件可跨会话保留，上下文窗口不能

### 12.2 强制文件化场景

| 场景 | 文件化要求 | 存储位置 |
|------|-----------|---------|
| 用户提出新需求 | 记录到每日日志 | `memory/YYYY-MM-DD.md` |
| 做出重要决策 | 记录到MEMORY.md | `MEMORY.md` |
| 制定行动计划 | 创建任务列表 | `todo_write` 或看板 |
| 发现新规律/洞察 | 知行合一沉淀 | Obsidian + MEMORY.md |
| 收到用户偏好 | 更新USER.md | `USER.md` |
| 完成系统升级 | 更新TOOLS.md | `TOOLS.md` |
| 出现错误/异常 | 记录到LEARNINGS.md | `LEARNINGS.md` |
| 用户说"记住这个" | **立即**更新记忆文件 | 对应记忆文件 |

### 12.3 "记住这个"标准操作流程

当收到用户说"记住这个"、"记住X"、"别忘了X"时：

```
步骤1：确认理解
  → "好的，我记住了：[复述核心内容]"
  
步骤2：立即写入文件
  → 根据内容类型选择合适的记忆文件
  → 重要规则 → MEMORY.md
  → 偏好习惯 → USER.md  
  → 项目信息 → memory/projects.md
  → 日常事项 → memory/YYYY-MM-DD.md
  
步骤3：确认存储
  → "已存储到 [文件名]，下次对话我会自动读取。"
```

## 十三、记忆限制规则（对标OpenClaw #20 Memory is Limited）

> **核心原则**：记忆容量有限。所有需要留存的信息，必须写入文件。"脑记笔记"无法在会话重启后保留，但文件可以。

### 13.1 记忆限制规则定义

**记忆限制规则（Memory is Limited）**：
- 定义：AI的上下文窗口（工作记忆）有容量限制，会话结束后即丢失
- 目的：强制信息持久化，确保重要信息不因会话结束而丢失
- 本质：**文件是唯一可靠的长期记忆载体**

### 13.2 三大标准操作

| 触发条件 | 标准操作 | 存储位置 |
|---------|---------|---------|
| 收到"记住这个"指令 | **立即**更新记忆文件 | 根据内容类型选择 |
| 当学到新经验时 | 更新学习档案 | `LEARNINGS.md` |
| 当出现错误时 | 记录归档 | `memory/learnings/` |
| 对话产生重要洞察 | 知行合一沉淀 | MEMORY.md + Obsidian |
| 用户表达偏好 | 更新用户画像 | `USER.md` |

### 13.3 信息持久化决策树

```
这条信息重要吗？
    ├── 不重要 → 不需要持久化
    └── 重要 → 是哪种类型？
                ├── 用户偏好/习惯 → USER.md
                ├── 项目信息/状态 → memory/projects.md
                ├── 错误/经验教训 → LEARNINGS.md
                ├── 核心洞见/决策 → MEMORY.md
                ├── 日常记录/决策 → memory/YYYY-MM-DD.md
                └── 专业知识 → Obsidian知识库
```

### 13.4 与三向同步的整合

```
Obsidian（主知识库）     ← 专业知识、结构化文档
WorkBuddy（工作记忆）    ← 每日日志、项目状态
IMA（云端笔记）          ← 快速笔记、移动端访问
        ↕ 三向同步 ↕
MEMORY.md（长期记忆）    ← 核心洞见、关键决策（100行以内）
```
