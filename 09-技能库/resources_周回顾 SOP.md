# 周回顾 SOP - 标准操作流程

> 🔄 龙心 OS 周度复盘系统 · v1.0

---

## 📋 核心目标

每周花 30-60 分钟回顾本周工作，确保：
1. 新增笔记有双向链接
2. 知识网络持续完善
3. 目标与行动对齐
4. 持续反思与成长

---

## ⏰ 建议时间

- **固定时间**: 每周五下午 16:00-17:00 或 周日上午 9:00-10:00
- **时长**: 30-60 分钟
- **环境**: 安静、无干扰

---

## 🔧 操作流程

### 阶段一：数据收集 (5 分钟)

```powershell
# 1. 查看本周新增笔记
obsidian-agent recent 7

# 2. 查看孤立笔记
obsidian-agent orphans

# 3. 查看统计
obsidian-agent stats
```

**记录**:
- 本周新增笔记数量
- 新增孤立笔记数量
- 各类别笔记分布

---

### 阶段二：笔记整理 (15-25 分钟)

#### Step 1: 检查新增笔记

对于本周新增的每篇笔记：
- [ ] 是否有入向链接？
- [ ] 是否有出向链接？
- [ ] 标签是否完整？
- [ ] 是否归类到正确的目录？

#### Step 2: 添加双向链接

对于孤立的新增笔记：
1. 找到相关的已有笔记
2. 在新增笔记中添加 `[[相关笔记]]`
3. 在相关笔记中添加 `[[新增笔记]]`

**示例**:
```markdown
// 在新建的"人机协同方法论"Area 中
## 🔗 相关链接
- [[00-知识库总索引]]
- [[人机协同五象限]]
- [[知行合一]]

// 在"人机协同五象限"Resource 中
## 🔗 相关链接
- [[人机协同方法论]] ← 新增链接
```

#### Step 3: 更新总索引

编辑 [[00-知识库总索引]]：
- [ ] 将新 Area 添加到 Areas 列表
- [ ] 将新 Project 添加到 Projects 列表
- [ ] 将新 Resource 添加到 Resources 列表
- [ ] 更新"最近更新"部分

---

### 阶段三：目标回顾 (10-15 分钟)

#### Step 1: 检查上周目标

打开上周的周回顾笔记：
- [ ] 哪些目标完成了？
- [ ] 哪些目标未完成？原因是什么？
- [ ] 有什么意外收获？

#### Step 2: 填写本周回顾

使用 [[周回顾模板]]：
- [ ] 填写基本信息
- [ ] 回顾本周目标完成情况
- [ ] 记录本周成果
- [ ] 五色光思维分析
- [ ] 人机协同五象限反思
- [ ] 记录关键洞察

---

### 阶段四：下周计划 (5-10 分钟)

#### Step 1: 设定核心目标

基于本周回顾，设定 1-3 个下周核心目标：
- 目标应具体、可衡量
- 与长期方向对齐
- 考虑资源和时间约束

#### Step 2: 规划关键行动

为每个目标列出 2-3 个关键行动：
- [ ] 行动 1
- [ ] 行动 2
- [ ] 行动 3

#### Step 3: 识别需要整理的笔记

记录下周需要特别关注的笔记：
- [ ] 需要补充链接的笔记
- [ ] 需要合并的笔记
- [ ] 需要归档的笔记

---

## 📊 检查清单

### 每周必做
- [ ] 运行 `obsidian-agent recent 7` 查看新增
- [ ] 运行 `obsidian-agent orphans` 检查孤立笔记
- [ ] 为新增笔记添加双向链接
- [ ] 更新总索引
- [ ] 填写周回顾模板
- [ ] 设定下周目标

### 每月选做
- [ ] 运行 `obsidian-agent duplicates` 检查重复笔记
- [ ] 运行 `obsidian-agent broken-links` 检查断裂链接
- [ ] 整理归档旧笔记
- [ ] 审视 Areas 是否仍然相关

---

## 🎯 成功指标

### 量化指标
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 新增笔记链接率 | >80% | 新增笔记中有入向链接的比例 |
| 孤立笔记增长率 | <10%/周 | 每周新增孤立笔记不超过 10% |
| 周回顾完成率 | 100% | 每周按时完成回顾 |

### 质化指标
- 知识网络是否更加紧密？
- 查找信息是否更加高效？
- 目标与行动是否更加对齐？
