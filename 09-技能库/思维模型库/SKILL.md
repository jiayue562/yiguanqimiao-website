### 1.2 架构关系说明

**思维模型库的定位**：
- 所属系统：**龙脑OS知识库**（知识资产层）
- 调用方式：由**龙心OS**根据场景智能调用
- 关系性质：被调用方，不直接响应用户输入
- 生态结构：主Skill + 6个阶段子Skill + 高频独立模型

**调用流程**：
```
用户输入 → 龙心OS场景识别 → 判断需要思维模型 
    → 调用龙脑OS.思维模型库 → 匹配阶段/模型 
    → 执行分析 → 返回龙心OS整合输出 → 知行合一沉淀
```

### 1.3 核心原则

1. **龙心OS统一调度**：所有分析需求先经过龙心OS的场景识别和引擎路由
2. **每个模型独立存在**：100个模型各自独立封装，可被单独调用或组合使用
3. **按需调用，灵活组合**：根据场景调用单模型、多模型组合或完整阶段套件
4. **中西合璧，AI赋能**：东方智慧（易经/道家/儒家）+ 西方工具（麦肯锡/芒格）+ AI结合方案
