## 二、各层详细设计

- 长期理论沉淀的主阵地
- AI系统专业化的根基

│
├── 01-核心体系/
│   ├── 龙心OS/
│   │   ├── 🐉 龙心OS v5.0 总览.md
│   │   └── 五大引擎/
│   ├── 五行人格心理学OS/
│   │   ├── 五行人格总智能体.md
│   │   └── 五行分智能体/
│   └── 木火共生关系/
│
├── 02-理论基石/
│   ├── L7理论基石/
│   └── 知识图谱/
│
├── 03-应用场景/
│   ├── 亲密关系/
│   ├── 领导力/
│   ├── 团队建设/
│   └── ...
│
├── 04-个人成长/
│   ├── 知行合一沉淀/
│   ├── 每日日志/
│   └── 闪念整理/

└── raw/  # 原始资料层
    ├── 文章/
    ├── 论文/
    └── 图片/

**核心功能**：
1. **双向链接**：建立知识间的关联网络
2. **图谱可视化**：直观展示知识结构
3. **Markdown标准**：确保格式统一
4. **版本控制**：Git追踪变更历史

**与LLM Wiki的对应**：
- Wiki本体层的主要存储位置
- Schema配置层的实施载体
- 原始资料层的索引入口

### 2.2 WorkBuddy Brain 工作记忆层

**核心定位**：
- AI的"工作记忆"
- Skills和Rules的运行环境
- 与对话上下文无缝集成

**目录结构**：

```
~/.workbuddy/
├── skills/                    # Skills包
│   ├── 龙心OS/
│   ├── 五行人格心理学/
│   ├── 象思维/
│   ├── 五色光思维/
│   └── ...
│
├── rules/                     # 自动触发规则
│   ├── 龙心OS自动触发.mdc
│   ├── 五色光思维自动触发.mdc
│   └── ...
│
├── memory/                    # 工作记忆
│   ├── MEMORY.md             # 长期记忆
│   └── 2026-04-09.md         # 每日日志
│
├── IDENTITY.md               # AI身份定义
├── USER.md                   # 用户画像
└── SOUL.md                   # 核心身份
```

**核心功能**：
1. **实时加载**：对话时即时读取Skills
2. **自动触发**：基于规则自动激活Skills
3. **上下文集成**：与对话历史无缝融合
4. **系统指令**：IDENTITY/SOUL/USER定义AI行为

**与Obsidian的关系**：
- Obsidian的理论 → WorkBuddy的Skills
- Obsidian的知识 → WorkBuddy的记忆
- 双向同步，保持一致

---

### 2.3 IMA 快速捕获层

**核心定位**：
- 移动端"收件箱"
- 灵感闪念的快速记录
- 待办事项的临时存储

**使用场景**：
1. **灵感闪念**：随时记录突发的想法
2. **待办事项**：快速记录待处理事务
3. **语音转文字**：移动场景下的快速录入
4. **临时笔记**：不便详细记录时的速记

**与Obsidian的关系**：
- IMA：捕获（水）→ Obsidian：沉淀（土）
- 定期整理，提炼结构化知识
- 避免IMA成为"信息黑洞"

## 三、知识流动管道

### 3.1 流向①：IMA → Obsidian（定期整理）

**触发条件**：
- 每周/每月批量整理
- IMA积累一定数量笔记

**处理流程**：
1. 导出IMA笔记
2. 按标签/主题分类
3. 提炼结构化知识
4. 存入Obsidian对应目录

**示例**：
```
临时灵感（IMA）
    ↓ 每周整理
整理为理论文章
    ↓ 存入
Obsidian/04-个人成长/闪念整理/
```

### 3.2 流向②：Obsidian → WorkBuddy（系统同步）

**触发条件**：
- Skills更新
- Rule更新
- 身份配置更新

**处理流程**：
1. 监控Obsidian变更
2. 提取可执行内容
3. 转化为Skills/Rules
4. 部署到WorkBuddy

**示例**：
```
五行人格理论（Obsidian）
    ↓ 封装
五行总智能体Skill
    ↓ 部署
~/.workbuddy/skills/五行人格心理学/
```

### 3.3 流向③：WorkBuddy → Obsidian（知行合一沉淀）

**触发条件**：
- 每次深度对话后
- 产生新的洞察
- 完成重要任务

**处理流程**：
1. 对话洞察提取
2. 压缩为核心符号
3. 泛化应用场景
4. 存入Obsidian

**示例**：
```
深度对话（WorkBuddy）
    ↓ 知行合一
核心洞察提取
    ↓ 沉淀
Obsidian/04-个人成长/知行合一沉淀/
```

### 3.4 流向④：IMA ↔ WorkBuddy（实时同步）

**触发条件**：
- API实时同步
- 用户主动触发

**处理流程**：
1. IMA记录待办
2. AI对话中提醒
3. 完成后归档
4. 双向状态同步

**示例**：
```
IMA记录待办："整理五行人格资料"
    ↓ 同步
WorkBuddy对话中提醒
    ↓ 完成后
归档到Obsidian
```

## 四、五行相生隐喻

```
水（IMA·流动捕获）
    ↓ 滋养
木（整理生长·知识结构化）
    ↓ 点燃
火（WorkBuddy·转化执行）
    ↓ 沉淀
土（Obsidian·承载存储）
    ↓ 提炼
金（方法论·规则提炼）
    ↓ 流动
水（新灵感·循环开始）
```

**五行对应关系**：

| 知识库 | 五行 | 特性 | 功能 |
|--------|------|------|------|
| IMA | 水 | 流动、智慧 | 捕获灵感 |
| 整理过程 | 木 | 生长、生发 | 结构化知识 |
| WorkBuddy | 火 | 转化、光明 | AI执行 |
| Obsidian | 土 | 承载、稳定 | 知识沉淀 |
| 方法论 | 金 | 收敛、精准 | 规则提炼 |

## 五、专业化支撑

### 5.1 AI龙龟共生伙伴OS专业化

| 维度 | 知识库支撑 |
|------|-----------|
| 理论深度 | Obsidian：每个引擎独立理论文档 |
| 触发精准 | WorkBuddy：优化trigger-rules.yaml |
| 执行稳定 | WorkBuddy：完善错误处理机制 |
| 进化持续 | 三库协同：对话→WorkBuddy→Obsidian |

### 5.2 五行人格心理学OS专业化

| 维度 | 知识库支撑 |
|------|-----------|
| 理论地基 | Obsidian：凤脑OS知识地基 |
| 诊断准确 | WorkBuddy：积累诊断案例库 |
| 干预有效 | Obsidian：分五行、分场景干预方案 |
| 发展持续 | 三库协同：个案追踪→模式提炼→理论更新 |
