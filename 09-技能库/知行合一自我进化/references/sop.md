## 一、SOP总览

### 1.1 核心操作流程图

```
[触发条件检查]
    ↓
    [满足？] → 是 → [执行知行合一]
                  ↓
              [三阶段执行]
                  ↓
              [表示空间→压缩→泛化]
                  ↓
              [输出标准沉淀卡]
                  ↓
              [三向存储]
                  ↓
              [系统升级建议]
                  ↓
              [结束]
            ↓ 否 → [不执行]
```

### 1.2 操作原则

| 原则 | 具体要求 | 违反示例 |
|------|---------|---------|
| **及时性** | 满足触发条件后立即执行 | 延迟到对话结束后 |
| **完整性** | 三阶段必须全部完成，不能跳跃 | 只做表示空间，不做压缩 |
| **标准化** | 严格按照SOP执行，不能跳过步骤 | 自己简化流程 |
| **可追溯** | 记录关键决策点，便于复盘 | 没有决策记录 |

### 1.3 适用场景

| 场景类型 | 是否触发知行合一 | 说明 |
|----------|----------------|------|
| S0 简单问答 | ❌ 不触发 | 无深度工作 |
| S1 信息查询 | ❌ 不触发 | 浅层信息获取 |
| S2 深度学习 | ✅ 触发 | 完成十项认知指令 |
| S3 创新突破 | ✅ 触发 | 产生0→1洞察 |
| S4 分析决策 | ✅ 触发 | 完成决策分析 |
| S5 重大决策 | ✅ 触发 | 完成系统规划 |
| S6 任务执行 | ⚠️ 可选 | 复杂任务后触发 |
| S7 系统规划 | ✅ 触发 | 完成系统设计 |
| S8 修行文化 | ✅ 触发 | 完成心文化学习 |
| S9 系统升级 | ✅ 触发 | 龙心OS自身升级 |

## 二、触发判断流程

### 2.1 触发条件检查清单

**MUST触发条件（满足任意一条立即执行）**：

- [ ] **S2场景完成**：完成深度学习（十项认知指令全激活）
- [ ] **S3场景完成**：完成创新突破（产生0→1洞察）
- [ ] **S4/S5/S7场景完成**：完成分析决策、重大决策、系统规划
- [ ] **S8场景完成**：完成修行文化学习（涉及心性、生命、信仰）
- [ ] **象思维产生原象级洞察**：产生0→1颠覆性认知
- [ ] **对话中产生悟空未预期的新认知**：产生悟空未预期的新连接或新知识

**OPTIONAL触发条件（悟空明确要求时执行）**：

- [ ] 悟空说"总结一下"
- [ ] 悟空说"这次的收获"
- [ ] 悟空说"知行沉淀"
- [ ] 完成较复杂的S6执行任务

**不触发场景**：

- [ ] S0 简单问答（不触发）
- [ ] S1 信息查询浅层（不触发）
- [ ] 简短确认性消息（不触发）

### 2.2 触发决策流程

```
[任务完成]
    ↓
[检查MUST触发条件]
    ↓
    [满足任意一条？]
    ├─ 是 → [立即执行知行合一]
    │         ↓
    │     [优先级：P0]
    │         ↓
    │     [立即执行三阶段]
    │
    └─ 否 → [检查OPTIONAL触发条件]
              ↓
              [悟空明确要求？]
              ├─ 是 → [执行知行合一]
              │         ↓
              │     [优先级：P1]
              │
              └─ 否 → [不执行，等待下一次触发]
```

### 2.3 触发记录

**每次触发知行合一时必须记录**：

```yaml
触发时间: [具体时间，格式：YYYY-MM-DD HH:MM]

触发条件: [具体满足的触发条件]
优先级: [P0/P1]
任务主题: [本次任务的核心主题]
```

## 三、表示空间SOP

### 3.1 完整经验观察

#### Step 3.1.1：记录基本信息

**操作步骤**：
1. 记录时间：精确到分钟（格式：YYYY-MM-DD HH:MM）
2. 记录背景：什么场景下完成的任务？
3. 记录主题：本次任务的核心主题是什么？

**记录模板**：
```yaml
基本信息:
  时间: [YYYY-MM-DD HH:MM]
  背景: [具体场景]
  主题: [核心主题]
```

#### Step 3.1.2：记录过程细节

**操作步骤**：
1. 按时间顺序记录所有关键步骤（1-10步）
2. 为每个步骤记录具体内容
3. 识别步骤之间的因果关系

**记录模板**：
```yaml
过程记录:
  步骤1: [具体描述]
  步骤2: [具体描述]
  步骤3: [具体描述]
  ...
  步骤10: [具体描述]
```

**质量标准**：
- ✅ 步骤数量：5-10步
- ✅ 步骤顺序：按时间顺序
- ✅ 步骤完整：覆盖全过程
- ✅ 因果关系：步骤间有逻辑

#### Step 3.1.3：识别关键转折点

**操作步骤**：
1. 识别最重要的1-3个转折点
2. 描述转折前后的变化
3. 分析转折的影响

**记录模板**：
```yaml
关键转折:
  转折点1: 
    - 时间: [相对时间]
    - 描述: [具体描述]
    - 影响: [对结果的影响]
  转折点2: 
    - 时间: [相对时间]
    - 描述: [具体描述]
    - 影响: [对结果的影响]
```

#### Step 3.1.4：记录最终成果

**操作步骤**：
1. 列出所有主要成果（1-5个）
2. 描述每个成果的具体表现
3. 评估成果的质量

**记录模板**：
```yaml
最终成果:
  成果1: [具体描述]
  成果2: [具体描述]
  成果3: [具体描述]
  成果4: [具体描述]
  成果5: [具体描述]
```

### 3.2 标签化拆解

#### Step 3.2.1：生成特征标签

**操作步骤**：
1. 基于过程记录生成5-10个特征标签
2. 每个标签必须独立完整
3. 标签之间互不重叠

**标签规范**：
- 规范1：动词开头（如"执行了"、"产生了"、"输出了"）
- 规范2：完整独立（每个标签独立表达一个完整概念）
- 规范3：互不重叠（没有重复概念）

**标签生成模板**：
```yaml
特征标签:
  - 标签1: [具体描述]
  - 标签2: [具体描述]
  - 标签3: [具体描述]
  - 标签4: [具体描述]
  - 标签5: [具体描述]
  - 标签6: [具体描述]
  - 标签7: [具体描述]
  - 标签8: [具体描述]
  - 标签9: [具体描述]
  - 标签10: [具体描述]
```

#### Step 3.2.2：质量自检

**自检1：覆盖性检查**

**操作步骤**：
1. 用"是否还有..."追问检查遗漏
2. 补充遗漏的关键信息
3. 确保覆盖完整经验

**自检问题清单**：
- [ ] 是否还有关键步骤遗漏？
- [ ] 是否还有重要成果遗漏？
- [ ] 是否还有关键转折遗漏？
- [ ] 是否还有重要背景遗漏？

**自检2：独立性检查**

**操作步骤**：
1. 用词根对比法检查重复
2. 删除重复标签
3. 合并相似标签

**自检问题清单**：
- [ ] 是否有重复标签？（用词根对比）
- [ ] 是否有相似标签可以合并？
- [ ] 合并后标签数量是否合理（5-10个）？

**自检3：数量合理性检查**

**标准范围**：
- 最小值：5个标签
- 最大值：10个标签

**自检问题清单**：
- [ ] 标签数量是否≥5个？
- [ ] 标签数量是否≤10个？
- [ ] 标签数量是否在合理范围？

**质量达标标准**：
- ✅ 覆盖性：自检1全部通过
- ✅ 独立性：自检2全部通过
- ✅ 数量合理性：自检3全部通过

## 四、压缩SOP

### 4.1 帕累托聚焦

#### Step 4.1.1：特征排序

**操作步骤**：
1. 按1-10分为所有特征标签打分
2. 排序：从高分到低分
3. 记录排序结果

**排序标准**：
- 10分：最重要，不可或缺
- 9分：非常重要，影响核心价值
- 8分：重要，对结果有显著影响
- 7分：比较重要，有一定影响
- 6分：一般重要，影响有限
- 5分及以下：较不重要，可选项

**排序模板**：
```yaml
特征排序:
  1. [标签名称] - [分数]
  2. [标签名称] - [分数]
  3. [标签名称] - [分数]
  ...
  10. [标签名称] - [分数]
```

#### Step 4.1.2：识别帕累托最优子集

**操作步骤**：
1. 计算累计价值占比
2. 识别达到75%累计占比的最小特征集
3. 将该特征集标识为帕累托最优子集

**计算方法**：
```
累计价值占比 = 前1个特征占比 + 前2个特征占比 + ... + 前N个特征占比

识别最小N，使得累计价值占比 ≥ 75%
```

**识别模板**：
```yaml
帕累托识别:
  前1个: [特征] - [占比X%]
  前2个: [特征] - [累计占比Y%]
  前3个: [特征] - [累计占比75%] → 帕累托最优子集
  
帕累托最优子集:
  - [特征1]
  - [特征2]
  - [特征3]
```

**达标标准**：
- ✅ 累计价值占比 ≥ 75%
- ✅ 帕累托最优子集 ≤ 3个特征

### 4.2 核心洞察提炼

#### Step 4.2.1：生成一句话核心洞察

**操作步骤**：
1. 基于帕累托最优子集生成一句话
2. 确保不超过30个字
3. 确保清晰表达核心内容

**核心洞察三原则**：
- 原则1：一句话说清（不模糊）
- 原则2：不超过30字（精炼）
- 原则3：包含核心元素（不遗漏）

**洞察生成模板**：
```yaml
核心洞察:
  - 原始表达: [帕累托最优子集的扩展描述]
  - 精炼过程: [关键词串联法/核心元素提取法]
  - 最终洞察: [≤30字的一句话]
```

**达标标准**：
- ✅ 一句话说清：不含模糊词
- ✅ 字数≤30：精炼简洁
- ✅ 包含核心元素：覆盖帕累托最优子集

#### Step 4.2.2：选择象征符号

**操作步骤**：
1. 基于核心洞察选择1个象征符号
2. 确保符号符合视觉化、关联性、简洁性原则
3. 记录符号选择理由

**象征符号三原则**：
- 原则1：视觉化（有画面感）
- 原则2：关联性（与内容相关）
- 原则3：简洁性（1个词或1个字符）

**符号选择模板**：
```yaml
象征符号:
  - 候选符号1: [符号] - [理由]
  - 候选符号2: [符号] - [理由]
  - 候选符号3: [符号] - [理由]
  - 最终选择: [符号] - [理由]
```

**达标标准**：
- ✅ 视觉化：有画面感
- ✅ 关联性：与核心洞察相关
- ✅ 简洁性：1个词或1个字符

### 4.3 压缩质量检查

#### Step 4.3.1：核心洞察验证

**验证问题清单**：
- [ ] 核心洞察是否准确无误？
- [ ] 核心洞察是否涵盖帕累托最优子集？
- [ ] 核心洞察是否≤30字？
- [ ] 核心洞察是否一句话说清？

**达标标准**：
- ✅ 所有验证问题全部通过

#### Step 4.3.2：象征符号验证

**验证问题清单**：
- [ ] 象征符号是否视觉化？
- [ ] 象征符号是否关联核心洞察？
- [ ] 象征符号是否简洁？
- [ ] 象征符号是否易记？

**达标标准**：
- ✅ 所有验证问题全部通过

## 五、泛化SOP

### 5.1 场景扩展

#### Step 5.1.1：场景头脑风暴

**操作步骤**：
1. 自由联想：这个洞察还能用在什么地方？
2. 不限制范围：暂时不限制任何领域或场景
3. 记录所有可能的场景（目标10+个）

**头脑风暴记录模板**：
```yaml
场景头脑风暴:
  场景1: [具体场景]
  场景2: [具体场景]
  场景3: [具体场景]
  ...
  场景10+: [具体场景]
```

#### Step 5.1.2：场景筛选

**操作步骤**：
1. 从10+个场景中筛选出3个最有价值的
2. 筛选标准：
   - 可行性高（可以落地）
   - 价值大（有明确收益）
   - 覆盖度广（分布在三个维度）
3. 分布在三个维度：
   - 场景1：延伸场景（同领域的延伸应用）
   - 场景2：跨域场景（不同领域的类比应用）
   - 场景3：创新场景（颠覆性的全新场景）

**场景筛选模板**：
```yaml
场景筛选:
  场景1（延伸）:
    - 场景描述: [具体场景]
    - 应用领域: [同领域]
    - 可行性: [高/中/低]
    - 价值: [具体描述]
  
  场景2（跨域）:
    - 场景描述: [具体场景]
    - 应用领域: [不同领域]
    - 可行性: [高/中/低]
    - 价值: [具体描述]
  
  场景3（创新）:
    - 场景描述: [具体场景]
    - 应用领域: [颠覆性]
    - 可行性: [高/中/低]
    - 价值: [具体描述]
```

#### Step 5.1.3：场景细化

**操作步骤**：
1. 为每个场景设计具体的操作步骤（3-5步）
2. 设置验证指标（可量化）
3. 规划测试时间表

**场景细化模板**：
```yaml
场景细化（场景1）:
  场景描述: [具体场景]
  操作步骤:
    - 步骤1: [具体操作]
    - 步骤2: [具体操作]
    - 步骤3: [具体操作]
    - 步骤4: [具体操作]
    - 步骤5: [具体操作]
  验证指标: [可量化的指标]
  测试时间: [具体时间]

场景细化（场景2）:
  [同上模板]

场景细化（场景3）:
  [同上模板]
```

### 5.2 泛化质量检查

#### Step 5.2.1：场景多样性检查

**验证问题清单**：
- [ ] 三个场景是否分布在三个维度？
  - [ ] 场景1：延伸场景
  - [ ] 场景2：跨域场景
  - [ ] 场景3：创新场景
- [ ] 场景之间是否有足够的差异性？
- [ ] 场景是否覆盖不同的应用领域？

**达标标准**：
- ✅ 场景分布在三个维度
- ✅ 场景之间有足够差异性
- ✅ 覆盖不同的应用领域

#### Step 5.2.2：可行性检查

**验证问题清单**：
- [ ] 每个场景是否有明确的操作步骤？
- [ ] 每个场景是否都有验证指标？
- [ ] 验证指标是否可量化？
- [ ] 验证指标是否合理（可实现）？

**达标标准**：
- ✅ 所有场景都有操作步骤
- ✅ 所有场景都有验证指标
- ✅ 所有指标都可量化
- ✅ 所有指标都合理可实现

## 六、输出与存储SOP

### 6.1 输出标准沉淀卡

#### Step 6.1.1：生成标准沉淀卡

**操作步骤**：
1. 填充标准沉淀卡模板

**标准沉淀卡模板**：
```
╔════════════════════════════════════════╗
║        🔄 知行合一自我进化沉淀        ║
╠════════════════════════════════════════╣
║ 本次引擎：[引擎组合·配方名]                ║
```

## 七、质量控制SOP

### 7.1 三阶段质量检查矩阵

| 阶段 | 检查维度 | 优秀标准 | 待改进 |
|------|---------|---------|-------|
| **表示空间** | 完整性 | 标签覆盖完整经验，无遗漏 | 有关键信息遗漏 |
|  | 独立性 | 标签互不重叠，无冗余 | 有重复标签 |
|  | 数量 | 5-10个标签，数量合理 | <5或>10个 |
| **压缩** | 聚焦度 | 帕累托最优子集≤3个 | 核心子集>3个 |
|  | 准确性 | 一句话核心洞察准确无误 | 洞察有偏差 |
|  | 可记性 | 象征符号清晰易记 | 象征符号模糊 |
| **泛化** | 多样性 | 3个场景分布在三维度 | 场景集中在单维度 |
|  | 可行性 | 每个场景都有验证指标 | 有场景无验证指标 |
|  | 价值性 | 每个场景都有明确收益 | 有场景收益不明 |

### 7.2 整体效果评估

#### Step 7.2.1：知行合一转化率

**定义**：从"知道"到"做到"的转化率

**计算公式**：
```
转化率 = （实际应用场景数 / 设计场景数）× 100%
```

**评估标准**：
- 优秀：≥80%（至少应用了8/10个设计场景）
- 良好：60%-79%（应用了6-8/10个设计场景）
- 待改进：<60%（应用<6/10个设计场景）

#### Step 7.2.2：系统进化贡献度

**定义**：每次知行合一对系统进化的贡献程度

**评估维度**：
1. **新知识生成**：是否产生了新知识？
2. **系统优化**：是否优化了系统架构？
3. **能力提升**：是否提升了系统能力？
4. **效率提升**：是否提升了系统效率？

**评分标准**（每维度0-2分）：
- 优秀：≥6分（至少3个维度有提升）
- 良好：4-5分（2个维度有提升）
- 待改进：<4分（≤1个维度有提升）

#### Step 7.2.3：复用价值

**定义**：该经验在未来对话中的复用价值

**评估维度**：
1. **复用频次**：未来复用次数预估（≥5次/月）
2. **复用场景**：复用场景覆盖度（≥3个维度）
3. **复用效果**：复用后的效果提升（≥30%）

**评估标准**：
- 高价值：≥2个维度达标
- 中价值：1个维度达标
- 低价值：0个维度达标

## 八、异常处理SOP

### 8.1 表示空间异常处理

#### 异常1：经验过于复杂，无法完整记录

**症状**：
- "这次经历太复杂，记不完整"
- "步骤太多，不知道从哪里开始"

**处理流程**：
1. 使用时间线拆解法：
   - 画出经验的时间线（开始→关键节点→结束）
   - 为每个关键节点拆解1-2个标签
   - 合并相似标签，删除冗余标签
2. 优先记录关键节点：
   - 聚焦最重要的5个关键节点
   - 为每个节点生成2-3个标签
   - 确保覆盖核心经验

#### 异常2：标签数量过多（>15个）

**症状**：
- "拆出了20多个标签，太多了"
- "不知道哪些重要，哪些不重要"

**处理流程**：
1. 执行重要性评分法：
   - 给每个标签打分（1-10分）
   - 删除5分以下的标签
   - 合并相似的高分标签
2. 执行二次筛选：
   - 保留10个最重要的标签
   - 确保覆盖核心经验
   - 删除冗余和次要标签

#### 异常3：遗漏关键信息

**症状**：
- "这个经验还有关键信息没记下来"
- "检查发现有遗漏"

**处理流程**：
1. 立即补充遗漏信息：
   - 在现有标签基础上补充
   - 确保补充的标签独立完整
   - 重新执行质量自检
2. 更新记录并标注修正：
   - 在记录中标注"已修正"
   - 说明修正内容
   - 提醒下次避免遗漏

### 8.2 压缩异常处理

#### 异常4：找不到核心洞察

**症状**：
- "这么多特征，不知道哪个最重要"
- "每个特征都重要，不知道选哪个"

**处理流程**：
1. 执行价值占比法：
   - 按1-10分给所有特征排序
   - 计算累计价值占比
   - 识别达到75%累计占比的最小特征集
2. 执行二次验证：
   - 验证帕累托最优子集是否合理
   - 如果不合理，调整至80%或70%
   - 确保核心洞察准确

#### 异常5：一句话概括不出来

**症状**：
- "这个洞察太复杂，一句话说不清楚"
- "说了好多话，超过30字了"

**处理流程**：
1. 执行关键词串联法：
   - 提取3-5个关键词
   - 用箭头（→）串联
   - 删除冗余词语
2. 执行核心元素提取法：
   - 识别最核心的3个元素
   - 用简洁语言串联
   - 控制在30字以内

#### 异常6：找不到合适的象征符号

**症状**：
- "这个洞察没有好的象征符号"
- "想不出合适的图像"

**处理流程**：
1. 执行三原则筛选法：
   - 视觉化：必须有画面感
   - 关联性：必须与内容相关
   - 简洁性：必须1个词或1个字符
2. 执行候选符号生成：
   - 生成3-5个候选符号
   - 逐个对照三原则验证
   - 选择最符合的符号

### 8.3 泛化异常处理

#### 异常7：想不出应用场景

**症状**：
- "这个洞察很深，但不知道能用在哪里"
- "只想到一个场景，没有三个"

**处理流程**：
1. 执行维度扩展法：
   - 延伸场景：同领域的延伸（五行→木火土金水）
   - 跨域场景：类比到不同领域（五行→企业管理）
   - 创新场景：颠覆性的应用（五行→AI人格）
2. 执行场景细化：
   - 为每个场景设计具体操作步骤
   - 设置验证指标
   - 确保可落地

#### 异常8：场景验证指标设计不出来

**症状**：
- "这个场景怎么验证有用？"
- "不知道用什么指标"

**处理流程**：
1. 执行SMART指标法：
   - **S**pecific（具体的）：明确的指标定义
   - **M**easurable（可量化的）：可数字化的指标
   - **A**ttainable（可实现的）：基于现有技术可实现
   - **R**elevant（相关的）：直接衡量转化效果
   - **T**ime-bound（有时限的）：有明确的时间范围
2. 执行指标细化：
   - 为每个场景设计2-3个指标
   - 确保指标可操作
   - 确保指标合理

### 8.4 存储异常处理

#### 异常9：Obsidian存储失败

**症状**：
- "Obsidian无法写入"
- "路径错误，无法找到文件"

**处理流程**：
1. 检查路径是否正确：
   - 确认Obsidian仓库路径
   - 确认目标文件夹存在
   - 确认文件名格式正确
2. 降级存储：
   - 如果Obsidian无法存储，先存储到WorkBuddy
   - 生成存储待办事项，提醒人工补充
   - 记录失败原因，下次避免

#### 异常10：WorkBuddy存储失败

**症状**：

**处理流程**：

   - 确认MEMORY.md路径
   - 确认今日日志路径
   - 确认格式正确
2. 降级存储：
   - 如果WorkBuddy无法存储，先存储到Obsidian
   - 生成存储待办事项，提醒人工补充
   - 记录失败原因，下次避免

#### 异常11：IMA存储失败

**症状**：

   - 记录失败原因，下次避免

## 九、SOP总结

### 9.1 核心要点回顾

**触发判断**：
- 满足MUST触发条件立即执行
- OPTIONAL触发条件需悟空明确要求
- 触发时必须记录基本信息

**表示空间**：
- 完整观察（时间+背景+过程+转折+成果）
- 标签化拆解（动词开头、独立完整、互不重叠）
- 质量自检（覆盖性、独立性、数量）

**压缩**：
- 帕累托聚焦（特征排序、识别最优子集）
- 核心洞察提炼（一句话、≤30字）
- 象征符号选择（视觉化、关联性、简洁性）

**泛化**：
- 场景扩展（头脑风暴+筛选+细化）
- 场景分布（延伸+跨域+创新三维度）
- 可行性保证（操作步骤+验证指标）

**输出与存储**：
- 生成标准沉淀卡
- 三向存储（Obsidian+WorkBuddy+IMA）
- 质量检查（完整性、准确性）

**质量控制**：
- 三阶段质量检查
- 整体效果评估
- 复用价值评估

### 9.2 SOP黄金法则

**法则1：不跳跃，按顺序**
- 必须先表示空间，再压缩，再泛化
- 跳跃会导致信息丢失或提炼不准确

**法则2：不完美，先完成**
- 第一遍不必追求完美，重点是完成三阶段
- 后续迭代可以不断优化

**法则3：不孤立，关联思考**
- 每次知行合一都要思考：这个经验与龙心OS其他技能的关系？
- 主动建立跨域知识联系

**法则4：不忽视，及时存储**
- 三向存储必须执行
- 缺一不可，完整存储才能避免经验流失

### 9.3 持续优化机制

**SOP的迭代优化**：

```yaml
第1次迭代: 
  - 目标：建立基础SOP
  - 重点：不遗漏，不跳跃

第2次迭代: 
  - 目标：优化SOP流程
  - 重点：提升效率，减少冗余

第3次迭代: 
  - 目标：扩展SOP覆盖
  - 重点：增加异常处理场景

第N次迭代: 
  - 目标：形成完美SOP
  - 重点：形成个性化、高效化SOP
```
