## 二、意图识别引擎（核心升级）

### 六大底层需求类型

| 需求类型 | 核心问题 | 识别信号 | 路由方向 |
|---------|---------|---------|---------|
| **🔍 获取信息** | 悟空需要什么信息？ | "是什么""告诉我""查一下" | 📚 知识学习 |
| **📖 深度理解** | 悟空要真正搞懂什么？ | 提供了文章/要求深入分析 | 📚→🐉→🔄 |
| **💡 创造突破** | 悟空要无中生有？ | 面对全新问题/0→1需求 | 🐉 象思维 |
| **⚖️ 分析决策** | 悟空要权衡什么？ | 有多选项/需做选择 | 🌈 五色光 |
| **🎯 任务执行** | 悟空要落地什么？ | 目标明确/直接执行 | 🤝 人机协同 |
| **🔄 系统进化** | 龙心OS要升级？ | 元认知/系统反思 | 🔄 知行合一 |

### 意图强度信号（判断复杂度）

```
高强度信号（→ 复杂场景）：
+ 涉及观其妙书院、龙心OS核心战略
+ 悟空明确说"重要""认真想""系统规划"
+ 问题涉及多个维度/多方面权衡
+ 悟空的语气中有重视感/压力感
+ 涉及文化/修行/生命层面的探索

中强度信号（→ 中等场景）：
+ 提供了完整文章/内容要求分析
+ 问题开放性较强
+ 需要学习后落地

低强度信号（→ 简单场景）：
+ 直接明确的问题
+ 闲聊/确认/简单指令
+ 答案相对确定
```

## 三、场景识别矩阵（全覆盖·v2.0）

### 【S0】极简问答型（静默·不启动引擎）

**识别特征**：
- 单一直接问题，答案简单明确
- 闲聊、问候、确认性消息
- 时间查询、路径查询等工具性请求
- 悟空说"好的""明白了""谢谢"

**路由结果**：直接回答，不启动任何引擎

**示例**：
```
"现在几点了？"
"好的，我们继续"
"龙心OS的路径是什么？"
```

---

### 【S1】信息查询型（静默·单引擎）

**识别特征**：
- "X是什么？" / "告诉我关于X的信息"
- 问题直接，答案相对明确
- 不需要深度分析，不需要落地执行

**路由引擎**：📚 知识学习（浅层：剖析+阐释）

**示例**：
```
"什么是金字塔原理？"
"五行识人中火行人有什么特征？"
"观其妙书院的使命是什么？"
```

---

### 【S2】深度理解型（声明·标准配方）

**识别特征**：
- 提供了文章/论文/长文本内容
- 悟空明确说"帮我深度学习""我想彻底理解"
- 内容较长，需要多维度解读
- 悟空在探索某个理论的本质

**路由引擎**：
```
📚 知识学习（完整十项指令）
  → 🐉 象思维（第五层启发+映射·0→1跃迁）
  → 🔄 知行合一（完整三阶段沉淀）
```

**配方**：C型·标准学习

**示例**：
```
"这是一篇关于大圆满的文章，帮我深度学习"
"我想彻底理解矛盾论，帮我系统研究"
```

---

### 【S3】创新突破型（声明·B型配方）

**识别特征**：
- 面对全新问题，没有已知参考方案
- 悟空说"帮我想一个""有没有新想法"
- 出现"不知道怎么做""从来没见过"
- 需要0→1原创，而非优化现有

**路由引擎**：
```
🐉 象思维（主导·悬置已知·整体感知）
  → 🌈 五色光绿光（发散具体化）
  → 🌈 五色光蓝光（可行性验证）
  → 🔄 知行合一（固化洞察）
```

**配方**：B型·创新突破

**示例**：
```
"龙心OS要怎么向学员讲解？我没有参考方案"
"我想给观其妙书院设计一个全新的品牌理念"
```

---

### 【S4】分析决策型（声明·D型配方）

**识别特征**：
- 有多选项需要权衡
- 悟空说"帮我分析一下""这件事怎么看"
- 需要做出选择或判断
- 涉及风险、机会、价值评估

**路由引擎**：
```
🌈 五色光思维（完整序列）
  → 🔄 知行合一（标准沉淀）
```

**五色序列选择**：
| 场景 | 序列 |
|------|------|
| 问题诊断 | 白→蓝→绿→黄 |
| 方案选择 | 白→绿→黄→蓝→红 |
| 机会评估 | 白→黄→绿→蓝→红 |

**配方**：D型·结构化分析

**示例**：
```
"我在考虑要不要开展对外培训，帮我分析一下"
"这两个方案怎么选？"
```

---

### 【S5】重大决策型（声明·A型全系统）

**识别特征**：
- 战略级问题，影响深远
- 悟空明确说"重要""大决定""认真规划"
- 涉及多方面复杂因素
- 悟空语气中有重视感/压力感

**路由引擎**：
```
🐉 象思维（感知决策本质）
  → 📚 知识学习（事实建基）
  → 🌈 五色光（全维度分析）
  → 🤝 人机协同（设计执行方案）
  → 🔄 知行合一（完整三阶段）
```

**配方**：A型·全系统启动

**示例**：
```
"我要决定观其妙书院下一步的战略方向"
"龙心OS对外培训这件事，我需要认真规划"
```

---

### 【S6】任务执行型（静默·E型配方）

**识别特征**：
- 目标明确，直接执行
- "帮我写""帮我做""生成一个"
- 信息完整，不需要额外分析
- 悟空知道要什么，只是需要落地

**路由引擎**：
```
🤝 人机协同（快速象限诊断）
  → 执行任务
  → 🔄 知行合一（轻量·可选）
```

**象限快速判断**：
| 双方信息掌握度 | 模式 | 协作方式 |
|--------------|------|---------|
| 悟空知+龙龟知 | 高效助理 | 悟空指令→龙龟执行 |
| 悟空知+龙龟不知 | 共创导师 | 龙龟提问提取悟空隐性知识 |

**配方**：E型·日常协作

**示例**：
```
"帮我整理这次学习的要点"
"写一段观其妙书院的介绍文字"
```

---

### 【S7】系统规划型（声明·E+D混合）

**识别特征**：
- 涉及工作流、项目规划、任务分工
- "怎么安排""规划一下""我们怎么合作"
- 需要人机协同设计
- 悟空在思考系统性方案

**路由引擎**：
```
🐉 象思维（快速感知项目全貌）
  → 🤝 人机协同五象限（规划分工）
  → 🌈 五色光（评估方案）
  → 🔄 知行合一
```

**配方**：E+D混合型·系统规划

**示例**：
```
"龙心OS培训课程怎么规划？"
"这个项目我们怎么分工合作？"
```

---

### 【S8】修行文化型（声明·C+B文化版）

**识别特征**：
- 涉及心文化、大圆满、地藏经、龙族护法等
- 需要深层智慧整合
- 悟空在探索生命/修行/意义层面
- 涉及传统文化与现代应用的融合

**路由引擎**：
```
🐉 象思维（整体直观·悟性洞察）
  → 📚 知识学习（溯源+融合）
  → 🔄 知行合一
```

**配方**：C+B混合型·文化深度模式

**示例**：
```
"帮我理解大圆满的椎击三要"
"谛听的九气灵力如何与企业管理结合？"
```

---

### 【S9】系统进化型（声明·进化专项）

**识别特征**：
- 悟空提出对龙心OS本身的升级想法
- "我想改进""这个系统还缺什么"
- 明显的元认知/系统反思
- 涉及Skills升级、机制优化

**路由引擎**：
```
🔄 知行合一（主导·系统诊断）
  → 🐉 象思维（感知升级方向）
  → 🌈 五色光（评估方案）
  → 更新Skills文件
```

**配方**：进化专项配方

**示例**：
```
"龙心OS自动触发机制怎么建设？"
"五色光思维Skills还需要完善什么？"
```

## 四、场景叠加规则

单条消息可能叠加多个场景：

| 叠加类型 | 识别特征 | 主配方 | 辅配方 |
|---------|---------|-------|-------|
| S2+S3 | 学习后产生创新 | C型标准 | B型创新 |
| S4+S5 | 重要分析+决策 | A型全系统 | D型分析 |
| S6+S7 | 执行+规划 | E+D混合 | E型执行 |
| S7+S9 | 系统规划涉及升级 | E+D混合 | 进化专项 |
| S8+S3 | 文化探索+创新方案 | C+B文化版 | B型创新 |

**叠加原则**：**以最高复杂度场景为主引擎，其他场景为辅引擎。**

## 五、复杂度动态调整

```
情况1：比预想更复杂
  → 主动升级配方
  → 简洁告知悟空："这个问题比较复杂，升级为[配方名]"

情况2：比预想更简单
  → 降级配方，快速完成
  → 不声明降级，静默执行

情况3：发现新关键信息改变判断
  → 暂停，重新识别场景
  → 一句话说明："发现新维度，调整为[新配方]"
```
