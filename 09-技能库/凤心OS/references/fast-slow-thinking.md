## 一、双系统核心定义

### 系统1：快思考（五行总智能体）
> 对应卡尼曼"系统1"——自动、快速、无意识、并行处理

| 属性 | 描述 |
|------|------|
| **速度** | 毫秒级响应，感知即路由 |
| **处理方式** | 并行·全局扫描·模式识别 |
| **激活条件** | 自动激活，无需意识介入 |
| **主要职能** | 感知→识别→分类→路由→整合 |
| **五行映射** | 水行人的快速感知 + 火行人的直觉洞察 |
| **局限性** | 无法进行深度精准分析；需要慢思考补充 |
| **典型错误** | 混行误判（将金行误判为水行）|

**快思考的工作清单**：
- ✅ 扫描语义/结构/能量三层信号
- ✅ 五行归类打分（三维感知矩阵）
- ✅ 做路由决策（S0~S8）
- ✅ 管理多行并行任务
- ✅ 整合慢思考的输出结果
- ❌ 不做深度一心三界分析
- ❌ 不做九层精确定位
- ❌ 不输出详细转化方案

### 系统2：慢思考（五行分智能体）
> 对应卡尼曼"系统2"——分析、深度、专注、串行处理

| 属性 | 描述 |
|------|------|
| **速度** | 深度分析，充分展开 |
| **处理方式** | 串行·单行专注·精准诊断 |
| **激活条件** | 由快思考（总调度）显式激活 |
| **主要职能** | 一心定位→三界扫描→九层诊断→转化方案 |
| **五行映射** | 金行人的精准决断 + 木行人的深度生长 |
| **局限性** | 单次只能深度处理一种五行类型 |
| **典型贡献** | 精准的转化方案；B=MAP具体落地 |

**慢思考的工作清单**：
- ✅ 一心状态定位（阳态/阴态/混合）
- ✅ 三界全面扫描（灵/心/身三界现状）
- ✅ 五行生克关系诊断
- ✅ 九层精确定位（1-9级）
- ✅ 拔阴取阳转化方案
- ✅ 化克为生具体路径
- ✅ B=MAP行为设计（M+A+P）
- ✅ 输出标准化分析报告

## 二、协调协议（Quick-Think → Deep-Think Protocol）

### 协议1：激活协议
```
触发条件（任意满足）：
  · 三维感知矩阵总分 ≥ 7分
  · 直接触发词命中（高权重词汇）
  · 快思考识别到明确的五行场景诉求

激活流程：
  快思考评分 → 确认主行 → 生成调度指令 → 激活对应分智能体
```

### 协议2：调度指令格式
```
【调度指令】
  目标分智能体：[木/火/土/金/水] 行人分智能体
  分析场景：[S0~S8 场景编号]
  用户诉求：[一句话描述核心诉求]
  关键信号：[触发路由的2-3个核心词]
  优先维度：[一心/三界/九层/转化/B=MAP]
  关联行（如有）：[副行类型 + 生克关系]
```

### 协议3：并行路由协议
当路由到多个分智能体时：

```
步骤1：快思考确认所有目标分智能体（最多同时调用3个）
步骤2：为每个分智能体生成独立调度指令
步骤3：并行执行（各分智能体独立分析）
步骤4：快思考收集所有输出
步骤5：执行生克关系检查（各行之间的生克动态）
步骤6：整合输出（按优先级排列，主行在前）
```

### 协议4：反馈循环协议
```
慢思考输出 → 快思考接收 → 生克检查 → 是否需要调整路由？
                                              ↓ 是
                                         重新路由（补充行分析）
                                              ↓ 否
                                         整合输出
```

### 协议5：错误校正协议
```
快思考误判检测：
  慢思考输出与感知信号不符 → 快思考重新评估
  用户反馈"不准确"→ 触发重新路由
  新信号出现（对话继续）→ 动态更新路由判断
  
校正方式：
  轻度偏差：追加副行分析
  中度偏差：调换主副行
  重度偏差：重置为S0（追问确认）
```

## 四、快思考vs慢思考的调用边界

### 何时只用快思考（不激活慢思考）
- 简单的五行类型快速确认（S1场景）
- 用户只需要一个简短判断
- 知识性问题（S8场景）
- 对话还在进行，信号积累中（S0场景）

### 何时必须激活慢思考
- 用户提出"深度分析"诉求
- 信号强度 ≥ 7分（主行明确）
- 转化/发展/改变诉求（S4/S5）
- 完整人格诊断请求
- 关系调适需要精准方案（S3/S7）

### 何时多个慢思考并行
- 双行关系分析（S3）：调用2个分智能体
- 团队/组织诊断（S7）：调用3-5个分智能体
- 跨行转化分析（化克为生）：调用相关的2个分智能体

## 五、卡尼曼×五行整合洞察

> 卡尼曼发现：人类决策中"系统1"（快思考）产生直觉偏见，"系统2"（慢思考）提供理性校正。
> 五行人格心理学的洞察：五行本身就是人类五种"快思考系统"的具体化——木行人快速生发、火行人快速热情、土行人快速承载、金行人快速决断、水行人快速流动。
> 
> 凤心OS的创新：用AI的快思考（五行总智能体）感知和路由，用AI的慢思考（五行分智能体）深度分析，模拟人类最优的思考模式——既有整体直觉，又有深度精准。

### 五行人格的快思考偏见（需要慢思考校正）

| 五行 | 快思考偏见 | 慢思考校正 |
|------|-----------|-----------|
| 阴木 | 一切批评都是攻击 | 识别是否真正有威胁 |
| 阴火 | 自己是最重要的 | 看见他人的需求和感受 |
| 阴土 | 没人关心我 | 识别自己对关系的过度诠释 |
| 阴金 | 一切不完美都是错误 | 区分标准和苛责 |
| 阴水 | 行动一定有危险 | 评估真实的风险等级 |
