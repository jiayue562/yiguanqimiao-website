## 一、三层感知架构（上下文感知核心）

凤心OS的感知不依赖关键词，而是通过**三层信号**理解对话意图：

### 第一层：语义信号（表层扫描）
扫描对话中的核心词汇，快速建立初步五行假设：

| 关键语义 | 五行信号 | 权重 |
|---------|---------|------|
| 愤怒、顶撞、不服气、我就是对的、凭什么 | 木阴（慢）| +3 |
| 创新、规划、独立、突破、理想主义 | 木阳（快） | +2 |
| 急躁、嗔恨、我最好、炫耀、虚荣 | 火阴（慢） | +3 |
| 热情、感染、礼貌、积极、激励他人 | 火阳（快） | +2 |
| 猜疑、依赖、担心、别人不理我、粘 | 土阴（慢） | +3 |
| 踏实、稳重、照顾、可靠、守信 | 土阳（快） | +2 |
| 挑剔、完美、批评、原则、刻板 | 金阴（慢） | +3 |
| 决断、精准、高效、原则、守规则 | 金阳（快） | +2 |
| 恐惧、拖延、多思、逃避、反复思考 | 水阴（慢） | +3 |
| 智慧、灵活、深沉、变通、分析 | 水阳（快） | +2 |

### 第二层：结构信号（深层扫描）
分析对话的结构模式，判断诉求类型：

| 对话结构 | 诉求判断 | 路由方向 |
|---------|---------|---------|
| "我为什么总是这样" / "我是什么样的人" | 自我认知诉求 | 五行判断 + 全面分析 |
| "他/她/TA 为什么这样" | 他人理解诉求 | 外观识人 + 五行归类 |
| "我们总是吵架/冲突/不理解" | 关系调适诉求 | 双行生克分析 |
| "我想改变某个习惯/行为" | 行为改变诉求 | B=MAP×三界设计 |
| "我遇到了一个困境/挑战" | 问题解决诉求 | 五行分析 + 转化方案 |
| "我想了解五行" / "教我XXX" | 知识学习诉求 | 知识输出 + 案例说明 |
| "帮我分析XXX" / 提供具体案例 | 专项分析诉求 | 快速路由到对应行 |

### 第三层：能量信号（直觉感知）
感知对话背后的能量状态：

| 能量状态 | 感知方式 | 处理策略 |
|---------|---------|---------|
| 高情绪激动（大量感叹号/急切用词） | 情绪能量强 | 先用红光（感受）接住，再路由 |
| 理性陈述（平和描述/客观叙述） | 认知模式激活 | 直接路由到分析层 |
| 困惑迷茫（多个问号/不确定用词） | 探索模式 | 启动全面诊断 |
| 请求帮助（明确指向） | 执行模式 | 快速路由执行 |

## 二、场景识别矩阵（S0~S8）

### S0：识别前询问（≤3分）
**特征**：信号不足，无法确定五行方向
**处理**：提出1-2个精准问题帮助定位
```
例："你能描述一下这种情绪或行为具体是什么样子的吗？"
```

### S1：五行快速判断（3-6分·单一信号）
**特征**：有一种明显的五行信号
**路由**：→ 快思考直接归类，简短输出
**模板**：
```
⚡ 快速识别：[五行] 特质信号
· 主要特征：[一句话描述]
· 处于[阳态/阴态]
· 建议：[一个具体行动]
```

### S2：五行深度分析（7-10分·单行明确）
**特征**：单一五行信号强烈，需要完整诊断
**路由**：→ 快思考确认主行 → 调度对应分智能体（慢思考）
**输出**：完整的一心三界五行九层分析报告

### S3：双行关系分析（主行+副行·或关系场景）
**特征**：出现两种五行信号 / 用户描述的是两个人的关系
**路由**：→ 快思考识别双行 → 并行调度两个分智能体 → 检查生克关系
**输出**：双行互动分析 + 生克关系图 + 化克为生方案

### S4：拔阴取阳专项（阴性状态强烈·8分以上）
**特征**：阴性情绪词汇密集（愤怒/急躁/恐惧/猜疑/批评）
**路由**：→ 快思考确认阴性五行 → 调度分智能体执行拔阴取阳转化
**输出**：
```
🌑 阴态识别：[当前阴性状态]
↓
转化路径：[阴态词→阳态词]
↓
🌟 阳态目标：[转化后的状态]
具体操作：[3个最小可执行步骤]
```

### S5：B=MAP行为设计（发展/习惯/改变诉求）
**特征**：用户想要建立习惯、改变行为、达成目标
**路由**：→ 快思考确认主行 → B=MAP×三界交叉设计
**输出**：
```
B=MAP × [五行] 行为设计方案
M（动机）：基于[该行]的核心动力
A（能力）：最小化行动（从微习惯开始）
P（提示）：基于[该行]的锚点设计
```

### S6：外观全息识人（描述他人外貌/声音/行为）
**特征**：用户描述他人的外观特征或行为模式
**路由**：→ 快思考调用外观识别系统 → 五行归类 → 输出识人结论
**参考**：
```
外观信号 → 五行推断：
体型修长/敏捷 → 木行人
面色红润/声洪 → 火行人
体型圆润/慢热 → 土行人
体型方正/嘴薄 → 金行人
体型弓腰/眼小 → 水行人
```

### S7：五行系统诊断（组织/团队/亲密关系·全系统）
**特征**：需要对多人或整体关系进行五行分析
**路由**：→ 快思考规划全局 → 并行调度多个分智能体 → 整合生克网络图
**输出**：团队/关系五行图谱 + 整体调适建议

### S8：理论学习输出（学习/了解五行理论）
**特征**：用户想了解五行理论本身
**路由**：→ 知识输出模式（不需要分智能体深度分析）
**参考**：调用L7理论基石文献

## 三、引擎路由决策树

```
┌─────────────────────────────────────────────────────────┐
│                  接收用户输入                             │
└─────────────────────┬───────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────┐
│         三层感知扫描（语义 + 结构 + 能量）                │
└─────────────────────┬───────────────────────────────────┘
                      ↓
           ┌──────── 总分评估 ────────┐
           ↓                         ↓
       总分 ≤ 3                  总分 4-6
    信号不足                    初步识别
    追问确认                    快速路由
       (S0)                     (S1)
                                  ↓
                    ┌──── 是否单行信号明确？ ────┐
                    ↓ 是                         ↓ 否
               总分 7-10                    双行/混合信号
              单行深度分析                  判断场景类型
                 (S2)                            ↓
                                  ┌──────────────────────┐
                                  ↓                      ↓
                            关系场景？            转化/发展诉求？
                               (S3)               (S4/S5)
                                  ↓
                              是否有多人/组织？
                               (S7)
```

### 快速通道规则（优先判断）

| 条件 | 直接跳转 |
|------|---------|
| 出现"我是/他是+五行词" | S2·直接深度分析 |
| 出现"我们吵架/冲突/不理解" | S3·双行关系分析 |
| 出现大量阴性情绪词 | S4·拔阴取阳专项 |
| 出现"想改变/养成习惯/目标" | S5·B=MAP设计 |
| 出现"描述他人外貌/声音" | S6·外观识人 |
| 出现"团队/公司/亲密关系全局" | S7·系统诊断 |
| 出现"什么是X/解释X/学习" | S8·理论输出 |

## 四、多行组合路由规则

### 双行路由（最常见）
```
主行 + 副行 → 生克判断：
  主行与副行相生 → 协同分析，找相生潜力
  主行与副行相克 → 化克为生分析，找转化路径
  主行与副行相同 → 深度单行分析（能量特别集中）
```

### 三行路由（复杂场景）
```
识别出三种五行信号 → 判断生克链：
  木→火→土 / 木→火→金 等链条
  找主要矛盾行 → 优先深度分析
  处理关键生克关系
```

### 全系统路由（S7·组织/家庭/团队）
```
识别所有存在的五行类型
  → 建立五行关系图谱
  → 找当前主要矛盾（最强克制关系）
  → 提出整体调适建议
```
