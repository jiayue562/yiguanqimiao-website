## 四、复杂度评估

| 级别 | 分数 | 特征 | 引擎数 |
|------|------|------|--------|
| 简单 | 1-3分 | 单轮问答，无需复杂推理 | 无/单引擎 |
| 中等 | 4-7分 | 多轮交互或组合分析 | 单/双引擎 |
| 复杂 | 8-10分 | 全系统协同，深度思考 | 多引擎/全系统 |

## 六、动态调整机制

| 调整 | 条件 | 表现 | 行动 |
|------|------|------|------|
| 降级 | 比预想简单 | 反应平淡/问题已解决 | 降级到更轻量引擎 |
| 升级 | 比预想复杂 | 发现新问题/新要求 | 升级到更强引擎组合 |
| 转向 | 发现新关键信息 | 上下文重大变化 | 调整引擎组合 |
| 终止 | 高风险/要求停止 | 敏感内容/风险过高 | 暂停并询问 |

## 七、决策层完整执行流程

```python
class 决策层:
    def 决策(self, 上下文):
        # Step 1: 场景感知
        场景 = self.场景感知器.识别(上下文)
        
        # Step 2: 意图识别
        意图 = self.意图识别器.识别(上下文)
        
        # Step 3: 复杂度评估
        复杂度 = self.复杂度评估器.评估(上下文)
        
        # Step 4: 引擎路由
        引擎组合 = self.引擎路由器.路由(场景, 意图, 复杂度)
        
        # Step 5: 灵魂注入
        引擎组合 = self.灵魂层.乘(引擎组合)
        
        # Step 6: 返回决策结果
        return {
            "场景": 场景,
            "意图": 意图,
            "复杂度": 复杂度,
            "引擎组合": 引擎组合
        }
```
