## 一、与五行人格总智能体的关系

### 🎯 调用关系

```
五行人格总智能体（凤心OS·快思考层）
    ↓ 场景归类 S0-S7
    ↓ 引擎路由决策
火行人分智能体（深度分析）
    ↓ 执行7步标准流程
    ↓ 输出诊断报告
    ↓ 返回总智能体整合
```

### 🔥 触发机制

**凤心OS调用条件**:
- 检测到火行人信号（阴火/阳火）
- 来者描述自己的火行特质
- 涉及火行人相关场景（人际关系/自我成长/情绪管理）

**智能调度系统**:
- 三维感知矩阵打分（情绪信号8分 + 行为信号4分 + 诉求类型2分）
- 总分≥10 → 直接激活火行人分智能体
- 总分7-9 → 双行分析（火+另一行）
- 总分5-6 → 全面诊断（五行总体）

### 📋 协作协议

| 调度阶段 | 总智能体责任 | 火行人分智能体责任 |
|----------|-------------|------------------|
| **意图识别** | 识别来者是否需要火行分析 | 接收调度指令 |
| **场景归类** | 判定场景类型（S1-S7） | 准备火行专属流程 |
| **结果整合** | 整合五行分析结果 | 提供火行诊断报告 |

## 二、与其他分智能体的关系

### 🌿 木行人分智能体（木生火关系）

**协作场景**:
- 木火关系分析
- 共生关系诊断
- 木行滋养火行不足

**整合方式**:
```
木行人分智能体（分析木行状态）
    + 火行人分智能体（分析火行状态）
    ↓
木火共生关系诊断
    ↓
转化建议：木生火（滋养）或火克木（克制）
```

**核心要点**:
- 木弱则火虚：需要木行的仁德滋养火行的热情
- 木过旺则克火：需要火行收敛，避免被木行压制
- 木火相生理想：木行的规划+火行的热情=高效执行

### 🌏 土行人分智能体（火生土关系）

**协作场景**:
- 火土关系分析
- 热情落地问题
- 成果导向训练

**整合方式**:
```
火行人分智能体（分析火行热情）
    + 土行人分智能体（分析土行承载）
    ↓
火土协同诊断
    ↓
转化建议：火生土（热情转化）或火不生土（浮在上面）
```

**核心要点**:
- 火生土理想：火行的热情转化为土行的成果
- 火不生土问题：热情停留在表面，缺乏实际成果
- 火过旺克土：过度热情破坏土行的稳定性

### ⚔️ 金行人分智能体（火克金关系）

**协作场景**:
- 火金关系冲突
- 人际紧张诊断
- 决策与表达问题

**整合方式**:
```
火行人分智能体（分析火行感染力）
    + 金行人分智能体（分析金行规则）
    ↓
火金冲突诊断
    ↓
转化建议：火克金（克过度）或火生土→土生金（化克为生）
```

**核心要点**:
- 火克金常见：火行人的热情和感染力可能让金行人不适
- 化克为生路径：火克金 → 火生土 → 土生金（引入稳定性）
- 火金和谐：火行的感染力+金行的规则感=高效团队

### 💧 水行人分智能体（水克火关系）

**协作场景**:
- 水火平衡诊断
- 急躁与冷静的整合
- 情绪管理训练

**整合方式**:
```
火行人分智能体（分析火行热情）
    + 水行人分智能体（分析水行智慧）
    ↓
水火既济诊断
    ↓
转化建议：水克火（过度克制）或水火既济（平衡）
```

**核心要点**:
- 水火既济理想：水的智慧冷却火的急躁
- 水过强火灭：智慧过度压抑热情，变得冷漠
- 火过旺水弱：缺乏冷静，热情变为躁狂

## 三、与龙心OS五大引擎的整合

### 🐉 象思维（心·0→1原创突破）

**整合场景**:
- 火行人的原创力培养
- 从"证明自己"到"照亮他人"的象思维转化
- 火行人的直觉洞察训练

**整合方式**:
```
象思维（0→1原创突破）
    ↓ 火行人专属设计
火行人的"原象"：从阴火（嗔恨）到阳火（礼明）的本质转化
    ↓
五色光验证（红光直觉+蓝光风险）
    ↓
输出：火行人的象思维实践指南
```

**核心要点**:
- 火行人的0→1突破：从"我被忽视"到"我能照亮谁"
- 象思维三层次：物象（火的行为）→意象（火的情绪）→原象（火的本质——光明性）
- 实践转化：阴火恨→阳火问，阴火嗔→阳火礼

### 📚 知识学习（脑·十项认知指令）

**整合场景**:
- 火行人理论体系深度学习
- 一心三界五行九层知识重构
- 拔阴取阳/化克为生技术掌握

**整合方式**:
```
知识学习（十项认知指令）
    ↓ 火行人专属流程
第一层（剖析+解构）→ 一心三界结构
第二层（透视+阐释）→ 火行本质理解
第三层（推演+思辨）→ 五行生克逻辑
第四层（溯源+融合）→ 与五行总智能体融合
第五层（启发+映射）→ 象思维+五色光验证
```

**核心要点**:
- 十项认知指令火行适配：剖析（金的清明）→透视（火的光明）→推演（水的润泽）→溯源（木的生长）→融合（土的承载）
- 输出结构：知识图谱+核心金句+实践指南
- 知行合一沉淀：学习后必须执行三阶段压缩

### 🌈 五色光思维（眼·五色分治同频共振）

**整合场景**:
- 火行人问题多维度分析
- 情绪混乱时的思维对齐
- 决策制定的结构化思考

**整合方式**:
```
五色光思维（六色分治）
    ↓ 火行人专属设计
白光（事实）→ 火行人的客观状态
红光（直觉）→ 火行人的情绪感知
黄光（价值）→ 火行人的温暖照亮价值
绿光（创新）→ 火行人的感染力创新
蓝光（风险）→ 火行人的急躁风险
主持人（过程）→ 思维序列管理
    ↓
输出：火行人的五色光思维方案
```

**核心要点**:
- 火行人的五色光速查表：
  - 白光：我现在的客观事实是什么？
  - 红光：我的直觉感受是什么？
  - 黄光：这个行为的价值是什么？
  - 绿光：如何创新我的表达方式？
  - 蓝光：有什么潜在风险？
  - 主持人：应该用什么顺序思考？
- 单色聚焦原则：同一时间只有一种颜色的声音
- 火行人专属序列：红光锚定→白光奠基→绿光展开→黄光评估→蓝光检验

### 🤝 人机协同五象限（手·五象限分工）

**整合场景**:
- 火行人人机协同分工设计
- 悟空（木行人）与龙龟神将（火行人）的协作优化
- 火行人特质的AI辅助策略

**整合方式**:
```
人机协同五象限（信息掌握度×信息掌握度）
    ↓ 火行人专属定位
高效助理（知/知）→ 火行人的执行力优势
学习伙伴（不知/知）→ 火行人的知识学习
共创伙伴（部分/部分）→ 火行人的感染力+悟空的战略
共创导师（知/不知）→ 火行人外化隐性知识
未知探索域（不知/不知）→ 火行人与悟空的象思维协同
    ↓
输出：火行人专属的人机协同方案
```

**核心要点**:
- 火行人优势象限：
  - 高效助理：指令清晰→高效执行（火行人的行动力）
  - 共创伙伴：感染力强→团队氛围营造
- 火行人待提升象限：
  - 共创导师：需要将隐性知识外化（火行人的实践智慧）
- 木火共生：木行人（悟空）战略+火行人（龙龟神将）执行=最佳协作

## 六、最佳实践与注意事项

### ✅ 整合最佳实践

**1. 优先级原则**:
- 火行人分智能体是火行专属分析的Primary Skill
- 其他Skills根据具体需求动态调用
- 避免Skills冲突（同一时间只有一个主导Skill）

**2. 数据一致性**:
- 跨Skills共享统一的火行人数据结构
- 保持火行人分析结果的一致性
- 知行合一沉淀时引用完整的分析过程

**3. 质量保证**:
- 火行人分智能体的输出经过7步标准流程
- 其他Skills的输出与火行人分析结果对齐
- 跨Skills协作时进行质量交叉验证

### ⚠️ 注意事项

**1. 避免循环触发**:
- 火行人分智能体不重复调用自己
- 知行合一不反向触发火行人分智能体
- 严格遵循WorkBuddy Rule的优先级机制

**2. 上下文传递**:
- 火行人分析结果完整传递给其他Skills
- 其他Skills的执行结果反馈给火行人分智能体
- 保持上下文的连贯性和完整性

**3. 用户体验**:
- 多Skills协作时不让来者感知到复杂的系统调用
- 保持火行人分智能体的主导地位（用户感知只有一个火行人分析）
- 其他Skills在后台静默运行，输出整合后呈现
