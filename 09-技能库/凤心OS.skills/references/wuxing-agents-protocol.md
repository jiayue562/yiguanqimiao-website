## 🔗 调用机制

### 快思考层 → 慢思考层

当快思考层确定需要调用某个五行分智能体时：

**1. 激活对应Skills**
- 通过独立Skills包路径自动调用
- 不依赖全局命名，直接通过路径激活
- 确保调用的准确性和独立性

**2. 传递场景信息**
- 情绪信号（阴/阳 + 分值）
- 行为信号（分值）
- 诉求类型（S0~S8场景）
- 三维感知总分

**3. 接收分析结果**
- 标准七步流程输出
- 结构化报告格式
- 转化方案和行为设计

**4. 整合输出**
- 快思考层整合多个分智能体的分析结果
- 保持五行分析的专业性（不混行）
- 统一输出格式

## 📊 协同场景

### 场景1：单行分析

**触发条件**：
- 总分 ≥ 10
- 单一五行信号明显
- 单行精准诊断需求

**路由策略**：
- 激活1个对应分智能体
- 执行标准七步流程
- 输出单行完整报告

**示例**：
```
用户："我最近很烦躁，总跟同事吵架，他们就是不理解我"
分析：
  - 情绪信号：烦躁（阴火·8分）、吵架（阴火·8分）
  - 总分：16分
  - 路由：激活火行人分智能体
```

### 场景2：双行分析

**触发条件**：
- 总分 7-9
- 两种五行信号同时存在
- 主行+副行并行分析

**路由策略**：
- 激活2个分智能体（并行）
- 主行优先，副行辅助
- 输出双行关系诊断

**示例**：
```
用户："我既想创新，又担心风险"
分析：
  - 情绪信号：创新（阳木·6分）、担心（阴水·6分）
  - 总分：12分
  - 路由：激活木行人分智能体 + 水行人分智能体（并行）
```

### 场景3：全面诊断

**触发条件**：
- 总分 5-6
- 五行信号较弱但多样
- 五行总体分析需求

**路由策略**：
- 全面诊断五行状态
- 识别主行和副行
- 综合平衡建议

**示例**：
```
用户："我最近状态不太稳定"
分析：
  - 信号：状态不稳定（多维度弱信号）
  - 总分：5-6分
  - 路由：全面诊断，识别主行（可能是水或土）
```

### 场景4：多行并行

**触发条件**：
- S7全系统场景
- 复杂问题/系统规划
- 需要多维度深度洞察

**路由策略**：
- 激活多个分智能体（并行）
- 多维度综合分析
- 系统级输出

**示例**：
```
用户："帮我规划职业生涯"
分析：
  - 场景：S7（全系统）
  - 路由：激活木行人（规划）+ 土行人（稳定）+ 金行人（决断）
```

## ⚖️ 协调原则

### 1. 专业性原则

**快思考层**：
- 只做识别路由，不做深度分析
- 场景归类优先
- 三维感知打分准确

**慢思考层**：
- 分智能体独立分析
- 保持五行专业性
- 不混行，不跨界

### 2. 资源优化原则

- 简单场景（S0）不启动引擎
- 快速判断（S1）直接输出
- 复杂场景（S7）多行并行
- 避免过度调用，节省资源

### 3. 精准路由原则

- 根据信号强度精准选择
- 总分判定明确（≥10、7-9、5-6、≤4）
- 避免模糊路由
- 确保调用准确性

### 4. 结果整合原则

- 快思考层统一整合
- 保持五行分析专业性
- 结构化输出格式
- 识别跨行模式
