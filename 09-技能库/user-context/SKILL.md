# User Context Skill - 用户画像管理

## 技能概述
用户画像管理技能，跨会话维护结构化用户profile，记住用户的五行类型、历史状态、偏好设置等信息。

## 核心功能

### 1. 用户画像存储
- **五行类型记录**：记录用户的木/火/土/金/水类型
- **历史状态追踪**：记录用户在不同场景下的状态变化
- **偏好设置管理**：记录用户的沟通偏好、工作习惯
- **成长轨迹记录**：记录用户的层级变化和成长轨迹

### 2. 会话上下文管理
- **短期记忆**：当前会话的上下文保持
- **用户状态感知**：实时感知用户当前的情绪/能量状态
- **上下文恢复**：新会话开始时恢复关键历史信息

### 3. 画像更新机制
- **自动更新**：根据对话内容自动更新用户画像
- **显式更新**：用户明确提供信息时更新
- **增量更新**：只更新变化的部分，保留历史

## 数据结构

```yaml
用户画像:
  基础信息:
    - 用户ID
    - 用户名
    - 五行类型
    - 创建时间
    - 最后活跃时间
  
  五行特征:
    - 主五行
    - 阴阳状态
    - 九层级别
    - 健康区间
  
  状态历史:
    - 时间线
    - 状态变化
    - 触发事件
  
  偏好设置:
    - 沟通风格
    - 工作时间
    - 偏好工具
    - 学习方式
```

## 触发关键词
- 用户画像、用户信息、我的五行
- 记住我、记住偏好
- 状态追踪、历史记录

## 使用场景

### 场景1：用户首次自我介绍
```
用户：我是木行人，喜欢简洁直接的沟通
→ 自动创建用户画像，记录五行类型和偏好
```

### 场景2：跨会话状态恢复
```
新会话开始：读取用户画像，恢复上次的五行状态和未完成任务
```

### 场景3：状态变化追踪
```
用户今天能量很低 → 更新画像中的当前状态
```
