## 📌 一句话定义

**五色光思维**是由爱德华·德·波诺提出的平行思维工具，通过代表六种不同思考视角的"色光帽"，帮助个人或团队系统性地拓展思维边界，避免陷入片面或混乱的讨论。

---

## 🎯 核心定义

### 1.1 六种思维视角

| 色光 | 视角 | 核心问题 | 关注点 |
|------|------|---------|--------|
| **⚪ 白光** | 客观 | "我们知道什么？信息是否完整？" | 事实、数据、信息 |
| **🔴 红光** | 主观 | "我对此感觉如何？" | 情感、直觉、预感 |
| **🟡 黄光** | 积极 | "这个想法有什么好处？为什么它会行得通？" | 优点、价值、利益 |
| **🔵 蓝光** | 谨慎 | "这可能出错吗？问题在哪里？" | 风险、困难、问题 |
| **🟢 绿光** | 创新 | "我们还有其他办法吗？能否用不同方式做？" | 新想法、替代方案 |
| **🎯 主持人** | 控制 | "我们的目标是什么？下一步该思考什么？" | 过程控制、组织 |

### 1.2 核心价值

> "平行思维——所有人同时从同一角度看问题，避免对抗性争论"

### 2.2 各色光详解

#### ⚪ 白光（客观视角）
- **关注**：事实、数据、信息
- **问题**："我们知道什么？信息是否完整？"
- **特征**：中性、客观、不带感情色彩

#### 🔴 红光（主观视角）
- **关注**：情感、直觉、预感
- **问题**："我对此感觉如何？"
- **特征**：无需解释理由，自由表达直觉

#### 🟡 黄光（积极视角）
- **关注**：优点、价值、利益
- **问题**："这个想法有什么好处？为什么它会行得通？"
- **特征**：乐观评估，寻找正面价值

#### 🔵 蓝光（谨慎视角）
- **关注**：风险、困难、问题
- **问题**："这可能出错吗？问题在哪里？"
- **特征**：逻辑批判，识别潜在问题

#### 🟢 绿光（创新视角）
- **关注**：新想法、替代方案
- **问题**："我们还有其他办法吗？能否用不同方式做？"
- **特征**：创意发散，打破常规

#### 🎯 主持人（控制视角）
- **关注**：过程控制、组织
- **问题**："我们的目标是什么？下一步该思考什么？"
- **特征**：引导讨论，总结结论

### 2.3 操作步骤

```
┌─────────────────────────────────────────────────────────────┐
│                  五色光思维六步法                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1️⃣ 设定议题                                           │
│  ─────────────────────────────────────                      │
│  明确需要讨论或决策的问题                                    │
│                                                             │
│  Step 2️⃣ 选择思维顺序                                        │
│  ─────────────────────────────────────                      │
│  可以按特定顺序依次使用各顶色光子：                          │
│  主持人(定调) → 白光(摆事实) → 黄光(看优势)                 │
│  → 蓝光(找风险) → 绿光(想创新) → 红光(做决断)                │
│                                                             │
│  Step 3️⃣ 角色扮演讨论                                        │
│  ─────────────────────────────────────                      │
│  所有参与者在同一时刻只戴一顶色光帽                            │
│  从该色光子的视角进行思考和发言                               │
│                                                             │
│  Step 4️⃣ 依次切换                                          │
│  ─────────────────────────────────────                      │
│  按顺序切换色光帽，每轮只讨论一个视角                          │
│                                                             │
│  Step 5️⃣ 主持人总结                                          │
│  ─────────────────────────────────────                      │
│  由主持人负责引导讨论，并在最后进行总结                         │
│                                                             │
│  Step 6️⃣ 形成结论或行动计划                                   │
│  ─────────────────────────────────────                      │
│  基于各视角的讨论，形成综合结论                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 应用案例

**案例：团队讨论是否采用新的技术方案**

| 阶段 | 色光 | 讨论内容 |
|------|------|---------|
| 1 | 🎯 主持人 | 宣布讨论目标：评估是否采用新技术方案 |
| 2 | ⚪ 白光 | 列出技术参数、成本、行业采用率等**事实** |
| 3 | 🟡 黄光 | 畅想该技术可能带来的**效率提升和竞争优势** |
| 4 | 🔵 蓝光 | 尖锐指出技术不成熟的**风险、转换成本** |
| 5 | 🟢 绿光 | 思考如何**规避风险，或是否有改良方案** |
| 6 | 🔴 红光 | 每位成员凭**直觉投票** |
| 7 | 🎯 主持人 | 基于事实、优势、风险和创新想法，**做出决策** |

### 2.5 注意事项

| 要点 | 说明 |
|------|------|
| **平行思维** | 所有人同时从同一角度看问题 |
| **单色聚焦** | 同一时刻只戴一顶帽子 |
| **严禁批判** | 在创意阶段不批评，只创新 |
| **主持人关键** | 主持人确保讨论有序进行 |
