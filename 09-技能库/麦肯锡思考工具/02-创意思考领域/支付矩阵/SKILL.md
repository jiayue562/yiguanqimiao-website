## 📌 一句话定义

**支付矩阵**（收益矩阵/影响-可行性矩阵）是一种以两个关键评估指标为轴，将多个创意或方案放在矩阵中进行比较和排序，从而选出实现难度最低、效果最佳的创意的工具。

---

## 🎯 核心定义

### 1.1 核心结构

```
                    效果（影响力）
                       ↑
                       │
         高效果        │        高效果
         低可行性      │        高可行性
      【重大项目区】   │       【速赢区】
                       │
     ──────────────────┼─────────────────→ 可行性
                       │
         低效果        │        低效果
         低可行性      │        高可行性
       【无望区】     │       【填充区】
                       │
                       │
```

### 1.2 四个象限

| 象限 | 效果 | 可行性 | 策略 |
|------|------|--------|------|
| **速赢区** | 高 | 高 | 优先实施 |
| **重大项目区** | 高 | 低 | 需评估投入，长期推进 |
| **填充区** | 低 | 高 | 可顺手做，不占主资源 |
| **无望区** | 低 | 低 | 果断放弃 |

---

## 📊 详细解析

### 2.1 操作步骤

| 步骤 | 操作 | 说明 |
|------|------|------|
| **1** | 确定指标 | 选择两个最关键的评估维度 |
| **2** | 绘制矩阵 | 画二维坐标轴，横轴/纵轴各代表一个指标 |
| **3** | 定位创意 | 将每个创意放置在矩阵相应位置 |
| **4** | 分区决策 | 根据所在象限采取对应策略 |

### 2.2 常用指标组合

| 组合 | 适用场景 |
|------|---------|
| 效果 + 可行性 | 通用方案评估 |
| 短期收益 + 长期价值 | 战略规划 |
| 用户价值 + 技术可行性 | 产品开发 |
| 成本 + 收益 | 投资决策 |

### 2.3 应用案例

**案例：流程改进想法优先级排序**

| 想法 | 效果 | 可行性 | 象限 | 决策 |
|------|------|--------|------|------|
| A | 大幅提升效率 | 实施简单 | 速赢区 | 立即执行 |
| B | 革命性改变 | 需巨额投资 | 重大项目区 | 高层决策 |
| C | 小优化 | 易做 | 填充区 | 空闲时做 |
| D | 收益小 | 复杂 | 无望区 | 放弃 |
