### 1.2 调用协议

**协议名称**：`LongXin-to-LongZhao-Protocol-v1`

**触发条件**：
```yaml
自动路由条件:
  - 场景归类为 S7（系统规划）
  - 场景归类为 S9（系统升级）
  - 用户明确提及项目/工作流/SOP
  - 检测到项目相关关键词

关键词触发:
  高优先级:
    - "企业文化项目"
    - "五行人格项目"
    - "启动工作流"
    - "执行SOP"
  
  中优先级:
    - "项目"
    - "工作流"
    - "流程"
    - "标准化"
```

**输入参数**：
```typescript
interface LongXinToLongZhaoInput {
  // 场景信息
  scene: {
    category: "S7" | "S9" | string;
    complexity: "简单" | "中等" | "复杂";
    urgency: "紧急" | "普通" | "可延期";
  };
  
  // 用户意图
  intent: {
    type: "启动项目" | "执行步骤" | "查看状态" | "调整计划";
    projectType?: string;
    projectPhase?: string;
    requirements: string;
  };
  
  // 上下文
  context: {
    history: Message[];
    userPreferences: UserPreferences;
    systemState: SystemState;
  };
}
```

**输出参数**：
```typescript
interface LongZhaoToLongXinOutput {
  // 执行状态
  status: "成功" | "部分成功" | "失败" | "需要确认";
  
  // 项目信息
  project: {
    id: string;
    name: string;
    type: string;
    currentPhase: string;
    progress: number;  // 0-100
  };
  
  // 执行结果
  result: {
    completedSteps: number[];
    currentStep?: number;

  };
  
  // 下一步建议
  nextActions: {
    description: string;
    recommendedAction: string;

  };
}
```

### 1.3 思维模型选择逻辑

**选择流程**：
```
龙心OS场景识别
    ↓
判断是否需要项目执行
    ↓
是 → 路由到龙爪OS
    ↓
龙爪OS分解为工作流步骤
    ↓
每个步骤请求龙心OS选择模型
    ↓
龙心OS根据步骤特征选择
    ↓
返回模型组合建议
    ↓
龙爪OS调用龙脑OS执行
```

**选择矩阵**：

| 步骤类型 | 推荐模型 | 辅助模型 | 选择理由 |

| 战略规划 | 第一性原理 | 二阶思维 | 本质+长远 |

### 2.2 调用协议

**协议名称**：`LongZhao-to-LongNao-Protocol-v1`

**调用方式**：
```yaml
同步调用:
  适用场景: 单模型分析，快速返回
  示例: 白光思维收集数据
  
异步调用:
  适用场景: 多模型并行，复杂分析
  示例: 五色光完整序列分析
  
流式调用:
  适用场景: 长文本生成，实时反馈
  示例: 诊断报告撰写
```

**输入参数**：
```typescript
interface LongZhaoToLongNaoInput {
  // 调用信息
  call: {
    requestId: string;
    timestamp: Date;
    priority: "高" | "中" | "低";
  };
  
  // 模型需求
  models: {
    primary: string;      // 主要模型
    secondary?: string[]; // 辅助模型
    combination?: string; // 组合方式：顺序/并行/混合
  };
  
  // 输入数据
  data: {
    type: "文本" | "数据" | "混合";
    content: any;
    context?: string;     // 额外上下文
  };
  
  // 输出要求
  output: {
    format: "结构化" | "文本" | "图表" | "混合";
    length?: "简短" | "中等" | "详细";
    language?: "中文" | "英文";
  };
}
```

**输出参数**：
```typescript
interface LongNaoToLongZhaoOutput {
  // 调用状态
  status: "成功" | "失败" | "部分成功";
  error?: string;
  
  // 分析结果
  result: {
    model: string;
    output: any;
    confidence: number;  // 置信度 0-1
    reasoning?: string;  // 推理过程
  }[];
  
  // 元数据
  meta: {
    processingTime: number;  // 处理时间（毫秒）
    tokenUsage: number;      // Token使用量
    cacheHit: boolean;       // 是否命中缓存
  };
}
```

### 2.3 模型组合策略

**策略1：顺序执行**
```
适用场景：步骤间有依赖关系
示例：诊断分析
步骤：白光思维 → 蓝光思维 → 绿光思维 → 黄光思维
逻辑：先收集事实 → 再识别问题 → 再创新方案 → 最后评估价值
```

**策略2：并行执行**
```
适用场景：多维度独立分析
示例：人格测评
并行：一心分析 + 三界扫描 + 五行诊断 + 九层定位
整合：综合所有维度结果
```

**策略3：混合执行**
```
适用场景：复杂项目，既有依赖又有并行
示例：企业文化项目
阶段1：顺序（调研→分析→方案→报告）
阶段2：并行（多维度理念提炼）
阶段3：顺序（整合→评审→定稿）
```

## 六、错误处理与容错机制

### 6.1 错误分类与处理

| 错误类型 | 错误示例 | 处理策略 | 降级方案 |
|---------|---------|---------|---------|
| 模型调用失败 | 龙脑OS服务不可用 | 重试3次后切换备用模型 | 使用简化版分析 |
| 数据缺失 | 必要输入未提供 | 提示用户补充 | 使用默认值继续 |
| 分析超时 | 复杂分析超过时限 | 分段返回结果 | 简化分析范围 |
| 结果不一致 | 多模型结果冲突 | 人工确认 | 置信度加权整合 |
| 系统错误 | 内部异常 | 记录日志，优雅降级 | 返回错误信息 |

### 6.2 降级策略

```
正常流程：
龙爪OS → 龙心OS选择模型 → 龙脑OS执行 → 返回结果

降级流程1（龙脑OS不可用）：
龙爪OS → 龙心OS选择模型 → 使用内置简化逻辑 → 返回基础结果

降级流程2（龙心OS路由失败）：
龙爪OS → 使用默认模型组合 → 执行分析 → 返回结果

降级流程3（系统异常）：
龙爪OS → 返回错误信息 → 建议用户稍后重试
```
