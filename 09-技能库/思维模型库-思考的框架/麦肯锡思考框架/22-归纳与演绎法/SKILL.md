# 归纳与演绎法

> 版本: v1.0 | 创建: 2026-04-15 | 维护者: 龙龟神将

---

## 一、核心定义

### 1.1 一句话定义
归纳法从个别到一般（从事实提炼规律），演绎法从一般到个别（从规律推演事实），两者是逻辑思维的基石。

### 1.2 核心公式

```
归纳法：事实1 + 事实2 + 事实3 → 规律
演绎法：规律 → 事实推演
```

---

## 二、归纳法 (Induction)

### 2.1 定义
从个别实例中总结出一般规律

### 2.2 逻辑结构

```
观察1：A公司战略成功 → 案例1
观察2：B公司战略成功 → 案例2
观察3：C公司战略成功 → 案例3
━━━━━━━━━━━━━━━
归纳：此类战略普遍有效 → 规律
```

### 2.3 归纳类型

| 类型 | 定义 | 示例 |
|------|------|------|
| 枚举归纳 | 数量积累 | 观察100只天鹅都是白的，归纳"天鹅都是白的" |
| 排除归纳 | 排除不可能 | 所有可能原因都被排除，剩下就是答案 |
| 类比归纳 | 相似性推理 | A和B在X、Y上相似，B还有Z特性，推测A也有Z |

---

## 三、演绎法 (Deduction)

### 3.1 定义
从一般规律推导个别结论

### 3.2 逻辑结构

```
大前提：规律/原则 → 所有天鹅都是白的
小前提：事实/情况 → 这是一只天鹅
━━━━━━━━━━━━━━━
演绎：结论 → 这只天鹅是白的
```

### 3.2 三段论结构

```
大前提：所有M都是P
小前提：所有S都是M
━━━━━━━━━━━━━━━
结论：所有S都是P
```

---

## 四、归纳vs演绎

| 维度 | 归纳法 | 演绎法 |
|------|--------|--------|
| 方向 | 个别→一般 | 一般→个别 |
| 结论 | 或然性（可能对） | 必然性（一定对） |
| 用途 | 发现规律 | 验证/应用规律 |
| 风险 | 过度归纳 | 前提可能错误 |

---

## 五、应用场景

### 5.1 归纳法应用

```
市场调研：
 - 访谈30位用户，80%提到同样痛点
 - 归纳：这是目标用户的核心痛点

竞品分析：
 - 分析5个成功竞品，都有相同功能
 - 归纳：这个功能是成功的关键因素
```

### 5.2 演绎法应用

```
战略推导：
 - 大前提：行业趋势是智能化
 - 小前提：我们的行业正在智能化
 - 结论：我们需要智能化战略

问题诊断：
 - 大前提：客户流失是因为服务不好
 - 小前提：这个客户流失了
 - 结论：可能服务出了问题
```

## 六、常见错误

### 归纳谬误

| 错误类型 | 示例 |
|---------|------|
| 过度归纳 | 只看3个案例就得出普遍规律 |
| 幸存者偏差 | 只看成功案例忽略失败案例 |
| 错误类比 | 表面相似但本质不同 |

### 演绎谬误

| 错误类型 | 示例 |
|---------|------|
| 大前提错误 | 前提本身就不成立 |
| 小前提错误 | 事实判断失误 |
| 概念混淆 | 偷换概念 |

---

## 七、核心金句

> "归纳与演绎是思考的两翼。归纳让我们从现象中发现规律，演绎让我们用规律指导行动。两者结合，才是完整的逻辑思维。"
