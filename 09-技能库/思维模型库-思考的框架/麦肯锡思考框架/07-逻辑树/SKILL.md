## 一、核心定义

### 1.1 一句话定义

**逻辑树**是一种将复杂问题**结构化拆解**的可视化工具，通过树状图形将问题分解为相互独立、完全穷尽的子问题，确保分析无遗漏、逻辑无重复。

### 1.2 核心公式

```
逻辑树 = 问题核心 + 逻辑分支 + MECE原则
```

### 1.3 经典引用

> "把一个大问题拆成小问题，你就已经解决了问题的一半。"  
> — 麦肯锡方法论

## 二、三种逻辑树类型

### 2.1 问题树（Issue Tree）

**用途**：问题诊断与原因分析

**结构**：
```
            [核心问题]
              /    \
           Why?    Why?
           /  \    /  \
         ...  ... ...  ...
```

**示例**：
```
        [销售额下降]
           /    \
       需求侧   供给侧
       /   \    /    \
    客户  渠道  产品   服务
```

### 2.2 假设树（Hypothesis Tree）

**用途**：假设验证与快速决策

**结构**：
```
        [核心假设]
           /    \
         如果...  并且...
         /  \    /  \
       证据1 证据2 证据3 证据4
```

**示例**：
```
      [新市场进入可行]
         /         \
    如果需求存在    并且我们能赢
       /    \          /    \
    调研   竞品    能力   资源
    数据   分析    评估   评估
```

### 2.3 是否树（Yes/No Tree）

**用途**：决策判断与方案筛选

**结构**：
```
        [决策问题]
         /      \
       Yes      No
        |       |
      下一步    下一步
```

**示例**：
```
      [是否进入新市场？]
          /        \
      是           否
      ↓            ↓
   如何进入?    为何不进?
```

## 三、MECE原则（核心检验标准）

### 3.1 MECE定义

| 维度 | 含义 | 说明 |
|------|------|------|
| **Mutually Exclusive** | 相互独立 | 子问题之间无重叠 |
| **Collectively Exhaustive** | 完全穷尽 | 所有子问题覆盖全部 |

### 3.2 MECE检验清单

```
分解层级的MECE检验：

□ 子问题1与子问题2是否重叠？
□ 子问题1+2+3+...是否能覆盖所有情况？
□ 是否有一个子问题可以进一步分解？
□ 分解的层级是否足够（一般3-5层）？
□ 最底层的子问题是否可执行分析？
```

## 四、操作流程

### 4.1 构建步骤

```
第一步：明确核心问题
    ↓
第二步：确定分解维度（What/Why/How）
    ↓
第三步：应用MECE检验
    ↓
第四步：构建第一层分支
    ↓
第五步：逐层分解（每层2-7个分支）
    ↓
第六步：验证完整性
    ↓
第七步：标注关键假设
```

### 4.2 分支数量控制

| 层级 | 建议分支数 | 原因 |
|------|-----------|------|
| 第一层 | 2-5个 | 便于记忆 |
| 第二层 | 3-7个 | 符合认知 |
| 第三层+ | 3-7个 | 避免过细 |

## 五、实战模板

### 5.1 战略分析逻辑树

```
          [公司战略选择]
              /    \
         增长     稳健
          |       |
    ┌─────┼─────┐ ┌────┐
    ↓     ↓     ↓ ↓    ↓
  市场  产品  渠道 现有 深耕
  扩张  延伸  创新 业务 客户
```

### 5.2 利润分析逻辑树

```
           [利润下降]
              /    \
          收入     成本
          /  \     /  \
       价格  销量  固定  可变
        |     |    |     |
      竞品  渠道  房租  原料
      定价  效率  工资  损耗
```
