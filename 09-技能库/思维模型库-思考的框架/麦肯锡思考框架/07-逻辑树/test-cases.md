# 🧪 逻辑树 · 测试用例

## 测试场景一：客户流失分析

### 输入
**情境**：某电商平台发现月活用户流失率上升，构建逻辑树诊断原因

### 期望输出
```
【逻辑树 · 客户流失原因分析】

问题树结构：

              [客户流失率上升]
                    /    \
              用户侧     平台侧
              /    \     /    \
          新客   老客   产品   服务
           |      |      |      |
         获取   留存   体验   售后
         成本高  活跃低  浏览   物流
                      体验差  投诉

MECE检验：
✓ 各分支相互独立，无重叠
✓ 分支覆盖用户侧+平台侧全部
✓ 可进一步分解到可执行层级
```

### 验证标准
- [ ] 问题树结构清晰
- [ ] MECE检验通过
- [ ] 分解到可执行层级

## 测试场景二：成本超支分析

### 输入
**情境**：某建筑项目成本超支30%，用逻辑树分解原因

### 期望输出
```
【逻辑树 · 项目成本超支分析】

              [成本超支30%]
                    /    \
              直接成本     间接成本
              /    \       /    \
          材料   人工   管理   风险
           |      |      |      |
        涨价   窝工   协调   设计
        损耗   效率低  沟通   变更
                       低效

进一步分解（直接成本）：
- 材料涨价：市场波动、供应链问题
- 材料损耗：管理不当、技术问题
- 人工窝工：天气、调配、配合
- 人工效率：技能、培训、激励
```

## 测试场景三：新产品上市决策

### 输入
**情境**：评估是否推出新产品，构建假设树

### 期望输出
```
【假设树 · 新产品上市决策】

        [我们应该推出新产品X]
               /         \
      如果市场存在需求    并且我们能获胜
         /         \         /         \
    目标客户    需求规模   竞争优势   资源能力
      明确      可量化     可持续     可获取
       |          |          |          |
    调研数据   TAM/SAM   差异化   研发/渠道
               /SAM
```

### 验证标准
- [ ] 假设树逻辑清晰
- [ ] 每个分支有验证路径
- [ ] 决策结论可推导
