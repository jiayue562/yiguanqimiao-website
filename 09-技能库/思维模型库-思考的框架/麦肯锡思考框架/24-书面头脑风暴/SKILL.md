# 书面头脑风暴

> 版本: v1.0 | 创建: 2026-04-15 | 维护者: 龙龟神将

---

## 一、核心定义

### 1.1 一句话定义
书面头脑风暴是一种**异步、匿名、去层级**的创意产生方法，通过结构化流程，让每个参与者独立思考后再汇总，打破面对面会议的局限性。

### 1.2 核心公式

```
书面头脑风暴 = 独立思考 + 延迟评判 + 数量优先 + 跨界组合
```

---

## 二、标准流程

### 2.1 六步法

```
第一步：明确议题
    ↓
第二步：独立撰写（10-15分钟）
    ↓
第三步：收集汇总（匿名）
    ↓
第四步：分类整理
    ↓
第五步：初步筛选（快速投票）
    ↓
第六步：深化讨论
```

### 2.2 规则

| 规则 | 说明 |
|------|------|
| 独立思考 | 不讨论，不看他人想法 |
| 延迟评判 | 不批评，不否定任何想法 |
| 数量优先 | 数量比质量重要 |
| 跨界组合 | 不同背景组合产生创意 |

## 四、实战模板

### 场景：新产品方向头脑风暴

```
【书面头脑风暴 · 新产品方向】

■ 第一轮：发散（收集30+想法）

参与者：产品/研发/市场/销售各3人
时间：15分钟独立撰写
规则：不讨论，只写下想法

■ 第二轮：汇总分类

分类1：产品功能创新（12个）
分类2：用户场景拓展（8个）
分类3：商业模式创新（6个）
分类4：技术突破（5个）

■ 第三轮：快速筛选

方法：每人10票，投最喜欢的5个
统计：得票最高的8个进入下一轮

■ 第四轮：深化讨论

对每个候选创意：
 - 补充细节
 - 评估可行性
 - 资源需求
 - 风险分析
```
