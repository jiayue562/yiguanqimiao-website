# 客户旅程图 · 测试用例

## 测试场景一：银行APP用户体验优化

### 输入
情境：某银行优化APP的用户体验旅程

### 期望输出
```
【客户旅程图 · 银行APP】

阶段① 认知
 - 触点：广告、朋友推荐、网点
 - 情感：陌生、谨慎
 - 痛点：安全顾虑
 - 机会：信任背书展示

阶段② 考虑
 - 触点：APP Store、对比评测
 - 情感：犹豫、评估
 - 痛点：功能太多难理解
 - 机会：简洁功能展示

阶段③ 开户
 - 触点：线上开户流程
 - 情感：焦虑、期待
 - 痛点：流程繁琐、等待长
 - 机会：快速开户

阶段④ 使用（转账）
 - 触点：转账功能
 - 情感：便利、安心
 - 痛点：限额提示
 - 机会：智能限额管理

阶段⑤ 推荐
 - 触点：社交分享
 - 情感：满意
 - 痛点：推荐激励不足
 - 机会：邀请有礼

【优化优先级】
P0：开户流程简化
P1：转账体验优化
P2：信任体系展示
```

## 测试场景二：SaaS产品客户旅程

### 输入
情境：某B2B SaaS产品优化从免费到付费的旅程

### 期望输出
```
【客户旅程图 · SaaS产品】

阶段① 认知
 - 触点：SEO、内容营销、销售推广
 - 情感：好奇、有兴趣
 - 痛点：产品价值难感知
 - 机会：价值导向内容

阶段② 试用
 - 触点：注册、引导、使用核心功能
 - 情感：期待、探索
 - 痛点：上手困难
 - 机会：智能新手指引

阶段③ 激活
 - 触点：关键功能使用、数据看板
 - 情感：有价值感
 - 痛点：看不到明显效果
 - 机会：价值可视化

阶段④ 付费
 - 触点：升级引导、合同、付款
 - 情感：决策焦虑
 - 痛点：价格不透明
 - 机会：定价策略优化

阶段⑤ 续费/扩费
 - 触点：续费提醒、升级推荐
 - 情感：认可或犹豫
 - 痛点：续费优惠不足
 - 机会：客户成功干预
```
