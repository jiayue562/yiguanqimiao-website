# 🧪 七步问题解决法 · 测试用例

## 测试场景一：销售业绩下滑诊断

### 输入
**情境**：某公司Q3销售额同比下降15%，需要用七步法诊断问题并提出解决方案

### 期望输出
```
【七步问题解决法 · 销售下滑诊断】

■ 第一步：陈述问题
销售部面临的销售额同比下降15%的问题，
导致全年利润目标完成率降至75%，
需要在Q4内将销售额提升至目标水平。

■ 第二步：分解问题
销售下滑 = 客流量下降 × 转化率下降 × 客单价下降
                ↓           ↓           ↓
           市场获客      销售能力     产品组合
           渠道效率      谈判技巧     定价策略

■ 第三步：优先排序
★★★ 高影响×高可行：销售团队培训（能力提升）
★★★ 高影响×高可行：产品组合优化（聚焦高毛利）
★★☆ 高影响×低可行：渠道扩张（需较长周期）

■ 第四步：分析计划
| 分析项 | 方法 | 数据来源 | 负责人 |
|--------|------|---------|--------|
| 客流量分析 | 趋势对比 | CRM系统 | 数据组 |
| 转化率分析 | 漏斗分析 | 销售日报 | 运营组 |
| 竞品分析 | 走访调研 | 市场部 | 市场组 |

■ 第五步：深入分析
【核心发现】
1. 客流量同比-10%，但转化率同比-5%
2. 高毛利产品销售占比从40%降至25%
3. 竞品Q3推出3款新品，我方无应对

■ 第六步：综合提炼
【洞察】
- 渠道获客能力尚可，问题在转化
- 产品竞争力下降是根本原因
- 销售团队能力与产品知识不足

【结论】
销售下滑的根因是产品竞争力下降+销售能力不足

■ 第七步：陈述建议
【建议1：产品组合聚焦】
- 理由：资源有限，需聚焦高毛利产品
- 行动：下架低毛利产品，重点推广TOP5
- 预期：毛利额提升20%

【建议2：销售能力提升】
- 理由：提升一线战力
- 行动：产品知识培训+销售技巧训练
- 预期：转化率提升15%
```

### 验证标准
- [ ] 问题陈述包含量化指标
- [ ] 分解逻辑清晰（MECE）
- [ ] 优先排序有依据
- [ ] 建议可执行可落地

## 测试场景二：客户满意度下降

### 输入
**情境**：某SaaS产品客户NPS分数从45降至30，需要诊断并解决

### 期望输出
```
【七步问题解决法 · 客户满意度提升】

■ 第一步：陈述问题
客户成功部面临客户NPS下降15分的问题，
导致客户续费率降至70%，
需要在6个月内将NPS恢复至40以上。

（分析过程同场景一）

■ 核心洞察
1. 新功能需求响应慢（平均3个月）
2. 客服响应时间超过24小时
3. 文档与教程不完善

■ 解决方案
【短期】客服升级：7×24支持 + 智能客服
【中期】产品加速：敏捷迭代，2周一个版本
【长期】生态建设：API开放，合作伙伴赋能
```

## 测试场景三：员工离职率上升

### 输入
**情境**：某科技公司研发团队年离职率达25%，远超行业平均15%

### 期望输出
```
【七步问题解决法 · 研发团队稳定性提升】

■ 第一步：陈述问题
研发部面临的年离职率25%问题，
导致项目交付延期率增加40%，知识流失严重，
需要在一年内将离职率降至15%以下。

■ 第二步：分解问题
离职率 = 主动离职率 × 被动离职率
             ↓            ↓
         薪酬、发展、文化    绩效、管理

■ 第三步：优先排序
★★★ 薪酬竞争力 → 调研市场薪资，调整薪酬结构
★★★ 成长空间 → 完善职级体系，技术路线明确
★★☆ 工作文化 → 团建、氛围文化建设

（完整七步输出）
```
