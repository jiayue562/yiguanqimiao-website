# 🧪 流程图 · 测试用例

## 测试场景一：电商退货流程

### 输入
**情境**：设计电商退货处理流程，识别优化点

### 期望输出
```
【流程图 · 电商退货处理】

标准流程：
开始 → 客户申请 → 客服审核 → 仓库接收 → 质检 → 退款/换货 → 结束

泳道流程：

| 客户 | 客服 | 仓库 | 财务 |
|------|------|------|------|
|开始 | | | |
|→申请退货 | | | |
| | →审核 | | |
| | 合规? | | |
| | 否→ | →拒绝 |
| | 是→ | →接收 | |
| | | →质检 | |
| | | 合格? | |
| | ←←← | 否→ | →换货 |
| | | 是 | |
| | ←←←←← | | →退款 |
|结束 | | | |

【优化点识别】
1. 质检环节成为瓶颈 → 增设质检人员
2. 退款周期长 → 自动化退款系统
3. 客户等待焦虑 → 实时进度通知
```

### 验证标准
- [ ] 流程节点完整
- [ ] 决策点清晰
- [ ] 泳道划分合理
- [ ] 优化点有针对性

## 测试场景二：招聘流程优化

### 输入
**情境**：优化公司招聘流程，从30天缩短至15天

### 期望输出
```
【流程图 · 招聘流程优化】

优化前（21天）：
简历筛选(3d) → 初试(7d) → 复试(7d) → 发放offer(4d)

优化后（12天）：
简历筛选(2d) → AI预筛(1d) → 初试(3d) → 复试(3d) → 并行背调(3d) → offer

【关键优化】
1. AI预筛：自动过滤不符合条件简历
2. 并行背调：复试同时启动背调
3. 简化审批：取消HRD复核环节
```
