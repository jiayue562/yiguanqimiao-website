## 一、核心定义

### 1.1 一句话定义

**流程图**是用标准化的图形符号将**工作流程、业务逻辑、决策路径**可视化呈现的工具，帮助识别流程中的瓶颈、冗余和优化机会。

### 1.2 核心符号

| 符号 | 名称 | 含义 |
|------|------|------|
| ○ | 开始/结束 | 流程的起点和终点 |
| ▭ | 处理步骤 | 执行具体的操作或任务 |
| ◇ | 决策判断 | 分支判断（是/否） |
| → | 箭头 | 流程流向方向 |
| ▷ | 输入/输出 | 数据或信息的输入输出 |

---

## 二、流程图类型

### 2.1 线性流程图

**用途**：单一线性任务的可视化

```
开始 → 接收订单 → 审核订单 → 生产制作 → 质检出货 → 结束
```

### 2.2 泳道流程图（跨职能流程图）

**用途**：多部门/多角色协作的流程

| 角色A | 角色B | 角色C |
|-------|-------|-------|
| 开始 | | |
| →处理 | →接收→ | |
| | →判断→ | |
| | | →执行→ |
| | ←反馈← | |
| 结束 | | |

### 2.3 决策流程图

**用途**：复杂决策路径可视化

```
发现问题
    ↓
是否紧急？ → 否 → 排队处理
    ↓是
是否重要？ → 否 → 快速处理
    ↓是
立即行动
```

## 三、操作流程

### 3.1 构建步骤

```
第一步：明确流程边界（起点/终点）
    ↓
第二步：识别关键步骤（主要活动）
    ↓
第三步：确定决策点（分支判断）
    ↓
第四步：标注责任角色（泳道）
    ↓
第五步：识别优化点（瓶颈/冗余）
    ↓
第六步：输出流程文档
```

### 3.2 流程优化检查清单

| 检查项 | 问题信号 | 优化方向 |
|--------|---------|---------|
| **效率** | 等待时间过长 | 并行处理 |
| **冗余** | 重复审批 | 简化环节 |
| **瓶颈** | 某步骤堆积 | 增设资源 |
| **断点** | 流程中断 | 无缝衔接 |
| **决策** | 判断模糊 | 明确标准 |

```
| | ←←←←←← | |
| | →发货指令 | | |

```

### 4.2 审批流程优化

**优化前（5个节点）**：
```
员工 → 主管 → 经理 → 总监 → 总经理 → HR
```

**优化后（3个节点）**：
```
员工 → 主管（自动触发） → 经理（金额阈值）
           ↓
        小额（<1万）直接通过
        大额（>1万）→ 总监
```
