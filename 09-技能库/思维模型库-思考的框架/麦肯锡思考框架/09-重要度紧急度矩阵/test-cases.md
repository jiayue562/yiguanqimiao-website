# 🧪 重要度紧急度矩阵 · 测试用例

## 测试场景一：创业者每日任务管理

### 输入
**情境**：某创业公司CEO有10个待处理任务，需要用矩阵分类排序

### 期望输出
```
【重要度紧急度矩阵 · CEO每日任务】

任务清单与分类：

| 任务 | 重要度 | 紧急度 | 象限 | 行动 |
|------|--------|--------|------|------|
| A. 投资人会议（明天） | ★★★★★ | ★★★★★ | I | 立即做 |
| B. 核心员工离职危机 | ★★★★★ | ★★★★ | I | 立即做 |
| C. 产品战略规划 | ★★★★★ | ★★ | II | 计划做 |
| D. 下周发布会准备 | ★★★★ | ★★★★ | I | 立即做 |
| E. 政府关系维护 | ★★★★ | ★★ | II | 计划做 |
| F. 团队1v1沟通 | ★★★ | ★★★ | III | 委托/控制 |
| G. 媒体采访邀约 | ★★ | ★★★ | III | 委托 |
| H. 行业报告阅读 | ★★ | ★ | IV | 删除 |
| I. 办公室装修监督 | ★★ | ★★ | IV | 减少 |
| J. 个人健身 | ★★★ | ★ | IV | 删除/调整 |

时间分配：
- 第一象限：4小时（40%）
- 第二象限：3小时（30%）
- 第三象限：2小时（20%）
- 第四象限：1小时（10%）
```

### 验证标准
- [ ] 所有任务分类完成
- [ ] 每象限有明确行动指引
- [ ] 时间分配合理

## 测试场景二：项目经理周任务规划

### 输入
**情境**：某项目经理有一周时间处理以下任务

### 期望输出
```
【重要度紧急度矩阵 · 项目经理周规划】

第一象限（每天优先）：
- P0故障处理
- 核心模块交付（截止周五）
- 客户演示准备

第二象限（计划执行）：
周一：技术债务梳理
周二：团队能力培训
周三：架构设计评审
周四：代码review
周五：下周计划制定

第三象限（控制干扰）：
- 每日站会（限时15分钟）
- 跨部门协调（委托主持）
- 邮件处理（固定时间集中处理）

第四象限（警惕陷阱）：
- 无目的刷群消息
- 可以推迟的会议
```

## 测试场景三：学生期末复习

### 输入
**情境**：某大学生期末有5门考试，需要规划复习顺序

### 期望输出
```
【重要度紧急度矩阵 · 期末复习】

第一象限（紧急且重要 → 立即复习）：
- 明天考试：科目A（已复习30%）
- 后天考试：科目B（已复习20%）

第二象限（重要不紧急 → 计划复习）：
- 大后天考试：科目C（已复习50%）
- 制定整体复习计划

第三象限（紧急不重要 → 控制时间）：
- 同学求助解答问题
- 复习群里的讨论

第四象限（删除）：
- 刷剧放松（适度）
- 无目的刷手机

复习顺序建议：
1. 科目B（紧急+未复习）
2. 科目A（紧急+已复习一部分）
3. 科目C（按计划复习）
4. 其他科目
```
