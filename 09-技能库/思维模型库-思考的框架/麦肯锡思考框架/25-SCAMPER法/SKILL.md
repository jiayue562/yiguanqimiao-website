# SCAMPER法

> 版本: v1.0 | 创建: 2026-04-15 | 维护者: 龙龟神将

---

## 一、核心定义

### 1.1 一句话定义
SCAMPER是一种系统的**产品/服务创新检查清单**，通过七个维度（替代/组合/改造/调整/其他用途/消除/逆转）引导改进方向的思考。

### 1.2 核心公式

```
SCAMPER = S(Substitute) + C(Combine) + A(Adapt) + M(Modify) + P(Put to other uses) + E(Eliminate) + R(Reverse)
```

---

## 二、七维度详解

### 2.1 S - Substitute（替代）

**问题**：什么可以替代？

| 替代维度 | 思考方向 |
|---------|---------|
| 材料 | 用其他材料替代？ |
| 资源 | 用其他资源实现？ |
| 环节 | 跳过某个环节？ |
| 人员 | 其他人做？ |

### 2.2 C - Combine（组合）

**问题**：什么可以组合？

| 组合维度 | 思考方向 |
|---------|---------|
| 功能 | 组合哪些功能？ |
| 用户 | 哪些用户需求可以一起满足？ |
| 渠道 | 哪些渠道可以整合？ |
| 体验 | 哪些体验可以融合？ |

### 2.3 A - Adapt（适应）

**问题**：如何适应环境？

| 适应维度 | 思考方向 |
|---------|---------|
| 场景 | 可以用到其他场景？ |
| 形式 | 可以变成什么形式？ |
| 部分 | 可以借用哪些部分？ |

### 2.4 M - Modify（调整）

**问题**：如何调整/放大/缩小？

| 调整维度 | 思考方向 |
|---------|---------|
| 放大 | 更大/更强/更长？ |
| 缩小 | 更小/更轻/更简洁？ |
| 调整 | 改变某些特征？ |

### 2.5 P - Put to other uses（其他用途）

**问题**：还有什么其他用途？

| 其他用途 | 思考方向 |
|---------|---------|
| 用户 | 还有谁可以使用？ |
| 场景 | 还有什么其他场景？ |
| 价值 | 还有什么其他价值？ |

### 2.6 E - Eliminate（消除）

**问题**：什么可以消除？

| 消除维度 | 思考方向 |
|---------|---------|
| 简化 | 去掉哪些部分？ |
| 精简 | 简化哪些流程？ |
| 合并 | 哪些可以二合一？ |

### 2.7 R - Reverse（逆转）

**问题**：如何逆转/反向？

| 逆转维度 | 思考方向 |
|---------|---------|
| 顺序 | 逆向操作？ |
| 角色 | 角色互换？ |
| 视角 | 换个角度思考？ |

## 三、操作流程

### 3.1 标准流程

```
第一步：选择改进对象
    ↓
第二步：逐维度头脑风暴
    ↓
第三步：记录每个维度的想法
    ↓
第四步：初步筛选可行想法
    ↓
第五步：组合优化形成方案
```

### 3.2 检查清单

```
□ S（替代）：想到了3个以上替代可能？
□ C（组合）：找到了有价值的组合？
□ A（适应）：发现了新的使用场景？
□ M（调整）：考虑了大小/程度变化？
□ P（其他用途）：探索了新的用户/价值？
□ E（消除）：找到了可以简化的部分？
□ R（逆转）：尝试了反向思考？
```

## 四、实战模板

### 场景：共享单车创新

```
【SCAMPER · 共享单车创新】

S替代：
 - 塑料车身→金属车身（更耐用）
 - 机械锁→电子锁（更便捷）

C组合：
 - 单车+广告屏（流量变现）
 - 单车+手机充电（解决痛点）

A适应：
 - 从城市出行→景区旅游
 - 从个人使用→企业团购

M调整：
 - 车型加大（亲子需求）
 - 车型缩小（轻便折叠）

P其他用途：
 - 最后三公里通勤
 - 运动健身场景
 - 广告媒体载体

E消除：
 - 消除车筐（减少损坏）
 - 消除停车桩（无桩模式）

R逆转：
 - 从"找车"到"车找我"（预约制）
 - 从"用户找车"到"车找用户"
```
