# 决策矩阵

> 版本: v1.0 | 创建: 2026-04-15 | 维护者: 龙龟神将

---

## 一、核心定义

### 1.1 一句话定义
决策矩阵是一种**多方案量化评估工具**，通过设定评估维度、赋予权重、方案评分，计算综合得分，辅助选择最优方案。

### 1.2 核心公式

```
决策矩阵 = 评估维度 × 权重 × 方案得分 = 综合得分
```

---

## 二，操作流程

### 2.1 标准六步

```
第一步：确定评估维度（5-8个）
    ↓
第二步：赋予权重（总和100%）
    ↓
第三步：列出备选方案（3-5个）
    ↓
第四步：各方案各维度评分（1-10分）
    ↓
第五步：计算加权得分
    ↓
第六步：敏感性分析
```

### 2.2 评分标准

| 分数 | 标准 |
|------|------|
| 10 | 完美满足该维度 |
| 7-9 | 较好满足 |
| 4-6 | 基本满足 |
| 1-3 | 很少满足 |
| 0 | 完全不满足 |

---

## 三，矩阵模板

### 3.1 基本模板

| 评估维度 | 权重 | 方案A | 方案B | 方案C |
|---------|------|-------|-------|-------|
| 维度1 | % | | | |
| 维度2 | % | | | |
| 维度3 | % | | | |
| 维度4 | % | | | |
| 维度5 | % | | | |
| **加权得分** | 100% | | | |

### 3.2 计算方法

```
加权得分 = Σ(权重 × 评分)

示例：
方案A得分 = 20%×8 + 25%×7 + 15%×9 + 25%×6 + 15%×8
           = 1.6 + 1.75 + 1.35 + 1.5 + 1.2
           = 7.4
```

## 四、实战模板

### 场景：供应商选择

```
【决策矩阵 · 供应商选择】

| 评估维度 | 权重 | 供应商A | 供应商B | 供应商C |
|---------|------|---------|---------|---------|
| 价格 | 30% | 8 | 9 | 6 |
| 质量 | 25% | 9 | 7 | 8 |
| 交付 | 20% | 7 | 8 | 7 |
| 服务 | 15% | 6 | 7 | 9 |
| 合作历史 | 10% | 8 | 5 | 6 |
| **加权得分** | 100% | **7.85** | **7.55** | **7.05** |

【结论】选择供应商A（得分7.85）
 - 综合表现最优
 - 价格和服务平衡
 - 建议签订3年框架协议
```

## 五、敏感性分析

### 5.1 权重敏感性

| 维度 | 原权重 | 调整后 | 方案变化 |
|------|-------|--------|---------|
| 价格 | 30% | 40% | 供应商B变为最优 |
| 服务 | 15% | 25% | 供应商C迎头赶上 |

### 5.2 敏感性分析价值
- 验证结论是否稳健
- 识别关键决策因素
- 辅助权重设定合理性判断
