# 🧪 假设思考 · 测试用例

## 测试场景一：定价策略假设验证

### 输入
**情境**：某SaaS产品考虑从19.9元/月涨价至29.9元/月，需要验证假设

### 期望输出
```
【假设思考 · 定价策略决策】

■ 核心假设
H1: 涨价不会导致流失率显著上升
H2: 涨价带来的收入增长大于流失损失
H3: 品牌形象不会因涨价受损

■ 验证设计

H1验证：
方法：用户访谈 + 历史数据
  - 深度访谈20位高活跃用户
  - 分析竞品涨价后的用户行为
验证标准：预计流失率 < 15%

H2验证：
方法：财务建模
  - 计算不同流失率下的收入变化
  - 敏感性分析
验证标准：净利润提升 > 10%

H3验证：
方法：品牌形象调研
  - 涨价前后NPS对比
验证标准：NPS下降 < 5分

■ 决策建议
如果H1、H2、H3均验证通过 → 执行涨价
如果H1未通过 → 暂缓涨价
如果H2未通过 → 调整涨价幅度
```

### 验证标准
- [ ] 假设具体可验证
- [ ] 验证方法有可操作性
- [ ] 决策路径清晰

## 测试场景二：技术选型假设验证

### 输入
**情境**：某团队考虑从React迁移到Vue，需要验证迁移假设

### 期望输出
```
【假设思考 · 技术栈迁移决策】

■ 核心假设
H1: Vue的上手速度比React快30%以上
H2: Vue的招聘难度比React低
H3: Vue的性能满足业务需求

■ 验证设计

H1验证（上手速度）：
方法：对比实验
  - 选取3位熟悉React的工程师
  - 同时学习Vue和React一周
  - 对比任务完成时间
标准：Vue快30%+

H2验证（招聘难度）：
方法：招聘市场数据分析
  - 统计BOSS直聘上React/Vue职位数量
  - 对比平均薪资
标准：Vue职位多20%或薪资低10%

H3验证（性能）：
方法：技术测试
  - 用相同页面做性能benchmark
  - 对比首屏加载、白屏时间
标准：性能差距 < 10%

■ 决策建议
H1验证失败 → 继续React
H2验证失败 → 继续React
H3验证失败 → 继续React
```

## 测试场景三：客户成功策略假设验证

### 输入
**情境**：某ToB产品考虑引入客户成功经理（CSM）提升续费率

### 期望输出
```
【假设思考 · CSM引入决策】

■ 核心假设
H1: 客户流失主要原因是缺乏主动关怀
H2: CSM的主动服务能提升续费率
H3: CSM的成本可以被续费收入覆盖

■ 验证设计

H1验证（流失原因）：
方法：客户访谈（流失客户20家）
  - 询问流失原因
  - 归类分析
标准：60%+客户提到"缺乏关注"

H2验证（CSM效果）：
方法：AB测试
  - 50家客户分配CSM主动服务
  - 50家客户保持现状
  - 6个月后对比续费率
标准：CSM组续费率 > 对照组15%

H3验证（成本覆盖）：
方法：财务测算
  - CSM成本 = 薪资 × 人数
  - 续费率提升带来的收入增量
验证标准：ROI > 2

■ 决策建议
H1验证失败 → 重新诊断流失原因
H2验证失败 → 优化CSM服务模式
H3验证失败 → 调整CSM配置或暂缓
```
