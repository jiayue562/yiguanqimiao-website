# CLI个人指令体系 Skill

## 核心定位

个人AI OS的**原生操作入口**，锚定"人控AI行"的交互底层。

> 核心理念：人只需发高层决策指令，AI自动拆解为底层CLI操作执行。

---

## 触发条件

- 用户提到"CLI"、"命令行"、"指令体系"
- 用户要求"设置个人指令"、"创建专属命令"
- 用户说"按我的思维模型分析"、"按长期主义视角"
- 进入人机协同场景，需要发出决策指令

---

## 核心功能

### 1. 指令分类管理

| 类型 | 说明 | 示例 |
|------|------|------|
| **核心决策指令** | 人发出，带个性化参数 | `按GEO思维模型分析AI趋势` |
| **执行操作指令** | AI自主调用 | `检索信仰相关笔记`、`运行数据脚本` |
| **人格适配指令** | 带语气/人格特质 | `用理性逻辑分析`、`用生活化语言总结` |

### 2. 指令解析引擎

- 提取指令中的**个性化参数**（思维模型、价值观、文化标签）
- 解析**人格特质**（语气、风格偏好）
- 转换为**底层执行指令**链

### 3. 指令存储与调用

- 个人指令库位置：`Obsidian/03-记忆系统/CLI指令库/`
- 支持按标签检索（信仰类、思维模型类、人格类）
- 自动记录指令使用频率，优化推荐

## 预设指令模板

### 决策指令模板

```
[按/基于] [我的思维模型] [分析/研究/输出] [主题]
```

示例：
- `按第一性原理分析AI行业发展`
- `基于长期主义视角评估投资机会`
- `用GEO思维模型写一篇行业报告`

### 人格指令模板

```
[用] [理性/感性/生活化/专业] [语言/方式/风格] [输出/总结]
```

示例：
- `用理性逻辑分析这个问题`
- `用生活化语言总结这篇内容`
- `用专业术语撰写报告`

### 信仰/文化指令模板

```
[结合] [儒家/道家/佛学/大圆满] [理念/智慧] [分析/解读]
```

示例：
- `结合佛学缘起性空解读这件事`
- `用大圆满见地分析心性`

## 指令执行流程

```
人发出高层指令
    ↓
CLI指令体系Skill解析
    ↓ 提取个性化参数
    ↓ 识别人格特质
    ↓ 匹配思维模型
    ↓
转换为底层执行指令
    ↓
调用对应Skills执行
    ↓
结果反馈 + 记录优化
```

## 核心原则

1. **人少操作、AI多干活** - 人只发高层指令
2. **指令带人格** - 所有执行带个人特质
3. **信仰/文化渗透** - 思维模型、价值观作为默认参数
4. **持续优化** - 记录反馈，自动改进指令效果
