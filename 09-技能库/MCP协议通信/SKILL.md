# MCP协议通信Skill v1.0

## 🌐 技能定位

MCP（机器控制协议）是AI OS的**通用协同语言**，负责：
- 打通人机、机机的共生联动链路
- 定义"人与AI、AI与Skills、AI与AI、AI与全系统组件"如何通信、协同、同步数据
- 解决异构系统信息孤岛问题，让整个AI OS成为有机整体

> **核心定位**：MCP是个人AI OS的"协同总线"，是支撑"人机共生、高效联动"的核心通信基础
> **共生价值**：MCP让整个AI OS的人机联动、机内联动实现标准化、无歧义，人发出的每一个决策都能快速传递到AI的每一个执行环节
> **个性化价值**：将个人的个性化内核作为MCP协议的"默认参数"，让全系统的协同执行都"带着个人印记跑"

---

## 📋 三大核心MCP协议

### 1. 人机协同协议

#### 定义
规范人向AI发出的决策指令、AI向人反馈的执行结果的格式

#### 协议特点
- 支持"指令带个性化参数"
- 支持"结果带解释性说明"
- 让人能快速理解AI的执行逻辑
- 让AI能精准接收人的创意/决策

#### 协议格式（人→AI指令）

```json
{
  "version": "1.0",
  "protocol": "human-ai",
  "timestamp": "2026-03-22T15:20:00Z",
  "sender": "human",
  "receiver": "ai",
  "decision": {
    "type": "high-level",
    "content": "基于我的GEO思维模型分析AI行业趋势",
    "context": {
      "domain": "AI行业",
      "timeframe": "未来3-5年",
      "focus": "技术趋势 + 市场应用"
    }
  },
  "personal_params": {
    "思维模型": "GEO思维模型",
    "价值观": "长期主义",
    "人格特质": "偏理性逻辑",
    "信仰文化": ["五行识人", "象思维"]
  },
  "knowledge_tags": ["AI行业", "GEO思维模型", "五行识人"]
}
```

#### 协议格式（AI→人反馈）

```json
{
  "version": "1.0",
  "protocol": "human-ai",
  "timestamp": "2026-03-22T15:25:00Z",
  "sender": "ai",
  "receiver": "human",
  "result": {
    "status": "success",
    "content": "AI行业趋势分析结果",
    "data": {
      "核心观点": "...",
      "详细分析": "..."
    }
  },
  "explanation": {
    "skills_used": ["行业数据检索Skill", "个人思维模型分析Skill", "文案撰写Skill"],
    "logic_flow": "检索数据 → GEO模型分析 → 生成报告",
    "personal_alignment": "结果符合GEO思维模型框架和长期主义价值观"
  },
  "confidence": 0.85
}
```

### 2. 机内协同协议

#### 定义
规范AI与Skills、Skills与Skills之间的调用规则、数据同步格式

#### 协议特点
- 让AI能无缝组合Skills
- 保证数据在不同模块间的精准传递
- 无需格式转换，数据透明传递

#### 协议格式（AI→Skill调用）

```json
{
  "version": "1.0",
  "protocol": "internal",
  "timestamp": "2026-03-22T15:20:05Z",
  "caller": "Framework调度模块",
  "callee": "个人思维模型分析Skill",
  "task_id": "task-001",
  "input_data": {
    "raw_data": {
      "type": "行业数据",
      "content": "..."
    },
    "personal_params": {
      "思维模型": "GEO思维模型",
      "价值观": "长期主义"
    }
  },
  "output_format": {
    "type": "structured",
    "schema": "GEO-model-analysis"
  },
  "context": {
    "task_chain": ["数据检索", "模型分析", "文案生成"],
    "previous_results": {
      "data_retrieval": {...}
    }
  }
}
```

#### 协议格式（Skill→Skill数据传递）

```json
{
  "version": "1.0",
  "protocol": "internal",
  "timestamp": "2026-03-22T15:20:10Z",
  "from": "个人思维模型分析Skill",
  "to": "文案撰写Skill",
  "task_id": "task-001",
  "data": {
    "type": "analysis-result",
    "content": {
      "核心观点": "...",
      "详细分析": "..."
    }
  },
  "personal_params": {
    "表达风格": "偏理性逻辑",
    "语言风格": "生活化"
  },
  "context": {
    "task_chain": ["数据检索", "模型分析", "文案生成"],
    "next_step": "生成文案"
  }
}
```

- 保证个人AI OS的可扩展性
- 随AI能力进化和个人需求变化持续升级

#### 协议格式（外部工具接入）

```json

  "interface": {
    "type": "command-line",
    "protocol": "HTTP",

    "credential": "..."
  },

    "code-review",
    "unit-test-generation",
    "code-refactoring"

    "思维模型": "第一性原理",
```

## 🎯 核心功能

### 功能1：协议路由与调度

**触发**：收到协议消息

**执行流程**：
1. 解析协议类型（人机协同/机内协同/系统扩展）
2. 根据协议类型路由到对应的处理模块
3. 携带个性化参数传递
4. 确保协议数据的完整性和一致性

**输出格式**：
```
╔══════════════════════════════════════════╗
║          🌐 MCP协议路由日志            ║
╠══════════════════════════════════════════╣
║ 协议类型：[human-ai / internal / system-extension] ║
║ 时间戳：[timestamp]                  ║
║ 发送方：[sender]                       ║
║ 接收方：[receiver]                     ║
║                                       ║
║ 个性化参数：                           ║
║   - 思维模型：[参数]                  ║
║   - 价值观：[参数]                    ║
║   - 人格特质：[参数]                  ║
║   - 信仰文化：[参数]                  ║
║                                       ║
║ 路由结果：[成功/失败·说明]            ║
╚══════════════════════════════════════════╝
```

---

### 功能2：数据同步与一致性保证

**触发**：Skills之间需要数据同步时

**执行流程**：
1. 确保数据格式符合协议规范
2. 携带个性化参数传递数据
3. 验证数据完整性和一致性
4. 记录数据同步日志

**输出格式**：
```
╔══════════════════════════════════════════╗
║          📊 数据同步日志               ║
╠══════════════════════════════════════════╣
║ 数据类型：[类型]                       ║
║ 数据格式：[JSON/Markdown/其他]         ║
║ 源Skill：[source-skill]               ║
║ 目标Skill：[target-skill]             ║
║                                       ║
║ 数据完整性：[完整/不完整]             ║
║ 数据一致性：[一致/不一致]             ║
║                                       ║
║ 个性化参数传递：[成功/失败]           ║
║ 同步耗时：[X ms]                       ║
╚══════════════════════════════════════════╝
```

```
║   - [协议1]：[优化建议]                ║

║   - [协议3]：[优化建议]                ║
║                                       ║
║ 个性化参数权重调整：                   ║
║   - [参数]：[调整前] → [调整后]       ║
```
