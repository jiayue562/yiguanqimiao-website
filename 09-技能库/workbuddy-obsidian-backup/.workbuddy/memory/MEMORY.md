# WorkBuddy-Obsidian 备份系统长期记忆

## 系统概览

WorkBuddy-Obsidian 备份系统是一个自动化知识沉淀平台，负责将 WorkBuddy 中的所有内容（系统配置、技能库、记忆系统、对话记录等）备份到 Obsidian 知识库，实现知识资产的三向同步（WorkBuddy ↔ Obsidian ↔ IMA）。

## 核心架构

### 主要组件
1. **主备份脚本** (`backup_script.py`) - 执行完整备份流程
2. **配置文件** (`config.json`) - 定义备份路径和规则
3. **三向同步脚本** (`tri_sync.py`) - 实现多平台同步
4. **手动备份脚本** (`manual_backup.py`) - 简化备份操作
5. **批处理脚本** (`simple_backup.bat`) - Windows环境快速备份
6. **手动触发脚本** (`manual_trigger.py`) - 新创建，用于绕过权限限制（2026-03-27）

### 备份目标
- **系统配置**：IDENTITY.md, USER.md, SOUL.md, TOOLS.md, SKILLS.md 等
- **技能库**：所有用户技能和项目技能
- **记忆系统**：工作记忆、长期记忆、会话记忆
- **项目文件**：所有项目相关文档和配置
- **自动化配置**：调度任务和定时规则

## 问题诊断与解决方案

### 常见问题
1. **权限问题**：执行命令受限
   - **症状**：所有执行命令返回 "spawn EPERM"
   - **原因**：环境安全限制
   - **解决方案**：在外部环境中执行或使用系统级定时任务

2. **路径问题**：文件路径配置错误
   - **症状**：文件不存在或不可访问
   - **原因**：路径配置与实际位置不匹配
   - **解决方案**：验证配置文件中的路径

3. **序列化问题**：JSON 序列化失败
   - **症状**：Object of type datetime is not JSON serializable
   - **原因**：Python datetime 对象无法直接序列化
   - **解决方案**：使用 `_serialize_backup_history()` 方法转换

### 故障排除流程
```
1. 检查配置文件路径 → 2. 验证文件权限 → 3. 执行简单测试 
→ 4. 逐步执行备份 → 5. 生成详细报告 → 6. 优化配置
```
