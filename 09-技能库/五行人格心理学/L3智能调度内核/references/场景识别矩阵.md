# 五行人格心理学操作系统 - 场景识别矩阵 v1.0

## 概述

场景识别矩阵是L3智能调度内核的核心组件，负责将用户输入精确分类到S0-S9场景，并匹配最优引擎组合。

---

## S0-S9 场景分类定义

| 场景 | 名称 | 定义 | 典型输入 | 引擎组合 |
|------|------|------|---------|---------|
| S0 | 简单问答 | 直接回答，不需要引擎 | "什么是五行" | 静默 |
| S1 | 信息查询 | 简单信息获取 | "木行人有什么特点" | L2知识库 |
| S2 | 深度理解 | 需要系统性分析 | "详细分析木行人的成长路径" | L2+🐉 |
| S3 | 创意创新 | 需要新想法新方案 | "有什么创新应用" | 🐉+🌈绿蓝 |
| S4 | 分析决策 | 多角度利弊分析 | "木行vs火行，哪个适合创业" | 🌈完整序列 |
| S5 | 重大决策 | 影响人生方向的决策 | "我该选择什么职业" | 全系统 |
| S6 | 任务执行 | 具体任务完成 | "帮我写一份分析报告" | 🤝人机协同 |
| S7 | 系统规划 | 构建完整体系 | "如何建立五行管理体系" | 🐉+🤝+🌈 |
| S8 | 修行文化 | 涉及心性修行 | "如何通过五行修行提升心性" | 🐉+📚溯源 |
| S9 | 系统升级 | 优化改进系统本身 | "如何优化五行分析准确度" | 🔄+🐉+🌈 |

## 复杂度评估标准

### 低复杂度特征
- 单一问题
- 信息明确
- 不需要推理
- 答案直接

### 中复杂度特征
- 多角度问题
- 需要选择
- 需要推理
- 需要整合

### 高复杂度特征
- 多层次问题
- 涉及人生决策
- 需要系统思考
- 需要创新方案

## 引擎组合规则

### 单引擎使用
```
S0 → 静默（直接回答）
S1 → L2知识库检索
```

### 双引擎组合
```
S3 → 🐉象思维 + 🌈绿光（创新）
S4 → 🌈白光 + 🌈黄光（分析）
S8 → 🐉象思维 + 📚溯源（文化）
```

### 多引擎序列
```
S2 → L2 → 🐉 → 🔄
S7 → 🐉 → 🤝 → 🌈 → 🔄
```

### 全系统（A型配方）
```
S5 → 🐉 → 📚 → 🌈 → 🤝 → 🔄
S9 → 🔄 → 🐉 → 🌈 → 更新
```
