## 🔗 协同协议详解

### 协议1：企业文化OS知识库 ↔ 龙心OS总智能体

**协同方式**：被调用层

**调用流程**：
```
用户查询 → 龙心OS场景识别 → 判断为企业文化相关
              ↓
    路由至企业文化OS知识库
              ↓
    返回知识检索结果
              ↓
    龙心OS整合输出
```

**握手包格式**：
```yaml
调用方: 龙心OS总智能体
被调用方: 企业文化OS知识库
输入:
  - 查询意图: [用户问题]
  - 场景类型: [S0-S9]
  - 深度要求: [浅层/中层/深层]
输出:
  - 知识节点: [相关知识点列表]
  - 关联联系: [跨域知识联系]
  - 推荐引擎: [建议调用的子引擎]
```

---

### 协议2：企业文化OS知识库 ↔ 象思维

**协同方式**：知识支撑 + 创新突破

**协同场景**：
- 当需要企业文化0→1创新时，象思维提供原创突破
- 企业文化OS知识库提供理论基础和创新约束

**协同流程**：
```
创新需求 → 象思维激活（0→1模式）
              ↓
    查询企业文化OS知识库（理论约束）
              ↓
    象思维在约束下创新
              ↓
    输出创新方案
```

**示例**：
- 输入："如何设计有特色的企业文化仪式？"
- 象思维：从"物象→意象→原象"三层次创新
- 知识库：提供礼乐仪式理论、镜像神经元原理
- 输出：创新的文化仪式设计方案

---

### 协议3：企业文化OS知识库 ↔ 知识学习

**协同方式**：深度学习 + 知识重构

**协同场景**：
- 当用户需要深度理解企业文化理论时
- 知识学习执行十项认知指令

**协同流程**：
```
深度学习需求 → 知识学习激活
                  ↓
        从企业文化OS知识库获取理论
                  ↓
        执行十项认知指令:
          1. 剖析 - 拆解理论结构
          2. 解构 - 分析组成要素
          3. 透视 - 洞察深层逻辑
          4. 阐释 - 解释核心概念
          5. 推演 - 推导应用场景
          6. 思辨 - 批判性思考
          7. 溯源 - 追溯理论来源
          8. 融合 - 跨域知识关联
          9. 启发 - 生成新洞察
          10. 映射 - 应用到实际
                  ↓
        输出深度学习报告
```

### 协议4：企业文化OS知识库 ↔ 五色光思维

**协同方式**：多维分析 + 决策支持

**协同场景**：
- 企业文化问题诊断
- 文化方案评估决策

**协同流程**：
```
决策需求 → 五色光思维激活
              ↓
    查询企业文化OS知识库获取信息
              ↓
    五色光多维分析:
      ⬜ 白光: 企业文化现状数据
      🟥 红光: 直觉感受（文化痛点）
      🟨 黄光: 价值利益（改进收益）
      🟩 绿光: 创新方案（改进方案）
      🟦 蓝光: 风险问题（实施风险）
              ↓
    输出决策建议
```

**示例**：企业文化变革决策
- 白光：员工满意度65%，流失率15%
- 红光：员工普遍感到"缺乏归属感"
- 黄光：改进后预计满意度提升至80%，流失率降至8%
- 绿光：三步走变革方案
- 蓝光：变革阻力、时间成本、资源投入

**协同流程**：

```
      - 共创导师: 战略指导、经验传承
      - 未知探索: 创新突破、0→1创造
              ↓
    从企业文化OS知识库获取知识支撑
              ↓
    人机分工执行
              ↓
    输出执行结果
```

## 📋 调用规范

### 标准调用接口

```yaml
接口名称: 企业文化OS知识库查询
路径: /api/culture-os-kb/query
方法: POST
请求参数:
  query: string  # 查询内容
  scene: enum    # 场景类型 [S0-S9]
  depth: enum    # 深度 [shallow/medium/deep]
  format: enum   # 格式 [concise/structured/detailed]

响应格式:
  status: success/error
  data:
    knowledge_nodes: []  # 知识节点列表
    relations: []        # 关联联系
    related_skills: []   # 推荐调用的Skills
    suggested_next: []   # 建议的下一步操作
```

### 错误处理

| 错误码 | 含义 | 处理建议 |
|--------|------|---------|
| KB001 | 知识未找到 | 建议补充相关知识 |
| KB002 | 查询过于模糊 | 请澄清具体问题 |
| KB003 | 深度不足 | 建议切换到深度学习 |
| KB004 | 知识过时 | 触发知识更新流程 |
